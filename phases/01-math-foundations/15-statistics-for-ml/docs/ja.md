# 機械学習のための統計学

> 統計学とは、あなたのモデルが本当に機能しているのか、それとも単に運が良かっただけなのかを判断するための手段だ。

**タイプ:** Build
**言語:** Python
**前提条件:** フェーズ1、レッスン06（確率と分布）、07（ベイズの定理）
**所要時間:** 約120分

## 学習目標

- 平均・中央値・最頻値・標準偏差・パーセンタイル・IQR などの記述統計をゼロから実装する
- Pearson/Spearman 相関と共分散行列をゼロから計算する
- 仮説検定（t検定、カイ二乗検定）を実施し、p値と信頼区間を正しく解釈する
- 分布の仮定を必要とせず、任意の指標に対してブートストラップ再サンプリングで信頼区間を構築する
- 効果量の指標を用いて、統計的有意性と実践的有意性を区別する

## 問題

2つのモデルを訓練した。モデルAのテストセットのスコアは0.87、モデルBは0.89だ。あなたはモデルBをデプロイした。3週間後、本番環境の指標は以前より悪化していた。何が起きたのか？

モデルBは実際にはモデルAを上回っていなかった。0.02の差はノイズだった。テストセットが小さすぎるか、分散が大きすぎるか、あるいは両方だった。あなたは改善に見せかけたランダム性を本番に投入してしまったのだ。

これは常に起きている。Kaggleリーダーボードの順位変動。再現できない論文。数百サンプルで勝者を宣言するA/Bテスト。根本原因はいつも同じだ——誰かが統計をスキップしたのだ。

統計学はシグナルとノイズを区別するための道具を与えてくれる。ある差が本物かどうか、どの程度信頼すべきか、結果を信頼できるようになるまでにどれだけのデータが必要かを教えてくれる。すべてのMLパイプライン、すべてのモデル比較、すべての実験に統計は必要だ。それなしでは、あなたは推測しているだけだ。

## 概念

### 記述統計：データの要約

何かをモデル化する前に、データがどのような形をしているかを知る必要がある。記述統計はデータセットをその形状を捉えるいくつかの数値に圧縮する。

**中心傾向の指標**は「中心はどこか？」に答える。

```
平均:   全値の合計 / 個数
        mu = (1/n) * sum(x_i)

中央値: ソート後の中間の値
        外れ値に頑健。[1, 2, 3, 4, 1000] があれば、平均は202だが
        中央値は3になる。

最頻値: 最も頻繁に現れる値
        カテゴリデータに有用。連続データではほとんど情報を持たない。
```

平均はバランス点だ。中央値は折り返し地点だ。両者が乖離するとき、分布は歪んでいる。所得分布では平均 >> 中央値（億万長者による右歪み）となる。訓練中の損失分布では平均 << 中央値（簡単なサンプルによる左歪み）となることが多い。

**散布の指標**は「データはどれだけ分散しているか？」に答える。

```
分散:         平均からの偏差の二乗の平均
              sigma^2 = (1/n) * sum((x_i - mu)^2)

標準偏差:     分散の平方根
              sigma = sqrt(sigma^2)
              データと同じ単位なので、より解釈しやすい。

範囲:         最大値 - 最小値
              外れ値に敏感。単独ではほぼ役に立たない。

IQR:          Q3 - Q1（四分位範囲）
              データの中間50%の範囲。
              外れ値に頑健。箱ひげ図や外れ値検出に使われる。
```

**パーセンタイル**はソート済みのデータを100等分する。第25パーセンタイル（Q1）はデータの25%がこの点より下にあることを意味する。第50パーセンタイルは中央値だ。第75パーセンタイルはQ3だ。

```
レイテンシ監視の場合:
  P50 = 中央値レイテンシ        （典型的なユーザー体験）
  P95 = 第95パーセンタイル       （悪いが最悪ではないケース）
  P99 = 第99パーセンタイル       （テールレイテンシ、中央値の10倍になることも）
```

MLでは、推論レイテンシ、予測信頼度の分布、エラー分布の理解のためにパーセンタイルを重視する。平均誤差は低くてもP99誤差が酷いモデルは、安全性が重要なアプリケーションでは役に立たない可能性がある。

**標本統計と母集団統計。** 標本から分散を計算するとき、nではなく（n-1）で割る。これはベッセルの補正だ。標本平均が真の母平均ではないという事実を補正する。分母にnを使うと、真の分散を系統的に過小評価してしまう。（n-1）を使うと、推定値は不偏になる。

```
母分散: sigma^2 = (1/N) * sum((x_i - mu)^2)
標本分散: s^2   = (1/(n-1)) * sum((x_i - x_bar)^2)
```

実際には、nが大きい（数千サンプル）場合、差は無視できる。nが小さい（数十サンプル）場合は重要になる。

### 相関：変数がどのように一緒に動くか

相関は2つの変数間の線形関係の強さと方向を測定する。

**Pearson相関係数**は線形関連を測定する：

```
r = sum((x_i - x_bar)(y_i - y_bar)) / (n * s_x * s_y)

r = +1:  完全な正の線形関係
r = -1:  完全な負の線形関係
r =  0:  線形関係なし（ただし非線形関係がある場合も！）

範囲: [-1, 1]
```

Pearsonは関係が線形であり、両変数がほぼ正規分布していることを仮定する。外れ値に敏感で、一点の極端な値でrが0.1から0.9に変わることがある。

**Spearmanの順位相関**は単調関連を測定する：

```
1. 各値をその順位（1, 2, 3, ...）に置き換える
2. 順位に対してPearson相関を計算する

Spearmanは線形だけでなく、任意の単調関係を捉える。
y = x^3 の場合、PearsonはR < 1を返すが、SpearmanはRho = 1を返す。
```

**使い分け:**

```
Pearson:    両変数が連続で、ほぼ正規分布している。
            特に線形関係を知りたい場合。
            極端な外れ値がない。

Spearman:   順序データ（ランキング、評点）。
            データが正規分布でない。
            単調だが線形ではない関係が疑われる場合。
            外れ値が存在する。
```

**黄金律:** 相関は因果関係を意味しない。アイスクリームの売上と溺死者数は相関する——どちらも夏に増加するからだ。モデルの精度とパラメータ数は相関するが、パラメータを追加することで自動的に精度が向上するわけではない（参照：過学習）。

### 共分散行列

2つの変数間の共分散は、それらがどのように一緒に変動するかを測定する：

```
Cov(X, Y) = (1/n) * sum((x_i - x_bar)(y_i - y_bar))

Cov(X, Y) > 0:  XとYは一緒に増加する傾向がある
Cov(X, Y) < 0:  Xが増加すると、Yは減少する傾向がある
Cov(X, Y) = 0:  線形な共変動なし
```

d個の特徴量に対して、共分散行列CはC[i][j] = Cov(特徴量_i, 特徴量_j)となるd×d行列だ。対角成分C[i][i]は各特徴量の分散だ。

```
C = | Var(x1)      Cov(x1,x2)  Cov(x1,x3) |
    | Cov(x2,x1)  Var(x2)      Cov(x2,x3) |
    | Cov(x3,x1)  Cov(x3,x2)  Var(x3)     |

性質:
  - 対称: C[i][j] = C[j][i]
  - 半正定値: すべての固有値 >= 0
  - 対角成分 = 分散
  - 非対角成分 = 共分散
```

**PCAとの関係。** PCAは共分散行列を固有値分解する。固有ベクトルが主成分（最大分散の方向）だ。固有値は各成分が捉える分散の量を示す。これはレッスン10で扱った内容だが、なぜ共分散行列が分解すべき正しい対象なのかが分かる——それはデータ内のすべての対ごとの線形関係をエンコードしているからだ。

**相関行列との関係。** 相関行列は標準化された変数（各変数をその標準偏差で割ったもの）の共分散行列だ。相関は共分散を正規化して、すべての値が[-1, 1]に収まるようにする。

### 仮説検定

仮説検定は不確実性のもとで決定を下すためのフレームワークだ。主張から始め、データを収集し、データがその主張と矛盾しないかどうかを判断する。

**設定:**

```
帰無仮説 (H0):        デフォルトの仮定、通常は「効果なし」
対立仮説 (H1): 示そうとしていること

例:
  H0: モデルAとモデルBは同じ精度を持つ
  H1: モデルBはモデルAより高い精度を持つ
```

**p値**とは、H0が真であると仮定したとき、観測したデータと同じかそれ以上に極端なデータを見る確率だ。H0が真である確率ではない。これは統計における最も一般的な誤解だ。

```
p値 = P(このくらい極端なデータ | H0が真)

p値 < alpha（通常は0.05）の場合:
    H0を棄却する。結果は「統計的に有意」だ。
p値 >= alpha の場合:
    H0を棄却できない。証拠が不十分だ。
    これはH0が真であることを意味しない。
```

**信頼区間**はパラメータのもっともらしい値の範囲を与える：

```
平均の95%信頼区間:
    x_bar +/- z * (s / sqrt(n))

ここで、95%信頼度の場合 z = 1.96

解釈: この実験を何度も繰り返したとき、95%の計算された区間が
真の平均を含む。この特定の区間に真の平均が含まれる確率が
95%であることを意味しない。
```

信頼区間の幅は精度を示す。幅が広いほど不確実性が高い。幅が狭ければ推定値が精密なことを意味する（ただし、データが偏っている場合は正確とは限らない）。

### t検定

t検定は平均を比較する。いくつかの種類がある。

**一標本t検定:** 母平均は仮説値と異なるか？

```
t = (x_bar - mu_0) / (s / sqrt(n))

自由度 = n - 1
```

**二標本t検定（独立）:** 2つのグループの平均は異なるか？

```
t = (x_bar_1 - x_bar_2) / sqrt(s1^2/n1 + s2^2/n2)

これはWelchのt検定で、分散が等しいことを仮定しない。
等分散を仮定する特定の理由がない限り、常にWelchのt検定を使うこと。
```

**対応のあるt検定:** 測定値がペアになっている場合（同じモデルを同じデータ分割で評価する場合）：

```
各ペアに対して d_i = x_i - y_i を計算する
次に、d_i の値に対して mu_0 = 0 として一標本t検定を実施する
```

MLでは、対応のあるt検定が一般的だ——両方のモデルを同じ10分割交差検証で実行し、スコアをペアごとに比較する。

### カイ二乗検定

カイ二乗検定は、観測された頻度が期待された頻度と一致するかどうかを検査する。カテゴリデータに有用だ。

```
chi^2 = sum((観測値 - 期待値)^2 / 期待値)

例: 言語モデルの出力分布は、カテゴリ全体で
訓練分布と一致しているか？

カテゴリ    観測値   期待値
Positive       120        100
Negative        80        100
chi^2 = (120-100)^2/100 + (80-100)^2/100 = 4 + 4 = 8

自由度1では、chi^2 = 8 で p < 0.005 となる。
差は有意だ。
```

### MLモデルのA/Bテスト

MLにおけるA/BテストはウェブのA/Bテストとは異なる。モデル比較には特有の課題がある：

```
1. 同じテストセット:  両方のモデルを同一データで評価しなければならない。
                      異なるテストセットでは比較が無意味になる。

2. 複数の指標:        精度だけでは不十分。精度（Precision）、再現率（Recall）、
                      F1、レイテンシ、公平性の指標が必要だ。

3. 分散:              点推定だけでなく、交差検証またはブートストラップを使って
                      各指標の分散を推定する。

4. データ漏洩:        モデル選択中にテストセットが使用されていた場合、
                      比較は偏っている。最終テストセットを保留しておく。
```

**手順:**

```
1. 指標と有意水準を定義する（alpha = 0.05）
2. 両方のモデルを同じk分割交差検証で実行する
3. ペアのスコアを収集する: [(a1, b1), (a2, b2), ..., (ak, bk)]
4. 差分を計算する: d_i = b_i - a_i
5. 差分に対して対応のあるt検定を実施する
6. 確認: 平均差分は0と有意に異なるか？
7. 平均差分の信頼区間を計算する
8. 実践的有意性を判断するために効果量（Cohen's d）を計算する
```

### 統計的有意性と実践的有意性

結果は統計的に有意でも実践的には意味がない場合がある。データが十分にあれば、ごくわずかな差でも統計的に有意になる。

```
例:
  モデルAの精度: 0.9234
  モデルBの精度: 0.9237
  n = 1,000,000 テストサンプル
  p値 = 0.001

統計的に有意？ はい。
実践的に有意？ 0.03%の改善は、新しいモデルをデプロイする
エンジニアリングコストに見合う価値はない。
```

**効果量**はサンプルサイズに依存せず、差の大きさを定量化する：

```
Cohen's d = (mean_1 - mean_2) / pooled_std

d = 0.2:  小さい効果
d = 0.5:  中程度の効果
d = 0.8:  大きい効果
```

常にp値と効果量の両方を報告すること。p値は差が本物かどうかを示す。効果量は差が重要かどうかを示す。

### 多重比較問題

多くの仮説を検定すると、偶然「有意」になるものが出てくる。alpha = 0.05 で20項目をテストすると、何も本物でなくても偽陽性が1つは予想される。

```
P(少なくとも1つの偽陽性) = 1 - (1 - alpha)^m

m = 20 テスト, alpha = 0.05:
P(偽陽性) = 1 - 0.95^20 = 0.64

少なくとも1つの偽陽性が発生する確率は64%だ。
```

**ボンフェローニ補正:** alphaをテスト数で割る。

```
調整済み alpha = alpha / m = 0.05 / 20 = 0.0025

p値 < 0.0025 の場合のみH0を棄却する。
保守的だが単純。テストが独立している場合に機能する。
```

MLでは、複数の指標でモデルを比較したり、多くのハイパーパラメータ設定をテストしたり、複数のデータセットで評価したりする場合にこれが重要になる。

### ブートストラップ法

ブートストラップは、データを復元抽出することで統計量のサンプリング分布を推定する。基礎となる分布についての仮定は不要だ。

**アルゴリズム:**

```
1. n個のデータポイントがある
2. 復元抽出でn個のサンプルを取る（一部のポイントは複数回現れ、
   一部は全く現れない）
3. このブートストラップサンプルで統計量を計算する
4. B回繰り返す（通常B = 1000から10000）
5. ブートストラップ統計量の分布がサンプリング分布を近似する
```

**ブートストラップ信頼区間（パーセンタイル法）:**

```
B個のブートストラップ統計量をソートする
95% CI = [第2.5パーセンタイル, 第97.5パーセンタイル]
```

**MLにおけるブートストラップが重要な理由:**

```
- テストセットの精度は点推定だ。ブートストラップは
  信頼区間を与えてくれる。
- 指標の分布が正規分布であると仮定できない（特に
  AUC、F1、上位k件の適合率に対して）。
- ブートストラップは任意の統計量に対して機能する: 中央値、2つの平均の比、
  2つのモデル間のAUCの差。
- 閉じた形の公式は不要だ。
```

**モデル比較のためのブートストラップ:**

```
1. 同じテストセット上のモデルAとモデルBの予測値がある
2. 各ブートストラップ繰り返しで:
   a. 復元抽出でテストインデックスを再サンプリングする
   b. 再サンプリングされたセットでmetric_AとmetricBを計算する
   c. diff = metric_B - metric_A を保存する
3. 差分の95% CI:
   [差分の第2.5パーセンタイル, 差分の第97.5パーセンタイル]
4. CIが0を含まない場合、差は有意だ
```

これは分布の仮定を設けないため、対応のあるt検定よりも頑健だ。

### パラメトリック検定と非パラメトリック検定

**パラメトリック検定**は特定の分布（通常は正規分布）を仮定する：

```
t検定:          正規分布データを仮定（または中心極限定理による大標本）
ANOVA:          正規性と分散の等質性を仮定
Pearson r:      二変量正規性を仮定
```

**非パラメトリック検定**は分布の仮定を設けない：

```
Mann-Whitney U:     2つのグループを比較（独立t検定の代替）
Wilcoxon符号順位検定: 対応のあるデータを比較（対応のあるt検定の代替）
Spearman rho:       順位による相関（Pearsonの代替）
Kruskal-Wallis:     複数のグループを比較（ANOVAの代替）
```

**非パラメトリックを使う場合:**

```
- 標本サイズが小さく（n < 30）、データが明らかに非正規分布
- 順序データ（評点、ランキング）
- 除去できない強い外れ値がある
- 歪んだ分布
```

**パラメトリックを使う場合:**

```
- 大きな標本サイズ（中心極限定理により検定統計量が近似的に正規分布）
- データが外れ値なしにほぼ対称
- より高い検定力（実際の差を検出するのが得意）
```

ML実験では、通常nが小さい（5回または10回の交差検証フォールド）ため、Wilcoxon符号順位検定のような非パラメトリック検定の方がt検定よりも適切なことが多い。

### 中心極限定理：実践的な意味

中心極限定理（CLT）は、基礎となる母集団分布に関わらず、nが大きくなるにつれてサンプル平均の分布が正規分布に近づくと述べる。

```
X_1, X_2, ..., X_n が平均muと分散sigma^2を持つ独立同分布の場合:

    X_bar ~ Normal(mu, sigma^2 / n)    n -> 無限大のとき

ほとんどの場合、n >= 30 で機能する。
高度に歪んだ分布では、n >= 100 が必要な場合がある。
```

**MLにとっての意味:**

```
1. 集約された指標に対する信頼区間とt検定を正当化する
2. 個々のフォールドが大きく変動しても、交差検証フォールドの
   平均が安定した推定値を与える理由を説明する
3. ミニバッチ勾配降下法は、バッチ上の平均勾配が真の勾配を
   近似するため機能する（CLTの実践）
4. アンサンブル法: 多くのモデルの予測を平均することで、
   単一モデルよりも安定した出力が得られる
```

**CLTができないこと:**

```
- データを正規化しない。サンプルの平均を正規化する。
- 無限分散を持つ裾の重い分布には機能しない
  （コーシー分布）。
- 従属データには適用できない（補正なしの時系列）。
```

### ML論文でよくある統計的ミス

1. **訓練セットでテストする。** 過学習が保証される。訓練中にモデルが見ないデータを常に保留しておくこと。

2. **信頼区間がない。** 不確実性なしに単一の精度数値を報告すると、結果が再現不可能で検証できなくなる。

3. **多重比較を無視する。** 50の設定をテストして補正なしに最良のものを報告すると、偽陽性率が膨らむ。

4. **統計的有意性と実践的有意性を混同する。** 0.01%の精度改善でのp値0.001は意味がない。

5. **不均衡データでの精度を使う。** 99%が陰性クラスのデータセットで99%の精度は、モデルが何も学んでいないことを意味する。精度（Precision）、再現率（Recall）、F1、またはAUCを使うこと。

6. **指標を都合よく選ぶ。** 自分のモデルが勝つ指標だけを報告する。正直な評価はすべての関連指標を報告する。

7. **訓練/テスト分割をまたいで情報を漏洩させる。** 分割前に正規化したり、過去を予測するために未来のデータを使ったりすること。

8. **分散推定なしに小さなテストセットを使う。** 100サンプルで評価して2%の改善を主張するのはノイズであり、シグナルではない。

9. **データが独立でないのに独立を仮定する。** 同じ患者からの医療画像、同じ文書からの複数の文。グループ内の観測値は相関している。

10. **p-hacking。** p < 0.05 を得るまで異なる検定、サブセット、除外基準を試す。結果は探索の産物だ。

## 実装

以下を実装する：

1. **記述統計のゼロ実装**（平均、中央値、最頻値、標準偏差、パーセンタイル、IQR）
2. **相関関数**（Pearson と Spearman、共分散行列付き）
3. **仮説検定**（一標本t検定、二標本t検定、カイ二乗検定）
4. **ブートストラップ信頼区間**（任意の統計量、仮定不要）
5. **A/Bテストシミュレーター**（データ生成、検定、第I種エラーと第II種エラーの確認）
6. **統計的有意性と実践的有意性のデモ**（大きなnがすべてを「有意」にしてしまうことを示す）

すべて `math` と `random` のみを使ってゼロから実装する。numpy も scipy も使わない。

## 主要用語

| 用語 | 定義 |
|---|---|
| 平均（Mean） | 値の合計を個数で割ったもの。外れ値に敏感。 |
| 中央値（Median） | ソートされたデータの中間の値。外れ値に頑健。 |
| 標準偏差（Standard deviation） | 分散の平方根。元の単位で散布を測定する。 |
| パーセンタイル（Percentile） | データの指定された割合がその値より下に落ちる値。 |
| IQR | 四分位範囲。Q3からQ1を引いたもの。中間50%の散布。 |
| Pearson相関（Pearson correlation） | 2つの変数間の線形関連を測定する。範囲[-1, 1]。 |
| Spearman相関（Spearman correlation） | 順位を使って単調関連を測定する。 |
| 共分散行列（Covariance matrix） | すべての特徴量間の対ごとの共分散の行列。 |
| 帰無仮説（Null hypothesis） | 効果や差がないというデフォルトの仮定。 |
| p値（p-value） | 帰無仮説が真であるとき、このくらい極端なデータが得られる確率。 |
| 信頼区間（Confidence interval） | 指定された信頼水準でパラメータのもっともらしい値の範囲。 |
| t検定（t-test） | 平均が有意に異なるかどうかを検定する。t分布を使用する。 |
| カイ二乗検定（Chi-squared test） | 観測された頻度が期待された頻度と異なるかどうかを検定する。 |
| 効果量（Effect size） | サンプルサイズに依存しない差の大きさ。Cohen's dが一般的。 |
| ボンフェローニ補正（Bonferroni correction） | 偽陽性を制御するために、テスト数で有意水準のしきい値を割る。 |
| ブートストラップ（Bootstrap） | サンプリング分布を推定するための復元抽出による再サンプリング。 |
| 第I種エラー（Type I error） | 偽陽性。H0が真のときにH0を棄却する。 |
| 第II種エラー（Type II error） | 偽陰性。H0が偽のときにH0を棄却できない。 |
| 検定力（Statistical power） | 偽のH0を正しく棄却する確率。検定力 = 1 - 第II種エラー率。 |
| 中心極限定理（Central limit theorem） | サンプルサイズが大きくなるにつれて、サンプル平均は正規分布に収束する。 |
| パラメトリック検定（Parametric test） | データの特定の分布（通常は正規分布）を仮定する。 |
| 非パラメトリック検定（Non-parametric test） | 分布の仮定を設けない。順位や符号に基づいて機能する。 |
