# ノルムと距離

> 距離関数は「類似」の意味を定義する。間違えれば、下流のすべてが壊れる。

**タイプ:** 実装
**言語:** Python
**前提条件:** フェーズ1、レッスン01（線形代数の直感）、02（ベクトル・行列・演算）
**時間:** 約90分

## 学習目標

- L1、L2、コサイン、マハラノビス、Jaccard、編集距離の各関数をゼロから実装する
- 与えられた ML タスクに対して適切な距離指標を選択し、代替手段がなぜ失敗するかを説明できる
- L1・L2 ノルムと LASSO・Ridge 正則化、およびその幾何学的制約領域とのつながりを理解する
- 同じデータセットが異なる指標では異なる最近傍を生み出すことを実証する

## 問題の設定

2つのベクトルがある。単語埋め込みかもしれない。ユーザープロファイルかもしれない。ピクセル配列かもしれない。問いは一つ：それらはどれくらい近いのか？

答えは、どの距離関数を選ぶかに完全に依存する。ある指標では最近傍だった2つのデータ点が、別の指標では遠く離れることがある。KNN 分類器、推薦エンジン、ベクトルデータベース、クラスタリングアルゴリズム、損失関数——これらはすべてこの選択に依存している。間違えればモデルは誤った目標に向けて最適化される。

普遍的に最良の距離は存在しない。L2 は空間データに有効。コサイン類似度は NLP で主流。Jaccard は集合を扱う。編集距離は文字列を扱う。マハラノビスは相関を考慮する。Wasserstein は確率質量を移動させる。それぞれが「類似」の意味について異なる仮定をエンコードしている。

このレッスンでは、主要な距離関数をすべてゼロから構築し、それぞれがいつ適切なツールになるかを示し、同じデータから使う指標によって全く異なる最近傍が生まれることを実証する。

## 概念

### ノルム：ベクトルの大きさを測る

ノルムはベクトルの「大きさ」を測る。2つのベクトル間の距離関数はすべて、差分のノルムとして書ける：d(a, b) = ||a - b||。つまりノルムを理解することは距離を理解することだ。

### L1 ノルム（マンハッタン距離）

L1 ノルムは全成分の絶対値の和。

```
||x||_1 = |x_1| + |x_2| + ... + |x_n|
```

軸方向にしか移動できない都市のグリッド上でどれだけ歩くかを測るため、マンハッタン距離と呼ばれる。斜め移動はない。

```
Point A = (1, 1)
Point B = (4, 5)

L1 distance = |4-1| + |5-1| = 3 + 4 = 7

グリッド上では、東に3ブロック、北に4ブロック歩く。
```

L1 を使う場面：
- 高次元スパースデータ（テキスト特徴、one-hot エンコーディング）
- 外れ値への頑健性が必要なとき（単一の巨大な差が支配しない）
- 特徴選択問題（L1 正則化はスパース性を促進する）

L1 正則化（Lasso）との関係：損失関数に ||w||_1 を加えると、重みの絶対値の和にペナルティを課す。これにより小さな重みがちょうどゼロになり、自動的な特徴選択が行われる。L1 ペナルティは重み空間でダイアモンド形の制約領域を作り、ダイアモンドの角は一部の重みがゼロになる軸上にある。

損失関数との関係：平均絶対誤差（MAE）は予測値と目標値の間の L1 距離の平均。すべての誤差を線形にペナルティ化するため、MSE に比べて外れ値に対して頑健。

### L2 ノルム（ユークリッド距離）

L2 ノルムは直線距離。各成分の二乗和の平方根。

```
||x||_2 = sqrt(x_1^2 + x_2^2 + ... + x_n^2)
```

これは幾何学の授業で習った距離。n 次元のピタゴラス。

```
Point A = (1, 1)
Point B = (4, 5)

L2 distance = sqrt((4-1)^2 + (5-1)^2) = sqrt(9 + 16) = sqrt(25) = 5.0

グリッドを斜めに突き抜ける直線。
```

L2 を使う場面：
- 低〜中次元の連続データ
- 特徴のスケールが比較可能なとき
- 物理的距離（空間データ、センサー読み取り値）
- ピクセルレベルの画像類似度

L2 正則化（Ridge）との関係：損失関数に ||w||_2^2 を加えると、大きな重みにペナルティを課す。L1 とは異なり、重みをゼロに押しやらない。すべての重みをゼロに向けて比例的に縮小する。L2 ペナルティは円形の制約領域を作るため、軸上に角がない。重みは小さくなるが、ほとんどの場合ちょうどゼロにはならない。

損失関数との関係：平均二乗誤差（MSE）は L2 距離の二乗の平均。二乗化により、大きな誤差が小さな誤差よりも重くペナルティ化される。

```
MAE (L1 loss):  |y - y_hat|         線形ペナルティ。外れ値に頑健。
MSE (L2 loss):  (y - y_hat)^2       二次ペナルティ。外れ値に敏感。
```

### Lp ノルム：一般的な族

L1 と L2 は Lp ノルムの特殊ケース：

```
||x||_p = (|x_1|^p + |x_2|^p + ... + |x_n|^p)^(1/p)
```

p の値が異なると、「単位球」（原点から距離 1 にあるすべての点の集合）の形が変わる：

```
p=1:    ダイアモンド形      (軸上に角)
p=2:    円/球              (通常の丸い球)
p=3:    超楕円形            (丸みを帯びた正方形)
p=inf:  正方形/超立方体     (軸に沿った平らな辺)
```

### L-無限大ノルム（チェビシェフ距離）

p が無限大に近づくにつれ、Lp ノルムは最大絶対成分に収束する。

```
||x||_inf = max(|x_1|, |x_2|, ..., |x_n|)
```

2点間の距離は、最も差が大きい単一の次元によって決まる。他の次元はすべて無視される。

```
Point A = (1, 1)
Point B = (4, 5)

L-inf distance = max(|4-1|, |5-1|) = max(3, 4) = 4
```

L-無限大を使う場面：
- 任意の単一次元での最悪ケースの偏差が重要なとき
- ゲームボード（チェスのキングは L-無限大で移動：どの方向への1ステップもコスト1）
- 製造公差（すべての寸法が仕様内でなければならない）

### コサイン類似度とコサイン距離

コサイン類似度は2つのベクトル間の角度を測り、大きさを無視する。

```
cos_sim(a, b) = (a . b) / (||a||_2 * ||b||_2)
```

-1（逆方向）から +1（同方向）の範囲をとる。直交するベクトルのコサイン類似度は 0。

コサイン距離はこれを距離に変換する：cosine_distance = 1 - cosine_similarity。0（同方向）から 2（逆方向）の範囲。

```
a = (1, 0)    b = (1, 1)

cos_sim = (1*1 + 0*1) / (1 * sqrt(2)) = 1/sqrt(2) = 0.707
cos_dist = 1 - 0.707 = 0.293
```

NLP と埋め込みでコサインが主流な理由：テキストでは、文書の長さが類似度に影響してはならない。猫についての文書が、別の猫についての文書の2倍の長さであっても、「類似している」べきだ。コサイン類似度は大きさ（長さ）を無視し、方向だけを気にする。同じ単語分布で長さが異なる2つの文書は同じ方向を指し、コサイン類似度 1.0 を得る。

コサイン類似度を使う場面：
- テキスト類似度（TF-IDF ベクトル、単語埋め込み、文埋め込み）
- 大きさがノイズで方向がシグナルであるあらゆるドメイン
- 推薦システム（ユーザー嗜好ベクトル）
- 埋め込み検索（ベクトルデータベースはほぼ常にコサインかドット積を使う）

### ドット積類似度とコサイン類似度

2つのベクトルのドット積は：

```
a . b = a_1*b_1 + a_2*b_2 + ... + a_n*b_n
      = ||a|| * ||b|| * cos(angle)
```

コサイン類似度は両方の大きさで正規化したドット積。両方のベクトルが既に単位正規化されている（大きさ = 1）場合、ドット積とコサイン類似度は同一になる。

```
If ||a|| = 1 and ||b|| = 1:
    a . b = cos(angle between a and b)
```

違いが生まれるとき：ドット積は大きさの情報を含む。大きな大きさを持つベクトルはドット積スコアが高くなる。「人気」のあるアイテムを上位にランクさせたい検索システムでは、これが重要になる。大きさは暗黙的な品質または重要度のシグナルとして機能する。

```
a = (3, 0)    b = (1, 0)    c = (0, 1)

dot(a, b) = 3     dot(a, c) = 0
cos(a, b) = 1.0   cos(a, c) = 0.0

どちらも方向については一致するが、ドット積は大きさも反映する。
```

実際の使い方：
- 純粋な方向の類似度が欲しい場合はコサイン類似度を使う
- 大きさが意味のある情報を持つ場合はドット積を使う
- 多くのベクトルデータベース（Pinecone、Weaviate、Qdrant）はどちらかを選択できる
- 埋め込みが L2 正規化されている場合、選択は関係ない

### マハラノビス距離

ユークリッド距離はすべての次元を等しく扱う。しかし特徴が相関しているかスケールが異なる場合、L2 は誤解を招く結果を与える。

マハラノビス距離はデータの共分散構造を考慮する。

```
d_M(x, y) = sqrt((x - y)^T * S^(-1) * (x - y))
```

ここで S はデータの共分散行列。

直感的に：マハラノビス距離は、まずデータの相関を除去して正規化（白色化）し、その変換された空間で L2 距離を計算する。S が単位行列（無相関、単位分散の特徴）であれば、マハラノビス距離はユークリッド距離に帰着する。

```
例：身長と体重は相関している。
6フィート2インチで180ポンドは珍しくない。
5フィート0インチで180ポンドは珍しい。

ユークリッド距離は、どちらも平均から等距離と言うかもしれない。
マハラノビス距離は身長と体重の相関を考慮するため、
2番目が外れ値であることを正しく識別する。
```

マハラノビス距離を使う場面：
- 外れ値検出（平均からのマハラノビス距離が大きい点が外れ値）
- 特徴のスケールと相関が異なる場合の分類
- 信頼できる共分散行列を推定できるだけのデータがある場合
- 製造業の品質管理（多変量プロセス監視）

### Jaccard 類似度（集合に対して）

Jaccard 類似度は2つの集合の重複を測る。

```
J(A, B) = |A intersect B| / |A union B|
```

0（重複なし）から 1（同一の集合）の範囲。Jaccard 距離 = 1 - Jaccard 類似度。

```
A = {cat, dog, fish}
B = {cat, bird, fish, snake}

Intersection = {cat, fish}         size = 2
Union = {cat, dog, fish, bird, snake}  size = 5

Jaccard similarity = 2/5 = 0.4
Jaccard distance = 0.6
```

Jaccard を使う場面：
- タグ、カテゴリ、特徴の集合の比較
- 単語の存在（頻度ではなく）に基づく文書類似度
- ほぼ重複の検出（Jaccard の MinHash 近似）
- バイナリ特徴ベクトルの比較（存在/不在データ）
- セグメンテーションモデルの評価（Intersection over Union = Jaccard）

### 編集距離（レーベンシュタイン距離）

編集距離は、ある文字列を別の文字列に変換するために必要な1文字操作の最小数を数える。操作は：挿入、削除、置換。

```
"kitten" -> "sitting"

kitten -> sitten  (substitute k -> s)
sitten -> sittin  (substitute e -> i)
sittin -> sitting (insert g)

Edit distance = 3
```

動的計画法で計算する。エントリ (i, j) が文字列 A の最初の i 文字と文字列 B の最初の j 文字の間の編集距離である行列を埋める。

```
        ""  s  i  t  t  i  n  g
    ""   0  1  2  3  4  5  6  7
    k    1  1  2  3  4  5  6  7
    i    2  2  1  2  3  4  5  6
    t    3  3  2  1  2  3  4  5
    t    4  4  3  2  1  2  3  4
    e    5  5  4  3  2  2  3  4
    n    6  6  5  4  3  3  2  3
```

編集距離を使う場面：
- スペルチェックと修正
- DNA 配列アライメント（重み付き操作）
- ファジー文字列マッチング
- 雑然としたテキストデータの重複排除

### KL ダイバージェンス（距離ではないが、距離のように使われる）

KL ダイバージェンスはある確率分布が別の分布とどれだけ異なるかを測る。レッスン09でカバーされるが、真の距離ではないにもかかわらず「距離」として使われることがあるため、この議論に含める。

```
D_KL(P || Q) = sum(p(x) * log(p(x) / q(x)))
```

重要な性質：KL ダイバージェンスは対称ではない。

```
D_KL(P || Q) != D_KL(Q || P)
```

これは距離指標の基本要件を満たさないことを意味する。三角不等式も満たさない。ダイバージェンスであり、距離ではない。

前向き KL（D_KL(P || Q)）は「平均志向」：Q は P のすべてのモードをカバーしようとする。
逆向き KL（D_KL(Q || P)）は「モード志向」：Q は P の単一モードに集中する。

KL ダイバージェンスが使われる場面：
- VAE（ELBO の KL 項が潜在分布を事前分布に向けて押す）
- 知識蒸留（生徒が教師の分布に合わせようとする）
- RLHF（KL ペナルティがファインチューニング済みモデルをベースモデルに近く保つ）
- 方策勾配法（方策更新の制約）

### Wasserstein 距離（アースムーバー距離）

Wasserstein 距離は、ある確率分布を別の分布に変換するために必要な最小の「仕事」を測る。直感的に：ある分布が土の山で別の分布が穴だとしたら、どれだけの土をどれだけ遠く動かす必要があるか？

```
W(P, Q) = inf over all transport plans gamma of E[d(x, y)]
```

1次元分布に対しては、累積分布関数の絶対差の積分に簡略化される：

```
W_1(P, Q) = integral |CDF_P(x) - CDF_Q(x)| dx
```

Wasserstein が重要な理由：
- 真の距離指標（対称、三角不等式を満たす）
- 分布が重ならない場合でも勾配を提供する（KL ダイバージェンスは無限大になる）
- この性質が Wasserstein GAN（WGAN）の中心となり、元の GAN の学習不安定性を解決した

```
重ならない分布：

P: [1, 0, 0, 0, 0]    Q: [0, 0, 0, 0, 1]

KL divergence: infinity (log of zero)
Wasserstein: 4 (move all mass 4 bins)

Wasserstein は意味のある勾配を与える。KL は与えない。
```

Wasserstein を使う場面：
- GAN 学習（WGAN、WGAN-GP）
- 重ならない可能性がある分布の比較
- 最適輸送問題
- 画像検索（カラーヒストグラムの比較）

### なぜ異なるタスクに異なる距離が必要か

| タスク | 最適な距離 | 理由 |
|--------|------------|------|
| テキスト類似度 | コサイン | 大きさはノイズ、方向が意味 |
| 画像ピクセル比較 | L2 | 空間的関係が重要、特徴のスケールが同程度 |
| スパース高次元特徴 | L1 | 頑健、まれな大きな差を増幅しない |
| 集合の重複（タグ、カテゴリ） | Jaccard | データが本質的に集合値であり、ベクトルではない |
| 文字列マッチング | 編集距離 | 操作が人間の編集直感に対応する |
| 外れ値検出 | マハラノビス | 特徴の相関とスケールを考慮 |
| 分布の比較 | KL ダイバージェンス | Q の代わりに P を使うことで失われる情報を測る |
| GAN 学習 | Wasserstein | 分布が重ならない場合でも勾配を提供 |
| 埋め込み（ベクトル DB） | コサインまたはドット積 | 埋め込みは方向に意味をエンコードするよう学習される |
| 推薦 | ドット積 | 大きさが人気度や信頼度をエンコードできる |
| DNA 配列 | 重み付き編集距離 | 置換コストがヌクレオチドペアにより異なる |
| 製造品質管理 | L-無限大 | 任意の次元での最悪ケースの偏差が重要 |

### 損失関数との関係

損失関数は、予測値と目標値に適用された距離関数。

```
損失関数            使用する距離             挙動
MSE                 L2 二乗               大きな誤差を強くペナルティ化
MAE                 L1                    すべての誤差を均等にペナルティ化
Huber loss          大きな誤差には L1、    両方のベスト：外れ値に頑健、
                    小さな誤差には L2      ゼロ付近で滑らかな勾配
Cross-entropy       KL ダイバージェンス   分布の不一致を測る
Hinge loss          max(0, margin - d)    マージン以下のみペナルティ化
Triplet loss        L2（通常）            正例を近く、負例を
                                          遠ざける
Contrastive loss    L2                    類似ペアを近く、非類似
                                          ペアをマージンを超えて遠ざける
```

### 正則化との関係

正則化は損失関数に重みのノルムペナルティを加える。

```
L1 正則化（Lasso）:   loss + lambda * ||w||_1
  -> スパースな重み。一部の重みがちょうどゼロになる。
  -> 自動的な特徴選択。
  -> ゼロで微分不可能な角を持つ解。

L2 正則化（Ridge）:   loss + lambda * ||w||_2^2
  -> 小さな重み。すべての重みがゼロに向けて縮小する。
  -> 特徴選択なし（何もちょうどゼロにはならない）。
  -> どこでも滑らかな解。

Elastic Net:          loss + lambda_1 * ||w||_1 + lambda_2 * ||w||_2^2
  -> L1 のスパース性と L2 の安定性を組み合わせる。
  -> 相関した特徴グループがまとめて保持または削除される。
```

L1 はスパース性を生み出すが L2 はそうでない理由：2次元重み空間の制約領域を想像してほしい。L1 はダイアモンド、L2 は円。損失関数の等高線（楕円）はダイアモンドの角に触れることが最も多く、そこでは一部の重みがゼロになる。円には滑らかな点で触れるため、両方の重みが非ゼロになる。

### 最近傍探索

すべての距離関数は最近傍探索問題を意味する：クエリ点が与えられたとき、データセット内の最も近い点を見つける。

厳密な最近傍探索は、d 次元で n 点のデータセットに対してクエリ1件あたり O(n * d)。大規模データセットではこれは遅すぎる。

近似最近傍（ANN）アルゴリズムは少量の精度を犠牲にして大幅な速度向上を実現する：

```
アルゴリズム       アプローチ                       使用例
KD-trees          軸平行空間分割                    scikit-learn（低次元）
Ball trees        入れ子の超球                      scikit-learn（中次元）
LSH               ランダムハッシュ射影              ほぼ重複検出
HNSW              階層的ナビゲーブル                FAISS、Qdrant、Weaviate
                  スモールワールドグラフ
IVF               クラスタベース探索を用いた         FAISS（10億スケール）
                  転置ファイルインデックス
Product quant.    ベクトルを圧縮し、圧縮空間で       FAISS（メモリ制約あり）
                  探索
```

HNSW（Hierarchical Navigable Small World）は現代のベクトルデータベースにおける支配的なアルゴリズム。各ノードが近似最近傍に接続する多層グラフを構築する。探索は最上層（疎、長いジャンプ）から始まり、最下層（密、短いジャンプ）まで降下する。

## 実装

### ステップ1：すべてのノルムと距離関数

完全な実装は `code/distances.py` を参照。すべての関数は基本的な Python の数学のみを使ってゼロから構築されている。

### ステップ2：同じデータ、異なる距離、異なる近傍

`distances.py` のデモはデータセットを作成し、クエリ点を選び、距離指標によって最近傍がどのように変わるかを示す。L1 で「最も近い」点が L2 やコサインでは最も近くならない場合がある。

### ステップ3：埋め込み類似度検索

コードにはモック埋め込み類似度検索が含まれており、コサイン類似度と L2 距離を使ってクエリに最も類似した「文書」を見つけ、ランキングが異なり得ることを示す。

## 活用

最も一般的な実際の使用例：ベクトルデータベースで類似アイテムを見つける。

```python
import numpy as np

def cosine_similarity_matrix(X):
    norms = np.linalg.norm(X, axis=1, keepdims=True)
    norms = np.where(norms == 0, 1, norms)
    X_normalized = X / norms
    return X_normalized @ X_normalized.T

embeddings = np.random.randn(1000, 768)

sim_matrix = cosine_similarity_matrix(embeddings)

query_idx = 0
similarities = sim_matrix[query_idx]
top_k = np.argsort(similarities)[::-1][1:6]
print(f"Top 5 most similar to item 0: {top_k}")
print(f"Similarities: {similarities[top_k]}")
```

`model.encode(text)` を呼び出してベクトルデータベースを検索するとき、内部ではこれが起きている。埋め込みモデルがテキストをベクトルにマッピングする。ベクトルデータベースはクエリベクトルと保存されたすべてのベクトルの間のコサイン類似度（またはドット積）を計算し、ANN アルゴリズムを使ってすべてをチェックしないようにする。

## 演習

1. (1, 2, 3) と (4, 0, 6) の L1、L2、L-無限大距離を計算する。L-inf <= L2 <= L1 が任意の点のペアに対して常に成り立つことを確認する。この順序が保証される理由を証明せよ。

2. コサイン類似度が高い（> 0.9）が L2 距離が大きい（> 10）2つのベクトルを作成する。幾何学的に何が起きているかを説明する。次に、コサイン類似度が低い（< 0.3）が L2 距離が小さい（< 0.5）2つのベクトルを作成する。

3. データセットとクエリ点を受け取り、L1、L2、コサイン、マハラノビス距離の下での最近傍を返す関数を実装する。4つすべてが最も近い点について一致しないデータセットを見つけよ。

4. CDF 法を使って [0.5, 0.5, 0, 0] と [0, 0, 0.5, 0.5] の Wasserstein 距離を手計算する。次に [0.25, 0.25, 0.25, 0.25] と [0, 0, 0.5, 0.5] の間で計算する。どちらが大きく、なぜか？

5. 近似 Jaccard 類似度のために MinHash を実装する。100 個のランダムな集合を生成し、すべてのペアの正確な Jaccard を計算し、50、100、200 個のハッシュ関数を使った MinHash 近似と比較する。近似誤差をプロットせよ。

## 重要用語

| 用語 | 人々が言うこと | 実際の意味 |
|------|---------------|-----------|
| ノルム | 「ベクトルの大きさ」 | ベクトルを非負のスカラーにマッピングする関数で、三角不等式、絶対一次斉次性、ゼロベクトルのみゼロを満たす |
| L1 ノルム | 「マンハッタン距離」 | 絶対成分値の和。最適化においてスパース性を生む。外れ値に頑健 |
| L2 ノルム | 「ユークリッド距離」 | 二乗成分の和の平方根。ユークリッド空間での直線距離 |
| Lp ノルム | 「一般化ノルム」 | 絶対成分の p 乗の和の p 乗根。L1 と L2 は特殊ケース |
| L-無限大ノルム | 「最大ノルム」または「チェビシェフ距離」 | 最大絶対成分値。p が無限大に近づくときの Lp の極限 |
| コサイン類似度 | 「ベクトル間の角度」 | 両方の大きさで正規化したドット積。-1 から +1 の範囲。ベクトルの長さを無視 |
| コサイン距離 | 「1 マイナス コサイン類似度」 | コサイン類似度を距離に変換する。0 から 2 の範囲 |
| ドット積 | 「正規化されていないコサイン」 | 成分ごとの積の和。コサイン類似度に両方の大きさを掛けたものに等しい |
| マハラノビス距離 | 「相関を考慮した距離」 | データの共分散行列を使って白色化（相関除去と正規化）された空間での L2 距離 |
| Jaccard 類似度 | 「集合の重複」 | 積集合の大きさを和集合の大きさで割ったもの。ベクトルではなく集合に対して |
| 編集距離 | 「レーベンシュタイン距離」 | ある文字列を別の文字列に変換するための最小の挿入、削除、置換 |
| KL ダイバージェンス | 「分布間の距離」 | 真の距離ではない（非対称）。Q を使って P をエンコードする際の余分なビットを測る |
| Wasserstein 距離 | 「アースムーバー距離」 | ある分布から別の分布へ質量を輸送するための最小の仕事。真の距離指標 |
| 近似最近傍 | 「ANN 探索」 | 厳密な探索よりはるかに速く近似的な最近傍を見つけるアルゴリズム（HNSW、LSH、IVF） |
| HNSW | 「ベクトル DB アルゴリズム」 | Hierarchical Navigable Small World グラフ。高速な近似最近傍探索のための多層グラフ |
| L1 正則化 | 「Lasso」 | 重みの L1 ノルムを損失に加える。重みをゼロに誘導する（スパース性） |
| L2 正則化 | 「Ridge」または「重み減衰」 | 重みの L2 ノルムの二乗を損失に加える。スパース性なしで重みをゼロに向けて縮小 |
| Elastic Net | 「L1 + L2」 | L1 と L2 正則化を組み合わせる。相関した特徴グループをどちらか単独より上手く処理 |
