# 命令追従をアライメント シグナルとして

> 後の RLHF のすべての批評はこのパイプラインの何かに対する議論。最適化圧力がプロキシをどう歪めるかを学ぶ前に、プロキシを見る必要。InstructGPT（Ouyang et al.、2022）が参照アーキテクチャを定義：命令応答ペアでの教師あり ファインチューニング、ペアワイズ優先度ランキングで学習された報酬モデル、KL ペナルティで SFT ポリシーへの PPO。1.3B InstructGPT は 175B GPT-3 より優先。その単一結果が 2026年のすべてのフロンティア実験室が RLHF 型事後学習パイプラインをまだ配送する理由。

**タイプ:** Learn
**言語:** Python（stdlib、オモチャ 3 ステージ パイプライン）
**前提条件:** Phase 10 · 06 (SFT)、Phase 10 · 07 (RLHF)、Phase 10 · 08 (DPO)
**所要時間:** ~45分

## 学習目標

- InstructGPT パイプラインの 3 つのステージ各々で使用される損失の 3 つのステージを名前で述べ、各ステージで使用される損失を述べる。
- 1.3B 命令チューンモデルがなぜ生の 175B GPT-3 を人間優先度評価で打たしたかを説明。
- ステージ 3 の KL ペナルティが何に対して保護されており、それを削除するとなぜモード シーキング動作に崩壊するかを述べる。
- アライメント税を説明し、Ouyang et al. が それに対して使用した PPO-ptx 緩和を説明。

## 問題

事前学習言語モデルはテキスト完成。質問に答えない。GPT-3 に「リストを反転する Python 関数を書く」と聞いて、別のプロンプトを得ることが多い、ほとんどの学習分布がより多くのウェブテキストで続くウェブテキストであるため。モデルはその仕事をしている — 仕事が間違っている。

すべての真面目な実験室が使用した代理は人間優先度。2 つの完成が評者に行き、評者がより良いものを選ぶ、報酬モデルが評者を学習する。その後 RL ループが報酬モデルが高い評価をする出力への方策をシフト。それが完全な InstructGPT テーゼを 3 つの文で。残りはエンジニアリング。

## コンセプト

### ステージ 1：教師あり ファインチューニング（SFT）

プロンプト応答ペアを集める。応答は善意の人間が書くもの。Ouyang et al. はラベラーからの 13k プロンプとと OpenAI API を使用。標準クロス エントロピー損失でこのデータの上でベース モデルをファインチューン。

SFT がくれるもの：モデルは質問を続けるのではなく答える。くれないもの：複数が妥当な場合どの答えが評者が好むかについてのシグナル。

### ステージ 2：報酬モデル（RM）

各プロンプトについて、SFT モデルから K の完成をサンプル。ラベラーがそれらをランク。報酬モデルをスコアするため任意のプロンプト応答ペアで学習：`y_w` が `y_l` よりも優先されたペアについて：

```
L_RM = -log sigmoid(r(x, y_w) - r(x, y_l))
```

これは Bradley-Terry ペアワイズ優先度損失。RM は通常、LM ヘッドがスカラー ヘッドに置換された SFT モデルから初期化。

報酬モデルは小さい：6B は 175B InstructGPT に十分。また脆弱 — 論文のセクション 5 はほとんどが小さいスケールで現れた報酬ハッキング動作について。

### ステージ 3：KL ペナルティを持つ PPO

目的を定義：

```
J(pi) = E_{x~D, y~pi(.|x)} [ r(x, y) ] - beta * KL(pi(.|x) || pi_SFT(.|x))
```

PPO で最大化。KL ターム は `pi` が SFT ポリシーから遠くドリフトするのを保持。なしで、オプティマイザーは敵対的例を見つける — RM がそれらを見たことはないため RM の下で高スコアされるストリング、人間が実際にそれらを優先するため。

KL 係数 `beta` は単一最も重要な RLHF ハイパーパラメータ。低すぎる：報酬ハッキング。高すぎる：SFT 以上の改善なし。

### アライメント税

RLHF 後、モデルは人間に優先されるが標準ベンチマーク（SQuAD、HellaSwag、DROP）で回帰。Ouyang et al. はこれをアライメント税と呼び PPO-ptx で修正：事前学習勾配を RL 目的に混ぜるため、モデルは報酬されなかった下流タスク実行方法を忘れない。

```
J_ptx(pi) = J(pi) + gamma * E_{x~D_pretrain} [ log pi(x) ]
```

PPO-ptx は標準になった。Anthropic、DeepMind、Meta すべていくつかのバリアント使用。

### 結果

1.3B InstructGPT（SFT + RM + PPO-ptx）は 175B ベース GPT-3 上でラベラーにより約 70% のとき優先。ギャップは本番トラフィックから隠し テスト プロンプト上で拡大。この数字から 2 つのことを読む：

1. アライメントは能力から異なる軸。175B モデルはより多くの能力を持つ、1.3B モデルはより多くのアライメント、ラベラーはアラインされたものを優先。
2. 能力フロアはベース モデルで設定。ベース モデルが見たことがない事実への RLHF できない。

### これが Phase 18 の参考ポイント理由

後のレッスンのすべての批評 — 報酬ハッキング（レッスン 2）、DPO（レッスン 3）、シコファント（レッスン 4）、CAI（レッスン 5）、スリーパー エージェント（レッスン 7）、アライメント フェイキング（レッスン 9）— はこのパイプラインの何か部分に対する議論。報酬ハッキングはステージ 2 を攻撃。DPO ステージ 2 と 3 を崩壊。CAI は人間ラベラーを置換。シコファンシーはラベラーが偏ったシグナル。アライメント フェイキングはポリシーがステージ 3 全体を回路で行えることを示す。これらの批評のどれもあなたの頭にまずパイプラインなしで従う。

## これを使用

`code/main.py` はオモチャ優先度データで 3 つのステージをシミュレート。ベース「ポリシー」はアクション {A、B、C} 上のバイアスコイン。ステージ 1 SFT は 200 プロンプト上のラベラー アクション。ステージ 2 は 500 ペアワイズ ランキングから Bradley-Terry 報酬モデルをフィッテッド。ステージ 3 は簡略化された PPO 更新を SFT ポリシーへの KL ペナルティで実行。報酬の上昇、KL ダイバージェンスの成長、ポリシー ドリフト観察可能 — KL ターム をオフにして報酬ハッキング を 50 更新ステップ内で現れるのを見られる。

見るべき：

- `beta = 0.1` vs `beta = 0.0` で報酬トラジェクトリー。
- トレーニング ステップにわたる KL(pi || pi_SFT)。
- ラベラー優先度比較で最終アクション分布。

## 配送

このレッスンは `outputs/skill-instructgpt-explainer.md` を生成。RLHF パイプライン説明または論文抄録を指定で、3 つのステージのどれが修正されているか、各ステージで使用される損失、KL ペナルティまたは同等な正則化があるかを識別。

## 演習

1. `code/main.py` を実行。`beta = 0.0` を設定して 200 PPO ステップ後のアクション分布をレポート。1 段落でモード シーキング動作を説明。

2. 報酬モデルを アクション B のためのシミュレートされた報酬バグで修正（+0.5 バイアス）。`beta = 0.1` で PPO を実行。KL ペナルティはポリシーがバイアスを利用するのを防止？どの `beta` で利用が見える？

3. Ouyang et al.（arXiv:2203.02155）図 1 を読む。1、5、20、100 ステップで PPO を実行してラベラー優先度カーブを再現してラベラー優先度に対する測定。

4. 論文のセクション 4.3 は 1.3B InstructGPT が 175B GPT-3 を約 70% のとき打つのをレポート。ラベラー自身のプロンプトより隠し本番プロンプト上の比率がより高いだろう理由？

5. DPO（Phase 10 · 08）を同じ優先度データで PPO 損失を置換。最終的ポリシー ドリフト（SFT への KL）と最終的報酬を比較。マッチ報酬でどの方法がより遠くドリフト？

## キー用語

| 用語 | 人々が言うこと | 実際の意味 |
|------|-----------------|--------|
| SFT | 「命令チューニング」 | ステージ 1：プロンプト応答ペアでクロス エントロピー ファインチューン |
| 報酬モデル | 「RM」 | ペアワイズ ラベルで Bradley-Terry でスカラー回帰者（プロンプト、レスポンス） |
| Bradley-Terry | 「ペアワイズ優先度損失」 | -log sigmoid(r_w - r_l)、ペアワイズ ランキングをバイナリ分類に減少 |
| KL ペナルティ | 「正則化子」 | `beta * KL(pi \|\| pi_SFT)` — RL ポリシーを SFT アンカー近く保持 |
| PPO-ptx | 「事前学習ミックス PPO」 | 事前学習ログ尤度の一部を PPO 目的に追加。アライメント税オフセット |
| アライメント税 | 「RLHF 回帰」 | RLHF は対象とならなかった標準ベンチマーク上のポストHLRF ドロップ |
| ラベラー優先度 | 「真実」 | 人間ランキング のサンプル、RM はこれへの統計的プロキシ、「人間値」ではない |
