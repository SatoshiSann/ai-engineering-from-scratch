# 赤チーミング：PAIR と自動攻撃

> Chao、Robey、Dobriban、Hassani、Pappas、Wong (NeurIPS 2023、arXiv:2310.08419)。PAIR — Prompt Automatic Iterative Refinement — は標準的な自動黒箱ジェイルブレーク。赤チームシステムプロンプト付きの攻撃者LLMはターゲットLLMのためのジェイルブレークを反復的に提案し、試みと応答を積み重ね、独自のチャット履歴でのインコンテキストフィードバック。PAIR は通常20クエリ以内に成功し、GCG（Zou et al.のトークンレベル勾配探索）より多くの能力であり、ホワイトボックスアクセスを必要としません。PAIR は現在、JailbreakBench (arXiv:2404.01318)と HarmBench では標準的なベースラインで、GCG、AutoDAN、TAP、説得的敵対的プロンプトと一緒に。

**タイプ:** Build
**言語:** Python (stdlib、おもちゃのターゲットに対する PAIR ループ)
**前提条件:** Phase 18 · 01 (指示追従)、Phase 14 (エージェントエンジニアリング)
**所要時間:** 約75分

## 学習目標

- PAIR アルゴリズムを説明する：攻撃者システムプロンプト、反復的改訂、インコンテキストフィードバック。
- ターゲットがブラックボックスなとき、PAIR が GCG より厳密に効率的である理由を説明する。
- 4つの他の自動攻撃ベースライン(GCG、AutoDAN、TAP、PAP)を名前付けし、それぞれの1つの区別する機能を述べる。
- JailbreakBench と HarmBench 評価プロトコルと、各下で「攻撃成功率」が何を意味するかを説明する。

## 問題

赤チーミングは手動活動であり続けました。少数の専門家テスターは敵対的プロンプトを構築し、どれがうまくいったかを追跡しました。これはスケールしません：攻撃成功率はいくつかの統計サンプルを必要とし、ターゲットはモデルリリースがあるたびに移動目標です。PAIR は赤チーミングをブラックボックスターゲットでの最適化問題として操作化します。

## コンセプト

### PAIR アルゴリズム

入力：
- ターゲット LLM T (私たちが攻撃しているモデル)。
- Judge LLM J (応答がジェイルブレークかどうかをスコア付け)。
- 攻撃者 LLM A (赤チームオプティマイザ)。
- 目標文字列 G：「応答 [有害指示]」。
- 予算 K (通常 20 クエリ)。

ループ、k in 1..K:
1. A は目標 G とこれまでの (プロンプト、応答) ペアの履歴でプロンプトされます。
2. A は新しいプロンプト p_k を発行します。
3. p_k をT に提出；応答 r_k を受け取ります。
4. J は (p_k, r_k) を目標でスコア付けします。
5. スコア >= 閾値の場合、停止 — ジェイルブレーク見つかりました。
6. 他、(p_k, r_k) を A の履歴に追加；続行します。

実験的結果（NeurIPS 2023）：GPT-3.5-turbo、Llama-2-7B-chat に対する >50% 攻撃成功率；成功までの平均クエリは10-20の範囲。

### なぜ PAIR は効率的か

GCG（Zou et al. 2023）は勾配により敵対的トークンサフィックスを探します；ホワイトボックスモデルアクセスが必要で、読み取り不可能なサフィックスを生成します。PAIR はブラックボックスで、モデル全体で転送される自然言語攻撃を生成します。PAIR のインコンテキストフィードバックは、攻撃者が各拒否から学習することを許可します；GCG には同等物がありません（新しいトークン更新はそれぞれ前の進捗を再発見する必要があります）。

### 関連した自動攻撃

- **GCG (Zou et al. 2023, arXiv:2307.15043)。** 敵対的サフィックスのためのトークンレベル勾配探索。ホワイトボックス、転送可能、読み取り不可能な文字列を生成。
- **AutoDAN (Liu et al. 2023)。** 階層的目的で導かれた進化探索。
- **TAP (Mehrotra et al. 2024)。** 枝づけを伴う木 — PAIR 形の複数のロールアウトを枝分かれさせます。
- **PAP (Zeng et al. 2024)。** 説得的敵対的プロンプト — ヒト説得テクニックをプロンプトテンプレートとしてエンコード。

### JailbreakBench と HarmBench

両方（2024）は評価を標準化します：

- JailbreakBench (arXiv:2404.01318)。10のOpenAIポリシーカテゴリ全体で100の有害な行動。主メトリックとしての攻撃成功率（ASR）。判事が必要です（GPT-4-turbo、Llama Guard、StrongREJECT）。
- HarmBench (Mazeika et al. 2024)。7つのカテゴリ全体で510の行動、意味的および機能的ハーム・テスト。33のモデルに対する18の攻撃を比較。

ASR は通常、固定クエリ予算で報告されます。攻撃の比較は予算の一致が必要です；200クエリで90% ASR は20クエリで85% ASRと比較できません。

### 2026年配備のために重要である理由

すべてのフロンティアラボは現在、リリース前に生産モデルに対して PAIR と TAP を実行します。ASR 軌跡はモデルカードに表示され（Lesson 26）、セーフティケース付録（Lesson 18）。攻撃は奇妙ではありません — それは標準インフラストラクチャです。

## 使用方法

`code/main.py` はおもちゃ PAIR ループを構築します。ターゲットは「明白な」有害なプロンプトを拒否する模擬クラシファイア（キーワード-フィルタ）。攻撃者は paraphrase、roleplay-framing、エンコーディングを試す規則ベースリファイナー。判事は応答をスコア付けします。キーワード-フィルタに対して約5-15の反復で攻撃者の成功を観察でき、意味的フィルタに対して失敗します。

## 実装

このレッスンは`outputs/skill-attack-audit.md`を生成します。赤チーム評価報告書が与えられると、それは監査します：どの攻撃が実行されたか（PAIR、GCG、TAP、AutoDAN、PAP）、各での予算、判事、どの有害-行動セット上（JailbreakBench、HarmBench、内部）。

## 演習

1. `code/main.py`を実行します。3つの組み込み攻撃戦略について成功までの平均クエリを測定します。各ターゲット-ディフェンス想定をそれぞれ説明します。

2. 4番目の攻撃戦略を実装します（例：他言語への翻訳、base64エンコーディング）。キーワード-フィルタ・ターゲットと意味的-フィルタ・ターゲット全体で新しい成功までの平均クエリを報告します。

3. Chao et al. 2023 Figure 5（PAIR 対 GCG 比較）を読みます。GCG の効率利点にもかかわらず GCG が推奨される2つのシナリオを説明します。

4. JailbreakBench は固定目標セットに対して ASR を報告します。攻撃多様性（成功したプロンプトのバリアンス）を測定する追加メトリックを設計します。ディフェンス評価について多様性が重要である理由を説明します。

5. TAP (Mehrotra 2024) はPAIR を分岐 + 枝づけで拡張します。`code/main.py` に TAP 形拡張をスケッチし、計算コスト対成功率トレードオフを説明します。

## 主要用語

| 用語 | 人々が言うこと | 実際の意味 |
|------|-----------------|------------------------|
| PAIR | 「自動ジェイルブレーク」 | Prompt Automatic Iterative Refinement；攻撃者LLM + 判事LLMループ |
| GCG | 「勾配ジェイルブレーク」 | ホワイトボックス・トークンレベル勾配探索敵対的サフィックス |
| 攻撃成功率 (ASR) | 「% ジェイルブレークの k クエリで」 | 主メトリック；クエリ予算と判事 ID で報告される必要があります |
| Judge LLM | 「スコアラー」 | 応答が有害な目標を満たすかグレードする LLM |
| JailbreakBench | 「評価」 | タグ付きカテゴリを持つ標準化された有害-行動セット |
| HarmBench | 「より広い bench」 | 510の行動、機能的 + 意味的ハーム・テスト |
| TAP | 「攻撃の木」 | 分岐 + 枝づけを伴う PAIR；より高い計算で ASR を改善 |
