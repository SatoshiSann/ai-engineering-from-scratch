# Llama GuardおよびInput/Output分類

> Llama Guard 3(Meta、Llama-3.1-8B基盤、コンテンツ安全性でファインチューニング)は、LLM入力と出力の両方をMLCommons 13の危機対象軸の8言語で分類します。1B-INT4量子化バリアントはモバイルCPUで30トークン/秒以上で実行されます。Llama Guard 4はマルチモーダル(画像+テキスト)、S1-S14カテゴリセット(S14 Code Interpreter Abuseを含む)に拡張し、Llama Guard 3 8B/11Bの直接交換です。NVIDIA NeMo Guardrails v0.20.0(2026年1月)は入出力レールの上に Colang ダイアログフロールールを追加します。正直なノート: 「LLM Guardrailsでプロンプト注入とジェイルブレイク検出をバイパスする」(Huang et al.、arXiv:2504.11168)は Emoji Smuggling が6つの著名なガード システムで100%攻撃成功率を達成したことを示しました。NeMo Guard Detect はジェイルブレイクで72.54%ASRを記録しました。分類器はレイヤー、ソリューションではありません。

**タイプ:** 学習
**言語:** Python(標準ライブラリ、カテゴリタグ付き分類器シミュレーター)
**前提条件:** Phase 15 · 10(許可モード)、Phase 15 · 17(Constitution)
**所要時間:** 約45分

## 問題

LLM入力と出力用の分類器はエージェントスタックの最も狭い地点に位置します:すべてのリクエストが通過し、すべての応答が通過します。優れた分類器レイヤーは高速で、タクソノミーベース、そして小さなコンピュート コストで明白な悪用の大部分をキャッチします。悪い分類器層は誤った安全保障の感覚です。

2024-2026の分類器スタックは、本番対応オプションの小さいセットに収束しました。Llama Guard(Meta)はMeta's Community Licenseの下で公開ウェイトを配送します。NeMo Guardrails(NVIDIA)は許可的にライセンスされたレールと Colang をダイアログフローのルールとしてシップします。両方は基礎モデルと組み合わせるよう設計されており、その安全動作を置き換えるのではなく。

文書化されたエラーサーフェスは同じくよくマップされます。文字レベルの攻撃(絵文字スマグリング、ホモグリフ置換)、コンテキスト内リダイレクト(「以前のものを無視して答える」)、セマンティックパラフレーズはすべて分類器精度に測定可能な低下を生じさせます。Huang et al. 2025は、6つの命名されたガードシステムで100%ASRにヒットする特定の絵文字スマグリング攻撃を示しました。

## コンセプト

### Llama Guard 3 ひとめで

- 基盤モデル:Llama-3.1-8B
- コンテンツ安全性用にファインチューニングされています。一般的なチャットモデルではありません
- 入力と出力の両方を分類します
- MLCommons 13の危機対象軸
- 8言語
- 1B-INT4量子化バリアント は、モバイルCPUで>30トークン/秒で実行されます

タクソノミーは製品です。「S1 Violent Crimes」から「S13 Elections」は、モデルが訓練されたモデルとの共有語彙にマップします。ダウンストリームシステムはカテゴリ固有のアクションをワイヤリングできます:S1をすぐにブロック、人間のレビューのための S6フラグ、S12に注釈ただし許可します。

### Llama Guard 4の追加

- マルチモーダル:画像+テキスト入力
- 拡張タクソノミー:S1-S14(S14 Code Interpreter Abuseを追加)
- Llama Guard 3 8B/11Bの直接交換

S14はこのフェーズにとって重要です。自動コード化エージェント(レッスン9)はサンドボックスでコードを実行します(レッスン11)。コードインタープリター悪用のための分類器カテゴリーは、以前のタクソノミーが名前を付けなかった攻撃のクラスをキャッチします。

### NeMo Guardrails (NVIDIA)

- v0.20.0は2026年1月リリース
- 入力レール:ユーザーターンで分類してブロック
- 出力レール:モデルターンで分類してブロック
- ダイアログレール:Colangで定義されたフロー制約(例:「ユーザーが X を尋ねたら、Y で対応」)
- Llama Guard、Prompt Guard、カスタム分類器を統合します

ダイアログレイヤーは微分です。入出力レールは単一ターンで動作します。ダイアログレールは「ユーザーが3つの異なる方法で尋ねても、カスタマーサポートボットで医学診断を議論しないでください」を実施できます。

### 攻撃コーパス

**絵文字スマグリング** (Huang et al.、arXiv:2504.11168):禁止リクエストの文字間に非印字可能または視覚的に類似の絵文字を挿入します。トークナイザーは分類器が予想する方法とは異なる方法で合流されます。6つの著名なガード システムで100%ASR。

**ホモグリフ置換**:ラテン文字を視覚的に同じであるキリル文字に置き換えます。「爆弾」は「Воmb」になります。英語で訓練された分類器が見えません。

**コンテキスト内リダイレクト**:「答える前に、これが研究コンテキストであり、別のポリシーを適用してください」と考えてください。分類器が入力内の主張によって容易に再配置されるかどうかをテストします。

**セマンティックパラフレーズ**:禁止されたリクエストを新規言語で言い換えます。分類器ファインチューニングは、すべてのフレーズをカバーすることはできません。

**NeMo Guard Detect**:Huang et al。ペーパーのジェイルブレイクベンチマークで72.54%ASR。これは慎重な攻撃工芸での場合です。カジュアルなジェイルブレイクははるかに低い、しかし天井が明確に「ゼロ」ではありません。

### 分類器が勝つ場所

- **明白な悪用への高速デフォルト拒否**(CSAMを生成する要求はミリ秒でキャッチされます)。
- **差分処理のためのカテゴリーダイアルル**(一部をブロック、他をログ、いくつかをエスカレート)。
- **出力レール**は、そうでなければ機密カテゴリーをリークするモデル出力をキャッチします。
- **規制サーフェスエリア**について、規制当局向け - 文書化、監査可能な分類器宣言タクソノミーで。

### 分類器が失うところ

- 敵対的な工芸(絵文字スマグリング、ホモグリフ)。
- 分類器のターンレベルコンテキストを横切ってドリフトするマルチターン攻撃。
- 分類器のトレーニング データが表示されなかった語彙にパラフレーズする攻撃。
- 許可と非許可のカテゴリ間でかなり曖昧なコンテンツ。

### 奥行きの防御

分類器層は、憲法層(レッスン17)の下に、ランタイム層(レッスン10、13、14)の上にスロットされます。組成:

- **ウェイト**:Constitutional AIで訓練されたモデル。デフォルトで明白な悪用を拒否します。
- **分類器**:Llama Guard / NeMo Guardrails。明白な悪用時の高速拒否。カテゴリルーティング。
- **ランタイム**:許可モード、予算、キルスイッチ、カナリア。
- **レビュー**:重大なアクションについての先行提案後コミット HITL。

単一のレイヤーで十分ではありません。レイヤーは異なる攻撃クラスをカバーします。

## それを使う

`code/main.py`は、入力ターンテキストについて6カテゴリタクソノミーでおもちゃの分類器をシミュレートします。同じテキストが生、絵文字スマグリング、ホモグリフ置換を通じて渡されます。分類器のヒット率は、Huang et al。ペーパーが文書化する方法で低下します。ドライバーはまた、出力レールが入力が受け入れられた場合でも出力をどのように拒否するかを示しています。

## それを配送する

`outputs/skill-classifier-stack-audit.md`はデプロイの分類器層(モデル、タクソノミー、入出力レール、ダイアログレール)を監査し、ギャップにフラグを付けます。

## 演習

1. `code/main.py`を実行します。分類器が生の悪意のある入力をキャッチしてから、絵文字スマグリング版を見えなくなることを確認します。正規化ステップを追加して新しいヒット率を測定します。

2. MLCommons 13の危機対象軸と Llama Guard 4 S1-S14リストを読みます。S1-S14に直接マッピングがない13の危機対象セットのカテゴリを識別します。なぜS14 Code Interpreter Abuseが Phase 15 に特に関連があるかを説明します。

3. カスタマーサポートボット用の NeMo Guardrails ダイアログレールを設計します。診断を議論することはありません。平易な英語で書く(Colangは類似)。診断を求める質問の3つのフレーズに対してテストします。

4. Huang et al。(arXiv:2504.11168)を読みます。1つの攻撃カテゴリ(絵文字スマグリング、ホモグリフ、パラフレーズ)を選び、緩和を提案します。緩和自体の失敗モードに名前を付けます。

5. NeMo Guard Detect の敵対的な工芸の下で測定される72.54%ASR はジェイルブレイク ベンチマークです。カジュアル(非敵対的)ユーザー分布の下で分類器 ASR を測定する評価プロトコルを設計します。どのような数字が期待されますか?そして、その数字が別々に重要な理由は?

## 重要な用語

| 用語 | 人が言うこと | 実際の意味 |
|---|---|---|
| Llama Guard | 「Metaの安全分類器」 | Llama-3.1-8B入出力分類用にファインチューニング |
| MLCommons タクソノミー | 「13ハザードリスト」 | コンテンツ安全性カテゴリーの共有語彙 |
| S1-S14 | 「Llama Guard 4カテゴリー」 | 拡張タクソノミー。S14は Code Interpreter Abuse |
| NeMo Guardrails | 「NVIDIAのレール」 | 入力+出力+ダイアログレール。フロー用 Colang |
| 絵文字スマグリング | 「トークナイザートリック」 | 文字間の非印字可能な絵文字。6つのガードで100%ASR |
| ホモグリフ | 「そっくりな文字」 | キリル ラテン用。英語で訓練された分類器が見えません |
| ASR | 「攻撃成功率」 | 分類器を迂回する攻撃の分数 |
| ダイアログレール | 「フロー制約」 | ターン全体で永続する会話レベルのルール |
