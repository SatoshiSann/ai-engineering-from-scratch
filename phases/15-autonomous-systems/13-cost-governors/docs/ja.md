# アクション予算、イテレーション上限、コスト管理者

> 中規模のeコマースエージェントの月額LLMコストは、チームが「注文追跡」スキルを有効にした後、1,200ドルから4,800ドルに跳ねました。これは価格設定のバグではありません。新しいループを見つけて、その中で支出を続けるエージェントです。Microsoftのエージェント統治ツールキット（2026年4月2日）は、このクラスに対する防御を成文化します。要求ごとの`max_tokens`、タスクごとのトークンと予算、1日/月の上限、イテレーション上限、段階的なモデルルーティング、プロンプトキャッシング、コンテキストウィンドウ、高価なアクションでのHITLチェックポイント、予算違反時のキルスイッチ。AnthropicのClaude Code Agent SDKは、異なる名前で同じプリミティブを出荷します。金融速度制限（例えば10分以上50ドルでアクセスを切断）は、月間上限よりもループをより速くキャッチします。

**タイプ:** 学習
**言語:** Python（標準ライブラリ、層状コスト管理シミュレータ）
**前提条件:** Phase 15 · 10 (権限モード)、Phase 15 · 12 (耐久実行)
**所要時間:** 約60分

## 問題

自律エージェントはすべてのターンで実際のお金を費やします。チャットボットの悪い出力は悪い返信です。エージェントの悪いループは請求です。失敗モードの業界文書用語は「ウォレット拒否」です。エージェントは推論を続け、ツール呼び出しを続け、請求を続け、何も設計されていないため何も停止しません。

修正は1つの数字ではありません。異なる時間スケールと粒度での制限の積み重ねです。要求ごと、タスクごと、時間ごと、日ごと、月ごと。十分に設計されたスタックは、暴走ループを数分以内に、遅い漏れを数時間以内に、悪いリリースを1日以内にキャッチします。同じスタックは、エージェントが長地平で自律的な場合、予算を保持します。

これはエンジニアリングレッスンです。数学は些細で、規律はチームが失敗する場所です。以下の制限のリストはすべて、MicrosoftエージェントStatementツールキットまたはAnthropicClaude Code Agent SDKドキュメントで指定されています。

## コンセプト

### コスト管理者スタック

1. **要求ごとの`max_tokens`。** 単純。単一の呼び出しが無限の完成を発するのを防ぎます。
2. **タスク当たりのトークン予算。** 全体的な実行中は、Nトークンを超えない。上限でハード停止。
3. **タスク当たりのドル予算。** トークンと同じですが、通貨で。Claude Codeの`max_budget_usd`。
4. **ツール呼び出し上限ごと。** 最大N個の`WebFetch`呼び出し、N個の`shell_exec`呼び出しなど。
5. **イテレーション上限（`max_turns`）。** エージェントループの反復の合計。無限推論ループを防ぐ。
6. **分ごと/時間ごと/日ごと/月ごと上限。** ローリングウィンドウ。異なる時間スケールでの漏れをキャッチ。
7. **金融速度制限。** 例えば「10分以内に50ドルを超える場合、アクセスをカット」。月間上限が起動する前にループベースの燃焼をキャッチ。
8. **段階的なモデルルーティング。** より小さいモデルをデフォルト化。分類器がタスクが保証されると判断した場合のみ、より大きなモデルにエスカレートします。
9. **プロンプトキャッシング。** システムプロンプトと安定したコンテキストがプロバイダーキャッシュに保存されます。再送信のトークンコストはほぼゼロです。
10. **コンテキストウィンドウ。** アクティブなコンテキストを閾値を下回ったままにするための凝集/要約。直接的なトークンコスト削減。
11. **高価なアクションのHITLチェックポイント。** 高価であることが知られているアクション（長いツール呼び出し、大きなダウンロード、コストのかかるモデルのアップグレード）の前に、人間のタップが必要です。
12. **予算違反時のキルスイッチ。** 任意の上限が発火したときのセッション中止。上限が記録されます。別の再有効化パスが必要です。

### なぜ1つの上限スタックではなく

単一の月間上限は、ウォレットが失われた後にのみ暴走エージェントをキャッチします。単一の要求ごとの上限は、セッションレベルでは何もキャッチしません。異なる失敗モードは異なる時間スケールを必要とします。

- **暴走ループ**（5秒の再試行に立ち往生しているエージェント）：速度制限によってキャッチ。
- **遅い漏れ**（エージェントがタスクごとに約2倍の予期された作業を行う）：毎日の上限によってキャッチ。
- **悪いリリース**（新しいバージョンは5倍のトークンを使用）：週/月の上限によってキャッチ。
- **正当な急増**（実際の需要、バグではない）：時間/日の上限によってクリアログでキャッチ。

### Claude Codeの予算サーフェス

Claude Code Agent SDKは公開ドキュメントを公開します：

- `max_turns` — イテレーション上限。
- `max_budget_usd` — ドル上限。セッションは違反時に中止。
- `allowed_tools` / `disallowed_tools` — ツール許可リストと拒否リスト。
- ツール使用前のカスタムコスト会計のためのフックポイント。

権限モードラダーと組み合わせます（レッスン10）。`max_budget_usd`なしの`autoMode`セッションは、統治されない自律性です。Anthropicは、Auto Modeが予算制御を必要とすることを明示的にフレーム化します。分類器はコストと直交しています。

### EU AI法、OWASP Agent Top 10

MicrosoftのエージェントStatementツールキットはOWASP Agent Top 10およびEU AI法第14条（人間の監視）の要件をカバーしています。EUでの本番環境でのログと上限の実行はオプションではありません。

### 観察された1,200ドル→4,800ドルのケース

Microsoftドキュメント内の実際のケース：新しいツールが追加された後、月額コストが3倍になったeコマースエージェント。ツールにより、エージェントはすべてのセッション中に注文ステータスをポーリングできました。ループ検出なし。ツール当たりの上限なし。週ごとの成長のアラートなし。修正はツール当たりの上限と毎日の成長アラートでした。これはテンプレート：すべての新しいツールサーフェスは新しい潜在的なループです。すべての新しいツールには独自の上限と独自のアラートが必要です。

## 使用方法

`code/main.py`は、層状のコスト管理スタックがない場合とある場合の両方のエージェント実行をシミュレートします。シミュレートされたエージェントはいくつかのターン後のポーリングループに漂流します。層状スタックは速度ウィンドウ内でそれをキャッチしますが、単一の月間上限は数日後までは発火しません。

## 運用

`outputs/skill-agent-budget-audit.md`は、提案されたエージェントのデプロイのコスト管理スタックを監査し、欠落レイヤーをフラグします。

## 演習

1. `code/main.py`を実行します。ポーリングループの軌跡上のイテレーション上限の前に速度制限が発火することを確認します。次に速度制限を無効にし、イテレーション上限がそれをキャッチするまでエージェントが「支出」する量を測定します。

2. ブラウザエージェント用にツール当たりの上限セット（レッスン11）を設計します。どのツールが最も厳しい上限を必要としていますか？どのツールはリスクなく無限に実行できますか？

3. Microsoft Agent Governance Toolkitドキュメントを読みます。ツールキットが指定するすべての上限タイプをリストアップします。各失敗モードの1つにマップします（暴走ループ、遅い漏れ、悪いリリース、急増）。

4. 現実的なタスクの一晩の無人実行の価格（例えば、「リポジトリの50の問題をトリアージ」）。ポイント推定の2倍で`max_budget_usd`を設定してください。2倍を正当化してください。

5. Claude Codeの`max_budget_usd`はセッション集計コストで発火します。外部で実行する補完速度制限を設計します。何がカットオフをトリガーし、再有効化はどのように見えますか？

## キーターム

| タイプ | 人々が言うこと | それが実際に意味するもの |
|---|---|---|
| ウォレット拒否 | 「暴走請求」 | それを止めるには何も設計されていないため、キャップなしで支出を生成するエージェントループ |
| max_tokens | 「要求ごとの上限」 | 単一の完成のサイズの天井 |
| max_turns | 「イテレーション上限」 | セッション内のエージェントループの反復の上限 |
| max_budget_usd | 「ドルキルスイッチ」 | セッションコスト上限。違反時に中止 |
| 速度制限 | 「レート上限」 | 短いウィンドウごとの支出制限（例：$ 50 / 10分） |
| 段階的なルーティング | 「小さいモデル優先」 | 安いモデルデフォルト。分類器が保証するときのみエスカレート |
| プロンプトキャッシング | 「キャッシュされたシステムプロンプト」 | プロバイダー側キャッシュは再送信トークンコストをほぼゼロに削減 |
| HITLチェックポイント | 「人間承認ゲート」 | 高価なアクション前に人間タップが必要 |
