# 自動調整研究 (Anthropic AAR)

> Anthropic は、独立のサンドボックスで Claude Opus 4.6 自律調整研究者の並列チームを実行し、共有フォーラム経由で調整しました。ログはサンドボックス外に存在します (エージェントは独自のレコードを削除することはできません)。弱から強いトレーニング問題で、AAR は人間の研究者を上回りました。Anthropic 独自の概要は、処方されたワークフローがしばしば AAR 柔軟性を制約し、パフォーマンスを低下させることをフラグします。自動調整研究の自動化は、RSP が検出することになっている正確な非調整リスクにタイムラインを圧縮することです。

**タイプ:** 学習
**言語:** Python (標準ライブラリ、並列研究フォーラム シミュレータ)
**前提条件:** フェーズ 15 · 05 (AI Scientist v2)、フェーズ 15 · 04 (DGM)
**所要時間:** 約 60 分

## 問題

調整研究は人間の研究者の時間では高価です。スケール可能な監視、報酬仕様、または弱から強いトレーニングのような問題は、イテレーションあたり数週間かかる実験が必要です。最先端の能力が上昇するように、調整ワークロードは修飾研究者の供給より速く増加しています。

自動調整研究 (AAR) は、能力がアライメントを上回っている同じ最先端モデルがギャップを閉じるのに貢献できるかどうかを尋ねます。Anthropic の 2026 レポート、弱から強いトレーニング研究を実行 AAR で (alignment.anthropic.com/2026/automated-w2s-researcher/) は、展開されたシステムのこのクラスの最初のパブリック結果の 1 つです。

結果は本当に積極的です: AAR は人間のベースラインよりも研究問題をより良く解決しました。結果また、集中されます。このフェーズが開発した各懸念は、単一のシステムに。調整研究を自動化できると、成果物を損なうことのできない部品もできます。RSP のR &D 能力のしきい値はこのループを念頭に書き込みできます。

## コンセプト

### アーキテクチャ (公開の説明ように)

- **並列エージェント。** 複数の Claude Opus 4.6 インスタンスが AAR のように実行し、各独立のサンドボックスに。サンドボックスは、ファイルシステム、ネットワーク、プロセス表面を制限します。
- **共有フォーラム。** 見つかった は共有フォーラムにパブリッシュされます。ストレージ バックエンド はすべてのサンドボックスの外側にあります。エージェントはフォーラムを読むことができます。エージェントは独自のサンドボックス内からフォーラム レコード を削除または編集することはできません。
- **タスク配置。** 人間で設計されたタスク キューが研究問題を AAR に割り当てます。AAR は分解を提案し、実験を実行し、見つかった を作成します。
- **弱から強いトレーニング フォーカス。** パブリッシュされた研究は、弱から強いトレーニングに対処しました—弱いスーパーバイザー モデルを使用して、より強い学生モデルをトレーニングします。AAR はこのタスク で人間の研究者を上回りました。

### 理由サンドボックス ログ外は重要です

ログ完全性プロパティは、制御実験とおそらく破損している間の違いです。AAR がそれら独自のログを編集できる場合、彼らは失敗または疑わしい実験をカバーアップすることができます。Anthropic はこれを直接フラグします: プロパティは研究出力を信頼する必要な条件であり、検証される非自明です。

インフラストラクチャ用語のアナログは、外部ストレージ (S3 オブジェクト ロック、不変 Datalog ログ) への書き込みスルー を持つ追加のみログです。エージェントは過去のレコード を修正するために資格情報を持ちません。これは、2026 会社のセキュリティ チームがランサムウェア耐性監査証跡を行ういくつかの相同のパターンです。

### 処方されたワークフロー トレードオフ

Anthropic のレポートは、処方されたワークフローがしばしば AAR 柔軟性を制約し、パフォーマンスを低下させることを注記します。言い換えれば: 人間のデザイナーが ステップバイ ステップで研究計画を書く場合、AAR は高レベルの目標を与えられる場合よりは悪いことになります。これは、エージェント研究全体の結果 (LangGraph、Microsoft のエージェント フレームワーク) 全体で一貫しています—オーバー指定されたスキャホは有能なエージェントを傷つけます。

安全性の意味は微妙です。パフォーマンスを改善する柔軟性は、認可のない分解、新しい ツール 使用、目標ドリフト を許可する柔軟性でもあります。トレードオフは設計スペース決定です: どのくらいの柔軟性人間を倒すのに十分な能力を買いますが、エージェントに配布外の目標を一般化するための部屋を与えていません?

### 圧縮リスク

RSP v3.0 (レッスン 19) では、AI R &D 能力のしきい値を導入しています: 人間対AI ツール対競争コストで AI R &D パイプラインを完全に自動化するための能力。DeepMind の FSF v3 には同様の ML R &D 自動性レベルが含まれています。両方のフレームワークはこのしきい値を高度なコントロール のトリガーとして扱 います。

AAR は、いくつかのステップは短い: それは、パイプライン (特定、よくスコープされた タスク上の調整研究) の部分を自動化しますが、エンド ツー エンド能力開発ループではありません。タイムライン質問は、ギャップがどのくらい速く閉じるか です。

圧縮のタイムライン は複合障害の懸念です。調整研究と能力研究が同じレートで複合する場合、非調整リスク表面は少なくともつまり速い成長のように増加します。能力がより速く複合する場合 (履歴トレンド)、ギャップは拡大します。これは AAR が条件付きの良い理由の議論です: 各追加調整結果はリスク表面を減らす場合、研究プロセスが信頼できる場合のみ、ギャップが縮小する場合のみ。

### AAR が置き換えないもの

人間の研究者が、見つかった をレビューし、憲法権限を保持するタスク キューを設定します。AAR はパイプラインの仲介を加速化し、エンドではありません。Anthropic のパブリッシュされた出力には、AAR の貢献と、何をパブリッシュするか、何を撤回するか、何を精緻にするかに関する人間の研究者の判断が含まれます。

これは、レッスン 15 から提案し、その後コミット パターンと合 致します。これは研究自体に適用されます: AAR は提案します; 人間はコミット します。

## それを使用する

`code/main.py` はマルチ AAR 研究フォーラムをシミュレートします。3 つの独立した「研究エージェント」は並列に実行し、オプションのタンパーアテンプト を持つ付加のみフォーラムに見つかった を投稿します。

見てください:

- サンドボックス外ロギング がログタンパーアテンプト をどのようにキャッチ します。
- タスク配置は、固定ワークフローより難しい問題でより自由な分解を実行する方法。
- 単一の侵害された AAR がどのようにフォーラムの信号を汚染でき、ログが改ざん耐性でない場合。

## それを納品する

`outputs/skill-aar-deployment-review.md` は、自動調整研究パイプラインの事前展開レビュー です。

## 演習

1. `code/main.py` を実行します。「固定ワークフロー」対「自由分解」設定を比較します。自由分解は常に勝利、またはいくつかの問題クラスは固定ワークフローは競争力がありますか?

2. 1 つのエージェント がログタンパーアテンプト を試行するようにシミュレータを変更します。付加ロギングのみ がそれを検出することを確認します。検出署名がログに正確に見えるかを説明する 1 段落を書きます。

3. Anthropic の弱から強い AAR レポートを読みます。AAR が人間の研究者の上でビート した具体的なサブ タスク を識別します。自動化にそれを何が適合させたか?

4. AAR 柔軟性 (より良い結果) と処方されたワークフロー制約 (より簡単な監査) のバランスを取るタスク キューの配置ポリシー を設計します。2 つを A/B テスト する方法を説明します。

5. RSP v3.0 の AI R &D-4 のしきい値を読みます。AAR は現在ではなく、何が越えるかを 1 段落で説明します。

| 用語 | 人々は何と言うか | それは実際には何を意味するのか |
|---|---|---|
| AAR | 「自動調整研究者」 | Claude Opus 4.6 インスタンス は調整問題で自動的に操作 |
| 弱から強いトレーニング | 「より弱いスーパーバイザー でより強いモデル をトレーニング」 | 古典的スケール可能監視ベンチマーク AAR が人間の上でビート しました |
| 共有フォーラム | 「エージェント は見つかった をパブリッシュする場所」 | 付加のみ、サンドボックス外ストレージ |
| サンドボックス外ログ | 「エージェント は独自の レコード を編集できない」 | 外部ストレージ への改ざん耐性 書き込みスルー |
| 処方されたワークフロー | 「人間のデザイナーからのステップバイ ステップ計画」 | AAR を制約; しばしば自由分解 対のパフォーマンスを低下 |
| 自由分解 | 「エージェント がタスク を破 る方法を決定」 | より有能、より難しい監査 |
| AI R &D のしきい値 | 「RSP/FSF 能力レベル」 | 競争コストで完全に自動化 R &D パイプライン |
| 圧縮タイムライン | 「調整対能力レース」 | 能力が調整より速く複合される場合、非調整リスク増加 |
