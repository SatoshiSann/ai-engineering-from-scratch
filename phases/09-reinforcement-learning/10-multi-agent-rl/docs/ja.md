# マルチエージェントRL

> 単一エージェントRLは環境が定常であると仮定。同じ世界に2つの学習エージェントを入れて、その仮定は崩れ：各エージェントはもう一方の環境の一部であり、両方が変わっている。マルチエージェントRLはマルコフ仮定がもはや成立しないとき、学習が収束するようにさせるトリックの集合。

**タイプ:** Build
**言語:** Python
**前提条件:** Phase 9 · 04 (Q-learning)、Phase 9 · 06 (REINFORCE)、Phase 9 · 07 (Actor-Critic)
**所要時間:** ~45分

## 問題

部屋を操作して学習ロボットは単一エージェントRL問題。サッカーチームはそうではない。AlphaStarとStarCraft対手は違う。入札エージェントのマーケットプレイスはそうではない。四方停止を交渉する2台の車もそうではない。多くのマルチエージェント現実世界の問題はそうではない。

すべてのマルチエージェント設定で、どのエージェントの視点からでも、他のエージェント *は* 環境の一部。彼らが学習して行動を変えるにつれ、環境は非定常になる。マルコフプロパティ — 「次状態は現在状態と私のアクションのみに依存」 — は違反される。なぜなら次状態は他のエージェントが選んだものにも依存し、彼らのポリシーは動く目標。

これはテーブル型収束証明を破壊する（Q-learningの保証は定常環境を仮定）。それはナイーブなディープRLも破壊：エージェントはループで互いを追い、安定したポリシーに決して収束しない。マルチエージェント固有のテクニックが必要：一元化訓練 / 分散実行、反事実ベースライン、リーグプレイ、自己プレイ。

2026アプリケーション：ロボットスワーム、トラフィックルーティング、自動運転フリート、市場シミュレーター、マルチエージェントLLMシステム (Phase 16)、複数の知能プレイヤーを持つすべてのゲーム。

## コンセプト

![4つのMARLレジーム：独立、一元化クリティック、自己プレイ、リーグ](../assets/marl.svg)

**形式化：マルコフゲーム。** MDPの一般化：状態 `S`、関節アクション `a = (a_1, …, a_n)`、遷移 `P(s' | s, a)`、エージェント単位報酬 `R_i(s, a, s')`。各エージェント `i` は自身のポリシー `π_i` の下で自身の報酬を最大化。報酬が同一なら、それは **完全に協力的**。ゼロサムなら、それは **対抗的**。混合なら、**一般サム**。

**中核的課題：**

- **非定常性。** エージェント `i` のビューから `P(s' | s, a_i)` は `π_{-i}` に依存し、変わっている。
- **信用配分。** 共有報酬で、どのエージェントがそれを起こしたか？
- **探索調整。** エージェントは補完的戦略を探索、同じ状態を冗長に探索しない。
- **スケーラビリティ。** 関節アクション空間は `n` で指数的に成長。
- **部分的観測性。** 各エージェントはそれ自身の観測のみ見る；グローバル状態は隠れている。

**4つの支配的レジーム：**

**1. 独立Q-learning / 独立PPO (IQL、IPPO)。** 各エージェントは自身のQ またはポリシーを学習、他を環境の一部として扱う。シンプル、時々それは機能（特に経験リプレイが滑らかなエージェントモデリングトリックとして機能）。理論的収束：なし。実践：よく疎結合タスク、悪い密結合タスク。

**2. 一元化訓練、分散実行 (CTDE)。** 最も一般的な最新パラダイム。各エージェントは自身の*ポリシー* `π_i` を持つ、ローカル観測 `o_i` を条件に — 配置での標準分散実行。*訓練*中、一元化クリティック `Q(s, a_1, …, a_n)` はグローバル状態と関節アクション条件。例：
- **MADDPG** (Lowe et al. 2017)：一元化クリティック付きDDPG、エージェント単位。
- **COMA** (Foerster et al. 2017)：反事実ベースライン — 「私が代わりにアクション `a'` を取っていたら私の報酬は何だったか」 — 私の貢献を分離。
- **MAPPO** / **IPPO** 共有クリティック付き (Yu et al. 2022)：一元化値関数を持つPPO。2026年のマルチエージェント協力で支配的。
- **QMIX** (Rashid et al. 2018)：値分解 — `Q_tot(s, a) = f(Q_1(s, a_1), …, Q_n(s, a_n))` 単調混合付き。

**3. 自己プレイ。** 同じエージェントの2つのコピーが互いにプレイ。対手のポリシー *は* 過去スナップショットからのポリシー。AlphaGo / AlphaZero / MuZero。OpenAI Five。ゼロサムゲームで最も機能；訓練信号は対称。

**4. リーグプレイ。** 一般サム / 対抗環境への自己プレイの拡張：過去と現在のポリシーのポピュレーション保つ、リーグからサンプル対手、それに対して訓練。エクスプロイター（現在のベストに打ち負かすのに特化）と主エクスプロイター（エクスプロイターに打ち負かすのに特化）を追加。AlphaStar (StarCraft II)。ゲームが「ロック・ペーパー・はさみ」戦略サイクルを認めるときに必要。

**通信。** エージェントが互いに学習メッセージ `m_i` を送信することを許す。協力設定で機能。Foerster et al. (2016) は、微分可能なエージェント間通信をエンドツーエンド訓練できることを示した。今日のLLMベースのマルチエージェントシステム (Phase 16) は本質的に自然言語で通信。

## 構築する

このレッスンは6×6 GridWorldを2つの協力エージェントで使う。彼らは反対隅から開始し、共有ゴールに到達する必要。共有報酬：いずれかのエージェントがまだ移動している間 `-1` ステップあたり、両方が到達したときに `+10`。`code/main.py` を参照。

### ステップ1：マルチエージェント環境

```python
class CoopGridWorld:
    def __init__(self):
        self.size = 6
        self.goal = (5, 5)

    def reset(self):
        return ((0, 0), (5, 0))  # two agents

    def step(self, state, actions):
        a1, a2 = state
        new1 = move(a1, actions[0])
        new2 = move(a2, actions[1])
        done = (new1 == self.goal) and (new2 == self.goal)
        reward = 10.0 if done else -1.0
        return (new1, new2), reward, done
```

*関節* アクション空間は `|A|² = 16`。グローバル状態は2つの位置。

### ステップ2：独立Q-learning

各エージェントは関節状態をキーとしたQ-テーブルを実行。各ステップで：両方がε-グリーディアクション選択、関節遷移収集、各自が共有報酬で自身のQを更新。

```python
def independent_q(env, episodes, alpha, gamma, epsilon):
    Q1, Q2 = defaultdict(default_q), defaultdict(default_q)
    for _ in range(episodes):
        s = env.reset()
        while not done:
            a1 = epsilon_greedy(Q1, s, epsilon)
            a2 = epsilon_greedy(Q2, s, epsilon)
            s_next, r, done = env.step(s, (a1, a2))
            target1 = r + gamma * max(Q1[s_next].values())
            target2 = r + gamma * max(Q2[s_next].values())
            Q1[s][a1] += alpha * (target1 - Q1[s][a1])
            Q2[s][a2] += alpha * (target2 - Q2[s][a2])
            s = s_next
```

この課題で機能するのは報酬が密で整列しているから。密結合タスク（例えば、1つのエージェントが他を*待つ*必要な場所）では失敗。

### ステップ3：分解値更新を持つ一元化Q

関節アクション上の1つのQ `Q(s, a_1, a_2)` 使う。共有報酬から更新。実行で分散化する周辺化によって：`π_i(s) = argmax_{a_i} max_{a_{-i}} Q(s, a_1, a_2)`。指数関数的関節アクション空間を*正確な*グローバルビューとトレード。

### ステップ4：シンプル自己プレイ（対抗2エージェント）

同じエージェント、2つのロール。エージェントAをエージェントBに対して訓練；`K` エピソード後、AのウェイトをBにコピー。対称訓練、一貫した進捗。AlphaZeroレシピをミニチュア。

## 落とし穴

- **非定常リプレイ。** 独立エージェントの経験リプレイはシングルエージェントより悪い。なぜなら古い遷移は今は時代遅れの対手によって生成されたから。修正：リラベル、または最近性で重み付け。
- **信用配分曖昧性。** 長いエピソード後に共有報酬；どのエージェント貢献したか言う明確な方法なし。修正：反事実ベースライン (COMA)、または単位エージェント報酬シェーピング。
- **ポリシードリフト / 追跡。** 各エージェントの最良応答は他のそれぞれの更新で変わる。修正：一元化クリティック、低い学習率、または1つずつ凍結。
- **調整を通した報酬ハッキング。** エージェント設計者が予期しなかった調整利用を見つける。オークションエージェントはゼロ入札に収束。修正：慎重な報酬設計、行動制約。
- **探索冗長性。** 両方のエージェントは同じ状態アクションペアを探索。修正：単位エージェントエントロピーボーナス、またはロール条件付け。
- **リーグサイクル。** 純自己プレイは優位サイクルで立ち往生できる。修正：多様な対手プール (リーグプレイ、Lesson 10)。
- **サンプル爆発。** `n` エージェント × 状態空間 × 関節アクション。関数近似で近似；分解アクション空間 (エージェント単位ポリシー出力ヘッド)。

## 使う

2026年のMARLアプリケーションマップ：

| ドメイン | 方法 | ノート |
|--------|--------|-------|
| 協力ナビゲーション / 操作 | MAPPO / QMIX | CTDE；共有クリティック + 分散アクター。 |
| 2プレイヤーゲーム（チェス、囲碁、ポーカー） | MCTS付き自己プレイ (AlphaZero) | ゼロサム；対称訓練。 |
| 複雑マルチプレイヤー (Dota、StarCraft) | リーグプレイ + 模倣事前訓練 | OpenAI Five、AlphaStar。 |
| 自動車フリート | 注意を持つCTDE MAPPO / PPO | 部分観測；変数チームサイズ。 |
| オークション市場 | ゲーム理論平衡 + RL | `n` → ∞ のとき平均場RL。 |
| LLMマルチエージェントシステム (Phase 16) | 自然言語通信 + ロール条件付け | RL ループはエージェント計画レイヤーに。 |

2026年で、MARLの最大成長領域はLLMベース：言語モデルエージェント群が交渉、討論、ソフトウェア構築。RLはトークンレベルではなく、*軌跡レベル* 出力での優先度最適化として表示 (Phase 16 · 03)。

## 演習

1. **簡単。** 独立Q-learningを2エージェント協力GridWorldで訓練。平均リターン > 0 までの何エピソード？関節学習曲線をプロット。
2. **中程度。** 「調整」タスク追加：ゴールは両方のエージェントが同じターンで段上にステップするときのみ到達。独立Qはまだ収束？何が破裂？
3. **難しい。** 調整タスク上でマップポスタイル訓練のための一元化クリティックを実装して収束速度を独立PPOと比較。

## 重要用語

| 用語 | 人が言うこと | 実際の意味 |
|------|-----------------|-----------------------|
| マルコフゲーム | "マルチエージェントMDP" | `(S, A_1, …, A_n, P, R_1, …, R_n)`；各エージェントは自身の報酬。 |
| CTDE | "一元化訓練、分散実行" | 訓練時の関節クリティック；各エージェントのポリシーはローカル観測のみ使う。 |
| IPPO | "独立PPO" | 各エージェントは別々にPPO実行。シンプルベースライン；しばしば過小評価。 |
| MAPPO | "マルチエージェントPPO" | 一元化値関数で条件付けPPO、グローバル状態。 |
| QMIX | "単調値分解" | `Q_tot = f_monotone(Q_1, …, Q_n)` は分散 argmax を許す。 |
| COMA | "反事実マルチエージェント" | アドバンテージ = 私のQ、マイアクション上周辺化期待Qを引いた。 |
| 自己プレイ | "エージェント対過去自己" | シングルエージェント、2ロール；ゼロサムゲームの標準。 |
| リーグプレイ | "ポピュレーション訓練" | 過去ポリシーキャッシュ、プール見本対手；戦略サイクル処理。 |
