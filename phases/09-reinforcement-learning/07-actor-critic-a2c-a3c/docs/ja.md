# アクター・クリティック — A2C と A3C

> REINFORCE はノイズが多い。`V̂(s)` を学習する批評家を追加して、リターンから差し引くと、同じ期待値を持つが分散がはるかに低いアドバンテージが得られる。それがアクター・クリティック。A2C は同期的に実行; A3C はスレッド全体で実行。どちらも、あらゆる現代的な深い RL メソッドの精神モデルです。

**タイプ:** Build
**言語:** Python
**前提条件:** Phase 9 · 04 (TD 学習)、Phase 9 · 06 (REINFORCE)
**所要時間:** ~75分

## 問題

Vanilla REINFORCE は機能しますが、その分散は恐ろしい。モンテカルロ リターン `G_t` はエピソード間で 10 倍もの幅で変動する可能性があります。そのノイズを `∇ log π` で乗算して平均化すると、ポリシーを同じ距離で移動するのに遠くない DQN 更新で移動できるのに、数千のエピソードが必要な勾配推定量が生じます。

分散は生のリターンを使用することから生じます。ベースライン `b(s_t)` を差し引く場合 — 学習された価値を含む、状態の任意の関数 — 期待値は変わらず、分散は低下します。最適なトラクタブルなベースラインは `V̂(s_t)` です。現在、`∇ log π` を乗算する量は*アドバンテージ*です:

`A(s, a) = G - V̂(s)`

行動は、平均を上回るリターンを生み出した場合は良好。下回った場合は悪い。学習された批評家を備えた REINFORCE は*アクター・クリティック*。批評家は俳優に低分散の教師を与えます。これは 2015 年以降のあらゆる深いポリシー メソッド (A2C、A3C、PPO、SAC、IMPALA) です。

## コンセプト

![アクター・クリティック: ポリシーネット + 価値ネット、TD残差をアドバンテージとして](../assets/actor-critic.svg)

**2 つのネットワーク、1 つの共有損失:**

- **俳優** `π_θ(a | s)`: ポリシー。行動するためにサンプリング。ポリシー勾配でトレーニング。
- **批評家** `V_φ(s)`: 状態からの期待リターンを推定します。`(V_φ(s) - target)²` を最小化するようにトレーニングされます。

**アドバンテージ。** 2 つの標準的な形式:

- *MC アドバンテージ:* `A_t = G_t - V_φ(s_t)`。不偏、高分散。
- *TD アドバンテージ:* `A_t = r_{t+1} + γ V_φ(s_{t+1}) - V_φ(s_t)`。バイアス (使用 `V_φ`)、はるかに低い分散。また、*TD 残差* `δ_t` とも呼ばれます。

**n ステップ アドバンテージ。** 2 つの間を補間:

`A_t^{(n)} = r_{t+1} + γ r_{t+2} + … + γ^{n-1} r_{t+n} + γ^n V_φ(s_{t+n}) - V_φ(s_t)`

`n = 1` は純粋な TD。`n = ∞` は MC。ほとんどの実装は Atari に `n = 5`、MuJoCo の PPO に `n = 2048` を使用します。

**一般化アドバンテージ推定 (GAE)。** Schulman et al. (2016) は、すべての n ステップ アドバンテージにわたる指数加重平均を提案しました:

`A_t^{GAE} = Σ_{l=0}^{∞} (γλ)^l δ_{t+l}`

`λ ∈ [0, 1]` 付き。`λ = 0` は TD (低分散、高バイアス)。`λ = 1` は MC (高分散、不偏)。`λ = 0.95` は 2026 年のデフォルト — バイアス/分散ダイアルがあなたが望むところになるまでチューニング。

**A2C: 同期的なアドバンテージ アクター・クリティック。** `N` 個の並列環境で `T` ステップを収集します。各ステップのアドバンテージを計算します。統合バッチで俳優と批評家を更新します。繰り返します。A3C のより単純でスケーラブルなシブリング。

**A3C: 非同期的なアドバンテージ アクター・クリティック。** Mnih et al. (2016)。`N` 個のワーカースレッドを生成します。各ワーカーは自独立したロールアウトで局所的に勾配を計算し、非同期的に共有パラメータサーバーに適用します。リプレイ バッファは必要ありません — ワーカーは異なる軌跡を実行することで相関を取ります。A3C は CPU でスケールでトレーニングできることを証明しました。2026 年では、GPU ベースの A2C (バッチされた並列環境) が支配的です。GPU は大きなバッチを求めているため。

**結合損失。**

`L(θ, φ) = -E[ A_t · log π_θ(a_t | s_t) ]  +  c_v · E[(V_φ(s_t) - G_t)²]  -  c_e · E[H(π_θ(·|s_t))]`

3 つの項: ポリシー勾配損失、価値回帰、エントロピー ボーナス。`c_v ~ 0.5`、`c_e ~ 0.01` は標準的な開始点です。

## 構築

### ステップ 1: 批評家

線形批評家 `V_φ(s) = w · features(s)` を MSE で更新:

```python
def critic_update(w, x, target, lr):
    v_hat = dot(w, x)
    err = target - v_hat
    for j in range(len(w)):
        w[j] += lr * err * x[j]
    return v_hat
```

表形式環境では、批評家は数百のエピソードで収束します。Atari では、線形批評家を共有 CNN トランク + 価値ヘッドに置き換えます。

### ステップ 2: n ステップ アドバンテージ

長さ `T` のロールアウトとブートストラップされた最終 `V(s_T)` が与えられた場合:

```python
def compute_advantages(rewards, values, gamma=0.99, lam=0.95, last_value=0.0):
    advantages = [0.0] * len(rewards)
    gae = 0.0
    for t in reversed(range(len(rewards))):
        next_v = values[t + 1] if t + 1 < len(values) else last_value
        delta = rewards[t] + gamma * next_v - values[t]
        gae = delta + gamma * lam * gae
        advantages[t] = gae
    returns = [a + v for a, v in zip(advantages, values)]
    return advantages, returns
```

`returns` は批評家のターゲットです。`advantages` は `∇ log π` を乗算するものです。

### ステップ 3: 結合更新

```python
for step_i, (x, a, _r, probs) in enumerate(traj):
    adv = advantages[step_i]
    target_v = returns[step_i]

    # critic
    critic_update(w, x, target_v, lr_v)

    # actor
    for i in range(N_ACTIONS):
        grad_logpi = (1.0 if i == a else 0.0) - probs[i]
        for j in range(N_FEAT):
            theta[i][j] += lr_a * adv * grad_logpi * x[j]
```

オンポリシー、ロールアウトごとに 1 つの更新、俳優と批評家の学習率を分離。

### ステップ 4: 並列化 (A3C vs A2C)

- **A3C:** `N` 個のスレッドを起動します。各は自環境と自前方パスを実行します。定期的に勾配更新をマスターに共有にプッシュします。マスターにはロックはありません — レースは大丈夫、それらはノイズを追加するだけです。
- **A2C:** 単一プロセスで `N` の環境インスタンスを実行し、観測値を `[N, obs_dim]` バッチにスタックし、バッチされた前方パス、バッチされた後方パス。より高い GPU 利用率、決定論的、推論しやすい。2026 年のデフォルト。

私たちのおもちゃのコードは明確さのために単一スレッドです; バッチされた A2C への書き直しは numpy の 3 行です。

## 落とし穴

- **アクター勾配の前の批評家バイアス。** 批評家がランダムな場合、ベースラインは非情報的で、純粋なノイズについてトレーニングしています。ポリシー勾配をオンにする前に、数百ステップの批評家をウォームアップするか、遅い俳優学習率を使用します。
- **アドバンテージ正規化。** アドバンテージをバッチごとにゼロ平均/ユニット std に正規化します。トレーニングをほぼ０ のコストで大規模に安定化させます。
- **共有トランク。** 画像入力で俳優と批評家に共有フィーチャー抽出器を使用します。別々のヘッド。共有フィーチャーは両方の損失に相乗りします。
- **オンポリシー契約。** A2C はデータを正確に 1 つの更新で再利用します。それ以上ですと、勾配がバイアスされます (重要度サンプリング補正が PPO で追加するもの)。
- **エントロピー崩壊。** `c_e > 0` なしで、ポリシーは数百回の更新で近似決定論的になり、探索を停止します。
- **報酬スケール。** アドバンテージの大きさは報酬スケールに依存します。報酬を正規化します (例えば、実行中の std 分割) タスク全体で一定の勾配の大きさのため。

## 使用

A2C/A3C は 2026 年で最後の選択ですが、それらはアーキテクチャです。その後のあらゆるもの改良:

| メソッド | A2C への関係 |
|--------|----------------|
| PPO | A2C + マルチエポック更新のためのクリップされた重要度比 |
| IMPALA | A3C + V-trace オフポリシー補正 |
| SAC (Phase 9 · 07) | オフポリシー A2C with soft-value 批評家 (次のレッスン) |
| GRPO (Phase 9 · 12) | 批評家なしの A2C — グループ相対アドバンテージ |
| DPO | A2C は好み順位付け損失に折り畳まれた、サンプリングなし |
| AlphaStar / OpenAI Five | リーグトレーニング + 模倣事前トレーニングを備えた A2C |

2026 年の論文で "advantage" を見たら、アクター・クリティック について考えます。

## 演習

1. **簡単。** MC アドバンテージ (`G_t - V(s_t)`) で 4×4 GridWorld にアクター・クリティックをトレーニン グします。レッスン 06 の実行平均ベースラインを備えた REINFORCE へのサンプル効率を比較します。
2. **中程度。** TD 残差アドバンテージ (`r + γ V(s') - V(s)`) に切り替えます。アドバンテージバッチの分散を測定します。どの程度低下しますか?
3. **難しい。** GAE(λ) を実装します。`λ ∈ {0, 0.5, 0.9, 0.95, 1.0}` をスイープします。最終的なリターン対サンプル効率をプロット します。このタスクでのバイアス/分散スイートスポットはどこですか?

## キーワード

| 用語 | 人々が言う | 実際の意味 |
|------|-----------------|-----------------------|
| 俳優 | "ポリシー ネット" | `π_θ(a\|s)`、ポリシー勾配で更新。 |
| 批評家 | "価値ネット" | `V_φ(s)`、リターン / TD ターゲットへの MSE 回帰で更新。 |
| アドバンテージ | "平均よりどの程度良いか" | `A(s, a) = Q(s, a) - V(s)` またはその推定量。`∇ log π` の乗算。 |
| TD 残差 | "δ" | `δ_t = r + γ V(s') - V(s)`; ワンステップ アドバンテージ推定。 |
| GAE | "補間ノブ" | n ステップ アドバンテージの指数加重合計、`λ` でパラメータ化。 |
| A2C | "同期的なアクター・クリティック" | 環境全体でバッチ; ロールアウトごとに 1 つの勾配ステップ。 |
| A3C | "非同期的なアクター・クリティック" | ワーカースレッドは共有パラム サーバーに勾配をプッシュします。元の論文; 2026 年ではあまり一般的ではありません。 |
| ブートストラップ | "水平線で V を使用" | ロールアウトを切り詰める、γ^n V(s_{t+n}) を追加して合計を閉じます。 |
