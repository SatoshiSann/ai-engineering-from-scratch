# バッチ API — 50% 割引は業界標準

> あらゆるメジャープロバイダー は約 24 時間ターンアラウンドで 50% 割引を使用した非同期バッチ API を出荷。OpenAI、Anthropic、Google、推論プラットフォーム（Fireworks バッチティア、Together バッチ）のほとんどは同じパターンを実装。バッチをプロンプト キャッシング とスタックし、オーバーナイト パイプラインは 同期キャッシュなし コストの 約 10%に低下。ルール は無慈悲にシンプル：インタラクティブ でなけれ ば、バッチで属する。コンテンツ生成 パイプライン、ドキュメント 分類、データ抽出、レポート生成、バルク ラベリング、カタログ タグ付け — 24 時間 レイテンシ を許容するものはすべて、バッチへの移動までテーブルに金銭が残る。2026 年本番 パターン は あらゆる新しい LLM ワークロード を 3 つのレーンへトリアージ：インタラクティブ（同期 キャッシング付き）、セミインタラクティブ（非同期キュー フォールバック付き）、バッチ（オーバーナイト、キャッシュ入力スタック付き）。インタラクティブを装うが分 のレイテンシ を許容するワークロード は ほとんどを浪費。

**タイプ:** Learn
**言語:** Python (stdlib、おもちゃのバッチ対同期 コストシミュレータ)
**前提条件:** Phase 17 · 14（プロンプト & セマンティック キャッシング）
**所要時間:** 約 45 分

## 学習目標

- 3 つのプロバイダー バッチ API（OpenAI、Anthropic、Google）と一般的 50% 割引 + 24h ターンアラウンド 保証を名前で述べてください。
- オーバーナイト 分類 ワークロード で バッチ + キャッシュ入力の コストを計算 し、同期キャッシュなし ベースラインと比較してください。
- ワークロードを インタラクティブ / セミインタラクティブ / バッチ へトリアージ し、レーンを正当化してください。
- 2 つの罠を名前で述べてください：部分的 インタラクティブ（ユーザーは 24h より早いを予想）と出力スキーマ ドリフト（バッチファイル フォーマット はプロバイダー単位で異なる）。

## 問題

チーム はオーバーナイト レポート生成 パイプライン を出荷。50,000 ドキュメント、各々を要約、要約をクラスタ化、エグゼクティブ ブリーフを作成。同期的に実行 すると 4 時間 $2,000/夜。バッチ API について聞く。

バッチ は 50% オフを得ます。また システム プロンプト（すべての 50k 呼び出し全体で共有）でプロンプト キャッシング を有効。スタックされた、請求書は $180/夜に低下 — ベースラインの 約 9%。同じ パイプライン、3 つの設定変更。

バッチ は LLM コストツールキット で最安 レバー で、誰も引っ張らない。理由 は主に組織的：チーム は「リアルタイム」を考える とき、SLA 実は「朝までに」。このレッスンは 90% 請求書をテーブルに残してないことについて。

## コンセプト

### 3 つのバッチ API

**OpenAI Batch API**：リクエスト リスト付き JSONL ファイル アップロード。約 24 時間ターンアラウンド 約束（実践で通常 2-8 時間）。入力 と出力 トークン で 50% 割引。`/v1/batches` エンドポイント。キャッシュ適格入力 もキャッシュ入力価格設定 上に得られます。

**Anthropic Message Batches**：JSONL アップロード。24 時間ターンアラウンド。50% 割引。`cache_control` をサポート — キャッシュ書き込み は明示的、読み込みはバッチ内に自動で発生。

**Google Vertex AI Batch Prediction**：BigQuery または GCS 入力。Gemini の同様 50% 割引。Vertex パイプライン と統合。

### セマンティック：非同期、遅くない

バッチ は「24 時間以内に返却を約束」— 「これは 24 時間かかります」ではない。典型的 P50 は 2-6 時間。プロバイダー はGPU インベントリ が不完全利用中はオフピーク ウィンドウ中にバッチ をスケジュール。

### キャッシング とスタック

同じ 4K トークン システム プロンプト 付き 50k ドキュメント 要約：

- 同期キャッシュなし：50000 × ($入力 × 4000 + $出力 × 200）フルレートで。
- 同期 キャッシュ：最初の書き込み後 システム プロンプト はキャッシュされた；残り 49999 は 10 倍チープ入力を得ます。
- バッチ キャッシュ：上記すべてプラス 読み込みと 書き込みの両方で 50% 割引。

スタック：バッチ + キャッシュ = 同期 キャッシュなし 請求書の 約 10%。オーバーナイト 実行し、共有 システム プロンプト を持つ あらゆるワークロード はこれを使用すべき。

### ワークロード トリアージ

**インタラクティブ** — ユーザーはレスポンス を待つ。TTFT が重要。プロンプト キャッシング 付き 同期 呼び出し。バッチ化 できない。

**セミインタラクティブ** — ユーザー はタスク を送信、分 でチェック バック。非同期キュー フォールバック付き 同期 へ。中程度 ボリューム RAG インデックス化を考える。

**バッチ** — ユーザー は「朝までに」または「次の時間」を予想する結果。コンテンツ パイプライン、スケール分類、オフラインanalysis。常にバッチ、常にキャッシング スタック。

一般的 誤り：パイプライン が本番環境 だからすべてをインタラクティブ として分類。本番環境 は レイテンシ spec ではない — SLA はそうだ。

### 部分的インタラクティブ罠

機能 はインタラクティブ のようだが 5-10 分を許容。例：「リフレッシュ」ボタン 付き オーバーナイト 顧客健康 レポート。ユーザーはリフレッシュ をクリック；10 分待つ は良い。チーム は同期 として出荷。50 個同時 リフレッシュ はバッチおよび電子メール経由で提供されるもの が要するコストの 10 倍。

尋ねるべき質問：「このユーザーに 24 時間は何を意味しますか？」答えが「彼らは気付かない」の場合、バッチ化。

### 出力スキーマ 罠

バッチファイル形式 はプロバイダー単位で異なる：

- OpenAI：JSONL、1 行 1 リクエスト。
- Anthropic：JSONL、1 行 1 メッセージ；レスポンス フォーマット 埋め込まれた。
- Vertex：BigQuery テーブル または GCS プレフィックス TFRecord 付き。

「1 つのバッチ クライアント」 をプロバイダー 全体で書く はプロバイダー単位 アダプタ コード を意味。複数プロバイダー バッチ（Portkey、LiteLLM 一部ティア）を広告するゲートウェイ はまだ 生のフォーマット をシンラッピング。

### 覚えるべき数字

- プロバイダー 全体バッチ割引：入力 + 出力で 50% フラット。
- ターンアラウンド SLA：24 時間保証、2-6 時間典型的 P50。
- スタック バッチ + キャッシュ入力：同期 キャッシュなし コストの 約 10%。
- ワークロード トリアージ ルール：24h レイテンシ が容認される 場合、常にバッチ。

## 使用方法

`code/main.py` は 50k ドキュメント ワークロード 全体で同期、同期+キャッシュ、バッチ、バッチ+キャッシュ でコスト を計算。$ と パーセント での 節約 を報告。

## 出荷方法

このレッスンは `outputs/skill-batch-triager.md` を生成します。ワークロード 特性 が与えられた場合、インタラクティブ/セミ/バッチ へトリアージ し、節約 を推定。

## 演習

1. `code/main.py` を実行。100k ドキュメント パイプライン で 3K トークン システム プロンプト と 500 トークン 出力、フル スタック（バッチ + キャッシュ）対 同期 ベースラインの 節約 を計算。
2. 知っている 実製品 から 3 つの機能 を選択。各々をインタラクティブ/セミ/バッチ へトリアージ。
3. ユーザーが 彼らのレポート が 3 時間 を取ったと不平。それは バッチ ミストリアージ または 正当な インタラクティブ か？決定 基準を書く。
4. バッチ API 返却 SLA は 24h だが P99 は 20 時間。ユーザー へこれを どう伝える — エッジケース での下流 システム 動作 は何か？
5. Break-even を計算：独自 予約 GPU でオーバーナイト 実行より バッチ + キャッシュ がチープ になる共有接頭辞 長さ はどこか？

## 主要用語

| 用語 | 人々の言い方 | 実際の意味 |
|------|----------------|------------------------|
| バッチ API | "非同期割引" | 50% オフ 24h ターンアラウンド 付き |
| JSONL | "バッチ形式" | 1 行 1 JSON リクエスト；OpenAI/Anthropic 標準 |
| Message Batches | "Anthropic バッチ" | Anthropic のバッチ API 製品名 |
| Batch prediction | "Vertex バッチ" | Vertex AI のバッチ API 製品 |
| ターンアラウンド SLA | "24h 約束" | 保証、典型的 ではない；典型的 は 2-6h |
| ワークロード トリアージ | "インタラクティブ決定" | インタラクティブ / セミ / バッチ ルーティング 決定 |
| 出力スキーマ | "レスポンス フォーマット" | プロバイダー単位 JSONL レイアウト；ポータブル でない |
| スタック 割引 | "バッチ + キャッシュ" | キャッシュなし 同期 請求書の 約 10%（両方適用時） |
