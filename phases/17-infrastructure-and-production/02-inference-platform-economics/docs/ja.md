# 推論プラットフォーム経済学 — Fireworks、Together、Baseten、Modal、Replicate、Anyscale

> 2026 年の推論市場はもはや GPU タイム レンタルではありません。カスタムシリコン（Groq、Cerebras、SambaNova）、GPU プラットフォーム（Baseten、Together、Fireworks、Modal）、API ファースト マーケットプレイス（Replicate、DeepInfra）に二分化。Fireworks は 2026 年 5 月 1 日に 1 GPU 当たり $1/時間の価格引き上げ、1 日 10T+ トークンの $4B 評価額は、ボリューム駆動モデルが機能することを示しています。Baseten は 2026 年 1 月 $5B で Series E を $300M 調達。競争的ポジショニングルールは単純：Fireworks は遅延を最適化、Together はカタログ幅を最適化、Baseten はエンタープライズ品質を最適化、Modal は Python ネイティブ DX を最適化、Replicate はマルチモーダルに対応、Anyscale は分散 Python を最適化。このレッスンは創業者に渡せるマトリクスを提供。

**タイプ:** 学習
**言語:** Python（stdlib、簡単なコール当たりの経済学比較ツール）
**前提条件:** フェーズ 17 · 01（マネージド LLM プラットフォーム）、フェーズ 17 · 04（vLLM サービング インターナル）
**所要時間:** 約 60 分

## 学習目標

- 3 つのマーケットセグメント（カスタムシリコン、GPU プラットフォーム、API ファースト）の名前を挙げ、各ベンダーをセグメントにマップ。
- 「トークン当たり」API 価格設定モデルがハードウェアのコストカーブへでなく、サービングエンジンのコストカーブへ圧縮される理由を説明。
- 少なくとも 3 ベンダー間でコール当たりの実効コストを計算し、トークン当たり（Baseten、Modal）がいつ 1 分当たりを上回るかを説明。
- 所定のワークロード（サーバーレス バースト、高スループット定常、ファインチューン変種、マルチモーダル）に対して適切なデフォルトプラットフォームを識別。

## 問題

マネージド ハイパースケーラー プラットフォームを評価。より狭く高速なプロバイダーが必要と判断 — 遅延用 Fireworks、幅用 Together、ファインチューンカスタムモデル用 Baseten。6 つの実際の選択肢があり、価格ページが揃っていません。Fireworks は $/M トークン表示。Baseten は $/分。Modal は $/秒。Replicate は $/予測。ワークロードをモデル化なしで頭対頭の比較はできません。

さらに悪いことに、各価格ページの背後のビジネスモデルが異なります。Fireworks は独自のカスタムエンジン（FireAttention）を共有 GPU で実行。トークン当たりレートは利用率曲線を反映。Baseten は Truss + 専用 GPU を提供。分当たりレートは排他性を反映。Modal は真の Python サーバーレス — サブ秒コールドスタート付きの秒単位請求。同じ出力（LLM 応答）、3 つの異なるコスト関数。

このレッスンは 6 つをモデル化し、各々がいつ勝つかを示します。

## コンセプト

### 3 つのセグメント

**カスタムシリコン** — Groq（LPU）、Cerebras（WSE）、SambaNova（RDU）。通常 GPU ベースクラスタより 5-10 倍高速デコード（同じモデル）。トークン当たり価格高い（Groq は 2025 年後期に Llama-70B で約 $0.99/M）ですが、遅延に敏感なユースケース。Groq は音声エージェント、リアルタイム翻訳用の本番ピック。

**GPU プラットフォーム** — Baseten、Together、Fireworks、Modal、Anyscale。NVIDIA（H100、H200、B200 in 2026）または時々 AMD で実行。「ロー GPU レンタル」（RunPod、Lambda）と「ハイパースケーラー マネージドサービス」（Bedrock）の経済レイヤー。

**API ファースト マーケットプレイス** — Replicate、DeepInfra、OpenRouter、Fal。広いカタログ、コール当たり支払い或いは秒当たり支払い、初回コール時間を強調。

### Fireworks — 遅延最適化 GPU プラットフォーム

- FireAttention エンジン（カスタム）。同等設定で vLLM より 4 倍低遅延と宣伝。
- バッチティア：非インタラクティブワークロード用、サーバーレスレートの約 50%。
- ファインチューンモデルが基本モデルと同じレートで配信 — LoRA に対してプレミアム請求するプロバイダーとの実際の差別化要因。
- 2026 年中盤：2026 年 5 月 1 日時点で有効なオンデマンド GPU レンタル $1/時間値上げ。ボリューム価格はスケールで交渉可能。
- 財務シグナル：$4B 評価額、1 日 10T+ トークン処理。

### Together — 幅最適化

- 200+ モデル、アップストリーム公開の数日以内のオープンソースリリース含む。
- Replicate 同等品の 50-70% 安い — 「AI ネイティブクラウド」ポジショニングはボリュームとカタログ。
- 推論 + ファインチューニング + 1 つの API での訓練。

### Baseten — エンタープライズ品質最適化

- Truss フレームワーク：1 つのマニフェストでの依存性、シークレット、サービング設定を持つモデルパッケージング。
- T4 を通じて B200 までの GPU 範囲。リーズナブルなコールドスタート軽減付きの分単位請求。
- SOC 2 Type II、HIPAA 対応。一般的なフィンテック、医療ピック。
- $5B 評価額、2026 年 1 月 Series E（$300M from CapitalG、IVP、NVIDIA）。

### Modal — Python ネイティブ最適化

- インフラストラクチャ・アズ・コード（純粋 Python）。`@modal.function(gpu="A100")` で関数を装飾、1 コマンドでデプロイ。
- 秒単位請求。2-4 秒のコールドスタート、事前ウォーミング付き。小さいモデル向け < 1s。
- $87M Series B at $1.1B 評価額（2025）。独立したサーベイで最強の開発者体験スコア。

### Replicate — マルチモーダル幅

- コール当たり支払い。画像、ビデオ、オーディオモデルのデフォルトプラットフォーム。
- 統合エコシステム（Zapier、Vercel、CMS プラグイン）。
- LLM トークン当たりレートでは競争力が低いですが、マルチモーダル多様性で勝利。

### Anyscale — Ray ネイティブ

- Ray 上に構築。RayTurbo は Anyscale の独自推論エンジン（vLLM と競争）。
- 推論ステップがより大きなグラフの 1 つのノードである分散 Python ワークロードに最適。
- マネージド Ray クラスタ。Ray AIR、Ray Serve との密接な統合。

### トークン当たり vs 分当たり — 各々がいつ勝つか

トークン当たりは遅延に敏感でないバースト ワークロードで意味があります — 使用したものに対してのみ支払い。分当たりは利用率が高く予測可能なときに意味があります — GPU を飽和させると、トークン当たりを上回ります。

大まかなルール：専用 GPU の約 30% 以上の持続的利用率のワークロードでは、分当たり（Baseten、Modal）がトークン当たり（Fireworks、Together）を上回り始めます。それ以下では、アイドル支払いを避けるため、トークン当たりが勝つ。

### カスタムエンジンが実際のお堀

vLLM と SGLang の上のすべてのプラットフォームはカスタムエンジンを主張。FireAttention、RayTurbo、Baseten の推論スタック。カスタムエンジン主張はマーケティングの影。正直なフレーミングは vLLM + SGLang が本番オープンソース推論の約 80% を表し、プラットフォームレイヤーでの差別化要因は DX、属性、SLA です。

### 覚えるべき数値

- Fireworks GPU レンタル：2026 年 5 月 1 日時点で有効 $1/時間値上げ。
- Fireworks 主張：同等設定で vLLM より 4 倍低遅延。
- Together：LLM で Replicate より 50-70% 安い。
- Baseten 評価額：$5B（Series E、2026 年 1 月、$300M ラウンド）。
- Modal 評価額：$1.1B（Series B、2025）。
- 分当たりがトークン当たりを上回る：約 30% 以上の持続的利用率。

## 使用

`code/main.py` は 6 つのベンダーを価格モデル全体で合成ワークロード上で比較。$/日、実効 $/M トークンを報告。実行して、トークン当たりと分当たりの損益分岐点を見つけてください。

## 配布

このレッスンは `outputs/skill-inference-platform-picker.md` を生成。ワークロード プロファイル、SLA、予算を考えると、主推論プラットフォームを選択し、ランナーアップに名前を付けます。

## 演習

1. `code/main.py` を実行。1 つの H100 上の 70B モデルで、Baseten（分当たり）が Fireworks（トークン当たり）を上回る持続的利用率はどのくらい？自分で損益分岐点を導出し、経験則と比較。
2. 製品は画像生成 + チャット + 音声・テキスト・変換を配信。各モダリティ用にプラットフォームを選択し、それらを統合するゲートウェイパターンに名前を付け。
3. Fireworks が主モデルの価格を $1/時間引き上げ。40% のトラフィックがバッチティア（50% 割引）に移動するなら、ブレンド コスト影響をモデル化。
4. 規制顧客が SOC 2 Type II + HIPAA + 専用 GPU を要求。3 つのプラットフォームが実行可能でどれが FinOps で勝つか？
5. Llama 3.1 70B について Fireworks サーバーレス、Together オンデマンド、Baseten 専用、Replicate API で 1,000 予測当たりのコストを比較。1 日 10 予測で一番安いのは？1 日 10,000 で？

## 重要用語

| 用語 | 人々が言うこと | 実際の意味 |
|------|-------------|----------|
| カスタムシリコン | 「非 GPU チップ」 | Groq LPU、Cerebras WSE、SambaNova RDU — デコード用最適化 |
| FireAttention | 「Fireworks エンジン」 | カスタムアテンション カーネル。vLLM より 4 倍低遅延と宣伝 |
| Truss | 「Baseten フォーマット」 | モデルパッケージング マニフェスト。依存性 + シークレット + サービング設定 |
| トークン当たり | 「API 価格」 | トークン消費で請求。アイドルに支払わない |
| 分当たり | 「専用価格」 | 実時間 GPU 時間で請求。高利用率で勝つ |
| 予測当たり | 「Replicate 価格」 | モデル呼び出し当たり請求。画像/ビデオで一般的 |
| RayTurbo | 「Anyscale エンジン」 | Ray 上の独自推論。Ray クラスタで vLLM と競争 |
| バッチティア | 「50% オフ」 | 非インタラクティブキューが削減レート。Fireworks、OpenAI で一般的 |
| ファインチューンで基本レート | 「Fireworks LoRA」 | LoRA 配信リクエストを基本モデルレートで請求（差別化要因） |
