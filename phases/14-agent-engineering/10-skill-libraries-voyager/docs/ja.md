# スキルライブラリとライフロング・ラーニング（Voyager）

> Voyager（Wang et al., TMLR 2024）は実行可能なコードをスキルとして扱う。スキルは名前付き、検索可能、合成可能であり、環境フィードバックで改善される。これはClaude Agent SDKのスキル、skillkit、および2026年のスキルライブラリパターンの参照アーキテクチャである。

**タイプ:** ビルド
**言語:** Python（stdlib）
**前提条件:** Phase 14 · 07（MemGPT）、Phase 14 · 08（Letta Blocks）
**所要時間:** 約75分

## 学習目標

- Voyagerの3つの構成要素（自動カリキュラム、スキルライブラリ、反復プロンプティング）を名前付けでき、それぞれの役割を説明できる。
- Voyagerがアクション空間をプリミティブコマンドではなくコードとする理由を説明できる。
- 登録、検索、合成、失敗駆動の改善機能を備えたstdlibスキルライブラリを実装できる。
- Voyagerのパターンを2026年Claude Agent SDKのスキルとskillkitエコシステムにマッピングできる。

## 問題

毎回ゼロから機能を再構築するエージェントは3つの問題を起こす:

1. **トークンを無駄にする。** すべてのタスクで同じ推論を再現する。
2. **進歩を失う。** セッションAで学んだ修正がセッションBに移行しない。
3. **長期的な合成に失敗する。** 複雑なタスクには機能階層が必要であり、ワンショットプロンプトではそれを表現できない。

Voyagerの答え：再利用可能な機能を、ライブラリに格納された名前付きコードチャンク、相似度で検索可能、他のスキルと合成可能、実行フィードバックで改善される形で扱う。

## コンセプト

### 3つの構成要素

Voyager（arXiv:2305.16291）はエージェントを次のようにして構成する:

1. **自動カリキュラム。** 好奇心駆動の提案機が、エージェントの現在のスキルセットと環境状態に基づいて次のタスクを選ぶ。探索はボトムアップ。
2. **スキルライブラリ。** 各スキルは実行可能なコード。タスクが成功すると新しいスキルが追加される。スキルはクエリから説明への相似度で検索される。
3. **反復プロンプティング機構。** 失敗時、エージェントは実行エラー、環境フィードバック、自己検証出力を受け取り、スキルを改善する。

Minecraft評価（Wang et al., 2024）: ベースラインに比べ、ユニークアイテムが3.3倍、石のツールが8.5倍高速、鉄のツールが6.4倍高速、マップ走破が2.3倍長い。数字はMinecraft固有だが、パターンは移行する。

### アクション空間 = コード

ほとんどのエージェントはプリミティブコマンドを出力する。VoyagerはJavaScript関数を出力する。スキルは:

```
async function craftIronPickaxe(bot) {
  await mineIron(bot, 3);
  await mineStick(bot, 2);
  await placeCraftingTable(bot);
  await craft(bot, 'iron_pickaxe');
}
```

サブスキルから合成される。説明と埋め込みをキーにして格納される。プロンプトではなくプログラムとして検索される。

これが2026年Claude Agent SDKスキル：エージェントがオンデマンドで読み込む名前付きで検索可能なコードチャンク＋指示。

### スキル検索

新しいタスク「ダイヤピッケルを作る」。エージェント:

1. タスク説明を埋め込む。
2. スキルライブラリにトップkの類似スキルをクエリする。
3. `craftIronPickaxe`、`mineDiamond`、`placeCraftingTable`などを検索する。
4. 検索したプリミティブから新しいスキルを＋新しいロジックで合成する。

これはMCPリソース（Phase 13）とAgent SDKスキルが実装するパターン：知識/コード表面上の検索、現在のタスクに限定。

### 反復改善

Voyagerのフィードバックループ:

1. エージェントがスキルを書く。
2. スキルは環境に対して実行される。
3. `success`、`error`（スタックトレース付き）、`self-verification failure`の3つの信号のいずれかが返される。
4. エージェントはその信号をコンテキストとして使ってスキルを書き直す。
5. 成功またはマックスラウンドまでループ。

これはSelf-Refine（レッスン05）を環境グラウンドな検証でコード生成に適用したもの。CRITIC（レッスン05）は外部ツールを検証者として使う同じパターン。

### カリキュラムと探索

Voyagerのカリキュラム・モジュールは「湖の近くにシェルターを建てる」のようなタスクを、エージェントが持っているもの、まだやっていないことに基づいて提案する。提案機は環境状態＋スキル在庫を使って、現在の能力のちょうど上のタスク（探索の最適スポット）を選ぶ。

本番エージェント向けには、これは「何が足りないか」演算子に変わる：現在のスキルライブラリとドメインを考えると、まだカバーしていないスキルは何か？チームは通常これを手動でカリキュラムレビューとして実装する。

### このパターンがうまくいかない場合

- **スキルライブラリの老朽化。** 同じスキルが説明が少し異なる状態で10回追加される。書き込み時に重複排除を追加；検索は1つだけ返す。
- **合成スキルドリフト。** 親スキルが改善された子に依存する。スキルをバージョン管理する；v1にピンされた親はv3を魔法のように選ばない。
- **検索品質。** ライブラリがスキル数百を超えて成長するにつれて、スキル説明上のベクトル検索は低下する。タグフィルタと厳密な制約（「category=toolingのスキルのみ」）で補足。

## ビルドする

`code/main.py`はstdlibスキルライブラリを実装する:

- `Skill` — 名前、説明、コード（文字列として）、バージョン、タグ、依存関係。
- `SkillLibrary` — 登録、検索（トークン重複）、合成（依存関係のトポロジカルソート）、改善（更新時のバージョンバンプ）。
- 3つのプリミティブスキルを登録し、4番目を合成し、失敗にぶつかり、v2改善するスクリプト化されたエージェント。

実行する:

```
python3 code/main.py
```

トレースはライブラリ書き込み、検索、合成、失敗した実行、v2改善を示す—Voyagerのループ端から端まで。

## 使う

- **Claude Agent SDKスキル**（Anthropic）—2026年の参照：各スキルは説明、コード、指示を持つ；エージェントセッション中にオンデマンドで読み込まれる。
- **skillkit**（npm: skillkit）—32以上のAIコーディングエージェント向けのクロスエージェントスキル管理。
- **カスタムスキルライブラリ** — ドメイン固有（データエージェント向けSQLスキル、インフラエージェント向けTerraformスキル）。Voyagerパターンはスケールダウン。
- **OpenAI Agents SDK `tools`** — 低端；各ツールは軽量スキル。

## 出荷

`outputs/skill-skill-library.md`はVoyager型スキルライブラリを生成し、任意のターゲットランタイムに向けて登録、検索、バージョン管理、改善を接続。

## 演習

1. `compose()`に依存サイクル検出器を追加。スキルAがBに依存し、BがAに依存する場合は？エラーか警告か？
2. スキルごとのバージョンピン留めを実装。親スキルが子`crafting@1`を合成するとき、`crafting@2`への改善は親を暗黙的にアップグレードしてはならない。
3. トークン重複検索をsentence-transformers埋め込みに置き換える（またはBM25 stdlibの実装）。50スキルおもちゃライブラリでretrieval@5を測定。
4. 「カリキュラム」エージェントを追加：現在のライブラリとドメイン説明を考えると、5つの欠けているスキルを提案。週ごとに呼び出す。
5. AnthropicのClaude Agent SDKスキルドキュメントを読む。おもちゃライブラリをSDKのスキルスキーマにポート。発見可能性について何が変わるか？

## キー用語

| 用語 | 人々が言うこと | 実際に意味すること |
|------|----------------|------------------------|
| スキル | 「再利用可能な機能」 | 名前付きコードチャンク＋説明、相似度で検索可能 |
| スキルライブラリ | 「エージェントのハウツーメモリ」 | スキルの永続ストア、検索可能で合成可能 |
| カリキュラム | 「タスク提案者」 | ボトムアップ目標ジェネレータ、現在の能力ギャップで駆動 |
| 合成 | 「スキルDAG」 | スキルがスキルを呼び出す；実行時にトポロジカルソート |
| 反復改善 | 「自己修正ループ」 | 環境フィードバック＋エラー＋自己検証が次バージョンに折り込まれ |
| アクション空間コード | 「プログラマティックアクション」 | プリミティブコマンドではなく関数を出力、時間的に拡張された行動のため |
| 書き込み時に重複排除 | 「スキル崩壊」 | ほぼ重複した説明が1つの正規スキルに崩壊 |
