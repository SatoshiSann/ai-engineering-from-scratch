# Tree of ThoughtsとLATS：意図的検索

> 単一のchain-of-thoughtトラジェクトリにはバックトラックの余地がありません。ToT (Yao et al., 2023)は推論を各ノードで自己評価を備えたツリーに変えます。LATS (Zhou et al., 2024)はToT、ReAct、Reflexionをモンテカルロツリー検索下で統一します。Game of 24は4%(CoT)から74%(ToT)に上昇；LATSはHumanEvalで92.7%のpass@1に達します。

**タイプ:** ビルド
**言語:** Python (stdlib)
**前提条件:** Phase 14 · 01 (Agent Loop)、Phase 14 · 03 (Reflexion)
**所要時間:** 約75分

## 学習目標

- 推論をサーチとしてフレーミング：ノードは「思考」、エッジは「拡張」、値は「どの程度有望か」。
- 自己評価スコアリングを備えた標準ライブラリToT形式BFSツリーサーチを実装する。
- Select/Expand/Simulate/Backpropagateを備えたおもちゃLATS MCTSループに拡張する。
- 検索がトークン乗数の価値がある場合(Game of 24、コード生成)と、単一のトラジェクトリが十分である場合(シンプルなQ&A)を判断する。

## 問題

Chain-of-thoughtは線形ウォークです。最初のステップが間違っている場合、その後のすべてのステップは悪い前提で機能します。Game of 24 (4つの数字を+−×÷で24にする)では、GPT-4 CoTは4%の精度に達します。モデルは早期に間違った部分式を選択し、リカバリできません。

推論が必要とするのは、複数の候補を提案し、それらを評価し、有望なものを選択し、行き止まりが現れたときにバックトラックする能力です。それはサーチです。Tree of ThoughtsとLATSは2つの標準的な定式化です。

## コンセプト

### Tree of Thoughts (Yao et al., NeurIPS 2023)

各ノードは一貫した中間ステップ(「思考」)です。各ノードはK個の子思考に拡張できます。LLMはスコアリングプロンプトで各ノードを自己評価します。サーチはツリーを探索します - BFS、DFS、またはビーム。

```
                     (root: "find 24 from 4 6 4 1")
                    /               |            \
           ("6 - 4 = 2")    ("4 + 1 = 5")    ("4 * 6 = 24")  <- Score: HIGH
              /   \              |                  |
          ...    ...          ...                finish
```

自己評価は荷物を運ぶコンポーネントです。論文は3つのバリアントを示しています：`sure / likely / impossible`分類、`1..10`数値スコア、候補間の投票。3つすべてがCoTを実質的にGame of 24で上回ります(4% -> GPT-4で74%)。

### LATS (Zhou et al., ICML 2024)

LATSはToT、ReAct、Reflexionを MCTSの下で統一します。LLMは3つの役割を果たします：

- **ポリシー**: 候補の次のアクションを提案(ReAct形式)。
- **値関数**: 部分的なトラジェクトリをスコア化(ToT形式自己評価)。
- **自己リフレクター**: 失敗時に、自然言語省察を書き、将来のロールアウトを再播種するために使用します(Reflexion形式)。

環境フィードバック(観測)は値関数に混在するため、サーチはモデルの意見ではなく、実際のツール結果によって情報を取得されます。論文時の結果：HumanEvalでGPT-4での92.7%のpass@1(SOTA)、WebShopでGPT-3.5での平均75.9(勾配ベースのファインチューニングに接近)。

### MCTS、最小限

イテレーションごとに4つのフェーズ：

1. **Select** — UCT(ツリー上の上界信頼)を使用してルートからリーフへ歩行します。
2. **Expand** — ポリシーを使用してK個の子を生成します。
3. **Simulate** — ポリシーを使用して子からロールアウト、値関数(または環境報酬)でリーフをスコア化します。
4. **Backpropagate** — 訪問数と値推定をパスアップで更新します。

UCT式：`Q(s, a) + c * sqrt(ln N(s) / N(s, a))`。最初の項は搾取；第2は探索です。タスクごとに`c`を調整します。

### コスト現実

サーチはトークンを爆発させます。Game of 24のToTはCoTの100-1000倍のトークンを使用します。LATSは同様です。これは無料ではありません；サーチを予約してください：

- 単一のトラジェクトリが明らかに不十分なタスク(Game of 24、複雑なコード)。
- 壁時計が正確性よりも重要度が低いタスク。
- 安価で信頼できる値関数を持つタスク(コード用のユニットテスト、数学用の明示的ターゲット)。

タスクが単一の正しい答えを持ち、ノイズの多い評価器を持っている場合、サーチはしばしば状況を悪化させます。それは「良好スコア」の間違った答えを見つけます。

### 2026年のポジショニング

ほとんどの本番エージェントはLATSを実行していません。彼らはツール根拠検証を備えたReActを実行します(CRITIC、レッスン05)。サーチは専門的なニッチで現れます：

- テストを値関数として実行するコーディングエージェント(HumanEval形式)。
- 複数のクエリパスを探索するディープリサーチエージェント。
- LangGraphサブグラフ内の計画集約的なワークフロー。

AlphaEvolve(レッスン11)は2025年の極端です：コード上の進化サーチ、機械チェック可能なフィットネス、フロンティア利得(56年間で最初の4x4 matmul改善)。

## ビルド

`code/main.py`は以下を実装します：

- スタイル化された「算術演算を選択」タスクでの小さなToT BFS。
- 同じタスクでのおもちゃLATS MCTSループ(Select/Expand/Simulate/Backpropagate) UCT選択付き。
- シンボリックスコアと自己評価スコアを構成する値関数。

実行：

```
python3 code/main.py
```

トレースは、MCTSを使用してLATSが最高のロールアウトに収束するのに対して、BFSを使用して3つのノードを拡張するToTを示しています。両方のトークン数を印刷します。

## 使う

LangGraphはToT形式の探索をサブグラフパターンとして出荷します；LangChainチームのLATSに関するブログ(2024年5月)は参照チュートリアルです。LlamaIndexは`TreeOfThoughts`エージェントを出荷します。ほとんどの2026年本番エージェントでは、このパターンは`if task_complexity > threshold: use_search()`ゲートの背後に存在します — レッスン05の評価器最適化パターンを参照してください。

## 出荷

`outputs/skill-search-policy.md`は、タスク形、予算、および評価器忠実度が与えられたリニアReAct、ToT、LATS、および進化探索の間で選択します。

## 演習

1. おもちゃLATSをUCT c=0.1対c=2.0で実行します。トレースで何が変わりますか？
2. 値関数をより騒々しいスコアラーと交換(ランダムジッターを追加)。MCTSはまだ最高のリーフを見つけますか？それが許容する最小信号対雑音は何ですか？
3. ビームサーチToT(各レベルで上位kを保つ)を実装し、BFSと比較します。どちらがタイトなトークン予算でより良いですか？
4. LATS Section 5.1を読みます。HumanEvalトラジェクトリ数を再現：報告されたpass@1に到達するのに何回のロールアウトが必要ですか？
5. LATS論文の「LATSがより少ない助けになる」に関する議論を読みます。タスク形状をサーチ戦略にマッピングするための1段落の決定ルールを書きます。

## キーターム

| ターム | 人が言うこと | 実際の意味 |
|--------|------------|----------|
| Tree of Thoughts | 「分岐CoT」 | Yao et al. — 自己評価を備えた思考ノードのツリー |
| LATS | 「LLMのためのMCTS」 | Zhou et al. — ToT + ReAct + Reflexionを MCTSの下で統一 |
| UCT | 「上界信頼」 | 搾取(Q)と探索(ln N / n)を平衡する選択式 |
| Value function | 「この状態はどのくらい良いのか」 | プロンプトされたLLMスコアまたは環境報酬；バックプロップに給料 |
| Policy | 「アクション提案者」 | ReAct形式ジェネレーター；候補の次の思考/アクションを発行 |
| Rollout | 「シミュレートされたトラジェクトリ」 | ポリシーを使用してノードからリーフへ歩行、値でスコア |
| Backpropagate | 「祖先を更新」 | リーフの報酬をパスアップで推し、訪問数とQを更新 |
| Search cost | 「トークン爆発」 | Game of 24で100-1000x CoT；採用前に予算 |
