# Reflexion：バーバル強化学習

> 勾配ベースのRLは故障モードを修正するために数千回の試行とGPUクラスターを必要とします。Reflexion (Shinn et al., NeurIPS 2023)はそれを自然言語で行います：失敗した試行ごとに、エージェントは省察を書き、エピソディックメモリに保存し、そのメモリを条件に次の試行を行います。これはLettaのスリープタイム計算、Claude CodeのCLAUDE.md学習、およびpro-workflowのlearn-ruleの背後にあるパターンです。

**タイプ:** ビルド
**言語:** Python (stdlib)
**前提条件:** Phase 14 · 01 (Agent Loop)、Phase 14 · 02 (ReWOO)
**所要時間:** 約60分

## 学習目標

- Reflexionの3つのコンポーネント(Actor、Evaluator、Self-Reflector)と、エピソディックメモリの役割を名前付けする。
- 二値評価器、省察バッファ、新規再試行を備えた標準ライブラリReflexionループを実装する。
- 与えられたタスクのスカラー、ヒューリスティック、自己評価されたフィードバックソース間で選択する。
- バーバル強化が勾配ベースのRLが修正するのに数千回の試行を必要とするエラーを捉える理由を説明する。

## 問題

エージェントがタスクに失敗します。標準的なRLでは、さらに数千回の試行を実行し、勾配を計算し、重みを更新します。費用がかかり、遅く、ほとんどの本番エージェントは毎回の失敗のためのトレーニング予算を持っていません。

Reflexion (Shinn et al., arXiv:2303.11366)は異なる質問をします：エージェントが失敗した理由を考え、そう思いながら再び試したらどうでしょうか？重み更新なし。勾配なし。試行間に保存される自然言語だけです。

結果：ALFWorldではReActおよび他の非ファインチューニングベースラインに勝ります。HotpotQAではReActを改善します。コード生成(HumanEval/MBPP)では当時の最新技術を設定します。単一の勾配ステップさえもなく。

## コンセプト

### 3つのコンポーネント

```
Actor         : トラジェクトリを生成(ReAct風ループ)
Evaluator     : トラジェクトリをスコア化 — バイナリ、ヒューリスティック、または自己評価
Self-Reflector: 失敗に関するナチュラルランゲージ省察を書く
```

プラス1つのデータ構造：

```
エピソディックメモリ: 先行する省察のリスト、次の試行のプロンプトの前置き
```

1つの試行はActorを実行します。Evaluatorはそれをスコア化します。スコアが低い場合、Self-Reflectorは省察を生成します("I picked the wrong tool because I misread the question as asking about X when it was asking about Y")。省察はエピソディックメモリに入ります。次の試行は新規に開始しますが、省察を見ます。

### 3つの評価器タイプ

1. **スカラー** — 外部バイナリ信号。ALFWorldは成功または失敗します。HumanEvalテストはパスまたは失敗します。最もシンプル、最高信号。
2. **ヒューリスティック** — 事前定義された失敗シグネチャ。「エージェントが同じアクションを連続して2回生成した場合、スタックとしてマークします。」「トラジェクトリが50ステップを超える場合、非効率としてマークします。」
3. **自己評価** — LLMが独自のトラジェクトリをスコア化します。基準真実が利用できない場合に必要です。より弱い信号；ツール根拠検証と組み合わせるとうまく機能します(レッスン05 — CRITIC)。

2026年のデフォルトは混合です：利用可能な場合はスカラー、利用不可の場合は自己評価、ヒューリスティックは安全機構として。

### なぜこれは一般化するのか

Reflexionは新しいアルゴリズムというより、名前付きパターンです。ほぼすべての本番「自己修復」エージェントはいくつかのバリアントを実行します：

- Lettaのスリープタイム計算(レッスン08)：過去の会話を反映し、メモリブロックに書き込む別のエージェント。
- Claude CodeのCLAUDE.md/"save memory"パターン：学習として記録された省察は、将来のセッションに前置きされます。
- pro-workflowの`/learn-rule`コマンド：明示的なルールとして記録された修正。
- LangGraphの省察ノード：出力をスコア化し、必要に応じて洗練にルートするノード。

すべては同じ洞察から導出されます：自然言語は「失敗から学んだこと」を実行間で運ぶには十分に豊かなメディアです。

### いつそれが機能し、いつ機能しないか

Reflexionは以下の場合に機能します：

- 明確な失敗信号があります(テスト失敗、ツールエラー、間違った答え)。
- タスククラスは再現可能です(同じタイプの質問を再度質問できます)。
- 省察には、トラジェクトリを改善する余地があります(十分なアクション予算)。

Reflexionは以下の場合に役立たない：

- エージェントは初回から既に成功しています。
- 失敗は外部です(ネットワークダウン、ツール破損)。「ネットワークがダウンしていた」に関する省察は、将来の実行には役立ちません。
- 省察は迷信になります。1回限りのflaky実行に関するナラティブを保存します。

2026年の落とし穴：メモリ腐食。省察が蓄積；一部は廃止または間違っています；エピソディックバッファが成長するにつれて、再実行は遅くなります。緩和策：定期的なコンパクション(レッスン06)、省察のTTL、または別のスリープタイム学習エージェント(Letta)。

## ビルド

`code/main.py`はおもちゃのパズルでReflexionを実装します：3要素のリストを生成してターゲットに合計します。Actorは候補リストを発行；Evaluatorは合計をチェック；Self-Reflectorは何が間違ったかについての1行を書きます。省察は次の試行のためのエピソディックメモリに入ります。

コンポーネント：

- `Actor` — 省察を見るとき改善するスクリプト化されたポリシー。
- `Evaluator.binary()` — ターゲット合計でパス/フェール。
- `SelfReflector` — 失敗の1行診断を生成します。
- `EpisodicMemory` — TTLセマンティクス付きの制限付きリスト。

実行：

```
python3 code/main.py
```

トレースは3つの試行を示します。試行1が失敗し、省察が保存され、試行2は省察を見て改善しますが依然として失敗し、試行3が成功します。ベースライン実行と比較(省察なし)。それは試行1の答えで行き止まったままです。

## 使う

LangGraphは省察をノードパターンとして出荷します。Claude Codeの`/memory`コマンドとpro-workflowの`/learn-rule`はエピソディックバッファをmarkdownファイルとして外部化します。Lettaのスリープタイム計算は、プライマリエージェントがレイテンシ制限のままであるように、ダウンタイムで自己リフレクターを実行します。OpenAI Agents SDKはReflexionを直接出荷しません。スコアでトラジェクトリを拒否するカスタムガードレールと、実行にわたって存続するメモリセッションを使用して構築します。

## 出荷

`outputs/skill-reflexion-buffer.md`は、省察キャプチャ、TTL、および重複排除を備えたエピソディックバッファを作成および保守します。タスククラスと失敗を与えられたので、実際に次の試行に役立つ省察を発行します(汎用的な「もっと注意してください」ではなく)。

## 演習

1. バイナリから距離メトリクスを返すスカラー評価器に切り替えます(ターゲットからどれだけ離れているか)。それはより速く収束しますか？
2. 省察にTTL 10試行を追加します。その点の後で古い省察は傷つきますか、それとも助けますか？
3. ヒューリスティック評価器を実装：同じアクションが繰り返す場合、試行がスタックとしてマークされます。これはSelf-Reflectorとどのように相互作用しますか？
4. 省察を無視する敵対的なActorでReflexionを実行します。Actorに気づかせるための最小限の省察プロンプトエンジニアリングは何ですか？
5. Reflexion論文のセクション4をAlfWorldで読みます。概念的に130%の成功率改善を再現します：バニラReActと比較してキーデルタは何ですか？

## キーターム

| ターム | 人が言うこと | 実際の意味 |
|--------|------------|----------|
| Reflexion | 「自己修正」 | Shinn et al. 2023 — Actor、Evaluator、Self-Reflectorプラスエピソディックメモリ |
| Verbal reinforcement | 「勾配なしで学習」 | ナチュラルランゲージ省察は次の試行のプロンプトに前置きされます |
| Episodic memory | 「タスク別省察」 | 1つのタスククラスのための先行する省察の制限付きバッファ |
| Scalar evaluator | 「バイナリ成功信号」 | 基準真実からパス/フェールまたは数値スコア |
| Heuristic evaluator | 「パターンベース検出」 | 事前定義された失敗シグネチャ(例えば、スタック・ループ、ステップが多すぎる) |
| Self-evaluator | 「LLM as judge on own trace」 | 基準真実がない場合の低信号フォールバック。ツール根拠検証と組み合わせます |
| Memory rot | 「陳腐な省察」 | エピソディックバッファは廃止されたエントリで満杯になります；コンパクション/TTLで修正 |
| Sleep-time reflection | 「非同期自己省察」 | プライマリエージェントが速く留まるように、ホットパスをオフの自己リフレクターを実行します |
