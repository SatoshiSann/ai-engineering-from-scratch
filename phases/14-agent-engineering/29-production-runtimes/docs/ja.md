# 本番環境ランタイム: キュー、イベント、Cron

> 本番環境エージェントは 6 つのランタイム シェイプで実行されます: リクエスト レスポンス、ストリーミング、耐久実行、キューベース バックグラウンド、イベント駆動、スケジュール設定。フレームワークを選択する前にシェイプを選択します。オブザーバビリティはすべてのシェイプで負荷を支えています。

**タイプ:** Learn
**言語:** Python (stdlib)
**前提条件:** Phase 14 · 13 (LangGraph)、Phase 14 · 22 (Voice)
**所要時間:** 約 60 分

## 学習目標

- 6 つの本番環境ランタイム シェイプに名前を付け、それぞれをフレームワーク / 製品パターンにマッチできる。
- 長期タスクにおいて耐久実行 (LangGraph) が重要な理由を説明できる。
- イベント駆動ランタイムと Claude Managed Agents がいつ適合するかを説明できる。
- マルチステップ エージェント用のオブザーバビリティ負荷支え主張を説明できる。

## 問題

本番環境エージェントは Jupyter ノートブックが表示しない方法で失敗します: ステップ 37 でのネットワーク タイムアウト、ユーザーが音声通話の途中でハングアップ、cron ジョブがマシン再起動時に停止、バックグラウンド ワーカーがメモリ不足。ランタイム シェイプは、どの障害が生き残り可能かを決定します。

## コンセプト

### リクエスト レスポンス

- 同期 HTTP。ユーザーは完了を待ちます。
- 短いタスク (<30s) のみ実行可能。
- スタック: Agno (Python + FastAPI)、Mastra (TypeScript + Express/Hono/Fastify/Koa)。
- オブザーバビリティ: 標準 HTTP アクセスログ + OTel スパン。

### ストリーミング

- プログレッシブ出力用の SSE または WebSocket。
- LiveKit これを WebRTC に拡張して音声/ビデオ用 (Lesson 22)。
- スタック: ストリーミング サポート付きのあらゆるフレームワーク + SSE/WS を処理するフロントエンド。
- オブザーバビリティ: チャンクごとのタイミング、最初トークン レイテンシー、テール レイテンシー。

### 耐久実行

- 状態は各ステップ後にチェックポイント; 障害時に自動再開。
- AutoGen v0.4 アクター モデルは障害を 1 つのエージェントに分離 (Lesson 14)。
- LangGraph のコア差別化要因 (Lesson 13)。
- ステップ カウントが不明で回復コストが高い場合に必須。

### キューベース / バックグラウンド

- ジョブがキューに入ります。ワーカーが拾います。結果は webhook または pub/sub 経由で流れ戻します。
- 長期エージェント (タスクごとに数十から数百のステップ、Anthropic のコンピューター使用発表により) に必須。
- スタック: Celery (Python)、BullMQ (Node)、SQS + Lambda (AWS)、カスタム。
- オブザーバビリティ: キュー深度、ジョブごとのレイテンシー分布、DLQ サイズ。

### イベント駆動

- エージェントはトリガーをサブスクライブ: 新しいメール、PR オープン、cron 発火。
- Claude Managed Agents はこれをボックスから外してカバー (Lesson 17)。
- CrewAI Flows (Lesson 15) はイベント駆動決定論的ワークフローを構成。
- オブザーバビリティ: トリガー ソース、イベント ツー スタート レイテンシー、エージェント レイテンシー。

### スケジュール設定

- 定期的に実行される Cron 形のエージェント。
- 耐久実行と組み合わせると、失敗した夜間実行は次のティックで再開。
- スタック: Kubernetes CronJob + 耐久フレームワーク; ホスト型 (Render cron、Vercel cron)。

### 2026 年デプロイメント パターン

- **CrewAI Flows** - イベント駆動本番環境用。
- **Agno** - Python マイクロサービス用のステートレス FastAPI。
- **Mastra** - サーバー アダプター (Express、Hono、Fastify、Koa) を埋め込み用。
- **Pipecat Cloud / LiveKit Cloud** - マネージド音声用 (Lesson 22)。
- **Claude Managed Agents** - ホスト型長実行非同期用。

### オブザーバビリティは負荷を支えています

OpenTelemetry GenAI スパン (Lesson 23) と Langfuse/Phoenix/Opik バックエンド (Lesson 24) がなければ、ステップ 40 で失敗したマルチステップ エージェントをデバッグすることはできません。これは本番環境用の選択肢ではありません。「高速にデバッグする」と「より多くのログを使用して最初からリプレイする」の違いです。

### 本番環境ランタイムが失敗する場所

- **間違ったシェイプの選択。** 5 分間のタスク用のリクエスト レスポンスを選択。ユーザーはハングアップ; ワーカーは積み重ねる; 再試行は複合。
- **DLQ なし。** デッドレター なしのキュー ワーカー。失敗したジョブは消えます。
- **不透明なバックグラウンド作業。** バックグラウンド エージェントはトレース エクスポートなしで実行。障害は、ユーザーがそれらをレポートするまで目に見えません。
- **耐久状態をスキップ。** 再起動を許可できない 30 秒を超える実行には耐久実行が必要。

## Build It

`code/main.py` は stdlib マルチシェイプ デモです:

- リクエスト レスポンス エンドポイント (プレーン関数)。
- ストリーミング ハンドラー (ジェネレーター)。
- DLQ 付きのキューベース ワーカー。
- イベント トリガー レジストリ。
- Cron 形のスケジューラー。

実行:

```bash
python3 code/main.py
```

出力: 同じタスク上の各シェイプの動作を示す 5 つのトレース。同じエージェント ロジック、異なる外側シェル。 耐久実行 (6 番目のシェイプ) は意図的に LangGraph チェックポイント付きの Lesson 13 でカバーされています。

## Use It

- **リクエスト レスポンス** - チャット形式 UX 用。
- **ストリーミング** - プログレッシブ レスポンス用。
- **耐久** - 長期タスク用。
- **キュー** - バッチ / 非同期 / 長実行用。
- **イベント** - エージェント反応性用。
- **Cron** - ハウスキーピング用 (メモリ統合、evals、コスト レポート)。

## Ship It

`outputs/skill-runtime-shape.md` はタスク用のランタイム シェイプを選択し、オブザーバビリティ要件を接続します。

## 演習

1. Lesson 01 ReAct ループをスタック内のすべての 6 つのシェイプにポート。 どのシェイプがどの製品サーフェスに適合しますか?
2. キューベース デモに DLQ を追加。 10% ジョブ失敗をシミュレート; DLQ サイズをサーフェス。
3. 夜間に上位 20 のトレースに対して実行される cron トリガーされた eval エージェントを記述。
4. バックプレッシャー付きストリーミングを実装: クライアントが遅い場合、エージェントを一時停止。 これはターン予算とどのように相互作用しますか?
5. Claude Managed Agents ドキュメントを読みます。 自ホスト型長期エージェントをマネージドに移動する場合は?

## 主要用語

| 用語 | 人々が言うこと | 実際の意味 |
|------|----------------|----------|
| リクエスト レスポンス | 「同期」 | ユーザーは待機; 短いタスクのみ |
| ストリーミング | 「SSE / WS」 | プログレッシブ出力; より良い UX; レイテンシーはチャンクごとに観察可能 |
| 耐久実行 | 「障害から再開」 | チェックポイント状態; 最後のステップで再起動 |
| キューベース | 「バックグラウンド ジョブ」 | プロデューサー / ワーカー プール / DLQ |
| イベント駆動 | 「トリガーベース」 | エージェント は外部イベントに反応 |
| DLQ | 「デッドレター キュー」 | 失敗したジョブ用の駐車場 |
| Claude Managed Agents | 「ホスト型ハーネス」 | Anthropic ホスト型長実行非同期 (キャッシング + 圧縮付き) |
