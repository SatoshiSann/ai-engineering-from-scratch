# メモリブロックとスリープタイム計算(Letta)

> MemGPTは2024年にLettaになりました。2026年進化は2つのアイデアを追加：モデルが直接編集できる離散機能メモリブロック、そしてプライマリエージェントがアイドル中に非同期でメモリを統合するスリープタイムエージェント。これは1つの会話を超えてメモリをスケーリングする方法です。

**タイプ:** ビルド
**言語:** Python (stdlib)
**前提条件:** Phase 14 · 07 (MemGPT)
**所要時間:** 約75分

## 学習目標

- Lettaが使用する3つのメモリティア(core、recall、archival)を名前付けし、各々の役割を名前付けする。
- メモリブロックパターンを説明：Humanブロック、Personaブロック、ユーザー定義ブロックとしての第1クラス型オブジェクト。
- スリープタイム計算が何か、なぜそれが重大パスをオフに座り、なぜそれがプライマリエージェントより強いモデルを実行できるのかを説明する。
- プライマリエージェントが応答を提供し、スリープタイムエージェントがターン間でブロックを統合するスクリプト化された2エージェントループを実装する。

## 問題

MemGPT(レッスン07)は仮想メモリ制御フローを解決しました。3つの本番問題が出現：

1. **レイテンシ。** すべてのメモリ操作は重大パスにあります。エージェントがプルーニング、要約、または調整してユーザーが待つ場合、テールレイテンシは吹き上がります。
2. **メモリ腐食。** 書き込みが蓄積します。矛盾した事実は滞在します。検索は陳腐なコンテンツで溺れています。
3. **構造喪失。** フラットアーカイバルストアは「Humanブロックは常にプロンプトにあります；Personaブロックは常にプロンプトにあります；Taskブロックはセッションごとに交換」を表現できません。

Letta (letta.com)は2026年の書き直しです。メモリブロックは構造を明示的にします；スリープタイム計算は統合を重大パスの外に移動します。

## コンセプト

### 3つのティア

| ティア | 範囲 | 場所 | 書き込み元 |
|--------|------|------|----------|
| Core | 常に表示 | メインプロンプト内 | エージェントツール呼び出し+スリープタイム書き直し |
| Recall | 会話履歴 | 取得可能 | 自動ターンログ |
| Archival | 任意の事実 | ベクトル+KV+グラフ | エージェントツール呼び出し+スリープタイム取得 |

Coreはメモリコアです。Recallは立退きテールを備えた会話バッファです。Archivalは外部ストアです。分割はMemGPTの2層オーバーロードをクリーンアップします。

### メモリブロック

ブロックは型付き、永続的、編集可能なコアティアのセクションです。元のMemGPT論文は2つを定義：

- **Humanブロック** — ユーザーに関する事実(名前、役割、好み、目標)。
- **Personaブロック** — エージェントの自己概念(アイデンティティ、トーン、制約)。

Lettaは任意のユーザー定義ブロックに一般化：現在の目標のための`Task`ブロック、コードベース事実のための`Project`ブロック、ハード制約のための`Safety`ブロック。各ブロックは`id`、`label`、`value`、`limit`(文字上限)、`description`(モデルがいつそれを編集するかを知る)を持っています。

ブロックはツール表面経由で編集可能：

- `block_append(label, text)`
- `block_replace(label, old, new)`
- `block_read(label)`
- `block_summarize(label)` — 限度の近いブロックを圧縮。

### スリープタイム計算

2025年Letta追加：2番目のエージェントをバックグラウンドで、重大パスをオフに実行。スリープタイムエージェントは会話トランスクリプトとコードベースコンテキストを処理し、`learned_context`を共有ブロックに書き、またはアーカイバルレコードを統合または無効化します。

落ちるプロパティ：

- **レイテンシコストなし。** プライマリ応答はメモリ操作を待ちません。
- **より強いモデルが許可されます。** スリープタイムエージェントはレイテンシ制約がないため、より高価でより遅いモデルであることができます。
- **自然統合ウィンドウ。** ユーザーが待たない場合にデデュプ、要約、矛盾した事実を無効化。

形状は人間がどのように機能するかと一致：タスクを行う、それで眠る、長期メモリは一晩に落ち着きます。

### Letta V1とネイティブ推論

Letta V1 (`letta_v1_agent`, 2026)は`send_message`/heartbeatとインライン`Thought:`トークンをネイティブ推論に賛成して廃止します。Responses API (OpenAI)とメッセージAPIは拡張思考(Anthropic)は別のチャネルで推論を発行し、ターン横切りに渡されます(本番環境ではプロバイダ横切りに暗号化)。制御ループはまだReActです。思考トレースは構造的、プロンプト形でなく。

### このパターンが間違う場所

- **ブロック膨張。** 無限`block_append`は速くLimit達します。キャップを超えて押すブロックサマライザーをワイヤー。
- **静かドリフト。** スリープタイムエージェントはブロックを書き直し、プライマリエージェントは決して気づきません。バージョンブロックとトレースで表面差分。
- **毒性統合。** スリープタイムエージェントは攻撃者到達可能コンテンツをコアに処理します。レッスン27はスリープタイム表面にも適用。

## ビルド

`code/main.py`は以下を実装：

- `Block` — id、label、value、limit、description。
- `BlockStore` — CRUD+`near_limit(label)`ヘルパー。
- 2つのスクリプト化されたエージェント — `PrimaryAgent`はターンを提供、`SleepTimeAgent`はターン間を統合。
- ブロック書き込みと、ブロック要約と陳腐な事実無効化を行うスリープタイムパスを含む3ターン会話を示すトレース。

実行：

```
python3 code/main.py
```

トランスクリプトは分割を示す：プライマリターンは速く、生書き込みを生成；スリープパスはコンパクトとクリーンアップします。

## 使う

- **Letta** (letta.com)参照実装のため。自己ホストまたは管理クラウド。
- **Claude Agent SDKスキル**ブロック形の知識として — スキルはエージェントがオンデマンドでロードする名前付き、バージョン管理されて、検索可能なブロック指示です。
- **カスタムビルド**ストレージバックエンドを制御したいチーム向け。Letta APIコントラクトを使用して、後で移行できます。

## 出荷

`outputs/skill-memory-blocks.md`は安全ルールと引用配線を含む任意のランタイムのためのLetta形メモリシステムを生成します。

## 演習

1. `near_limit`が真を返すときにブロック値をモデル生成要約で置き換える`block_summarize`ツールを追加。どのトリガー閾値が要約呼び出しとブロックオーバーフロー両方を最小化しますか？
2. アーカイバル上でスリープタイムデデュプを実装：テキストが>90%トークンオーバーラップを持つ2つのレコードは1つに崩壊。スリープパスでのみ行う、決して重大パスをしない。
3. バージョンブロック。すべての書き込みで古い値と差を記録。`block_history(label)`を公開して、オペレーターが「なぜエージェントX を忘れたのか」をデバッグできます。
4. スリープタイムエージェントを信頼されない作成者として扱う。Personaまたはセーフティブロックに触るときは、コミット前に2番目のエージェントレビューが必要。
5. Letta API (`letta_v1_agent`)を使用する例をポート。ブロックスキーマで何が変わり、ネイティブ推論はトレース形をどのように変えますか？

## キーターム

| ターム | 人が言うこと | 実際の意味 |
|--------|------------|----------|
| Memory block | 「編集可能プロンプトセクション」 | コアメモリの型付き、永続的、LLM編集可能セグメント |
| Human block | 「ユーザーメモリ」 | ユーザーに関する事実、コアに固定 |
| Persona block | 「エージェントアイデンティティ」 | 自己概念、トーン、制約、コアに固定 |
| Sleep-time compute | 「非同期メモリ作業」 | 重大パスをオフに統合を行う第2エージェント |
| Core / Recall / Archival | 「ティア」 | 3層メモリ分割：常に表示/会話/外部 |
| Block limit | 「キャップ」 | ブロックごとの文字制限；要約を強制 |
| Native reasoning | 「思考チャネル」 | プロバイダレベル推論出力、プロンプトレベル`Thought:`ではなく |
| Learned context | 「スリープ出力」 | スリープタイムエージェントが共有ブロックに書き込む事実 |
