# ハイブリッドメモリ：ベクトル+グラフ+KV (Mem0)

> Mem0 (Chhikara et al., 2025)はメモリを3つのストアとして並行で扱う — セマンティック類似性のためのベクトル、高速事実検索のためのKV、エンティティ関係推論のためのグラフ。スコアリング層は検索時に3つを融合させます。これは2026年の本番外部メモリの標準です。

**タイプ:** ビルド
**言語:** Python (stdlib)
**前提条件:** Phase 14 · 07 (MemGPT)、Phase 14 · 08 (Letta Blocks)
**所要時間:** 約75分

## 学習目標

- 単一ストア(ベクトルのみ、グラフのみ、KVのみ)がエージェントメモリに不十分である理由を説明する。
- Mem0の3つの並行ストアを名前付けし、各々が最適化するものを名前付けする。
- Mem0の融合スコアリング — 関連性、重要性、最新性 — を説明し、なぜそれが重みの合計で、階層ではなく。
- スタンダードライブラリにおもちゃの3ストアメモリを実装、すべてのストアに書き込む`add()`と結果を融合する`search()`。

## 問題

1つのストアは3つのクエリクラスの1つには間違っています：

- **セマンティック類似性** — 「先週エージェントドリフトについて何を議論しましたか？」ベクトルが勝利；KVとグラフが見逃し。
- **事実検索** — 「ユーザーの電話番号は何ですか？」KVが勝利；ベクトルは浪費、グラフはやり過ぎ。
- **関係推論** — 「どの顧客が同じ請求エンティティを共有していますか？」グラフが勝利；ベクトルとKVは回答できない。

本番エージェントは1つのセッションで3つすべてを発行します。単一ストアメモリは常に2つに対して間違っています。Mem0の貢献は、3つすべてを単一の`add`/`search`表面の背後にワイヤリング、それらを融合するスコアリング関数を使用します。

## コンセプト

### 3つのストアが並行

Mem0 (arXiv:2504.19413、2025年4月)オン`add(text, user_id, metadata)`：

1. テキストから候補事実を抽出(LLM駆動ステップ)。
2. 各事実をセマンティック検索のためのベクトルストアに書き込み(埋め込み)。
3. 各事実をO(1)検索のためにキーイングされた(user_id、fact_type、entity)KVストアに書き込み。
4. 各事実を関係クエリのための型付きエッジとしてグラフストア(Mem0g)に書き込み。

オン`search(query, user_id)`：

1. ベクトルストアは埋め込みコサインで上位kを返す。
2. KVストアはクエリ導出(user_id、type、entity)でキーイングされた直接ヒットを返す。
3. グラフストアはクエリエンティティから到達可能なサブグラフを返す。
4. スコアリング層は3つを融合させます。

### 融合スコアリング

```
score = w_relevance * relevance(q, record)
      + w_importance * importance(record)
      + w_recency * recency(record)
```

- **関連性** — ベクトルコサイン、KV正確マッチ、グラフパス重み。
- **重要性** — 書き込み時タグ付けまたは学習(一部の事実はより重要：名前、ID、ポリシー)。
- **最新性** — 最後の書き込みまたは読み取りからの時間を超えた指数減衰。

重みは製品ごとに調整されます。チャットエージェントの高い`w_recency`；コンプライアンスエージェントの高い`w_importance`；検索エージェントの高い`w_relevance`。

### Mem0gと時間推論

Mem0gは競合検出器を追加します。新しい事実が既存のエッジと矛盾する場合、既存のエッジは無効としてマークされますが、削除されません。時間クエリ(「ユーザーの都市は3月に何でしたか？」)は有効時間サブグラフを横断します。

これはLettaの無効化パターンが一般化するコンプライアンス級動作です。

### ベンチマーク数

Mem0論文レポート(2025)：

- **LoCoMo** (長形式会話メモリ)：91.6
- **LongMemEval** (長期エピソディックメモリ)：93.4
- **BEAM 1M** (1M トークンメモリベンチマーク)：64.1

比較ベースライン(フルコンテキスト128k LLM、フラットベクトルストア、フラットKV)はすべて10+ポイント失います。ベンチマーク単独は選択を正当化しません — 操作形がしますが — 数字は融合設計が丸め誤りではないことを示しています。

### スコープ分類

Mem0はスコープごとにメモリを分割：

- **ユーザーメモリ** — セッション間で永続、`user_id`でキー。
- **セッションメモリ** — 1つのスレッド内で永続。
- **エージェントメモリ** — エージェントインスタンスステート。

すべての書き込みは1つのスコープを選択します。検索はスコープ別重みを使用してスコープ横切りをクエリできます。スコープを思慮なく混合することは「アシスタントはAliceにBobのプロジェクトについて語った」インシデントを得る方法です。

### このパターンが間違う場所

- **埋め込みドリフト。** 最初の百クエリで正しく見える結果はコーパスが成長すると劣化します。定期的に上位Nレコード使用の再埋め込みを追加。
- **KVスキーマクリープ。** `(user_id, type, entity)`は単純に見えるまで、すべてのチームが独自の`type`を追加。`type`セットを四半期ごとに監査。
- **グラフ爆発。** 1つのノイズ抽出器がメッセージごとに50エッジを追加。`add`呼び出しごとのグラフ書き込みをキャップ；低信頼エッジをドロップ。

## ビルド

`code/main.py`は標準ライブラリに3ストアパターンを実装：

- `VectorStore` — 埋め込みスタンドイン用のナイーブトークンオーバーラップ類似性。
- `KVStore` — `(user_id, fact_type, entity)`でキーイングされた辞書。
- `GraphStore` — 型付きエッジ(subject、relation、object、valid)。
- `Mem0` — `add()`、`search()`、融合スコアリング、スコープ認識検索を備えた最上レベルファサード。
- マルチユーザ、マルチセッション会話への取り組みトレース。

実行：

```
python3 code/main.py
```

出力は3つの別個の回想パスと融合トップkを示しています。`main()`の上部のスコアリング重みをフリップし、ランキング変化を見ます。

## 使う

- **Mem0 (Apache 2.0)** — 本番レディ。Postgres+Qdrant+Neo4jで自己ホスト、または管理クラウドを使用。
- **Letta** — 3層core/recall/archival；独自のベクトルとグラフバックエンドを持つ。
- **Zep** — 時間KGと事実抽出を備えた商用の代替。
- **カスタムビルド** — 抽出器(コンプライアンス)または融合重み(最新性が支配する音声エージェント)を正確に制御する必要がある場合。

## 出荷

`outputs/skill-hybrid-memory.md`は融合スコアラー、スコープ分類、時間無効化配線付きで3ストアメモリ・スキャフォルドを生成します。

## 演習

1. おもちゃのベクトル類似性をリアル埋め込みモデル(sentence-transformers、Ollama、OpenAI埋め込み)で置き換え。合成長会話でrecall@10を測定。1000書き込み後にランキングドリフトしますか？
2. 時間クエリを追加：`search(query, as_of=timestamp)`。その時間以降有効なレコードのみを返す。どのストアが最も作業が必要ですか？
3. 競合検出器を実装：受信事実がグラフエッジと矛盾する場合、古いエッジを無効化し、両方をログ。「ユーザーはベルリンに住んでいる」 -> 「ユーザーはリスボンに住んでいる」で試験。
4. 融合スコアラーをゲーミング防止のための`user_feedback`次元(検索レコードで親指アップ)を含めるにポート。(エージェントが既に好きだったレコードのみを返す)ゲーミングをどのように防止しますか？
5. Mem0ドキュメント(docs.mem0.ai)を読みます。おもちゃを`mem0`クライアント呼び出しにポート。同じ20テストクエリで検索品質を比較。

## キーターム

| ターム | 人が言うこと | 実際の意味 |
|--------|------------|----------|
| Hybrid memory | 「ベクトルプラスグラフプラスKV」 | 3つのストアが並行で書き込み、検索時に融合 |
| Fact extraction | 「メモリ取得」 | テキストを(entity、relation、fact)タプルに分割するLLMステップ |
| Fusion scoring | 「関連性ランキング」 | 関連性、重要性、最新性の重み付き合計 |
| Scope | 「メモリネームスペース」 | user/session/agent — 誰が何を見るかを決定 |
| Mem0g | 「メモリグラフ」 | 関係クエリのための時間的有効性を備えた型付きエッジ |
| Temporal invalidation | 「ソフトデリート」 | 矛盾したエッジを無効としてマーク；決して削除しない |
| Embedding drift | 「検索腐食」 | ベクトル品質はコーパスが成長すると劣化；定期的に再埋め込み |
