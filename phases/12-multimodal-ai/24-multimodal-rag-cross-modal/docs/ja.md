# マルチモーダル RAG とクロスモーダル 収索

> ビジョンネイティブ ドキュメント RAG はワンスライス。本番 マルチモーダル RAG はより広い — テキスト、イメージ、オーディオ、ビデオ全体で収索するワークフロー「静かなビーガン ブランチで自然光を探す」、医療診療法「この写真+これらのノートに一致するけが」、eコマース「このセルフィに似たアウトフィット、自分のサイズ」、現場サービス「この エンジン音プラスパーツの写真を診断」のため。3つの2025調査 — Abootorabi et al., Mei et al., Zhao et al. — サブ問題を成文化：クロスモーダル 収索、収索融合、生成根拠、マルチモーダル 評価。本レッスンは調査を読み本番 パイプラインを設計。

**タイプ:** ビルド
**言語:** Python（stdlib、クロスモーダル retriever 融合 + 根拠付きジェネレータ）
**前提条件:** Phase 12 · 23（ColPali）、Phase 11（RAG 基本）
**所要時間:** 約180分

## 学習目標

- クロスモーダル 収索を設計：テキスト -> イメージ、イメージ -> テキスト、オーディオ -> ビデオ等。
- 3つの融合戦略を比較：スコア融合、注意ベース融合、MoE融合。
- 生成根拠を説明：何をソースがモダリティの混合の時「ソースを引用」するように見えるか。
- 2025年の3つの標準的マルチモーダル RAG 調査を名前付けし、彼らのサブ問題タクソノミー。

## 問題

単一モダリティ RAG は解決パターン：クエリを埋め込み、チャンクを埋め込み、取得、LLMに詰め込む。マルチモーダル RAG は必要：

1. 複数収索ヘッド（各モダリティは互換性のある空間の埋め込みが必要）。
2. モダリティ全体での収索結果の融合。
3. モダリティ全体でソースを引用する生成根拠。
4. クロスモーダル シグナルをカバーする評価メトリック。

2025調査はすべて同じタクソノミーに到着。

## コンセプト

### クロスモーダル 収索

モダリティA のクエリ与えモダリティB のドキュメントを取得。3つのパターン：

1. 共有埋め込み空間。CLIP と CLAP は テキスト+イメージ / テキスト+オーディオ 埋め込みを共有空間で生成。コサイン相似性はモダリティ全体で直接動作。CLIPトレーニング ペアに限定。

2. モダリティ毎エンコーダ + 翻訳。テキスト エンコーダ + イメージ エンコーダ + スペース間のマップ小さい翻訳モジュール。Gupta et al. による Sen2Sen とその他の2024設計。柔軟だが複雑性を加える。

3. VLM をエンコーダとして。VLM の隠れた状態を収索表現として使用。VLM がサポートする任意のモダリティが動作。より高い品質、より高価。

選択：テキスト+イメージの場合CLIP / SigLIP 2；テキスト+オーディオ用CLAP；先端品質でのクロスモーダル用VLM隠れた状態。

### 融合戦略

取得した10結果：5イメージ、3テキスト 段落、2オーディオ クリップ。どうマージ？

スコア融合（最安値）。各モダリティはそれ独自retriever、各スコアを返す。モダリティ内スコアを正規化その後合計。シンプル、しばしば動作。

注意ベース融合。すべての取得アイテムを連結、小さい注意ネットワークにそれらにウェイトさせ。訓練が必要。

MoE融合。ゲーティング ネットワーク モダリティ特定専門家にルート。異なるクエリ タイプが異なったルート — ビジュアル質問はイメージより高くウェイト。

本番デフォルト：スコア融合と、クエリの支配的モダリティへのわずかなバイアス。A/B がドメイン上で明確な勝利を示すならMoE にアップグレード。

### 生成根拠

LLM はどの取得アイテムが各クレームを駆動したかを引用すべき。マルチモーダルのため：

- テキスト ソース：標準引用 `[1]`。
- イメージ ソース：`[img 3]` 短いキャプション付き。
- オーディオ：`[audio 2 at 0:34]`。

根拠認識データでジェネレータを訓練：訓練ターゲットの各クレームはソース インデックス でタグ付け。推論で、モデルは自然に引用を発行。

### 2025調査

Abootorabi et al.（arXiv:2502.08826、「Any Modality でAsk」）：マルチモーダル RAG のタクソノミー。収索、融合、生成をカバー。最幅カバレッジ。

Mei et al.（arXiv:2504.08748、「A Survey of Multimodal RAG」）：サブ タスク ベンチマークと失敗モードに焦点。評価設計に有用。

Zhao et al.（arXiv:2503.18016）：ビジョン焦点調査。ColPali族の動作で強い。

すべての3つを読むことはあなたに2025年春の技術状態を与える。ほとんどのサブ問題は未だ開く。

### MuRAG — 基礎論文

MuRAG（Chen et al., 2022）はマルチモーダル RAG 最初。マルチモーダル KB から取得されたイメージ+テキスト、生成答え。可能性を示した VLM 波の前。モダン システム（REACT、VisRAG、M3DocRAG）がそれに構築。

### 本番旅行計画-例

クエリ：「静かなビーガン ブランチで自然光を探す。」

パイプライン：

1. クエリを分解。「静かな」-> オーディオ/レビュー キーワード；「ビーガン ブランチ」-> メニュー アイテム；「自然光」-> イメージ 特徴。
2. モダリティ毎取得：
   - テキスト 収索レビュー上：「ビーガン ブランチ、静かな雰囲気。」
   - イメージ 収索レストラン 写真上：「自然光、広々。」
   - オーディオ 収索環境音クリップ上：「低デシベル、音楽なし。」
3. スコアをフュージ。各レストランは複合スコアを持つ。
4. トップk レストラン -> VLM ジェネレータ すべての証拠付き -> 引用ですべての証拠で答え。

これはテキストRAG ずっと超え。各モダリティ はテキストは単独で見落とすシグナルを加える。

### エージェント的マルチモーダル RAG

マルチホップ：最初の収索 は高信頼答え を返さない場合、LLM は言い直し収索 再度。Phase 14 からのエージェント的 RAG パターンは適用。例：

- 取得初期トップ10 -> LLM は「あまりに騒々しい、< 40 dB でフィルター」を要求 -> 再収索。
- 取得イメージ -> LLM はメニュー持つ1つを見る -> テキストメニューを取得 -> 答え。

複雑性を加えるが、単発 収索 ができないクエリを処理。

### 評価

クロスモーダル 評価はまだ未成熟。通常的プロキシ：

- モダリティごとのRecall@k。
- フューズ トップk 精度。
- 人間が判定したエンドツーエンド 満足。
- タスク特定（完了した予約、購入）。

すべてのモダリティを綱ぐ標準ベンチマークなし。ほとんどの論文 ドメイン特定タスクで評価。

## 使用方法

`code/main.py`：

- 3つのモック retriever（テキスト、イメージ、オーディオ）はレストランの共有 コーパスで運用。
- スコア融合 設定可能ウェイトでモダリティ スコアを組み合わせ。
- 信頼が低い場合クエリを言い直す単純なエージェント的ループ付きジェネレータ スタブ。

## 提出

本レッスンは`outputs/skill-multimodal-rag-designer.md`を生成。マルチモーダル クエリ フローで製品スペックが与えられた場合、retriever、融合、ジェネレータ、評価を設計。

## 演習

1. 医療診療法 マルチモーダル RAG を提案：クエリ = 損傷の写真 + テキスト症状。何をモダリティが何 KB から取得？

2. スコア融合 は単純な加重合計。何の失敗モード MoE融合 は避ける？

3. Abootorabi et al. のタクソノミーを読む（Section 3）。3つの標準的サブ問題は何、そして彼らはあなたの選択製品にどうマップ？

4. 旅行計画-マルチモーダル RAG 用評価 仕様を設計。何のメトリック がイメージ再現率、オーディオ再現率、複合 正確性をカバー？

5. エージェント的 マルチホップ RAG はラウンドトリップあたり レイテンシ税を持つ。クエリ難易度でレイテンシ ゲイン を正当化する精度ゲインはどうなる？

## 主要な用語

| 用語 | 人々が言う | 実際の意味 |
|------|------------|---------|
| クロスモーダル 収索 | 「1つのモダリティをクエリ、別を取得」 | テキスト クエリ がイメージを取得；イメージ クエリ がテキストを取得；共有空間または翻訳が必要 |
| スコア融合 | 「スコアを組み合わせ」 | モダリティ毎 収索 スコアの加重合計；最も単純な融合 |
| MoE融合 | 「モダリティ ルート専門家」 | ゲーティング ネットワーク どのモダリティのスコアを信頼するか、クエリごとに選ぶ |
| 根拠付き生成 | 「ソースを引用」 | 回答の各クレームは ソース インデックスでタグ付け |
| MuRAG | 「マルチモーダル RAG 最初」 | マルチモーダル RAG パターンを確立した2022年の論文 |
| エージェント的マルチホップ | 「言い直し再試」 | LLM は最初パス 信頼が低い時 retriever に再クエリ |
