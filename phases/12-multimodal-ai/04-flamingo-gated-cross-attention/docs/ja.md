# Flamingo とゲートクロスアテンション (少数ショット VLM)

> DeepMind の Flamingo (2022) は他の誰かに先行 2つのことをした。それは単一モデルが画像、ビデオ、テキストの任意にインターリーブされたシーケンスを処理できることを示した。そして VLM がコンテキスト内で学習できることを示した — 3つの例 (画像、キャプション) ペアを持つ少数ショットプロンプトを与えて、勾配ステップなしで新しい画像をキャプション。メカニズム：ゲートクロスアテンションレイヤー、凍結 LLM の既存レイヤー間に挿入、学習された tanh ゲート初期化時ゼロなので LLM のテキスト機能は初期化で保持。このレッスンは Flamingo の Perceiver リサンプラーとゲートクロスアテンションアーキテクチャ — Gemini のインターリーブされた入力と Idefics2 の視覚トークンの祖先を説明。

**タイプ:** 学習
**言語:** Python (標準ライブラリ、ゲートクロスアテンション + Perceiver リサンプラーデモ)
**前提条件:** Phase 12 · 03 (BLIP-2 Q-Former)
**所要時間:** 約120分

## 学習目標

- ゲートクロスアテンションがどのように tanh(gate) = 0 経由で初期化で凍結 LLM のテキスト機能を保持するか説明。
- Perceiver リサンプラーを通る：N 画像パッチ → K 固定「潜在」クエリ (クロスアテンション経由)。
- Flamingo がキャジュアルマスキングを尊重する画像配置を使用してインターリーブされた画像テキストシーケンスを処理方法を説明。
- 少数ショットマルチモーダルプロンプト構造を再現 (3 画像キャプション例次にクエリ画像)。

## 問題

BLIP-2 は 32 個の視覚トークンを凍結 LLM の入力レイヤーに供給。単一画像あたりで機能。だが "here is image A, caption it; here is image B, caption it; now here is image C, caption it" のような、多くの画像をテキストでインターリーブするか？LLM の自己アテンションは単一ストリームの画像トークンとテキストトークンを処理する必要があり、どの位置がどの画像に注意できるかの質問は面倒。

Flamingo の答え：LLM の入力ストリームをまったく変えない。既存 LLM ブロック間に余分なクロスアテンションレイヤーを挿入。テキストトークンはいつものように LLM の因果自己アテンションを通じてフロー。すべてのいくつかの LLM ブロック間、テキストトークンもニューゲートレイヤー経由で画像特徴へのクロスアテンション。ゲート (初期化ゼロ) は段階ゼロで新規レイヤーがノーオプ — モデルは事前学習 LLM と同じように動作。トレーニング進行するとゲートが開き、視覚情報フロー開始。

2番目の質問 Flamingo は回答：可変数の画像 (0、1、または多く) あたりをプロンプトで処理か？Perceiver リサンプラー — 小さなクロスアテンションモジュール、パッチ任意数を受け取り、固定数の視覚潜在トークン生成。LLM クロスアテンションレイヤーはプロンプトに何個イメージがあるかに関わらず同じ形。

## コンセプト

### 凍結 LLM

Flamingo は凍結 Chinchilla 70B LLM から開始。すべての 70B 重みは手付かず。既存テキスト自己アテンションと FFN は通常運用。

### Perceiver リサンプラー

プロンプトの各イメージについて、ViT は N パッチトークン生成。Perceiver リサンプラーは K 固定学習可能潜在 (Flamingo は K=64 使用)。各リサンプラーブロックは2つのサブステップ：

1. クロスアテンション：K 潜在は N パッチトークン上で注意 (Q は潜在から、K/V はパッチから)。
2. 潜在内の自己アテンション + FFN。

6 リサンプラーブロック後、出力は K=64 視覚トークン (次元 1024)、ViT がいくつパッチ生成かかかわらず。224x224 画像 (196 パッチ) と 480x480 画像 (900 パッチ) 両方が 64 リサンプラートークンとして出力。

ビデオの場合、リサンプラーは時間的に適用：各フレームのパッチが 64 潜在生成、時間的ポジショナルエンコーディングがモデルを t=0 から t=N 区別させ。完全ビデオは T * 64 視覚トークンになる。

### ゲートクロスアテンション

凍結 LLM の M レイヤー間ごと (Flamingo は M=4 使用)、新規ゲートクロスアテンションブロック挿入：

```
x_after_llm_block = llm_block(x_before)
cross = cross_attn(x_after, resampler_output)
gated = tanh(alpha) * cross + x_after
x_before_next_block = gated
```

- `alpha` は学習可能スカラー初期化ゼロ。
- `tanh(0) = 0`、初期化時ゲートブランチ寄与ゼロ。
- `alpha` ゼロから遠ざかるとき、クロスアテンション寄与滑らかに成長。
- 残差接続は完全に開いたゲートでも LLM のテキスト表現を上書きしない；単に上の視覚情報追加。

これは Flamingo で単一最も重要な設計選択：視覚条件付けは加算、ゲート、初期化ゼロ。段階ゼロの Flamingo は テキストのみ入力で完璧な Chinchilla 70B。

### インターリーブされた入力用のマスクされたクロスアテンション

"<image A> caption A <image B> caption B <image C> ?" のようなプロンプトで、各テキストトークンはシーケンスで前に来た画像だけを見るべき。クロスアテンションマスクは強制：位置 `t` のテキストトークンは画像リサンプラートークン注意は画像インデックス `i < i_t` (ここで `i_t` は位置 `t` 前の最新画像)。「最新先行画像だけ見る」または「すべての先行画像を見る」両方が有効；Flamingo は前者を選択。

### コンテキスト内少数ショット学習

Flamingo プロンプトは次のようなもの：

```
<image1> A photo of a cat. <image2> A photo of a dog. <image3> A photo of a
```

モデルは完成パターンを見て出力 "bird" (または image3 が示すもの)。勾配ステップなし。凍結 LLM のコンテキスト内学習能力はゲートクロスアテンション経由で運ぶ — これはペーパーのパンチラインと重要理由。

### トレーニングデータ

Flamingo は3つのデータセット上でトレーニング：

1. MultiModal MassiveWeb (M3W)：43M ウェブページ、インターリーブ画像とテキスト、読取り順序再構築。
2. 画像テキストペア (ALIGN + LTIP)：4.4B ペア。
3. ビデオテキストペア (VTP)：27M 短ビデオクリップ。

OBELICS (2023) はインターリーブされたウェブコーパスの開放再現、Idefics、Idefics2、ほとんどの開く "Flamingo のような" モデルがトレーニング。

### OpenFlamingo と Otter

OpenFlamingo (2023) は開放再現。アーキテクチャ同一 (Perceiver リサンプラー + ゲートクロスアテンション凍結 LLaMA または MPT)。チェックポイント 3B、4B、9B。品質はより小さいベース LLM と少ないデータによる Flamingo の後ろ。

Otter (2023) は OpenFlamingo 上に構築し、MIMIC-IT (マルチモーダル命令のデータセット) による命令チューニング、ゲートクロスアテンションは命令追従でも動作を示す。

### 子孫

- Idefics / Idefics2 / Idefics3：Hugging Face のゲートクロスアテンション系統、次第にシンプル (Idefics2 は適応プール直接パッチトークンに有利なリサンプラーをドロップ)。
- Flamingo から Chameleon トランジション：2024年までに多くのチームは早期融合に移動 (Lesson 12.11)；Flamingo スタイルゲートクロスアテンション本番環境で残る バックボーン凍結が必要のとき。
- Gemini のインターリーブされた入力：概念的に Flamingo のインターリーブフォーマット柔軟性を引き継ぐ、正確なメカニズムは専有。

### BLIP-2 との比較

| | BLIP-2 | Flamingo |
|---|---|---|
| ビジュアルブリッジ | Q-Former 1回入力で | ゲートクロスアテンション M レイヤーごと |
| ビジュアルトークン | イメージあたり32 | クロスアテンションレイヤーあたり64イメージ |
| 凍結 LLM | はい | はい |
| 少数ショットコンテキスト内 | 弱 | 強 — ペーパー中心 |
| インターリーブされた入力 | ネイティブサポートなし | はい、設計対象 |
| トレーニングデータ | 130M ペア | 1.3B ペア + 43M インターリーブページ |
| パラメータカウント | 188M トレーニング | ～10B トレーニング (クロスアテンションレイヤー) |
| コンピュート | 8 A100s で日 | thousands TPUv4 で週 |

単一画像 VQA を予算で選択 BLIP-2。インターリーブ、少数ショット、または マルチ画像推論のために Flamingo/Idefics2 を選択。

## これを使用する

`code/main.py` は実演：

1. 36 偽パッチトークン上の Perceiver リサンプラー、8 学習可能潜在 (純粋 Python クロスアテンション)。
2. ゲートクロスアテンションステップ、`alpha = 0` → 出力等しい入力 (LLM 変更なし)、`alpha = 2.0` → 視覚寄与混入。
3. インターリーブマスク ビルダー、"(image 1) (text 1) (image 2) (text 2)" シーケンス向けの 2D アテンションマスク生成。

## 配布する

このレッスンは `outputs/skill-gated-bridge-diagnostic.md` を生成。開く VLM の設定 (リサンプラー Y/N、クロスアテンション頻度、ゲートスキーム) が与えられたとき、Flamingo 系統要素を特定し、凍結戦略を説明。ファインチューンがテキスト性能を劣化なぜ理由 (答え：ゲートが開きすぎた、あまりに速く)をデバッグするのに有用。

## 演習

1. Flamingo-9B の視覚パラメータカウント計算：9B LLM + 1.4B ゲートクロスアテンション + 64M リサンプラー。トレーニング済みパラメータの全体の何割か？

2. PyTorch で ゲート残差 `y = tanh(alpha) * cross + x` を実装。`alpha=0` で `y==x` は正確に初期化実験的に示す。

3. OpenFlamingo Section 3.2 (arXiv:2308.01390) 読む、各プロンプトが異なるイメージカウント持つ場合にバッチで複数画像処理方法。パディング戦略を説明。

4. Flamingo のクロスアテンションマスク なぜ テキストトークンに *最新先行イメージだけ* 注意させるのではなく、すべての先行イメージ？Flamingo ペーパー Section 2.4 を読み、トレードオフを説明。

5. コンテキスト内少数ショット："image → main object color" の4例を持つプロンプト構築、新規 Flamingo バリアント用。例の数が 0 から 8 に変化するとき、期待精度パターンを説明。

## キーターム

| 用語 | 人は何と言うか | 実際の意味 |
|------|----------------|----------|
| Perceiver リサンプラー | 「固定潜在クロスアテンション」 | 入力パッチの可変数から K 固定トークン生成モジュール |
| ゲートクロスアテンション | 「Tanh ゲートブリッジ」 | 残差レイヤー `y = tanh(alpha)*cross + x`、学習可能 alpha、初期化ゼロ |
| インターリーブされた入力 | 「混合シーケンス」 | 画像とテキスト自由に読取り順序で混合プロンプトフォーマット |
| 凍結 LLM | 「LLM 勾配なし」 | テキスト LLM の重みは更新されない；リサンプラー + クロスアテンションレイヤーだけトレーニング |
| 少数ショット | 「コンテキスト内例」 | 少数 (画像、答え) ペアをプロンプトで与える；モデルはファインチューニング無しで一般化 |
| OBELICS | 「インターリーブウェブコーパス」 | オープンデータセット 141M ウェブページ、画像とテキスト読取り順序 |
| Chinchilla | 「70B 凍結ベース」 | Flamingo の凍結テキスト LLM、DeepMind の Chinchilla ペーパーから |
| ゲートスケジュール | 「Alpha がどう動くか」 | トレーニング中のクロスアテンションゲートがどのレートで開くか |
| クロスアテンション頻度 | 「M レイヤーごと」 | ゲートクロスアテンションブロックが挿入どのくらい頻繁；Flamingo は M=4 使用 |
| OpenFlamingo | 「開放再現」 | MosaicML/LAION 開く 3-9B チェックポイント；アーキテクチャ同一 Flamingo |
