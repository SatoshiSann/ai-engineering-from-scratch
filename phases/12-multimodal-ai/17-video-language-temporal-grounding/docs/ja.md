# ビデオ言語モデル：時間トークンとグラウンディング

> ビデオは写真のスタックではない。5秒のクリップは因果順序、アクション動詞、イベント時間を画像モデルが表現できないもの。Video-LLaMA（Zhang et al.、2023年6月）は音声ビジュアルグラウンディングでの最初のオープンビデオLLMを出荷。VideoChatと Video-LLaVAはパターンをスケール。2025年でQwen2.5-VLのTMRoPEは最前線所有モデルとのギャップを閉じた。各システムは時間トークンを異なるように解決 — クリップあたりQ-former、フレームあたりconcat-pool、トークンあたりTMRoPE。このレッスンはパターンを読み、均一対動的フレームサンプラーを構築、時間グラウンディングタスクで評価。

**タイプ:** ビルド
**言語:** Python (stdlib、フレームサンプラー+時間グラウンディング評価者)
**前提条件:** Phase 12 · 08 (LLaVA-OneVision)
**所要時間:** 約180分

## 学習目標

- 時間ポジショナルエンコーディングがビジョンエンコーダから独立してビデオVLM性能を変更する理由を説明できる。
- 均一、動的FPS、イベント駆動フレームサンプリングをトークン毎秒対グラウンディング正確性で比較できる。
- Q-formerクリップあたり（Video-LLaMA）対プール化フレームあたり（Video-LLaVA）対M-RoPEトークンあたり（Qwen2.5-VL）デザインを説明できる。
- 4つのビデオベンチマークの名前：VideoMME、TempCompass、EgoSchema、Video-MMMU。

## 問題

30 FPSで1分ビデオは1800フレーム。ViT-Bで224フレームあたり196ビジュアルトークン、そりは352kトークン — 2024エラLLMコンテキストより大きい。

3つの削減戦略が存在：

1. サブサンプルフレーム（コンテンツに応じ1-8 FPS）。
2. 各フレームのパッチトークンを積極的にプール（3x3または4x4バイリニアプール）。
3. 16フレームクリップを取るQ-formerを通じて圧縮、64トークンを出力。

各トレードオフは異なる。サブサンプリングは時間詳細を失う。プーリングは空間詳細を失う。Q-formerは両方を少し失うが、トークンを節約。

時間ポジション エンコーディングは他の軸：モデルはフレーム5がフレーム6の前に来たことを知っているか？選択肢は単純な1D時間RoPE（Video-LLaMA）、学習時間埋め込み（Video-LLaVA）、TMRoPE（Qwen2.5-VL、完全3D）を含む。

## コンセプト

### Video-LLaMA: クリップあたりQ-former+オーディオブランチ

Video-LLaMA（2023）は最初のオープンビデオLLM。アーキテクチャ：

- 2 FPSで16フレームクリップ（したがって8秒）。
- フレームあたりViTフィーチャー -> ビデオQ-formerはすべての16フレームを横切る -> 32学習クエリ -> LLM。
- 並列オーディオブランチ：ウェーブフォーム -> ImageBind オーディオエンコーダ -> オーディオQ-former -> 32クエリ -> LLM。

強さ：オーディオビジュアル関節推論。弱さ：固定クリップ長、任意時間グラウンディングなし。

### VideoChat と Video-LLaVA

VideoChatはVideo-LLaMAアイデアを保ったが、オーディオをドロップ、単純化。Video-LLaVA（Lin et al.、2023）は単一ビジュアルエンコーダを画像とビデオフレーム両方で訓練（「投影前アライメント」）、統一表現を与える。両方は固定CLIP エンコーダ+MLP+LLM。

どちらは長ビデオを処理しない。両方は8-16フレームシステム。

### Qwen2.5-VL と TMRoPE

Qwen2.5-VLはTMRoPE — 時間モダリティ回転ポジション埋め込みを導入。各パッチトークンは（t、h、w）ポジション（フレームインデックスではなくtは実際のタイムスタンプ）をキャリー。

単純時間埋め込みからのキー差異：

- 絶対時間、インデックスではなく。モデルは「フレーム15」ではなく「4.2秒で」を見る。
- クリップあたり、トークンあたり。各ビジュアルトークンはそのタイムスタンプにより独立して回転。
- 動的FPSと互換。ここで2 FPSでサンプルしたら、TMRoPEは不均一スペーシングネイティブに処理。

TMRoPE「猫はどの秒にジャンプするか？」クエリを可能にする。モデルは「4.2秒で」を出力できる。Video-LLaMAは「クリップの早い」しか言うことができなかった。

### フレームサンプリング戦略

均一：期間を横切ってN フレームを均等にサンプル。単純、モーション頂上を失う。

動的FPS：モーション強度に基づいて適応的にサンプル。光学フロー、またはフレーム差異は高モーション分割をより密なサンプリングピック。Qwen2.5-VLはこれで訓練。

イベント駆動：軽量検出器を実行、アクションが起きる所でより多くサンプル。VideoAgentで使用。

キーフレーム+コンテキスト：ショット境界で、加えて隣接フレーム数をサンプル。映画的コンテンツで使用。

### フレームあたりプーリング

1 FPSとフレームあたり576トークン、5分クリップは172,800トークン。Qwen2.5-VL-72Bの128kコンテキストで可能しかし高価。

3x3バイリニアプール64トークンにフレームあたり削減 -> 5分のための19,200トークン。ほとんどのタスクのための甘い場所。

より積極的にプール（6x6 -> フレームあたり16トークン）エージェントワークフローのため空間詳細はより少ない。

### 4つのビデオベンチマーク

- VideoMME：包括的ビデオ理解、短い+中間+長い。
- TempCompass：細粒度時間推論、「前」/「後」質問。
- EgoSchema：長地平線一人称ビデオ。
- Video-MMMU：マルチモーダル多学問ビデオ質問。

完全ビデオVLM評価は全て4つを打つ。彼らは異なる軸を強調 — TempCompassはすべてについて順序、EgoSchemaは3+分推論について、VideoMMEは期間にまたがる。

### グラウンディング出力フォーマット

時間グラウンディングのための出力フォーマット：

- 自由テキスト：「猫は4秒マークの周り跳ぶ。」解析しやすいしかし不正確。
- 構造化JSON：`{"event": "jump", "start": 4.1, "end": 4.3}`。Qwen2.5-VLはこれを訓練。
- トークンベース：特別な`<time>4.1</time>`トークン回答で交互。Qwen2.5-VLの内部フォーマット。

トークンベースは下流ユースのため最も正確。Qwen2.5-VLのJSON出力フォーマットは直接解析。

### 2026最良実践

2026年のビデオVLMのため：

- エンコーダ：M-RoPEまたはTMRoPE（Qwen2.5-VL）でSigLIP 2。
- フレームサンプリング：動的FPS（モーションに応じ1-4）最大フレームキャップで。
- フレームあたりプーリング：3x3バイリニア。
- 出力：時間+イベントフィールドを持つ構造化JSON。
- ベンチマーク：一般のためのVideoMME+TempCompass；長地平線のためのEgoSchema。

## 使う

`code/main.py`は含む：

- 均一と動的FPSフレームサンプラー。
- おもちゃ時間グラウンディング評価者：時間Tと「基本真実」イベント与えられて、許容度で精度スコア。
- Video-LLaMA（16フレーム、Q-former）、Video-LLaVA（8フレーム、MLP）、Qwen2.5-VL（動的FPS+TMRoPE）を横切る比較。

## デプロイ

このレッスンは`outputs/skill-video-vlm-frame-planner.md`を作成。ビデオタスク（監視、アクション認識、時間グラウンディング、要約）を与えられて、それはフレームサンプラー、プーリング係数、出力フォーマット、予想正確性ティアをピック。

## 演習

1. 3分調理デモのため、均一対動的FPSをピック。トークン数で正当化。

2. TMRoPEは何を追加、単純な時間埋め込みテーブルが何ができず？

3. VLMが出力を学ぶことができるJSON スキーマを時間グラウンディングのため書く。エラーケースを含む。

4. Video-LLaVAのセクション3を「投影前アライメント」について読む。なぜこれは別の画像とビデオエンコーダを訓練するより良い？

5. VideoMMEリーダーボードを与えられて、2026年現在、最高オープンモデルと最高所有モデル間のギャップは何？そのギャップの多くは時間エンコーディング対ベースLLMスケールに帰属可能？

## キー用語

| 用語 | 人が言うこと | それが実際に意味すること |
|------|-----------------|------------------------|
| 時間グラウンディング | 「時間局所答」 | VLMはイベントが起きるときのために特定タイムスタンプ範囲を出力 |
| TMRoPE | 「時間マルチモーダルRoPE」 | 3D回転ポジション絶対タイムスタンプ、Qwen2.5-VLで使用 |
| 動的FPS | 「モーション認識サンプリング」 | 高モーション分割でより多くのフレーム、静的なものでより少ないサンプル |
| フレームプーリング | 「フレームあたり空間圧縮」 | LLM前にバイリニア補間でフレームあたりパッチを削減 |
| ビデオQ-former | 「クリップ圧縮器」 | Nフレームをk学習クエリにマッピング横切る注意ボトルネック |
| VideoMME | 「ビデオベンチ」 | 包括的短/中間/長ビデオベンチマーク、2500+サンプル |
