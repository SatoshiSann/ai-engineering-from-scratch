# CLIP から BLIP-2 へ — Q-Former をモダリティブリッジとして

> CLIP は画像とテキストを配置するが、キャプション生成、質問応答、会話をすることはできない。BLIP-2 (Salesforce, 2023) はそれを小さなトレーニング可能なブリッジで解決した：32個の学習可能なクエリベクトルが、クロスアテンションを介して凍結 ViT の特徴に対して注意を払い、その後、凍結 LLM のインプットストリームに直接スロット。188M パラメータのブリッジが 11B LLM を ViT-g/14 に接続した。2026 年までのすべてのアダプタベース VLM — MiniGPT-4、InstructBLIP、LLaVA の親戚 — は子孫である。このレッスンでは Q-Former のアーキテクチャを読み、その2段階のトレーニングを説明し、凍結テキストデコーダへの視覚トークンを供給するおもちゃバージョンを構築する。

**タイプ:** ビルド
**言語:** Python (標準ライブラリ、クロスアテンション + 学習可能クエリデモ)
**前提条件:** Phase 12 · 02 (CLIP)、Phase 7 (トランスフォーマー)
**所要時間:** 約180分

## 学習目標

- トレーニング可能なボトルネックが凍結視覚エンコーダと凍結 LLM の間でエンドツーエンドのファインチューニングよりもコストと安定性の面で勝つ理由を説明する。
- 固定された学習可能クエリのセットが外部画像特徴に対して注意を払うクロスアテンションブロックを実装する。
- BLIP-2 の2段階事前学習 (表現 ITC + ITM + ITG、次に生成 LM 損失、凍結デコーダ) を実行する。
- Q-Former を LLaVA で使用されるより単純な MLP プロジェクターと比較し、各選択がいつ勝つかを主張する。

## 問題

凍結 ViT があり、画像あたり 256 パッチトークン (次元 1408) を生成する。凍結 7B LLM があり、次元 4096 のトークン埋め込みを期待する。明白なブリッジ — 1408 から 4096 への線形層 — 機能するが、すべての 256 パッチトークンを LLM のコンテキストに供給することは、画像あたり 256 の追加トークンの費用がかかる。32 画像のバッチについて、それは視覚モダリティだけで 8192 トークン消費される。

BLIP-2 の質問：256 トークンの画像表現をはるかに少ないトークン (たとえば 32) に圧縮することができるが、LLM がキャプション、質問応答、画像について推論するのに十分な情報を保持することができるか？そして、凍結バックボーンに触れずにこのブリッジをトレーニングすることができるか、トレーニングコストをブリッジのパラメータだけに保つか？

答え：Q-Former。32 個の学習可能な「クエリ」ベクトル、ViT のパッチトークンにクロスアテンションし、LLM が消費する 32 トークン視覚要約を生成する。合計 188M パラメータ。凍結 LLM に触れる前に、対比、マッチング、生成目的でトレーニング。

## コンセプト

### 学習可能クエリ

Q-Former の中核的なトリック：LLM のテキストトークンが画像パッチに対して注意を払わせるのではなく、32 個の学習可能なクエリベクトル `Q` の新しいセットを導入し、*それらが* 画像パッチに対して注意を払わせる。クエリはモデルのパラメータである — それらはトレーニング中に学習され、同じ 32 個のクエリがすべての画像に使用される。

クロスアテンションの後、各クエリは画像の圧縮要約を保持する — 「メインオブジェクトを説明する」、「背景を説明する」、「オブジェクトを数える」など。クエリは文字通りセマンティックラベルに特化しない；それらはダウンストリーム損失を低下させる任意のエンコーディングを学習する。

### アーキテクチャ

Q-Former は小さなトランスフォーマー (12レイヤー、～100M パラメータ) で2つのパスがある：

1. クエリパス：32 クエリベクトルは自己アテンション (それら自身の間) を介してフロー、次にクロスアテンション (凍結 ViT のパッチトークンを超える)、次に FFN。
2. テキストパス：BERT のようなテキストエンコーダはクエリパスと自己アテンションと FFN の重みを共有。クロスアテンションはテキストパスで無効。

トレーニング時の両方のパスが実行される。クエリとテキストは共有自己アテンションを介して相互作用し、クエリはテキストを条件付けることができることを意味する (ITM、ITG が必要なタスク用)。VLM ハンドオフの推論時に、クエリだけが フロー、32 視覚トークンを生成。

### 2段階トレーニング

BLIP-2 は2段階で事前学習する：

段階1：表現学習 (LLM なし)。3つの損失：
- ITC (画像テキスト対比)：CLIP スタイルの対比 (プールされたクエリトークンとテキスト CLS トークン間)。
- ITM (画像テキストマッチング)：二項分類器 — この画像テキストペアはマッチか？困難なネガティブマイニング。
- ITG (画像グラウンデッドテキスト生成)：テキストのキャジュアル LM ヘッド、クエリ条件付き。クエリがテキスト生成可能コンテンツをエンコードするよう強制。

Q-Former だけがトレーニング。ViT は凍結。LLM は関与しない。

段階2：生成学習。凍結 LLM (OPT-2.7B または Flan-T5-XL など) をアタッチ。32 クエリ出力を小さな線形層を介して LLM の埋め込み次元に投影。それらをテキストプロンプトの前に付加。結合プロンプト + 画像 + キャプションシーケンスの LM 損失で線形投影と Q-Former だけをトレーニング。

段階2 の後、Q-Former + 投影が完全な視覚アダプター。推論時：画像 → ViT → Q-Former → 線形投影 → テキストに付加 → 凍結 LLM が出力を放出。

### パラメータ経済

BLIP-2 ViT-g/14 (1.1B、凍結) + OPT-6.7B (6.7B、凍結) + Q-Former (188M、訓練) = 8B 合計、188M トレーニング。Q-Former だけが完全なスタックのパラメータの～2.4%。トレーニングコストはこれを反映：A100s の一握りで日数対数週間 (エンドツーエンド)。

品質：BLIP-2 は Flamingo-80B 上でゼロショット VQA と一致またはそれを上回り、50倍小さい。ブリッジは機能する。

### InstructBLIP と命令認識クエリ

InstructBLIP (2023) は Q-Former を追加入力で拡張：命令テキスト自身。クロスアテンション時、クエリは現在イメージパッチと命令の両方へのアクセスを持つ。クエリは命令ごとに特化できる (「車を数える」、「気分を説明する」) むしろ単一の固定要約を学ぶことの。ホールドアウトタスク上のベンチマークゲイン。

### MiniGPT-4 とプロジェクターのみアプローチ

MiniGPT-4 は Q-Former を保持したが、他のすべてを凍結しながら出力線形投影だけをトレーニング。安価だが、コストは品質 — クエリは BLIP-2 の、あなたのではない。高速イテレーション用に良い、最高のアーキテクチャではない。

### LLaVA がなぜ単純になったか

LLaVA (2023、Lesson 12.05) は Q-Former を、すべての ViT パッチトークンを LLM 空間に投影する プレーン2レイヤー MLP に置き換えた — 24x24 グリッド用に 576 トークンすべてが LLM に供給。より悪い圧縮だが LLM は生のパッチに対して注意を払わせる。当時これは物議を醸す；2023年後半までには LLaVA-Instruct-150k（視覚命令データ）が MLP が十分なシグナルを保持するようトレーニングできることを証明したため、支配的。トレードオフ：LLaVA のコンテキストはより速く満たされるが、マルチ画像とビデオに自然にスケーリング。

2026年までに、フィールドは分割：Q-Former はトークン予算が重要なとき (長いビデオ、多くの画像) 生き残る；MLP プロジェクターは生データ品質トークンあたりが優先のとき支配的。

### ゲートクロスアテンション：Flamingo、祖先

Flamingo (Lesson 12.04) は BLIP-2 に先行し、同じクロスアテンションアイデアを使用したが、単一ブリッジではなく凍結 LLM レイヤーのすべてで。BLIP-2 は入力レイヤーのみに圧縮してもまだ機能することを示した。Gemini と Idefics は両方を結合：インターリーブされた入力トークン + コンテキスト内少数ショット用のオプショナルゲートクロスアテンション。

### 2026 の子孫

- Q-Former：BLIP-2、InstructBLIP、MiniGPT-4、ほとんどのビデオ言語モデル (トークン予算理由)。
- Perceiver リサンプラー：Flamingo のバリアント (Lesson 12.04)；Idefics ファミリー、Eagle、OmniMAE。
- MLP プロジェクター：LLaVA、LLaVA-NeXT、LLaVA-OneVision、Cambrian-1。
- アテンションプール：VILA、PaliGemma。

すべて4つが有効。決定質問はトークン予算に制約されているか、トークン品質あたりか。

## これを使用する

`code/main.py` は標準 stdlib Q-Former スタイルクロスアテンションを構築：

1. 256 イメージパッチトークン (次元 128) をシミュレート。
2. 32 個の学習可能クエリ (次元 128) をインスタンス化。
3. スケーリングドット積クロスアテンション実行 (Q はクエリ、K/V はパッチ)。
4. LLM-次元 (512) への線形層経由で投影。
5. 32 LLM-準備完了視覚トークン出力。

純粋な Python のすべての数学 (ベクトル上のネストループ)。おもちゃだが正しい形。アテンション重みマトリックスが印字されるので、各クエリがどのパッチから引き出されたかを見ることができる。

## 配布する

このレッスンは `outputs/skill-modality-bridge-picker.md` を生成する。ターゲット VLM 設定 (視覚エンコーダトークンカウント、LLM コンテキスト予算、デプロイメント制約、品質目標) が与えられたとき、Q-Former vs MLP vs Perceiver リサンプラーを簡潔な正当化とブリッジごとのパラメータカウント推定で推奨。

## 演習

1. PyTorch でクロスアテンションブロックを実装。32 クエリと 256 キー/値で、アテンション重みマトリックスが 32 x 256 で各行が softmax 後に 1 に合計することを検証。

2. BLIP-2 段階1では、Q-Former は3つの損失を同時に実行：ITC、ITM、ITG。各疑似コード前方署名を書く。どれがテキストエンコーダパスをアクティブにする必要があるか？

3. パラメータカウント比較：Q-Former (12レイヤー、768 隠し) vs 2層 MLP プロジェクター (1408 → 4096、2レイヤー)。LLM スケールでの 188M Q-Former コスト (トレーニング効率で) 払い戻しか？

4. BLIP-2 論文の Section 3.2 (arXiv:2301.12597) を読む Q-Former 初期化について。BERT-base から初期化 (ランダムではなく) 収束を加速なぜか説明。

5. 1 FPS サンプリングされた10分ビデオ 60 フレームについて、フレームあたりトークンコストを計算 (Q-Former → 32 トークン/フレーム) vs (MLP プロジェクター → 576 トークン/フレーム)。どちらが 128k トークン LLM コンテキストウィンドウにフィット？

## キーターム

| 用語 | 人は何と言うか | 実際の意味 |
|------|----------------|----------|
| Q-Former | 「クエリトランスフォーマー」 | 凍結 ViT 特徴上でクロスアテンション 32 学習可能クエリベクトル小さいトランスフォーマー |
| 学習可能クエリ | 「ビジョン用ソフトプロンプト」 | クロスアテンション クエリ側として機能するパラメータ固定セット；モデルごとに学習、すべての入力横断で共有 |
| クロスアテンション | 「Q はここから、K/V そこから」 | アテンション Q、K、V が異なるソースから；クエリがViT パッチを引き出し方法 |
| ITC | 「画像テキスト対比」 | Q-Former プールクエリ vs テキスト CLS に適用 CLIP スタイル損失 |
| ITM | 「画像テキストマッチング」 | 困難なネガティブマイニングペア上の二項分類器；クエリを細粒ミスマッチを判別するよう強制 |
| ITG | 「画像グラウンデッドテキスト生成」 | キャジュアル LM 損失テキストはクエリ条件付きで生成；クエリはテキストデコード可能コンテンツをエンコード強制 |
| 2段階事前学習 | 「表現次に生成」 | 段階1は Q-Former だけトレーニン (ITC/ITM/ITG)；段階2は凍結 LLM アタッチ、投影 + Q-Former だけトレーニング |
| 凍結バックボーン | 「ファインチューンしない」 | ビジョンエンコーダと LLM 重みは固定；ブリッジだけがトレーニング |
| 投影ヘッド | 「LLM 次元への線形」 | Q-Former 出力を LLM の埋め込み次元にマップ最終線形層 |
| Perceiver リサンプラー | 「Flamingo のバージョン」 | 類似の学習可能クエリクロスアテンション、Flamingo によって単一ブリッジではなく各レイヤーで使用 |
