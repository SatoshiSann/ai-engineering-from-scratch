# ColPali とビジョンネイティブ ドキュメント RAG

> 伝統的なRAG はPDFをテキストに解析、チャンク に分割、チャンクを埋め込み、ベクトルを保存。すべてのステップはシグナルを失う：OCR はチャート データをドロップ、チャンキング はテーブル行を破壊、テキスト埋め込みはフィギュアを無視。ColPali（Faysse et al., 2024年7月）は簡単な質問をした：なぜ、まず最初にテキストを抽出？ PaliGemma 経由でページイメージを直接埋め込み、収索用にColBERTスタイル後期相互作用を使用、ドキュメントが運ぶレイアウト、フィギュア、フォント、フォーマット すべてを保持。発表ベンチマーク：視覚的に豊か なドキュメント上のテキストRAGより20-40%優れた エンドツーエンド精度。ColQwen2、ColSmol、VisRAG はパターンを拡張。本レッスンはビジョンネイティブRAG 論文を読み、小さいColPaliのようなインデクサーを構築。

**タイプ:** ビルド
**言語:** Python（stdlib、マルチベクトル インデクサー + MaxSim スコアラー）
**前提条件:** Phase 11（LLM エンジニアリング — RAG 基本）、Phase 12 · 05（LLaVA）
**所要時間:** 約180分

## 学習目標

- バイ エンコーダ 収索（ドキュメント当たり1つのベクトル）と後期相互作用収索（ドキュメント当たり多くのベクトル）の違いを説明。
- ColBERT のMaxSim操作を説明し、どうColPali がテキストトークンからイメージパッチに一般化。
- 小さいColPaliのようなインデクサーを構築：ページ -> パッチ埋め込み -> クエリタームの埋め込みに対するMaxSim -> トップk ページ。
- インボイス/財務レポート ユースケースでColPali + Qwen2.5-VLジェネレータ 対 テキストRAG + GPT-4 を比較。

## 問題

PDFでのテキストRAG はほとんどのドキュメントを捨てる。財務レポートのQ3売上成長は通常、チャート にあり；医療レポートの見つけは注釈イメージにあり；法律契約の署名ブロックはレイアウト事実、テキスト事実ではない。

テキストRAG パイプライン：

1. PDF -> OCR / pdftotext 経由のテキスト。
2. テキスト -> 300-500トークン チャンク。
3. チャンク -> バイ エンコーダ埋め込み（1ベクトル）。
4. ユーザー クエリ -> 埋め込み -> コサイン相似性 -> トップk チャンク。
5. チャンク + クエリ -> LLM。

5つの損失ステップ。チャートはキャプチャされない。テーブルはチャンク全体で破壊。マルチ列レイアウトは平坦化。フィギュア注釈は消える。

ColPali の修正：OCR をスキップ、ページイメージを直接埋め込み。收索用ColBERTスタイル後期相互作用を使用して、モデルはクエリ時にファイングレイン パッチに注意可能。

## コンセプト

### ColBERT（2020）

ColBERT（Khattab & Zaharia、arXiv:2004.12832）はテキスト収索方法。ドキュメント当たり1つのベクトル の代わりに、トークン当たり1つのベクトルを生成。クエリ時間：

- クエリ トークンは独自の埋め込みを得る（N_q ベクトル）。
- ドキュメント トークンは埋め込みを得る（N_d ベクトル、通常キャッシュ）。
- スコア = コサイン相似性のクエリ トークン全体のマックス ドキュメント トークン全体のサム：Σ_i max_j cos(q_i, d_j)。

これはMaxSim 操作。各クエリ トークンはそれの最高一致ドキュメント トークンを「ピック」。最終スコアは合計。

長所：強い再現率、タームレベル セマンティックスを処理。短所：ドキュメント当たりN_d ベクトル、保存は高価。

### ColPali

ColPali（Faysse et al., arXiv:2407.01449）はColBERT パターンをイメージに適用。

- 各ページはPaliGemma（ViT + 言語）でパッチ埋め込みにエンコード：ページ当たりN_p ベクトル。
- 各ユーザー クエリ（テキスト）はクエリタームの埋め込みにエンコード：N_q ベクトル。
- スコア = Σ_i max_j cos(q_i, p_j)、すなわち、クエリテキストトークンと ページイメージパッチのMaxSim。
- 合計スコアでトップk ページを取得。

ドキュメント摂取時間：PaliGemma ですべてのページを埋め込み、すべてのパッチ埋め込みを保存。クエリ時間：クエリトークンを埋め込み、すべての保存ページ埋め込みに対するMaxSim を計算、トップk ページを返す。

長所：エンドツーエンド テキストRAG をビジュアル豊かドキュメント上で20-40%を打つ。各パッチベクトル は局所レイアウトと コンテンツをキャプチャ。

短所：ページ当たりN_p パッチ × 4バイト浮動小数点 × D-dim ベクトル = 保存は速く成長。PQ / OPQ 量子化で軽減。

### ColQwen2 とColSmol

ColQwen2（illuin-tech、2024-2025）は PaliGemma をQwen2-VLと入れ替え。より良いベース エンコーダ、より良い収索。

ColSmol はより小さいスケール バリアント ローカル / エッジ使用のため。~1B params でのColSmol retriever はコンシューマー GPU で走る。

### VisRAG

VisRAG（Yu et al., arXiv:2410.10594）は別バリアント：パッチ上のMaxSim の代わりに、VLM を持つ1つのベクトルに各ページをプール その後バイ エンコーダ収索。速い インデックス化 + より小さい保存、弱い再現率。

品質対コストトレード オフ：品質対ColPali、スケール対VisRAG。

### M3DocRAG

M3DocRAG（Cho et al., arXiv:2411.04952）はマルチモーダル収索をマルチページ マルチドキュメント推論に拡張。ドキュメント全体でページを取得、VLM のマルチページ コンテキストを構成。

### ViDoRe — ベンチマーク

ColPali のコンパニオン ベンチマーク。Visual Document Retrieval Evaluation。タスクは財務レポート、科学論文、行政ドキュメント、医療レコード、マニュアルを含む。メトリック：nDCG@5。

ColPali-v1 はViDoRe 上で~80% nDCG@5 スコア；テキストRAG は同じドキュメントで~50-60%。

### エンドツーエンド RAG パイプライン

ビジョンネイティブ RAG のため：

1. 摂取：PDF -> ページイメージ -> PaliGemma エンコーディング -> すべてのパッチ埋め込みを保存。
2. クエリ：ユーザー テキスト -> クエリターム 埋め込み -> すべてのインデックス ページに対するMaxSim -> トップk ページ。
3. 生成：トップk ページ イメージ + クエリ -> VLM（Qwen2.5-VL またはClaude）-> 回答。

OCR はどこもない。フィギュア、チャート、フォント、レイアウト はすべて回答に流れ込む。

### 保存の数学

50ページ財務レポート729パッチ/ページと128-dim 埋め込みで：

- ColPali：50 * 729 * 128 * 4 バイト = ~18 MB raw、PQ後~4 MB。
- テキストRAG：50 チャンク * 768-dim * 4 バイト = ~150 kB。

ColPali は~30倍より多くドキュメント当たり保存。スケールで、OPQ / PQ は~5-10倍、通常許容可能に持つ。

### テキストRAG が勝つ時

- レイアウト シグナルなしの純テキスト ドキュメント（wiki記事、チャット ログ）。テキストRAG はシンプルおよび保存安価。
- ストレージが コスト支配する数百万ページ アーカイブ。
- VLM 幻覚隣接抽出可能OCRテキスト要求を監査する厳密な規制要件。

2026年のすべての他 — 財務レポート、科学論文、法律契約、医療レコード、UX ドキュメント — ビジョンネイティブ RAG が勝つ。

## 使用方法

`code/main.py`：

- おもちゃパッチ エンコーダ：「ページ」（機能ベクトルの小さいグリッド）をパッチ埋め込みのアレイにマップ。
- MaxSim スコアラー：クエリ トークン埋め込みセットとページ パッチセット間のColBERTスタイル スコアを計算。
- 5つのおもちゃ ページをインデックス化、3つのクエリを実行、スコアでトップk を返す。

## 提出

本レッスンは`outputs/skill-vision-rag-designer.md`を生成。ドキュメント RAG プロジェクトが与えられた場合、ColPali / ColQwen2 / VisRAG / テキストRAG を選択しストレージをサイズ。

## 演習

1. 729パッチ/ページ、128-dim emb、4バイト浮動小数点での200ページ年報。生のストレージと PQ圧縮（8倍）ストレージを計算。

2. MaxSim はΣ_i max_j cos(q_i, p_j)。この合計が単純な平均相似性がキャプチャしない何をキャプチャ？

3. ColPali はページをパッチセットとしてインデックス化。単語レベルでインデックス化する代わりに何が変わる（ColBERT のように）？トレード オフ?

4. 1Mページ コーパスで500ミリ秒/クエリレイテンシ 予算とのエンドツーエンド パイプラインを設計。ColQwen2 / VisRAG を選択し正当化。

5. M3DocRAG を読む（arXiv:2411.04952）。マルチページ注意パターンを説明し、ワンページColPali 収索と異なる方法。

## 主要な用語

| 用語 | 人々が言う | 実際の意味 |
|------|------------|---------|
| 後期相互作用 | 「ColBERTスタイル」 | トークンごと またはパッチごとの埋め込み + MaxSim、単一ドキュメント ベクトルでなく収索 |
| MaxSim | 「Max-over-patches」 | 各クエリ トークンのため、最高相似性ドキュメント トークンをピック；クエリ全体にわたって合計 |
| バイ エンコーダ | 「単一ベクトル」 | ドキュメント当たり1ベクトル；速いがグラナリティを失う |
| マルチベクトル | 「ドキュメント/ページ当たり多くのベクトル」 | ドキュメント/ページ当たりN_p ベクトルを保存；保存コスト成長だが再現率改善 |
| パッチ埋め込み | 「ページ特徴」 | VLM エンコーダからページごと1ベクトル イメージ パッチ、ページごと キャッシュ |
| ViDoRe | 「ビジョン ドキュメント ベンチ」 | ビジュアル ドキュメント 収索用ColPali のベンチマーク スイート |
| PQ 量子化 | 「製品 量子化」 | 圧縮 ベクトル相似性を保ちながら保存~8倍縮小 |
