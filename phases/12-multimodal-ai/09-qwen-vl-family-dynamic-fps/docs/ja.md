# Qwen-VL ファミリーと動的 FPS ビデオ

> Qwen-VL ファミリー — Qwen-VL (2023)、Qwen2-VL (2024)、Qwen2.5-VL (2025)、Qwen3-VL (2025) — 2026 年で最も影響力あるオープンビジョン言語モデルの系統。各世代は単一決定的アーキテクチャ賭け — 残りオープンエコシステムは 12 ヶ月内にコピー：M-RoPE のネイティブ動的解像度、動的 FPS サンプリング、絶対時間配置、ViT でのウィンドウアテンション、構造化エージェント出力フォーマット。Qwen3-VL によって、レシピ安定化：2D-RoPE-ViT エンコーダネイティブアスペクト比入力、MLP プロジェクター大きな Qwen3 言語ベースの中へ、訓練段階は OCR、グラウンディング、エージェント振る舞いを第一クラスターゲットとして強調。このレッスンは家族を時系列で読むのであなたは理解なぜすべてのノブはそれがある。

**タイプ:** 学習
**言語:** Python (標準ライブラリ、M-RoPE エンコーダ + 動的 FPS サンプラー)
**前提条件:** Phase 12 · 06 (patch-n'-pack)
**所要時間:** 約120分

## 学習目標

- 計算 M-RoPE の3軸回転 (時間、高さ、幅)，すべての3つがなぜ必要か説明。
- ビデオの動的 FPS サンプリング戦略を選択，トークン/秒 vs イベント検出精度について推論。
- オーダーで 4つの Qwen-VL 世代アップグレード名付け，各有効。
- Qwen2.5-VL スタイル JSON エージェント出力フォーマット ワイヤ，VLM 応答から構造化ツール呼び出し解析。

## 問題

Qwen-VL は 2023 年 8 月に LLaVA-1.5 と BLIP-2 への直接応答として配布。ギャップ Qwen チームがターゲットは3重：解像度、ビデオ、構造化出力。

解像度：LLaVA-1.5 は 336x336 で実行。写真のために良い、密な中国言語請求書またはスプレッドシート スクリーンショット用に無用。Qwen-VL の最初の革新は 448x448 とグラウンディングバウンディングボックス出力、モノを指さしデルモ位置する許可。

ビデオ：Video-LLaMA は /フレームエンコーダスタックして LLM に供給。短いクリップに機能、多くテンポラル軸がシグナルである分ビデオではなく。Qwen チームは時間を理解する単一エンコーダを望んだ。

構造化出力：LLaVA は自由形式テキスト放出。エージェントは JSON が必要。Qwen-VL は明示的 JSON 出力フォーマット上で訓練，バウンディングボックス座標をテキストとして含む。

すべての Qwen-VL 世代は これら3軸の 1つを拡張。

## コンセプト

### Qwen-VL (2023 年 8 月)

最初の世代：OpenCLIP ViT-bigG/14 エンコーダ (2.5B パラメータ)、Llama 互換 Q-Former (1 ステップ 256 クエリ)、Qwen-7B ベース。貢献：

- 448x448 解像度 (当時開く VLM の SOTA)。
- グラウンディング：明示的座標トークン出力を持つイメージテキスト ペアで訓練。「猫は <box>(112、204)、(280、344)</box> にある」。
- 中国語 + 英語多言語トレーニング開始から。

時点でベンチマーク：英語で GPT-4V と競争的、中国語で支配的。グラウンディング教師ありは本当のヘッドライン。

### Qwen2-VL (2024 年 9 月) — M-RoPE とネイティブ解像度

Qwen2-VL は固定解像度 + Q-Former スタックをネイティブ動的解像度 ViT エンコーダに置き換え。キー変更：

- ネイティブ動的解像度。ViT は任意 HxW を受け入れ、28 で除せる (パッチ 14 /2x 空間マージ)。1120x672 (40x24 マージパッチ) 画像は 960 ビジュアルトークン生成。リサイズなし、タイリングなし、サムネイルなし。
- M-RoPE (Multimodal RoPE)。各トークンは 3D ポジション (t、h、w) 1D ではなく運ぶ。イメージ t=0、ビデオ t = frame_index。RoPE は軸ごとに周波数によってクエリ/キーベクトルを回転。ポジショナル埋め込みテーブルなし。
- MLP プロジェクター。Q-Former をドロップ；マージパッチトークン上で 2層 MLP を使用。
- 動的 FPS を持つビデオ。ビデオは デフォルト 1-2 FPS でサンプリング、だが モデルは任意フレームカウント受け入れ。

結果：Qwen2-VL-7B がいくつかマルチモーダルベンチマーク上で GPT-4o にマッチし、DocVQA (94.5 vs 88.4) で打つ。アーキテクチャ変更は決定的動き。

### Qwen2.5-VL (2025 年 2 月) — 動的 FPS + 絶対時間

Qwen2.5-VL の大きなシフトはビデオ。動的 FPS は「必要なときより多くフレーム サンプル」だけではない。ペーパーは形式化：

- 絶対時間トークン。ポジショナル指標 (フレーム 0、1、2...) の代わり、実タイムスタンプ使用。「0:04 で、猫がジャンプ。」モデルは `<time>0.04</time>` トークンインターリーブ、フレームトークンで見。
- 動的 FPS。遅いフットの 1 FPS でサンプル、アクション 4+ FPS。ユーザーまたはトレーナーが選択；M-RoPE は適応。
- ViT でウィンドウアテンション。空間アテンションはウィンドウ (ブロック内ローカル) スループット；全アテンション各いくつレイヤー。
- 明示的 JSON 出力フォーマット。ツールコール データで訓練："{\"tool\": \"click\"、\"coords\": [380、220]}"。エージェント準備すぐ。
- MRoPE-v2 スケーリング。ポジション max 入力サイズでスケール、10 分ビデオは周波数範囲の外に実行しない。

ベンチマーク：Qwen2.5-VL-72B はほとんど ビデオベンチマーク上で GPT-4o を打つ、ドキュメント上に Gemini 2.0 マッチ，GUI グラウンディングのためにオープン VLM SOTA を設定 (ScreenSpot：84% 精度 vs 38% GPT-4o 用)。

### Qwen3-VL (2025 年 11 月)

Qwen3-VL はアーキテクチャ ではなく コンソリデートする段階的アップグレード：より大きい LLM バックボーン (Qwen3-72B)、拡張トレーニングデータ、改善 OCR、Qwen3 「思考モード」経由より強い推論。ViT と M-RoPE は滞在。ペーパーは コンピュート とトレーニング改善上に集中アーキテクチャをいない。

系統の仕事：2025 までに、Qwen-VL アーキテクチャ安定化していた。追加 世代はコンピュートとデータスケール、プリミティブではない。

### M-RoPE 数学的に

古典 RoPE は クエリ `q` ディメンション `d` ポジション `m` 使用ペア座標で回転：

```
q_rot[2i]   = q[2i]   * cos(m * theta_i) - q[2i+1] * sin(m * theta_i)
q_rot[2i+1] = q[2i]   * sin(m * theta_i) + q[2i+1] * cos(m * theta_i)
theta_i     = 10000^(-2i/d)
```

M-RoPE は隠しディム 3つのバンドに分割。言う `d = 96`。32 ディム をテンポラル、32 高さ、32 幅に指定。各バンドは軸ポジション自身で回転。パッチ (t=5、h=10、w=20) で回転 `R_t(5)`、`R_h(10)`、`R_w(20)` それの3つのバンドに適用。

テキストトークンは `t = text_index、h = 0、w = 0` (または正規化選択) を使用、互換性保つ。ビデオフレームは `t = frame_time、h = 行、w = 列` を使用。単一イメージは `t = 0` を使用。

利点：1つのポジション符号化はテキスト、イメージ、ビデオ をハンドル分岐コード または異なるポジション テーブルなし。

### 動的 FPS サンプリング論理

ビデオ期間 `T` 秒とターゲット トークン予算 `B` 与えられたとき：

1. あなたが支払うことができる最大 FPS を計算：`fps_max = B / (T * tokens_per_frame)`。
2. `{1、2、4、8}` からターゲット FPS をピック、それは `fps <= fps_max` を満たし。
3. 動きが高い場合 (光フロー ヒューリスティック または明示的ユーザー要求)、より高い FPS をピック。動きが低い、より低い FPS をピック。
4. 選択された FPS で均等サンプル；フレーム間に `<time>t</time>` トークンを挿入。

Qwen2.5-VL はこのロジック 暗黙的に訓練；推論時にユーザーは `fps` パラメータ経由を制御。60 秒 アクション シーケンス 4 FPS /フレーム 81 トークン = 19440 トークン、32k コンテキスト管理可能。

### 構造化エージェント出力

Qwen2.5-VL のエージェント訓練は明示的に構造化ツール呼び出しをターゲット：

```
{
  "tool": "mouse_click",
  "coords": [1024, 512],
  "button": "left",
  "modifier": null
}
```

解析は決定的：モデルの出力上で JSON.parse。自由形式「(1024、512) でクリック」と比べて、regex と曖昧性処理が必要。シフトは Qwen2.5-VL の ScreenSpot スコア Qwen2-VL の 55% から 84% にジャンプなぜ。

## これを使用する

`code/main.py` は実装：

- テキスト、イメージパッチ、ビデオフレーム混合する詰めたシーケンスのための M-RoPE ポジション計算。
- 動的 FPS サンプラー：(期間、予算、motion_level) 与えられたとき、FPS をピック，フレームタイムスタンプ発行。
- おもちゃ Qwen2.5-VL JSON-出力パーサー，座標フィールド /ツール呼び出し応答をハンドル。

それを実行して、5 分ビデオ上で固定 FPS から動的 FPS へスワップしたときそれ違いを感じ。

## 配布する

このレッスンは `outputs/skill-qwen-vl-pipeline-designer.md` を生成。ビデオタスク (監視、エージェント、アクション認識、アクセス) が与えられたとき、それは Qwen2.5-VL 設定 (フレーム予算、FPS 戦略、ウィンドウアテンション旗、エージェント出力モード) とレイテンシー推定を発行。Qwen-VL ファミリーモデルをビデオ製品向けに展開するたびそれを使用。

## キーターム

| 用語 | 人は何と言うか | 実際の意味 |
|------|----------------|----------|
| M-RoPE | 「マルチモーダル RoPE」 | 隠しディムでテンポラル、高さ、幅バンド /3D 回転ポジション埋め込み |
| 動的 FPS | 「スマートサンプリング」 | フレームサンプリング率 /ビデオに基づきピック、期間、トークン予算 /動き |
| 絶対時間トークン | 「タイムスタンプトークン」 | `<time>t</time>` シーケンスでインターリーブ、モデルは実秒を見ないフレームインデックス |
| ウィンドウアテンション | 「ローカルアテンション」 | 空間自己アテンションスピード用にウィンドウに制限；周期的に追加 全アテンション |
| 構造化エージェント出力 | 「JSON モード」 | 訓練データ教師ありを放出するおかげで VLM を教える解析可能 JSON /座標とツール名 |
| min_pixels / max_pixels | 「解像度範囲」 | Qwen2.5-VL /リクエスト制御/トークン ハード上限 / 非常に小さい /非常に大きい入力。 |
| グラウンディング | 「ポイント-it」 | バウンディングボックス座標をテキストトークンとして出力；Qwen-VL v1 以来使用 |
