# Show-o と離散拡散統一モデル

> Transfusionは連続と離散表現を混ぜ。Show-o（Xie et al.、2024年8月）は反対方向に行く：テキストトークンは因果次トークン予測を使用、画像トークンはMaskGITの精神で離散拡散をマスク。両方は1つのトランスフォーマー内に1つの混合注意マスク。結果は1つのバックボーン上でVQA、テキスト画像、修復、混合モダリティ生成を統一、モダリティあたり1つのトークナイザー、1つの損失定式化（次トークンをマスク予測に拡張）。このレッスンはShow-oデザインを説明 — なぜマスク離散拡散は並列、数ステップ画像生成者 — Transfusionと Emu3で対比。

**タイプ:** 学習
**言語:** Python (stdlib、マスク離散拡散サンプラー)
**前提条件:** Phase 12 · 13 (Transfusion)
**所要時間:** 約120分

## 学習目標

- マスク離散拡散を説明できる：トークンを一様にマスクしてトランスフォーマーに回復を聞くスケジュール。
- 並列画像デコーディング（Show-o、MaskGIT）を自動回帰画像デコーディング（Chameleon、Emu3）と速度と品質で比較できる。
- Show-oが1つのチェックポイントで処理する3つのタスク：T2I、VQA、画像修復の名前を挙げられる。
- マスキングスケジュール（余弦、線形、切り詰め）を選び、サンプル品質への効果に関する理由をつけられる。

## 問題

Transfusionの2損失訓練は機能するが、より厳しい力学を持つ — 連続拡散損失は離散NTP損失から異なる数値スケールに住む。損失重みをバランスすることはハイパーパラメータ探索。アーキテクチャは効果的しかし複雑。

Show-oの答え：両方のモダリティを離散に保つ（Chameleonのように）、しかし順次ではなく並列でマスク離散拡散を通して画像を生成。訓練目標は次トークン予測を自然に一般化する単一マスク-トークン予測になる。

## コンセプト

### マスク離散拡散（MaskGIT）

元々のChang et al.（2022）MaskGITのコツはエレガント。完全にマスクされた画像から始まる（すべてのトークンは特別な`<MASK>` id）。各ステップで、すべてのマスクされたトークンを並列に予測、その後最も確実な予測のトップKを保つ、残りを再マスク。約8-16反復後、すべてのトークンは埋める。ステップあたりいくつのトークンをアンマスクするかのスケジュールはチューンされる — 余弦スケジュール良く働く。

訓練は単純：マスキング比率を一様に[0,1]からサンプル、それを画像のVQトークンに適用、マスクされたトークンを回復するトランスフォーマーを訓練。正確に何BERTはテキストのしたが、画像生成にスケール。

### Show-o: 1つのトランスフォーマー、混合マスク

Show-oはMaskGITを因果言語モデルトランスフォーマー内に置く。注意マスク：

- テキストトークン：因果（標準LLM）。
- 画像トークン：完全双方向画像ブロック内（マスクされたトークンが予測中他のすべての画像トークンを見ることができるように）。
- テキスト画像：テキストは過去の画像に注意、画像は過去のテキストに注意。

訓練は交互：
1. テキストシーケンスの標準NTP。
2. T2Iサンプル：テキスト→マスクされた画像トークンの画像、マスク-トークン予測損失。
3. VQAサンプル：画像→マスクされたテキストトークンのテキスト（本当に NTPのみ）。

統一損失は`<MASK>`トークン上の交差エントロピー、テキストNTP（最後のトークンのみが「マスク」）と画像マスク拡散（ランダムサブセットはマスク）をカバー。

### 並列サンプリング

Show-oは約16ステップ（トークンあたりの自動回帰）または約20（拡散）の代わりに画像を生成。各ステップで、すべてのマスクされたトークンを並列に予測；最確実なトップKをコミット；繰り返し。

比較：
- Chameleon / Emu3（トークン上自動回帰）：N_tokensの前方パス、典型的1024-4096画像あたり。
- Transfusion（連続拡散）：約20ステップ、それぞれ完全トランスフォーマーパス。
- Show-o（マスク離散拡散）：約16ステップ、それぞれ完全トランスフォーマーパス。

Show-oは類似スケールモデルでChameleonより速く、段階数Transfusionとおおよそマッチし、より低いステップあたりコスト（離散ボキャブラリロジット対連続MSE損失）。

### 1つのチェックポイント内のタスク

Show-oは推論で4つのタスク、プロンプト形式で選択：

- テキスト生成：標準自動回帰テキスト出力。
- VQA：画像インテキスト出。
- T2I：テキスト、マスク離散拡散を通して画像アウト。
- 修復：いくつかのトークンがマスクされた画像、埋めイン。

修復能力はマスク予測訓練から自由でくる。VQ-トークングリッドの領域をマスク、テキストプロンプト加え残り、マスクされたトークンを予測。

### マスキングスケジュール

ステップあたりいくつのトークンをアンマスクするかのスケジュール品質を形。Show-oは余弦を推奨：

```
mask_ratio(t) = cos(pi * t / (2 * T))   # t = 0..T
```

ステップ0で、すべてマスク（比率1.0）。ステップT、ナシマスク。余弦は予測が最も情報的である中域比率で集中。線形スケジュールも機能するがより速くプラトー。

### Show-o2

Show-o2（2025フォローアップ、arXiv 2506.15564）はShow-oをスケール：より大きいLLMベース、より良いトークナイザー、改善マスクスケジュール。同じアーキテクチャパターン。

### Show-oが座る所

2026分類法：

- 離散トークン+NTP：Chameleon、Emu3。単純しかし低い推論。
- 離散トークン+マスク拡散：Show-o、MaskGIT、LlamaGen、Muse。並列サンプリング、依然としてトークナイザーでロッシー。
- 連続+拡散：Transfusion、MMDiT、DiT。最高品質、より複雑な訓練。
- 連続+VLMでフロー一致：JanusFlow、InternVL-U。最新。

タスクで選ぶ。Show-o 1つのオープンモデルで T2I+修復+VQAを望むとき合理的速度；品質が最重要でありあなたが2損失配管を余裕ができるときTransfusion。

## 使う

`code/main.py`はShow-oサンプリングをシミュレート：

- 16 VQトークンのおもちゃグリッド。
- プロンプトと現在アンマスクされたトークンに基づくロジットを予測する模擬「トランスフォーマー」。
- 余弦スケジュール8ステップ上の並列マスク予測。
- 中間状態（マスクパターン進化）と最終トークンを印刷。

それを実行、ステップバイステップ溶解マスクを見守る。

## デプロイ

このレッスンは`outputs/skill-unified-gen-model-picker.md`を作成。理解（VQA、キャプショニング）と生成（T2I、修復）の両方が必要な製品とオープンウェイト制約を与えられて、ピック Show-oファミリー、Transfusion/MMDiTファミリー、Emu3 / Chameleonファミリー間、具体的トレードオフで。

## 演習

1. マスク離散拡散サンプル約16ステップ。なぜ1ではない？ステップ0ですべてをアンマスクするとき何が壊れ？

2. 修復はマスク拡散で自由。Show-oの修復が特化モデルを打ち負かす製品ユースケース（本当または仮想）を提案。

3. 余弦スケジュール対線形スケジュール：T=8の各ステップでアンマスクされたトークンの数をトレース。どちらがより平衡？

4. 512x512 Show-o画像は1024トークン。K=16384でボキャブラリ、モデルは1024 * log2(16384) = 14,336ビット（約1.75 KiB）のデータを出力。Stable Diffusionは512*512*24ビット = 6,291,456ビット（約768 KiB）生ピクセルを出力。圧縮比は何でありどんな品質をそれが買？

5. LlamaGen（arXiv:2406.06525）を読む。LlamaGenのクラス条件自動回帰画像モデルはShow-oのマスク接近と異なるように何？

## キー用語

| 用語 | 人が言うこと | それが実際に意味すること |
|------|-----------------|------------------------|
| マスク離散拡散 | 「MaskGITスタイル」 | マスクされたトークンを予測するよう訓練；推論で、反復的に最も確実な予測をアンマスク |
| 余弦スケジュール | 「アンマスクスケジュール」 | 推論ステップ上でマスク比率の減衰；中域で確信成長を集中 |
| 並列デコーディング | 「すべてのトークンは一度」 | すべてのステップは1つの前方パスで完全マスクトークンシーケンスを予測、その後トップKをコミット |
| 混合注意 | 「因果+双方向」 | テキストトークン上因果とビジョンブロック内双方向マスク |
| 修復 | 「埋め込み生成」 | いくつかのトークンがマスクされた画像に条件、失うものを予測；訓練目標から自由 |
| コミットメント率 | 「ステップあたりトップK」 | いくつのトークンが反復あたり「終わり」宣言；推論対品質トレードオフを制御 |
