# オープンウェイト VLM レシピ：実際に重要なもの

> 2024-2026 オープンウェイト VLM 文献はアブレーションテーブル森。Apple の MM1 は 13 の組み合わせテスト：画像エンコーダ、コネクター、データ混合。Allen AI の Molmo は詳細な人間キャプションが GPT-4V 蒸留を打つ証明。Cambrian-1 は 20+ エンコーダ比較実行。Idefics2 は5軸設計空間を形式化。Prismatic VLM は制御ベンチマークで 27 訓練レシピを比較。すべてのそのノイズのうち、小さい結果セット保持ペーパー横断：画像エンコーダはコネクターアーキテクチャより物質が多く、データ混合は両者より物質が多く、詳細な人間キャプションが蒸留合成データを打つ。このレッスンはそのテーブルを読むので、あなたはしない必要。

**タイプ:** 学習 + ラボ
**言語:** Python (標準ライブラリ、アブレーションテーブルパーサー + レシピピッカー)
**前提条件:** Phase 12 · 05 (LLaVA ベースライン)
**所要時間:** 約180分

## 学習目標

- 5軸 VLM 設計空間命名：画像エンコーダ、コネクター、LLM、データ混合、解像度スケジュール。
- MM1 / Idefics2 / Cambrian-1 アブレーションテーブルを読み、与えられたベンチマークノブはどれかを予測。
- 新規 VLM 用 レシピ (エンコーダ、コネクター、データ、解像度) 計算予算とタスク混合与えられ選択。
- なぜ詳細な人間キャプションが GPT-4V 蒸留を同じトークンカウント打つか説明。

## 問題

数百のオープンウェイト VLM が存在。ほとんど「良い」と「最先端」間のギャップはアーキテクチャではない。それはデータ、解像度スケジュール、エンコーダ選択。あなたのモデルが性能不足のとき、どのノブを最初に回すかを知る 500万 GPU 時間の誤りを保存。

2023 ウェーブ (LLaVA-1.5、InstructBLIP、MiniGPT-4) はキャプションペア事前訓練 + LLaVA-Instruct-150k で実行。良好なベースライン。MMMU の約 35% で上限。

2024 ウェーブ (MM1、Idefics2、Molmo、Cambrian-1、Prismatic VLM) は拡大的アブレーション実行。結果は驚き、実用的。

## コンセプト

### 5軸設計空間

Idefics2 (Laurençon et al.、2024) 軸命名：

1. 画像エンコーダ。CLIP ViT-L/14、SigLIP SO400m/14、DINOv2 ViT-g/14、InternViT-6B。エンコーダはパッチサイズ、解像度、事前訓練目的で異なる。
2. コネクター。MLP (2-4 層)、Q-Former (32 クエリ + クロスアテンション)、Perceiver Resampler (64 クエリ)、C-Abstractor (畳み込み + 二線形プール)。
3. 言語モデル。Llama-3 8B / 70B、Mistral 7B、Phi-3、Gemma-2、Qwen2.5。LLM サイズはドミナント param コスト。
4. トレーニングデータ。キャプション ペア (CC3M、LAION)、インターリーブ (OBELICS、MMC4)、命令 (LLaVA-Instruct、ShareGPT4V、PixMo、Cauldron)。
5. 解像度スケジュール。固定 224/336/448、AnyRes、ネイティブ動的。訓練中にランプまたは定数。

各本番 VLM は各軸で選択を行う。ほとんどの MMMU スコア分散は軸 1、4、5 で説明 — どのコネクター選択ではなく。

### 軸1：エンコーダ > コネクター

MM1 Section 3.2 は示した：CLIP ViT-L/14 から SigLIP SO400m/14 へスワップ MMMU 3+ ポイント追加。MLP から Perceiver Resampler へコネクタースワップは 1 ポイント未満追加。Idefics2 複製：SigLIP > CLIP、Q-Former ≈ MLP ≈ Perceiver 同じトークンカウント。

Cambrian-1 「Cambrian Vision Encoders Match-Up」 (Tong et al.、2024) は 20+ エンコーダビジョン中心ベンチマーク (CV-Bench) で実行。リーダーボードのトップは DINOv2 と SigLIP の混合；CLIP はパック中央；ImageBind と ViT-MAE は低い。CLIP ViT-L から DINOv2 ViT-g/14 へのギャップは CV-Bench の ~5-7 ポイント。

2026 デフォルトエンコーダ開く VLM の SigLIP 2 SO400m/14 セマンティック + 密な特徴、時々 DINOv2 ViT-g/14 特徴が連結 (Cambrian の「Spatial Vision Aggregator」はこれをする)。

### 軸2：コネクター設計は無視

MM1、Idefics2、Prismatic、MM-Interleaved はすべて同じ結論に達した：固定ビジュアルトークンカウントで、コネクターアーキテクチャはほとんど重要でない。同じトークン予算で平均プール パッチの 2層 MLP は32 クエリ Q-Former の1ポイント以内で実行。

物質は何かトークンカウント。より多くビジュアルトークン = より多く LLM コンピュート = より良い性能，ある点まで、次に収穫逓減。イメージあたり64トークンは OCR に低すぎる。576-1024 トークンはほとんどの開く VLM 向けスイートスポット。2048+ はドキュメントとチャートだけでヘルプ。

Q-Former vs MLP はコスト質問、品質質問ではない：Q-Former は イメージ解像度かかわらず32-64 トークン上限；MLP は すべてパッチトークン発行。高解像度入力で、Q-Former は LLM コンテキスト保存；低解像度で、差異はノイズ。

### 軸3：LLM サイズは天井設定

LLM の 7B から 13B へダブル MMMU 横断すべての VLM ペーパーで確実に2-4 ポイント追加。70B で最もベンチマーク飽和。VLM のマルチモーダル推論天井は LLM のテキスト推論天井— ビジョンエンコーダはそれを供給できるだけ、ためではなく推論。

これが Qwen2.5-VL-72B と Claude Opus 4.7 が MMMU-Pro と ScreenSpot-Pro をクラッシュなぜ：言語ブレイン大きい。7B VLM は 70B VLM の代用はできない巧妙なコネクター設計を通じて。

### 軸4：データ — 詳細な人間キャプション vs 蒸留

Molmo + PixMo (Deitke et al.、2024) は 2024 結果みんな読むべき。Allen AI はヒト注釈者画像を1-3分の密度音声テキスト通す、712K 密キャプション画像得、利回る。GPT-4V 蒸留はトレーニングデータの場所どこも。

Molmo-72B は Llama-3.2-90B-Vision 11 11 ベンチマーク上で打つ。デルタはアーキテクチャではない — それはキャプション品質。詳細人間キャプションは短いウェブキャプション比べて 5-10 倍より情報を含む、GPT-4V 蒸留が幻覚する事実地面に留まる。

ShareGPT4V (Chen et al.、2023) と Cauldron (Idefics2) は同じ遊戯本に従った、混合人間 + GPT-4V キャプション。傾向明白：2026 フロンティア、キャプション密度 > キャプション数量 > 蒸留利便性。

### 軸5：解像度とスケジュール

Idefics2 のアブレーション：384 -> 448 1-2 ポイント追加。448 -> 980 イメージ分割 (AnyRes) 次の 3-5 ポイント OCR ベンチマーク上。フラット解像度訓練は中程度精度で高原；解像度ランプ (開始 224、終了 448 またはネイティブ) より速い訓練と より高い終了。

Cambrian-1 は解像度 vs トークン トレードオフを実行：固定コンピュート、低い解像度でより多くトークンがあるか、高い解像度でより少ないトークン。高い解像度 OCR 勝つ；低解像度より多くトークンは一般的シーン理解勝つ。

2026 本番レシピ：段階1 384 固定で訓練、段階2 OCR-ヘビー タスク 1280 まで動的解像度。

### Prismatic 制御比較

Prismatic VLM (Karamcheti et al.、2024) はペーパーはすべての軸制御。同じ13B LLM、同じ命令データ、同じ評価 — 1軸は時間同じ違い。結果：

- イメージあたりビジュアルトークンカウント説明 ~60% 分散。
- エンコーダ選択説明 ~20%。
- コネクターアーキテクチャ説明 ~5%。
- 他すべて (データ混合、スケジューラ、LR) 残り ~15%。

これは荒い分解、だがそれは「最初は何を切るべきか」に対する文献最も潔い答え。

### 2026 用ピッカー

証拠で与えられ、デフォルト開く VLM レシピ新規プロジェクト 2026：

- エンコーダ：SigLIP 2 SO400m/14 ネイティブ解像度 NaFlex 持つ、密な特徴セグメンテーション/グラウンディング必要場合 DINOv2 ViT-g/14 と連結。
- コネクター：パッチトークン 2層 MLP。トークン制約されない限り Q-Former スキップ。
- LLM：Qwen2.5 / Llama-3.1 / Gemma 2、7B コスト、70B 品質、選択ターゲットレイテンシ。
- データ：PixMo + ShareGPT4V + Cauldron、トップ-タスク固有命令データで。
- 解像度：動的 (256 分、1280 ピクセル/長辺最大)。
- スケジュール：段階1 配置 (プロジェクターのみ)、段階2 完全微調、段階3 タスク固有微調。

それらの各デフォルトはこのレッスンの最後に引用論文で測定アブレーションにトレース バック。

## これを使用する

`code/main.py` はアブレーションテーブルパーサーとレシピピッカー。それは MM1 と Idefics2 アブレーションテーブル符号化 (凝縮) と質問を許可：

- 「与えられ予算 X とタスク Y、どのレシピ勝つ？」
- 「7B Llama で SigLIP の CLIP をスワップ、期待 MMMU デルタ何か？」
- 「どの軸最初に切るべき 80% 信頼答え？」

出力はランク付けレシピリスト期待ベンチマークデルタと「最初に切ります」推奨。

## 配布する

このレッスンは `outputs/skill-vlm-recipe-picker.md` を生成。ターゲットタスク混合、計算予算、レイテンシ目標が与えられたとき、それは完全レシピ (エンコーダ、コネクター、LLM、データ混合、解像度スケジュール) 発行、各選択を正当化する引用を持つアブレーション。エンジニアが毎 VLM プロジェクトが開始するたびに Idefics2 アブレーション テーブル再発明するのを停止。

## 演習

1. MM1 Section 3.2 読む。固定 2B LLM で予算 50M 画像、どのエンコーダ勝つ？答えは13B LLM でフリップ？なぜ？

2. Cambrian-1 は DINOv2 + SigLIP 連結が視覚中心ベンチマーク上いずれかだけを上回るが、MMMU 上シグナル追加しないことを見つける。ゲイン予測ベンチマークどれ、フラットどれ。

3. あなたのターゲットはモバイル UI エージェント 2B LLM。エンコーダ、コネクター、解像度、データ混合を選択。各選択を特定のアブレーションテーブルで正当化。

4. Molmo は 4B と 72B モデルに配布。4B は競争的クローズド 7B VLM；72B は Llama-3.2-90B-Vision 11/11 ベンチマーク上を打つ。LLM-サイズ高原仮説について何を示す？

5. LLM-品質からエンコーダ品質を隔離するため、アブレーション テーブル設計 7B VLM。訓練最小実行どのくらい？4 軸設定提案。

## キーターム

| 用語 | 人は何と言うか | 実際の意味 |
|------|----------------|----------|
| アブレーション | 「1ノブを回す」 | 複数実行訓練、ちょうど1つデザイン空間軸で異なるが、他すべて定数をホールド |
| コネクター | 「ブリッジ」 / 「プロジェクター」 | ビジョンエンコーダ出力を LLM のトークン空間にマップ訓練可能モジュール (MLP、Q-Former、Perceiver) |
| 詳細人間キャプション | 「密キャプション」 | マルチ文人間書く記述 (典型的 80-300 トークン) ウェブ alt テキストより豊か |
| 蒸留 | 「GPT-4V キャプション」 | より強い専有 VLM で生成訓練データ；利便性だがが継承された幻覚傾向 |
| AnyRes / 動的 res | 「高解像度パス」 | エンコーダ のネイティブ解像度より大きい画像フィード戦略 (タイリング経由または M-RoPE) |
| 解像度ランプ | 「カリキュラム」 | 訓練スケジュール低解像度から増加開始、配置学習を加速 |
| ビジョン中心ベンチ | 「CV-Bench / BLINK」 | 評価言語ヘビー推論ではなく、細粒ビジュアル知覚強調 |
| PixMo | 「Molmo のデータ」 | Allen AI の 712K 密キャプション画像データセット；ヒト音声転写密キャプション。
