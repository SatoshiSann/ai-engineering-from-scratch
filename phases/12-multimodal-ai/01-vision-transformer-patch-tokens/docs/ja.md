# Vision Transformers とパッチトークンプリミティブ

> 何か マルチモーダルの前に、画像はトランスフォーマーが食べられるトークン列になる必要。2020年ViTペーパーはこれに16x16ピクセルパッチ、線形射影、位置埋め込みで答えた。5年後、すべての2026年フロンティアモデル(Claude Opus 4.7 at 2576pxネイティブ、Gemini 3.1 Pro、Qwen3.5-Omni)はまだこのようにします — エンコーダーはViTからDINOv2へからSigLIP 2へ変わった、記録トークン追加、位置スキームが2D-RoPEになった、しかしプリミティブは保持。このレッスンはパッチトークンパイプラインを終わりまで読み、stdlib Pythonでそれをビルド。Phase 12の残りが「ビジュアルトークン」のコンクリート心的モデルを持つ。

**タイプ:** 学習
**言語:** Python(stdlib、パッチトークナイザー + ジオメトリ計算)
**前提条件:** Phase 7 (Transformers)、Phase 4 (Computer Vision)
**所要時間:** 約120分

## 学習目標

- HxWx3画像を正しい位置エンコーディングでパッチトークン列に変換
- 与えられた(パッチサイズ、解像度、隠れDim、深さ)のViTの列長、パラメータカウント、FLoPを計算
- 2020 ViTから2026本番 3つアップグレードを命名：自教教(DINO / MAE)、記録トークン、ネイティブ解像度パック
- ダウンストリームタスク用CLSプール、平均プール、記録トークンのいずれかで選ぶ

## 問題

トランスフォーマーはベクトル列で実行。テキストは既に列(バイトまたはトークン)。画像は3色チャネル付き2D ピクセルグリッド — 列ではない。すべてのピクセルをフラット化すると、224x224 RGB 画像は150,528トークンになり、その長さ自己アテンション非開始(列長でクアドラティック)。

2020年前のアプローチはCNN特徴抽出をボルト：ResNetは7x72048-dim ベクトル特徴マップを生成、それら49トークンをトランスフォーマーに給与。これは機能しますが、CNNバイアス(並進等分散、局所受信場)を継承し、トランスフォーマーのスケール欲望を失う。

Dosovitskiyら。(2020)は率直な質問を求めた：CNNをスキップする場合はどうか？画像を固定サイズパッチに分割(例えば16x16ピクセル)、各パッチを線形射影、位置埋め込み追加、列をバニラトランスフォーマーに給与。当時これは異端 — 対流なしビジョン。十分データと(JFT-300M、その後LAION)と一致し、ResNetを倒しImageNetで保つ改善。

2026年までに、ViTプリミティブは異議なしの基礎。すべてのオープンウェイト VLMのビジョンタワーはいくつかの子孫(DINOv2、SigLIP 2、CLIP、EVA、InternViT)。質問はもはや「パッチを使うべきか？」から「パッチサイズ何、解像度スケジュール何、事前学習目標何、位置エンコーディング何」。

## コンセプト

### パッチ as トークン

イメージ`x`シェイプ`(H, W, 3)`とパッチサイズ`P`を与え、イメージをグリッド`(H/P) x (W/P)`非重複パッチにカーブ。各パッチは`P x P x 3` ピクセルキューブ。各キューブを`3 P^2`ベクトルにフラット。スケール線形射影`W_E`シェイプ`(3 P^2, D)`モデルの隠れDimに`D`各パッチをマップ。

ViT-B/16正規設定では：
- 解像度224、パッチサイズ16 → グリッド14x14 → 196パッチトークン。
- 各パッチは`16 x 16 x 3 = 768`ピクセル値、`D = 768`に射影。
- 習得`[CLS]`トークン追加 → 列長197。

パッチ射影は数学的に2D対流と同一、カーネルサイズ`P`、ストライド`P`、`D`出力チャネル。本番コードは実際にそのようにそれを実装 — `nn.Conv2d(3, D, kernel_size=P, stride=P)`。「線形射影」フレーミングは概念的；カーネルフレーミングは効率的。

### 位置埋め込み

パッチはネイティブ順序なし — トランスフォーマーはそれらをバッグとして見る。初期ViT は習得1D位置埋め込み追加(1位置で768-dimベクトル、197それら)。動作しますが、訓練解像度にモデルを結ぶ：推論で、グリッド変更のならイメージ位置テーブルを補間する必要があります。

モダン ビジョンバックボーン使用2D-RoPE(Qwen2-VLのM-RoPE、SigLIP 2のデフォルト)または因数分解2D位置。2D-RoPEはクエリとキー ベクトルをパッチ(行、列)インデックス、推論相対2D位置に基づきに回転、このため回転角度なし位置テーブル。モデルは推論で任意グリッドサイズを処理。

### CLSトークン、プール出力、記録トークン

イメージレベル表現は何？3つの選択が共存：

1. `[CLS]`トークン。習得ベクトルをパッチ列に前置。すべてのトランスフォーマーブロック後、CLSトークンの隠れ状態がイメージ表現。BERTから継承。元ViT、CLIPで使用。
2. 平均プール。パッチトークン出力隠れ状態を平均。SigLIP、DINOv2、ほとんどモダン VLMで使用。
3. 記録トークン。Darce etら。(2023)は観察、CLSトークンなしトレーニング ViTは高ノルム「アーティファクト」パッチを開発、自己アテンション ハイジャック。4-16習得記録トークン追加がこのロードを吸収し、密予測品質を改善(分割、深さ)。DINOv2 と SigLIP 2 の両方記録付きで送信。

チョイスダウンストリームタスク問題。CLSが分類用に良好。VLMがパッチトークンをLLMに給与する、プール全然スキップ — すべてのパッチはLLMインプットトークンになる。記録がハンドオフ前に捨てる(スキャフォルド、コンテンツではない)。

### 事前学習：教師、対比、マスク、自己蒸留

2020年ViTはJFT-300M上教師分類で事前訓練。迅速に代替：

- CLIP(2021)：400M対の対比イメージテキスト。レッスン12.02。
- MAE(2021、He etら)：パッチの75%マスク、ピクセル再構築。自教、純イメージで。
- DINO(2021) / DINOv2(2023)：学生先生自蒸留、ラベルなし、ラベルなし。2023 DINOv2 ViT-g/14は最強純粋ビジュアルバックボーン、「密機能」ユースケース用デフォルト。
- SigLIP / SigLIP 2(2023、2025)：CLIPシグモイド喪失、ネイティブアスペクト比用NaFlex。2026オープンVLM支配ビジョンタワー(Qwen、Idefics2、LLaVA-OneVision)。

あなたの事前訓練チョイスはバックボーンが何のためか良好：テキスト意味マッチに対してCLIP/SigLIP、密ビジュアル機能用DINOv2、ダウンストリーム微調整用スタートポイントとしてMAE。

### スケーリング法則

ViTスケーリング(Zhai etら2022)ViTの品質が予測可能法則に従うことを確立、モデルサイズ、データサイズ、計算で。固定計算：
- より大きいモデル + より多データ →より良い品質。
- パッチサイズは列長対忠誠度のレバー。パッチ14(DINOv2/SigLIP SO400m用典型)がパッチ16より多くの列トークン生成；OCRと密タスク用より良い忠誠度、速度用より悪い。
- 解像度は他の大きなレバー。224から384から512へは、ほぼ常に助け、FLoPで二次クラッシュで。

ViT-g/14(1B params、パッチ14、解像度224 → 256トークン)とSigLIP SO400m/14(400Mパラム、パッチ14)は2つのワークホース エンコーダー2026オープンVLM。

### ViTのパラメータカウント

完全計算`code/main.py`の住居。ViT-B/16 at 224：

```
patch_embed = 3 * 16 * 16 * 768 + 768  =  591k
cls + pos    = 768 + 197 * 768          =  152k
block        = 4 * 768^2 (QKVO) + 2 * 4 * 768^2 (MLP) + 2 * 2*768 (LN)
             = 12 * 768^2 + 3k          =  7.1M
12 blocks    = 85M
final LN    = 1.5k
total       ≈ 86M
```

すべてのViTをこのやり方ごと球場。バックボーンサイズがすべてのダウンストリームVLMVRAM床を設定。

### 2026本番設定

ほとんどのオープンVLM が2026で出荷するエンコーダーはSigLIP 2 SO400m/14、ネイティブ解像度(NaFlex)。それは：
- 400Mパラメーター。
- パッチサイズ14、デフォルト解像度384 → 729パッチトークン/イメージ。
- イメージレベルタスク用平均プール；VQA用すべての729パッチはLLMに流。
- 4記録トークン、LLMハンドオフ前に破棄。
- 2D-RoPEネイティブアスペクト比用イメージレベルスケーリング。

そのコンフィグ内すべてのデシジョンはあなたが読める論文をトレース。

## 使用方法

`code/main.py`はパッチトークナイザーとジオメトリ計算。それは(イメージH、W、パッチP、隠れD、深さL)を取得し、レポート：

- グリッドシェイプと列長パッチング後。
- トークン列シンセティック8x8ピクセルおもちゃイメージ(フラット + 射影路を歩く)。
- パラメータカウントパッチ埋め込み、位置埋め込み、トランスフォーマーブロック、頭部で破れた。
- ターゲット解像度でリクエストあたりFLOP。
- ViT-B/16 @ 224、ViT-L/14 @ 336、DINOv2 ViT-g/14 @ 224、SigLIP SO400m/14 @ 384の比較テーブル。

それを実行。パラメータカウントを公開番号と照合。パッチサイズと解像度でプレイ、トークンカウントコストを感じ。

## 出荷

このレッスンは`outputs/skill-patch-geometry-reader.md`を生成。ViTコンフィグ(パッチサイズ、解像度、隠れDim、深さ)を与えると、正当化を持つトークンカウント、パラメータカウント、VRAM見積りを生成。このスキルをいつでも使用VLM用ビジョンバックボーン選ぶ — それは「トークンは爆発し、LLMコンテキスト満杯」驚き防ぐ。

## 演習

1. ネイティブ1280x720入力でパッチサイズ14でQwen2.5-VLパッチトークン列長計算。CLSのみ表現と比較方法。

2. 1080p フレーム(1920x1080)パッチ14でいくつかのトークンを生成？30 FPSで5分のビデオでのトークン総数？何があなたを節約最も：プール、フレームサンプリング、またはトークンマージ？

3. Pure Pythonでパッチトークン平均プールを実装。平均プール196 DINOv2出力トークンが、モデルの`forward`が返すものと合致確認プール埋め込みを求める時。

4. 「Vision Transformers Need Registers」(arXiv:2309.16588) セクション3を読む。記録が吸収するアーティファクトと密予測ダウンストリーム問題。

5. `code/main.py`をサポートに修正パッチ-n'-パック：異なる解像度イメージリストを与え、単一パック列とブロック対角アテンション マスクを生成。Lesson 12.06に到達する時検証。

## キーワード

| 用語 | 人々が言うこと | 実際の意味 |
|------|----------------|------------------------|
| Patch | "16x16ピクセル正方形" | 固定サイズ非重複イメージ領域；1トークンになる |
| Patch embedding | "線形射影" | 習得マトリックス(またはConv2dストライド=P)フラット化パッチピクセルをD-dimベクトルにマップ |
| CLS token | "クラストークン" | 前置習得ベクトル、最終隠れ状態がイメージを表現；2026年オプション |
| Register token | "シンク トークン" | 特別な習得トークン高ノルムアテンション アーティファクトを吸収ViT事前訓練開発 |
| Position embedding | "位置情報" | 列順序を作る振る舞えるベクトル/回転；2D-RoPEはモダンデフォルト |
| Grid | "パッチグリッド" | (H/P) x (W/P) 2D解像度とパッチサイズ与えられたパッチ配列 |
| NaFlex | "ネイティブ フレキシブル解像度" | SigLIP 2機能：単一モデル複数アスペクト比、解像度なし再訓練|
| Backbone | "ビジョンタワー" | VLMのLLMにパッチトークン出力給与する事前訓練イメージエンコーダー |
| Pooling | "イメージレベル要約" | パッチトークンを1ベクトルに変えます戦略：CLS、平均、アテンション プール、記録ベース |
| Patch 14 vs 16 | "より細かい対粗いグリッド" | パッチ14 / イメージあたり多くトークン生成、OCR用品質、速度用より悪い；パッチ16クラシックデフォルト |
