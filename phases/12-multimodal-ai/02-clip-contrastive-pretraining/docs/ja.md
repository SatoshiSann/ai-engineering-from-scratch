# CLIP と対比学習ベースのビジョン言語事前学習

> OpenAI の CLIP (2021) は、その後5年間を支配するほど大きなアイデアを証明した。単一のアイデア：ノイジーなウェブの画像キャプションペアと対比損失のみを使用して、画像エンコーダとテキストエンコーダを同じベクトル空間に配置する。教師付きラベルなし。4億ペア。生成された埋め込み空間は、ゼロショット分類、画像テキスト検索を実行し、2026年のすべての VLM にその視覚タワーとして接続される。SigLIP 2 (2025) は softmax をシグモイドに置き換え、より低いコストで CLIP を上回る規模に達した。このレッスンでは InfoNCE からシグモイド対ペア損失までの数学を説明し、標準 Python でトレーニングステップを構築する。

**タイプ:** ビルド
**言語:** Python (標準ライブラリ、InfoNCE + シグモイド損失実装)
**前提条件:** Phase 12 · 01 (ViT パッチ)、Phase 7 (トランスフォーマー)
**所要時間:** 約180分

## 学習目標

- 相互情報量から InfoNCE 損失を導出し、数値的に安定したベクトル化バージョンを実装する。
- シグモイド対ペア損失 (SigLIP) が softmax の全ギャザーオーバーヘッドなしでバッチ 32768+ までスケーリングする理由を説明する。
- テキストテンプレート (`a photo of a {class}`) を構築し、コサイン類似度に対して argmax を取ることで、ゼロショット ImageNet 分類を実行する。
- CLIP / SigLIP 事前学習が提供する4つのレバー（バッチサイズ、温度、プロンプトテンプレート、データ品質）を名前付ける。

## 問題

CLIP 前のビジョンは教師付きだった。ラベル付きデータセット (ImageNet: 120万画像、1000クラス) を収集し、CNN を訓練し、配布する。ラベルは高価で、ラベルはラベラーが同意できるものにバイアスを持つ、ラベルはファインチューニングなしで新しいタスクに転送されない。

画像キャプション web には、無料で10億以上のゆるくラベル付けされたペアがある。黄金のレトリーバーの写真とキャプション "my dog Max in the park" は、教師ありシグナルを持つ — テキストが画像を説明する。質問：これを有用な訓練に変えることはできるだろうか？

CLIP の答え：画像キャプションペアをマッチングタスクとして扱う。N個の画像と N個のキャプションのバッチが与えられたとき、各画像をそれ自体のキャプションと N-1 個のディストラクターに対してマッチングすることを学ぶ。教師ありシグナルは「これらの2つはまとまっている；これら N-1 は そうではない」。クラスラベルなし。人間による注釈なし。対比損失だけ。

結果として生じる埋め込み空間は、CLIP が訓練された以上のことを行う。明示的にキャット とラベル付けされたことのない、ネコの画像に近い "a photo of a cat" が埋め込まれるため、ImageNet ゼロショットが機能する。これは 2026 年のすべての VLM を生み出した賭けである。

## コンセプト

### 双エンコーダ

CLIP には2つのタワーがある：

- 画像エンコーダ `f`：ViT または ResNet、画像ごとに D 次元ベクトルを出力
- テキストエンコーダ `g`：小さなトランスフォーマー、キャプションごとに D 次元ベクトルを出力

両方のタワーはそれらの出力を単位長に正規化する。類似度は `cos(f(x), g(y)) = f(x)^T g(y)` である (両方が単位ノルムであるため)。

N個の(画像、キャプション)ペアのバッチについて、類似度行列 `S` を形 `(N, N)` で構築する：

```
S[i, j] = cos(f(x_i), g(y_j)) / tau
```

ここで `tau` は学習された温度である (CLIP は 0.07 に初期化；ログ空間で学習)。

### InfoNCE 損失

CLIP は行と列の対称クロスエントロピーを使用する：

```
loss_i2t = CE(S, labels=identity)     # 各画像の正例はそれ自体のキャプション
loss_t2i = CE(S^T, labels=identity)   # 各キャプションの正例はそれ自体の画像
loss = (loss_i2t + loss_t2i) / 2
```

これが InfoNCE である。CE の softmax は各画像をそのキャプションをバッチ内の他のすべてのキャプションより多くマッチングするよう強制する。「ネガティブ」はすべての他のバッチアイテムである。より大きなバッチ = より多くのネガティブ = より強いシグナル。CLIP はバッチ 32k で訓練；スケールが重要である。

### 温度

`tau` は softmax の鋭さを制御する。低い tau → 鋭い分布、困難なネガティブマイニング効果。高い tau → 柔らかい、すべてのサンプルが寄与する。CLIP は log(1/tau) を学習し、崩壊を防ぐためにクリップされる。SigLIP 2 は初期の tau を固定し、代わりに学習されたバイアスを使用する。

### シグモイドがより良くスケーリングする理由 (SigLIP)

Softmax は全類似度行列を同期させる必要がある。分散学習では、すべての埋め込みをすべてのレプリカに全ギャザーし、softmax を実行する必要がある。これは通信でワールドサイズの2乗。

SigLIP は softmax を要素ごとのシグモイドで置き換える。各ペア `(i, j)` について、損失は「これらはマッチングペアか？」の二項分類である。正例ラベルは対角線、他はすべてネガティブ。損失は：

```
L = -1/N sum over (i, j) [ y_ij log sigmoid(S[i,j]) + (1-y_ij) log sigmoid(-S[i,j]) ]
```

`y_ij = 1` if `i == j`、else 0。各ペアの損失は独立。全ギャザー不要。各 GPU はその局所ブロックを計算して合計。SigLIP 2 はバッチ 32k-512k に安くスケーリング；CLIP は比例してより多くの通信が必要となる。

### ゼロショット分類

N個のクラス名が与えられたとき、各クラスのテキストテンプレートを構築する：

```
"a photo of a {class}"
```

テキストエンコーダで各テンプレートを埋め込む。画像エンコーダで画像を埋め込む。コサイン類似度に対して argmax = 予測クラス。ターゲットクラスでの訓練なし。

プロンプトテンプレートが重要である。CLIP のオリジナル論文は、クラスごとに80個のテンプレート (通常、芸術的、写真、絵画など) を使用し、埋め込みを平均化した。ImageNet で +3 ポイント。現代の使用は通常、1つまたは2つのテンプレートを選ぶ。

### リニアプローブとファインチューニング

ゼロショットはベースライン。リニアプローブ (凍結 CLIP 特徴の上にターゲットクラスのための1つの線形層をトレーニング) は、ドメイン内タスクでゼロショットを上回る。完全なファインチューニングはドメイン内でリニアプローブを上回るが、ゼロショット転送に害を与えることができる。3つのレジームで3つのトレードオフ。

### SigLIP 2: NaFlex と密な特徴

SigLIP 2 (2025) は以下を追加する：
- NaFlex：単一モデルは可変アスペクト比と解像度を処理
- セグメンテーションと深度推定のためのより良い密な特徴、VLM の凍結バックボーンとしての使用を目標
- 多言語：100+ 言語で訓練 (CLIP は英語のみ)
- 1B パラメータスケール (CLIP は 400M に上限)

2026 年のオープン VLM では、SigLIP 2 SO400m/14 がデフォルト視覚タワーである。CLIP は、特定の LAION-2B トレーニング分布がクエリパターンに一致する純粋な画像テキスト検索の場合のデフォルトのままである。

### ALIGN、BASIC、OpenCLIP、EVA-CLIP

ALIGN (Google、2021)：CLIP と同じアイデア、1.8B ペアスケール、90% ノイジー。ノイジーなデータがスケーリングされることを証明した。OpenCLIP (LAION)：LAION-400M / 2B 上での CLIP の開放再現、複数のスケール、開いているチェックポイントの候補。EVA-CLIP：マスク画像モデルから初期化；VLM のための強いバックボーン。BASIC：Google の CLIP+ALIGN ハイブリッド。すべて同じファミリー、異なるデータとチューニング。

### ゼロショット天井

CLIP クラスモデルは約 76% ImageNet ゼロショット (CLIP-G、OpenCLIP-G) で上限である。それを超えるには、より大きなデータ (SigLIP 2 は 80%+ を取得) またはアーキテクチャの変更 (教師付きヘッド、より多くのパラメータ) が必要である。ベンチマークは飽和しつつある；本当の価値はダウンストリーム VLM が消費する埋め込み空間である。

## これを使用する

`code/main.py` は以下を実装する：

1. おもちゃの双エンコーダ (ハッシュベースの画像特徴、テキスト文字特徴) なので、numpy なしで InfoNCE 形を見ることができる。
2. 純粋な Python での InfoNCE 損失 (log-sum-exp 経由の数値安定性)。
3. 比較のためのシグモイド対ペア損失。
4. ゼロショット分類ルーチン：テキストプロンプトのセットに対するコサイン類似度を計算し、予測のために argmax を実行。

それを実行して損失曲線を見よう。絶対数はおもちゃ；形は実際の CLIP トレーナーが放出するものに一致する。

## 配布する

このレッスンは `outputs/skill-clip-zero-shot.md` を生成する。画像のセット (パス経由) とターゲットクラスのリストが与えられたとき、CLIP テンプレートでテキストプロンプトを構築し、両側を指定されたチェックポイント (例：`openai/clip-vit-large-patch14`) で埋め込み、上位1 / 上位5の予測と類似度スコアを返す。スキルはプロンプトリストにないクラスについてクレームを作成することを拒否する。

## 演習

1. 4ペアに対して手動で InfoNCE を実装する。4x4 類似度行列を構築し、softmax を実行し、対角線を選び出し、クロスエントロピーを計算する。Python 実装をこの手計算と照合して検証する。

2. SigLIP は温度に加えてバイアスパラメータ `b` を使用する：`S'[i,j] = S[i,j]/tau + b`。バッチが大きなクラス不均衡を持つ場合 (行あたりより多くのネガティブよりはるかに多い) `b` はどのような役割を果たすか？SigLIP Section 3 (arXiv:2303.15343) を読む。

3. ネコと犬の分類器のゼロショットを構築。2つのプロンプトテンプレートを試す：`a photo of a {class}` と `a picture of a {class}`。100のテスト画像で精度を測定。テンプレートのアンサンブルはシングルを上回るか？

4. softmax InfoNCE とシグモイド対ペアの通信コストを 512-GPU実行でバッチ 32k で計算。どちらが O(N)、どちらが O(N^2) にスケーリングするか？SigLIP Section 4 を引用。

5. OpenCLIP スケーリング則論文を読む (arXiv:2212.07143, Cherti et al.)。図からデータスケーリングの結論を再現：固定モデルサイズで ImageNet ゼロショット精度とトレーニングデータサイズの間のログ線形関係は何か？

## キーターム

| 用語 | 人は何と言うか | 実際の意味 |
|------|----------------|----------|
| InfoNCE | 「対比損失」 | バッチの類似度行列に対するクロスエントロピー；各アイテムの正例は対になったアイテム、ネガティブはすべて他 |
| シグモイド損失 | 「SigLIP 損失」 | 対ごとの二項クロスエントロピー；softmax なし、全ギャザーなし、分散トレーニングで安くスケーリング |
| 温度 | 「tau」 | softmax/シグモイド前のロジットをスケーリングするスカラー；分布の鋭さを制御 |
| ゼロショット | 「ノーファインチューンしょぶん分類」 | テキストプロンプトを使用してクラス埋め込みを構築し、コサイン類似度で分類；ターゲットクラスでの訓練なし |
| プロンプトテンプレート | 「a photo of a ...」 | クラス名の周りのテキストスカフォルド；ゼロショット精度に1-5ポイント影響 |
| 双エンコーダ | 「ツータワー」 | 1つの画像エンコーダ + 1つのテキストエンコーダ、共有 D 次元空間に出力 |
| 困難なネガティブ | 「難しいディストラクター」 | 正例に十分似ているネガティブ、モデルがそれらを分離するために機能する必要がある |
| リニアプローブ | 「凍結 + 1レイヤー」 | 凍結特徴の上にのみ線形分類器をトレーニング；特徴品質を測定 |
| NaFlex | 「ネイティブ柔軟解像度」 | SigLIP 2 リサイズなしで任意のアスペクト比と解像度の画像を取り込む能力 |
| 温度スケーリング | 「ログパラメータ化 tau」 | CLIP は `log(1/tau)` をパラメータ化するため、勾配は動作；崩壊を防ぐためにクリップ |
