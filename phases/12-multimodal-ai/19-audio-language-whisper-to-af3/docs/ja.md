# オーディオ言語モデル：WhisperからAudio Flamingo 3へ

> Whisper（Radford et al., 2022年12月）は音声認識を確立 — 68万時間の弱教師付き多言語音声、シンプルなエンコーダ・デコーダ・トランスフォーマー、その後のASRリリースすべてを引用させたベンチマーク。しかし認識は推論ではない。「この録音に何楽器が？」「スピーカーが表現する感情は？」「3分目に何が？」というには、オーディオ理解が、文字起こしではなく必要。Qwen-Audio、SALMONN、LTU、NVIDIAのAudio Flamingo 3（AF3、2025年7月）は段階的にそのスタックを構築：Whisper級エンコーダを保持、Q-フォーマーを取り付け、オーディオテキスト指示データで訓練、連鎖思考推論を追加。本レッスンはそのアークを走る。

**タイプ:** ビルド
**言語:** Python（stdlib、log-Mel スペクトログラム + オーディオQ-former スケルトン）
**前提条件:** Phase 6（音声とオーディオ）、Phase 12 · 03（Q-Former）
**所要時間:** 約180分

## 学習目標

- 波形からlog-Melスペクトログラムを計算：ウィンドウイング、FFT、フィルターバンク、対数変換。
- エンコーダオプションを比較：Whisper エンコーダ、BEATs、AF-Whisper ハイブリッド。各が勝つとき。
- オーディオQ-formを構築：N個の学習可能なクエリはスペクトログラムパッチに交差注意。
- カスケード（Whisper-then-LLM）対エンドツーエンドオーディオLLM訓練を説明：なぜエンドツーエンドが推論のスケールで良い。

## 問題

音声認識はWhisperで解決。オーディオのOCRはコモディティ。しかし「コモディティ」は文字起こしで止まる。モデルが聞いたことに推論できない — タイミング、スピーカー、感情、音楽構造、環境音 — 文字起こしだけは製品機能を駆動できない。

3つの明白なルート：

1. カスケード：Whisper が文字起こし、LLMが文字起こしに推論。純粋な音声シナリオで動作。音楽、環境オーディオ、マルチスピーカーオーバーラップ、感情で失敗。

2. エンドツーエンドオーディオLLM：オーディオエンコーダが文字起こしをスキップしてLLMに直接オーディオトークンを供給。音響情報（感情、スピーカー、環境）を保持。新訓練データが必要。

3. ハイブリッド：文字起こしと推論の両方ができるオーディオエンコーダ + テキストデコーダ。Qwen-AudioとAudio Flamingoはこのルートを選択。

## コンセプト

### Log-Mel スペクトログラム：入力機能

すべてのオーディオエンコーダは同じ機能から始まる：log-Mel スペクトログラム。

1. 16kHzにリサンプル。
2. 25msウィンドウ、10msホップでの短時間フーリエ変換。
3. FFT結果の大きさを取得。
4. Melフィルターバンク（通常0-8000 Hzで対数スペース80フィルター）を適用して知覚周波数に歪む。
5. 動的範囲のために対数圧縮（log(1 + x)）。

結果：形状（T、80）の2Dアレイ。T は時間フレーム数。100 Hzフレームレートでの30秒クリップ：（3000、80）。

### Whisperのエンコーダ

Whisper のエンコーダは、log-Mel スペクトログラムを時間フレームシーケンスとして処理する12層ViT スタイルトランスフォーマー。出力：フレームごとに1つの隠れ状態ベクトル。

ASRでは、Whisper のデコーダはエンコーダ出力に条件付けされたテキストトークンを生成する交差注意トランスフォーマー。標準エンコーダ・デコーダ。

ALM（オーディオLLM）では、エンコーダ出力を別のLLMへの入力にしたい。パターン：Whisper エンコーダ凍結、Q-former 訓練可能、LLM 凍結またはチューニング。

### BEATsとオーディオ特有のエンコーダ

Whisperは音声優位データで訓練。音楽と環境オーディオで弱い。

BEATs（Chen et al., 2022）はAudioSetで訓練された自己教師付きトランスフォーマー。同じパラメータ数でWhisperより音楽と環境音を良く捕捉。

AF-Whisper（Audio Flamingo 3のハイブリッド）：Whisper + BEATs機能をオーディオ入力として連結。Whisperは言語信号を、BEATs は音響信号を運ぶ。

### オーディオQ-former

BLIP-2のビジュアルQ-formと同じパターン。固定数の学習可能なクエリ（通常32または64）はオーディオエンコーダ出力フレームに交差注意。クエリはLLMが消費するオーディオトークンになる。

訓練の配置ステージ：Q-form のみ、対比損失+キャプション損失（AudioCaps、Clotho）。指示ステージ：エンドツーエンド、LLMをアンフリーズ、指示データで訓練。

### アーク — SALMONN、Qwen-Audio、AF3

SALMONN（Tang et al., 2023）：Whisper + BEATs + Q-form + LLaMA。真剣な推論能力を持つ最初の開いたオーディオLLM。MMAUベンチマークでは~0.55複合。

Qwen-Audio（Chu et al., 2023）：類似のアーキテクチャ、豊かなデータセットで訓練、多ターンダイアログにチューニング。MMAU ~0.60。

LTU — Listen, Think, Understand（Gong et al., 2023）：明確な推論データ、オーディオクリップ上の連鎖思考に焦点。小さいがより集中。

Audio Flamingo 3（Goel et al., 2025年7月）：現在の開いたSOTA。8B LLMバックボーン（Qwen2 7B）、Whisper-largeエンコーダ連結BEATs、64クエリQ-form、1M以上のオーディオテキスト指示ペアで訓練。MMAU 0.72、いくつかのサブタスクで独自界に一致。

AF3 は、オーディオのオンデマンド連鎖思考も導入：モデルはオプションで最終回答の前に思考トークン（「まず楽器を特定してみましょう：...」）を発行。複雑な推論タスクでの精度は思考が有効で3-5ポイント向上。

### カスケード対エンドツーエンド

カスケード パイプライン：

1. Whisper がオーディオを文字起こし → テキスト。
2. LLMがテキストに推論。

「このポッドキャストを要約」で完璧に動作。失敗：
- 「この歌の気分は？」— 気分は音にあり、言葉ではない。
- 「誰が話している、AliceまたはBob？」— スピーカー識別が必要。
- 「爆発は何秒で起きる？」— テキストで時間的接地が失われた。
- 「本物か生成オーディオ？」— ディープフェイク検出は音響機能が必要。

エンドツーエンドは音響信号を保持。Qwen-AudioとAF3は音楽、環境、感情をネイティブに処理。

### 2026年本番レシピ

新しいオーディオ理解製品のために：

- カスケード：文字起こしが目標、音楽がない、感情推論がない。
- AF3 / Qwen-Audio-family：音楽、感情、マルチスピーカー、または複雑なオーディオ推論。

カスケードは安価でシンプル。エンドツーエンドはより有能。

### MMAU — オーディオ推論ベンチマーク

MMAU（Massive Multimodal Audio Understanding）は2024-2025年のオーディオ推論ベンチマーク：

- 音声、音楽、環境音全体で1万オーディオテキストQAペア。
- 分類、時間推論、因果推論、オープン型QAをカバー。
- カスケード パイプラインが体系的に見逃すものをテスト。

開いたSOTA（AF3）は0.72；独自界~0.78（Gemini 2.5 Pro、Claude Opus 4.7）。ギャップはVideoMMEの開いたvs.クローズド デルタより小さい。オーディオLLMが成熟を示す。

## 使用方法

`code/main.py`：

- stdlib でlog-Mel スペクトログラム計算を実装：ウィンドウイング、ナイーブDFT、Mel フィルターバンク。
- オーディオQ-form スケルトン：エンコーダ出力フレームから、Q、K、V、注意を計算、N トークンを発行。
- おもちゃのタスクでのカスケード vs エンドツーエンド比較。

## 提出

本レッスンは`outputs/skill-audio-llm-pipeline-picker.md`を生成。オーディオタスク（文字起こし、音楽タグ付け、感情推論、マルチスピーカーダイアライゼーション、環境分類）が与えられた場合、カスケード、エンドツーエンドAF3、またはハイブリッドを選択。

## 演習

1. 16kHz、25msウィンドウ、10msホップ、80 Mel bin での30秒クリップのlog-Mel スペクトログラム寸法を計算。48kHzでこれはどう変わる？

2. Whisper がなぜ音楽で過小評価か。BEATs がWhisper よりキャプチャする音響機能は何か？

3. 64クエリ対32クエリのオーディオQ-form：何タスク複雑度で64が利益か？32は何でコンピュートを保存？

4. AF3 Section 4でのオンデマンド思考を読む。連鎖思考が最もオーディオタスクを助ける3つを提案。

5. AF3 出力を使用してミニマル ダイアライゼーション パイプラインを実装。スピーカー変更をどう通知？

## 主要な用語

| 用語 | 人々が言う | 実際の意味 |
|------|------------|---------|
| Log-Mel スペクトログラム | 「Mel 機能」 | Mel フィルターバンク後のlog-大きさ値の2D（時間、周波数）アレイ |
| オーディオQ-former | 「オーディオ Perceiver」 | オーディオエンコーダ出力からLLMに供給する固定長クエリへの交差注意ボトルネック |
| カスケード | 「ASR-then-LLM」 | Whisper が文字起こしし、テキストLLMが推論；音響情報が失われるパイプライン |
| エンドツーエンド | 「オーディオLLM」 | オーディオ機能はQ-form を通じてLLMに直接；音響信号を保持 |
| BEATs | 「オーディオ AudioSet エンコーダ」 | AudioSetで訓練されたSSLトランスフォーマー；Whisper と同じパラメータ数で音楽+環境音が強い |
| MMAU | 「オーディオ推論ベンチ」 | 音声、音楽、環境全体の1万QAペア；2024評価基準 |
| オンデマンド思考 | 「オーディオ CoT」 | モデルは最終回答の前にオプションで推論トークンを発行、精度3-5ポイント向上 |
