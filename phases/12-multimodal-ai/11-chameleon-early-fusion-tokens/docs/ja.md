# Chameleon と早期フュージョントークンのみマルチモーダルモデル

> これまで見た全てのVLMは画像とテキストを別々に保つ。ビジュアルトークンはビジョンエンコーダから来て、プロジェクターに流れ込み、その後LLM内でテキストに会う。ビジョンとテキストの語彙は決して重複しない。Chameleon（Meta、2024年5月）は聞いた：もし彼らが重複したら？VQ-VAEを訓練して画像を共有語彙から離散トークンのシーケンスに変える。すべてのマルチモーダル文書は今1つのシーケンス — テキストトークンと画像トークンが交互に混在、単一の自動回帰損失。副作用：モデルは混合モダリティ出力を生成できる — 単一の推論呼び出しで交互のテキストと画像トークン。このレッスンは早期フュージョン論文を読み、エンドツーエンドでおもちゃのバージョンを構築。

**タイプ:** ビルド
**言語:** Python (stdlib、VQ-VAEトークナイザー+交互デコーダ)
**前提条件:** Phase 12 · 05、Phase 8 (生成AI)
**所要時間:** 約180分

## 学習目標

- 共有語彙+単一損失がモデルができることを変更する理由を説明できる。
- VQ-VAEが画像を離散シーケンスにトークン化してトランスフォーマーの次トークン目標と互換性を持つ方法を説明できる。
- Chameleonの訓練安定性のコツを名前付け：QK-Norm、ドロップアウト配置、LayerNorm順序。
- Chameleon対BLIP-2のQ-Formerアプローチを比較し、各々がいつ正しい選択かを説明できる。

## 問題

アダプターベースのVLM（LLaVA、BLIP-2、Qwen-VL）はテキストと画像を2つの異なるもの扱う。テキストトークンは`embed(text_token)`を通る；画像は`visual_encoder(image) → projector → ... pseudo_tokens`を通る。モデルは途中で合わせる2つの入力パスがある。

3つの結果：

1. LLMは画像を消費できるだけで、出力できない。出力はテキストのみ。
2. 混合モダリティ文書（交互の段落と画像、記事のように）はぎこちない — マルチモーダル入力をモデルの外で解析するか生成を連鎖。
3. 分布ミスマッチ。ビジュアルトークンとテキストトークンは隠れ空間の異なる領域に住む、微妙なアライメント問題を作成。

Chameleonは前提を否定：画像は単に共有語彙から離散トークンのシーケンス。モデルを交互文書で訓練、1つの損失、1つの自動回帰デコーダ、無料で混合モダリティ生成をアンロック。

## コンセプト

### VQ-VAEを画像トークナイザーとして

トークナイザーはベクトル量子化変分オートエンコーダ。アーキテクチャ：

- エンコーダ：CNN + ViTは画像を空間フィーチャーマップ、32x32フィーチャー暗い256にマップ。
- コードブック：K個のベクトル（Chameleonは8192を使用）の学習語彙、同じ暗い256。
- 量子化：各空間フィーチャーについて、L2距離で最も近いコードブックエントリを検索。連続フィーチャーを整数インデックスで置き換え。
- デコーダ：CNNは量子化フィーチャーをピクセルに戻す。

訓練：VAE再構成損失+コミットメント損失+コードブック損失。コードブックインデックスは画像の離散アルファベットを形成。

Chameleonについて：1つの画像は32*32 = 1024トークン8192の語彙から引き出される。テキストトークンで連結（LLMのBPE語彙、32000という）。最終語彙：40192。トランスフォーマーは1つのシーケンス、1つの損失を見る。

### 共有語彙

Chameleonの語彙はテキストトークン、画像トークン、モダリティ分離子を組み合わせ。各トークンは単一のIDを持つ。入力埋め込みレイヤーはすべてのIDをD-暗い隠れベクトルにマップ。出力投影は隠れをボキャブラリロジットに戻す。Softmaxは次のトークンを選ぶ、どんなモダリティでも。

分離子は重要：`<image>`と`</image>`タグは画像トークンシーケンスをブラケット。生成時に、モデルが`<image>`を出力する場合、下流ソフトウェアは次の1024トークンがピクセルレンダリング用デコーダに送信するVQインデックスを知っている。

### 混合モダリティ生成

推論は共有語彙での次トークン予測。プロンプト例：「猫を描いて説明して。」Chameleonは出力：

```
<image> 4821 1029 2891 ... (1024 image tokens) </image>
The cat is orange, sitting on a windowsill...
```

モデルは順序を自律的に選ぶ — 画像を出力した後テキスト、テキスト後画像、または交互。同じデコーダ、同じ損失。

アダプターVLMを比較、生成がテキストのみ。Chameleonはモデル出力モダリティの質問を再度開く。

### 訓練安定性 — QK-Norm、ドロップアウト、LayerNorm順序

早期フュージョン訓練はスケールで不安定。Chameleonの論文3つのコツを文書化：

- QK-Norm。LayerNormを照会投影と鍵投影内で注意内に適用、ドット積前。深さで対数大きさ爆発を防ぐ。複数の事後2024大きなモデルで使用。
- ドロップアウト配置。すべての残差追加後ドロップアウト、注意とMLP後だけではなく。画像トークンからの勾配が支配できるとき、より多くの正則化が必要。
- LayerNorm順序。残差枝にPre-LN（標準）、加えて最後のブロックのスキップ接続に余分なLN。最終層勾配フローを安定。

これらのコツなし、34Bパラメータ Chameleon訓練は複数のチェックポイントで発散。それらと共に、それは収束。訓練レシピはアーキテクチャのようにしてもしなくても貢献。

### トークナイザー再構成の上限

VQ-VAEはロッシー。8192コードブックエントリと1024トークン512x512画像あたり、再構成PSNRは26-28 dB周辺で上限。これは認識可能な画像生成に十分だが、連続空間拡散より目に見える悪い（安定な拡散3は32+ dBを達成）。

トークナイザーはボトルネック。より良いトークナイザー（MAGVIT-v2、IBQ、SBER-MoVQGAN）上限を持ち上げる。Emu3（レッスン12.12）はより良いトークナイザーのみで SDXL品質生成を達成。

### Chameleon対BLIP-2/LLaVA

Chameleon（早期フュージョン、共有語彙）:
- 1つの損失、1つのデコーダ。
- 混合モダリティ出力を生成。
- トークナイザーは品質の上限。
- 高価：生成画像ごとの推論パスでのVQ-VAEデコーダ。

BLIP-2 / LLaVA（後期フュージョン、別タワー）:
- ビジョンインの、テキスト出力のみ。
- 事前訓練されたLLMを再利用。
- 理解のためのトークナイザーボトルネックなし。
- 安い：単一の前方パス。

タスクで選ぶ。画像生成が必要な場合、Chameleonファミリー。理解のみが必要な場合、アダプターVLMはよりシンプルで、より事前訓練された計算を再利用。

### Fuyu と AnyGPT

Fuyu（Adept、2023）は関連アプローチ：完全に別のビジョンエンコーダをスキップ、LLMの入力投影を通して直接生の画像パッチをフィード、トークナイザーなし。Chameleonより単純、共有語彙出力生成を失う。

AnyGPT（Zhan et al、2024）は4つのモダリティにChameleonを拡張：テキスト、画像、スピーチ、音楽。各々のための同じVQ-VAEトリック、共有トランスフォーマー。任意対任意生成。レッスン12.16でもっとカバー。

## 使う

`code/main.py`はおもちゃのエンドツーエンド早期フュージョンモデルを構築：

- 8x8パッチをコードブックインデックスにマップする小さいVQ-VAE風量子化器（K=16）。
- （テキストID 0..31）+（画像ID 32..47）+（分離子48、49）の共有語彙。
- 合成キャプション+画像トークンシーケンスで訓練された小さい自動回帰デコーダ（バイグラムテーブル）。
- プロンプト与えられてテキスト+画像トークンを交互に出力するサンプリングループ。

コードは意図的にトランスフォーマーを小さく保つ（バイグラム）から、エンドツーエンド信号フローをトレースできる。

## デプロイ

このレッスンは`outputs/skill-tokenizer-vs-adapter-picker.md`を作成。製品仕様（理解のみ対理解+生成、必要な画像品質、コスト予算）を与えられて、それはChameleonファミリー（早期フュージョン）とLLaVAファミリー（後期フュージョン）間で選び、定量的経験則で正当化。

## 演習

1. Chameleonは K=8192コードブックエントリと1024トークン512x512画像あたり。24ビットRGB画像対の圧縮比を推定。ロッシー？どのくらいロッシー？

2. 4K画像（3840x2160）同じVQ-VAE密度で多くの画像トークンを作成？Chameleonスタイルモデルは1つの推論呼び出しで4K画像を生成できる？何が最初に壊れる — コンテキスト、トークナイザー品質、またはKVキャッシュ？

3. 純粋なPythonでQK-Normを実装。64-暗い照会と鍵与えられて、ドット積をLayerNorm前後で表示。なぜ大きさ制御は深さで重要？

4. Chameleonセクション2.3を訓練安定性について読む。QK-Normなし34Bで論文が観察した正確な失敗モードを説明。「ノルム爆発」シグネチャは何だった？

5. テキストのみプロンプト与えられた混合モダリティ応答を出力するおもちゃのデコーダを拡張。モデルが訓練データ分布60%テキスト最初/40%画像最初与えられた画像最初対テキスト最初を選ぶ頻度を測定。

## キー用語

| 用語 | 人が言うこと | それが実際に意味すること |
|------|-----------------|------------------------|
| 早期フュージョン | 「統一トークン」 | 画像はステップ1からトランスフォーマーの語彙を共有する離散トークンに変換 |
| VQ-VAE | 「画像トークナイザー」 | CNN + ViT +コードブック、画像を整数インデックスにマップしてトランスフォーマーが予測可能 |
| 共有語彙 | 「1つの辞書」 | テキスト+画像+モダリティ分離子をカバーする単一トークンIDスペース |
| QK-Norm | 「注意安定化器」 | LayerNormはドット積前に照会と鍵に適用、ノルム吹き上がりを防ぐ |
| 混合モダリティ生成 | 「テキスト+画像出力」 | 自律的に1つのパスで交互テキストと画像トークンを作成する推論 |
| コードブックサイズ | 「K個エントリ」 | VQ-VAEが量子化できる離散ベクトル数；圧縮と忠実度をトレード |
| トークナイザー上限 | 「再構成制限」 | VQトークンをデコードすることで達成可能な最高PSNR；モデルの画像品質を制限 |
