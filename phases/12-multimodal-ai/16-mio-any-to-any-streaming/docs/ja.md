# MIO と任意対任意ストリーミングマルチモーダルモデル

> GPT-4oは大部分のオープンモデルが複製できない製品を出荷：エージェントが音声を聞き、ビデオを見て、リアルタイムで戻る話す。2024年後期でのオープンエコシステムの答えはMIO（Wang et al.、2024年9月）。MIOはテキスト、画像、音声、音楽をトークン化、交互シーケンスで1つの因果トランスフォーマーを訓練、任意のモダリティを任意のモダリティに生成。AnyGPT（Zhan et al.、2024年2月）は証明の概念；MIOはスケールアップ；Unified-IO 2（Allen AI、2023年12月）はビジョン+アクショングラウンディングの従兄弟。このレッスンは任意対任意パターン — 4つのトークナイザー、1つのトランスフォーマー、ストリーミングフレンドリーデコード — を読む。

**タイプ:** 学習
**言語:** Python (stdlib、4モダリティトークンアロケータ+ストリーミングデコードループ)
**前提条件:** Phase 12 · 11 (Chameleon)、Phase 6 (音声とオーディオ)
**所要時間:** 約120分

## 学習目標

- テキスト、画像、音声、音楽トークンをホストする衝突なしで共有語彙を設計できる。
- SEED-トークナイザー（画像）とSpeechTokenizer残差VQ（音声）を圧縮+再構成トレードオフで比較できる。
- 任意対任意生成を構築する4段階カリキュラムを説明できる。
- 3つのオープン任意対任意レシピ及びそれらのメイントレードオフを名前付け：MIO、AnyGPT、Unified-IO 2。

## 問題

統一マルチモーダルモデルはスケールで主張しやすく、構築は難しい。ほとんどの「任意対任意」システムまで2024はパイプライン：ビジョンモデル → テキスト表現 → 音声モデル → オーディオ。各ホップは情報を失い、レイテンシを追加、訓練を複雑にする。GPT-4oのデモビデオは秒以下応答での単一モデル代替を示；オープンシステムは月で後ろを続けた。

エンジニアリング課題：

- トークナイザーは全てのモダリティに存在する必要、再構成のために十分なロスレスに圧縮、トランスフォーマーが消費できるレートでトークンを作成。
- 単一語彙はテキスト空間を割り当てる必要（32k+）、画像（16k+）、音声（4k+）、音楽（8k+）。最小40,000以上エントリ。
- 訓練データはすべての入力出力ペア（text→image、image→speech、speech→image等）をカバーする必要またはモデルは組み立てなければならない。
- 推論はストリームアウトトークン十分速く会話レイテンシ（<500ms time-to-first-audio-byte）のため。

## コンセプト

### 4モダリティのための4つのトークナイザー

MIOのトークナイザースタック：

- テキスト：標準BPE、ボキャブラリ約32000。
- 画像：SEED-トークナイザー（2023）—  離散コードブック付き量子化VAE、4096エントリ、32x32トークン画像あたり。
- 音声：SpeechTokenizer残差VQ（2023）— 16kHzウェーブフォームを8階層的コードブックにエンコード；最初レベルは粗いコンテンツ、後レベルは韻律スピーカーアイデンティティを追加。
- 音楽：類似残差VQ（MetaのMusicGen / Encodecファミリー）、4-8コードブック。

各モダリティ整数トークンを作成。トークンは共有語彙で不連続IDセットを得る：

```
text:   0..31999
image:  32000..36095  (4096 image tokens)
speech: 36096..40191  (4096 speech base tokens, plus residual layers)
music:  40192..48383  (8192 music tokens)
sep:    48384..48390  (<image>, <speech>, <music>, </...>, etc.)
```

合計：約48k語彙。入力埋め込みと出力投影はすべてそれを横切る。

### ストリーミングデコード

音声生成は残差VQを使用。トランスフォーマーはベース（レイヤー0）音声トークンを予測；並列デコード残差量子化器は後続レイヤーを予測。各レイヤー0トークンはおおよそ16kHzで50msのオーディオ。

ストリーミングパターン：

1. ユーザーはマイクに話す；リアルタイムオーディオトークナイザーは50msごと音声トークンを出力。
2. MIOは到着するトークンを消費（プロンプトプリフィル+増分前方）。
3. 出力トークンはストリーム生成；並列音声デコーダはそれらをオーディオサンプル約50-150msレイテンシに変換。
4. Time-to-first-audio-byte：MIO論文で約300-500ms、GPT-4oの約250msにアプローチ。

Mini-Omni（arXiv:2408.16725）、GLM-4-Voice（arXiv:2412.02612）、Moshi（arXiv:2410.00037）は補足ストリーミング音声LLMデザイン。Moshi特に単一GPUで160msラウンドトリップを達成。

### 4段階カリキュラム

MIOの訓練カリキュラム：

1. ステージ1 — アライメント。大規模モダリティペアコーパス：テキスト画像、テキスト音声、テキスト音楽。各ペアは独自トークン語彙セグメントを使用。共有語彙を訓練。
2. ステージ2 — 交互。マルチモダリティ交互文書（画像+ビデオを持つブログ、トランスクリプトを持つポッドキャスト等）。クロスモダリティコンテキストを訓練。
3. ステージ3 — 音声強化。テキスト能力を失わずに音声品質を持ち上げるため特別オーディオデータ。
4. ステージ4 — SFT。モダリティを横切る命令チューニング：VQA、キャプショニング、ナレーション、音声対音声対話。

ステージを逃すと特定能力をデグレード：ステージ2をスキップ、モデルはクロスモダリティコンテキストを失う；ステージ3をスキップ、音声は貧弱。

### 視覚的思考の連鎖

MIOは視覚的思考の連鎖を導入：モデルは推論ステップとして中間画像トークンを出力。「猫は木を登っているか？」のために、モデル：

1. シーン（入力画像またはスケッチから）レンダリング`<image>`トークンを出力。
2. スケッチを分析するテキストを出力。
3. 最終回答を出力。

レンダリング中間画像はスクラッチパッドとして機能。ベンチマークは空間推論タスクを改善。アイデアはテキスト推論のための思考の連鎖をミラー。

### 任意対任意で競う

- AnyGPT（arXiv:2402.12226）：4 モダリティ（テキスト、画像、音声、音楽）、類似デザイン。
- Unified-IO 2（arXiv:2312.17172）：ビジョンアクション出力、深さ、ノーマルを追加。より多いタスク多様性、より小さいスケール。
- NExT-GPT（arXiv:2309.05519）：LLM+モダリティ固有拡散デコーダ。単一モデルアプローチではない。
- CoDi（arXiv:2305.11846）：合成可能な拡散；共有隠れを通じた任意対任意。

MIOは純粋トークン任意対任意に最も近い。AnyGPTはその概念的祖先。

### レイテンシ予算

会話製品のため、すべてのコンポーネントのレイテンシは重要：

- マイクからオーディオトークン：約50ms。
- プリフィル（オーディオトークン+履歴）：8Bモデルで約100ms。
- 最初出力トークン：約50ms。
- 並列残差VQ+音声デコーダ：100-150ms。

合計time-to-first-audio-byte：最小約300ms。GPT-4oは約250msを主張。Moshiは160msを主張。MIO/AnyGPT公開ベンチマーク範囲の400-600msはい。

### なぜ任意対任意は難しいのまま

2026年でも、オープン任意対任意モデルは2つの軸で閉じたトレイル：

- 音声品質。残差VQトークナイザーはロッシー；会話音声はElevenLabs級音声と比較してロボットに聞こえる。
- クロスモダリティ推論。「あなたが見るものについて歌う」ことについてモデルに聞く、それでも純粋ビジョンタスクより多くの場合失敗。

これらはオープン研究問題。Qwen3-Omni（レッスン12.20）は2025年で最も高度なオープン試み。

## 使う

`code/main.py`:

- 4モダリティ語彙割り当てを定義、それを印刷。
- マルチモーダル入力（テキスト、画像、オーディオクリップ、音楽）のリストをトークナイザールーターを通してルート。
- レイテンシカウントでテキスト音声応答のストリーミングデコードをシミュレート。
- エンコーダ、プリフィル、デコーダレイテンシ与えられた期待time-to-first-audio-byteを計算。

## デプロイ

このレッスンは`outputs/skill-any-to-any-pipeline-auditor.md`を作成。会話製品仕様（イン、イン、レイテンシターゲット）を与えられて、それはMIOファミリーデザイン選択を監査、レイテンシ予算を計算。

## 演習

1. あなたの製品は音声入力を受け入れて音声出力を返す。エンドツーエンドレイテンシ予算ターゲットは何？時間を過ごすコンポーネントをリスト。

2. SpeechTokenizer残差VQ 8コードブックを使用。並列デコーディング残差レベルが必要な理由（対順序）及びどんなレイテンシ節約をそれが持ち込むか提案。

3. あなたの語彙は32k テキスト+4k画像+4k音声を持つ。8k音楽と~10分離を追加。埋め込みマトリックスパラメータコストは隠れ暗4096？

4. 視覚的思考の連鎖は中間画像を出力。どんな質問は恩恵？どんなは特別トークンに傷つけられる？

5. Moshi（arXiv:2410.00037）を読む。その「内部モノローグ」テクニックを説明、MIOの視覚的思考の連鎖と比較。

## キー用語

| 用語 | 人が言うこと | それが実際に意味すること |
|------|-----------------|------------------------|
| 任意対任意 | 「マルチモーダルイン/アウト」 | テキスト、画像、音声、音楽をどんな方向でも受け入れ出力する単一モデル |
| 残差VQ | 「音声トークナイザースタック」 | マルチコードブックトークン化、各レイヤーは情報を追加；ベースレイヤーはコンテンツ、後レイヤーは韻律 |
| SEED-トークナイザー | 「画像コード」 | MIOで使用される4096エントリコードブック付き離散画像トークナイザー |
| 視覚的思考の連鎖 | 「ビジュアルスクラッチパッド」 | モデルは最終回答前の推論ステップとして中間画像を生成 |
| Time-to-first-audio-byte | 「TTFAB」 | ユーザー音声から最初オーディオ出力へのレイテンシ；会話感のため<500ms |
| 4段階カリキュラム | 「訓練レシピ」 | アライメント -> 交互 -> 音声強化 -> SFT、その順序で |
