# 体現VLA：RT-2、OpenVLA、π0、GR00T

> ロボットがウェブサイトからレシピを読んで、キッチンロボットで実行した最初の時間はRT-2（Google DeepMind、2023年7月）。RT-2はアクションをテキストトークンに離散化、ウェブデータとロボット行動データでVLMをコー細調整、ウェブスケール視言語知識がロボット制御に転送することを証明。OpenVLA（2024年6月）は開きの7B参照をシップ。Physical Intelligence のπ0シリーズ（2024-2025）はフロー一致アクション専門家を加え。NVIDIAのGR00T N1（2025年3月）ヒューマノイド ロボットでスケールでデュアル・システム（System 1 / System 2）制御を配信。VLAプリミティブ — 視言語行動、見て、読んで、機能する単一モデル — は本フェーズの理解モデルとPhase 15の自律システムのブリッジ。

**タイプ:** 学習
**言語:** Python（stdlib、アクション トークナイザー + VLA 推論 スケルトン）
**前提条件:** Phase 12 · 05（LLaVA）、Phase 15（自律システム、参照）
**所要時間:** 約180分

## 学習目標

- アクション トークン化を説明：離散ビン エンコーディング（RT-2）、FAST効率的アクション トークン、連続フロー一致アクション（π0）。
- なぜコー細調整がウェブ+ロボット データで一般知識転送を新しいタスクに保持するか説明。
- OpenVLA（開き7B Llama+VLM）、π0（フロー一致）、GR00T N1（デュアル・システム）を同じロボットタスクで比較。
- Open X-Embodiment データセットを名前付けし、そのRT-Xトレーニング コーパスとしての役割。

## 問題

自然言語指示から家事をするロボットは1970年代から研究目標。2020年代の答え：視言語行動（VLA）モデル。VQA に使用される同じVLMアーキテクチャだがアクション（関節トルク、エンドエフェクタ姿勢、離散コマンド）出力はテキストの代わりに。

VLAに特有のチャレンジ：

1. アクション空間は連続（関節角、力）で高次元（7-DOF アーム + 3-DOF グリッパー = 30 Hz で10次元）。
2. ロボット特定訓練データは希少。Open X-Embodiment は~1M 軌跡；ウェブテキスト画像は5B+。
3. 制御周波数は重要。30 Hz 制御ループは1アクションあたり33ミリ秒予算を意味。
4. 安全性。間違ったアクションはハードウェア、人間、または財産を損傷。

## コンセプト

### アクション トークン化（RT-2）

RT-2のトリック：各関節ターゲットを量子化テキストトークンとして表現。正規化[-1、1]範囲を256ビンに離散化、各ビンを語彙IDにマップ。10-DOF アクションは各制御ステップで10トークンになる。

PaLM-X VLMでミックスにコー細調整：

- ウェブ画像テキスト ペア（キャプション、VQA）。
- ロボット デモンストレーション、アクションはトークンとして。

モデルは「赤いキューブをピック」（言語）-> イメージ（ビジョン）-> 10トークン アクション シーケンス（離散関節ターゲット）を見る。ウェブ事前訓練は一般知識転送を保持：RT-2は「高速移動オブジェクトに向かって移動」に従うことができ、「高速移動」がトレーニング データにない場合でも。

推論はRT-2論文で3-5 Hz、VLM自己回帰デコードで限定。

### OpenVLA — オープン7B参照

OpenVLA（Kim et al., 2024年6月）はオープンウェイトRT-2相当。7B Llamaバックボーン、DINOv2 + SigLIP デュアル ビジョン エンコーダ、256ビンを超えるアクション トークン化。

Open X-Embodiment（22ロボット全体970kの軌跡）で訓練。LoRA 細調整サポートで新しいロボットに適応するために船。

推論：量子化でA100で4-5 Hz。遅い操作に十分ですが、高周波制御ではない。

### FAST トークナイザー — より速いアクション デコード

Pertsch et al.（2024）は離散ビン トークン化は非効率 — ほとんどのアクションはビン空間の小さな領域でクラスター。FAST（周波数領域アクション シーケンス トークナイザー）はDCTとコエフィシエント量子化経由でアクション シーケンスを圧縮。

30ステップ アクション 軌跡は離散ビン トークンの代わりに~10 FAST トークンになる。推論は3-5倍高速化、品質損失なし。

### π0 とフロー一致アクション

Physical Intelligence のπ0（Black et al., 2024年10月）はフロー一致アクション専門家で離散アクション トークンを置き換え：

- 小さいアクション トランスフォーマーはVLMの隠れた状態を読み、修正フロー経由で連続50ステップ アクション シーケンスを出力。
- アクション ヘッドはフロー一致損失で訓練；VLM事前訓練は変わらない。
- 推論：~5デノイズ ステップで完全なアクション シーケンス、効果的に50 Hz 制御。

π0のクレーム：操作タスクの広いスイートでOpenVLAとOctoを打つ。連続アクション公式化は離散化が破壊するスムーズネスを保持。

π0.5 とπ0-FAST は段階的なアップグレード。π0-FAST はFAST トークン化をフロー一致と組み合わせ。

### GR00T N1 — ヒューマノイド向けのデュアル・システム

NVIDIA のGR00T N1（2025年3月）はヒューマノイド ロボット（>30 DOF、全身）向けに建設：

- System 2：大きいVLMは現場+指示を読み、~1 Hz で高レベル サブゴールを生成。
- System 1：小さいアクション・ヘッド トランスフォーマーはサブゴールに条件付けされた低レベル50-100 Hz ジョイント コマンドを生成。

分割はKahnemanの高速・遅い思考にマップ：System 2は計画、System 1は行為。利点：遅いVLMサイズの計画は速い制御をブロックしない；System 1は小さいままレイテンシのために。

GR00T N1.7（2025年後期）はデータ スケーリングを改善。GR00T は Omniverse からのsim-to-realデータで細調整。

### Open X-Embodiment

訓練データ。RT-X（2023年10月）は22ロボット全体1M軌跡をカバーする22データセットをアッセンブル。Open X-Embodiment は皆が使う コーパス：

- ALOHA / Bridge V2 / Droid / RT-2 Kitchen / Language Table。
- 各サンプル：（ロボット 状態、カメラ ビュー、指示、アクション シーケンス）。
- 訓練衛生：アクション 空間を統一、ジョイント 範囲を正規化、カメラをリサイズ。

OpenVLAとπ0は Open X-Embodiment で訓練。任意の特定のロボットへのドメイン ギャップは100-1000タスク特定デモでのLoRA細調整で閉じられ。

### コー細調整対ロボットのみ

コー細調整はウェブVQAデータをロボット軌跡と混合。比率は重要：VQAが多すぎるとモデルはアクションを忘れ；ロボット データが多すぎるとモデルは一般知識を失う。

RT-2の比率：~1:1。OpenVLA：~0.5:1 web-to-robot。π0：類似。正確な比率は各データセット サイズごとにチューニングするハイパーパラメータ。

ロボットのみ訓練は分布外の指示に失敗するタスク特定モデルを生成。コー細調整は「赤いキューブをピック（デモで）」と「左から3番目の最大オブジェクトをピック（新しいフレーズ）」の違い。

### 安全性とアクション制限

すべての本番VLAは船：

- ハード ジョイント 制限（仕様を超えてトルクできない）。
- ベロシティ 制限（ソフト クリッピング）。
- ワークスペース バウンズ（エンドエフェクタはテーブルを離れることはできない）。
- 新しいタスクのための人間ループ承認。

これらはVLAの外側の制御層チェックに座る。VLA出力は提案、コマンドではない。

## 使用方法

`code/main.py`：

- 256ビン アクション トークン化とデ・トークン化を実装。
- FAST トークナイザーのスケッチはDCT + 量子化に基づく。
- アクション ステップごとのトークンカウントを比較（離散ビン、FAST、連続フロー）。
- RT-2 -> OpenVLA -> π0 -> GR00Tの系統図サマリーを印刷。

## 提出

本レッスンは`outputs/skill-vla-action-format-picker.md`を生成。ロボット タスク（操作、ナビゲーション、ヒューマノイド全身）が与えられた場合、離散ビン + RT-2、FAST + OpenVLA、フロー一致 + π0、またはデュアル・システム + GR00T を選択。

## 演習

1. 30 Hz 制御レート上の10-DOF アーム。256ビンで離散ビン トークン化は秒あたりいくつのトークンを発行？7B VLM は キープアップ できる？

2. FAST トークン化は30ステップ軌跡を~10トークンに圧縮。ユーザーが高周波モーション（例、ドラミング）で軌跡を持つ場合、何を失う？

3. π0のフロー一致ヘッドは~5ステップでデノイズ。4-5 Hz でのOpenVLAの自己回帰デコード にスループットを比較。

4. GR00T の System 1 / System 2 分割は Kahneman にマップ。双足歩行ウォーキングを助けるかもしれない異なる分割（System 3？）を提案。

5. Open X-Embodiment Section 4でのデータセット キュレーション を読む。ドメイン リークを防ぐ3つのキュレーション ルールを名前付け。

| 用語 | 人々が言う | 実際の意味 |
|------|------------|---------|
| VLA | 「視言語行動」 | 画像+指示を入力しアクション コマンド を出力するモデル |

| コー細調整 | 「ウェブ+ロボット を混合」 | ウェブVQAデータ隣接ロボット デモで訓練して一般知識を保持 |
| フロー一致アクション ヘッド | 「π0連続出力」 | 修正フロー経由で50ステップ アクション シーケンスを出力する小さなトランスフォーマー |
