# Transfusion: 1つのトランスフォーマーでの自動回帰テキスト+拡散画像

> ChameleonとEmu3は全てを離散トークンに賭けた。彼らは機能するが、量子化ボトルネックは見える — 画像品質は連続空間拡散モデルの下でプラトー。Transfusion（Meta、Zhou et al.、2024年8月）は反対の賭け：画像を連続に保つ、VQ-VAEを完全にドロップ、2つの損失で1つのトランスフォーマーを訓練。テキストトークンは次トークン予測を得る。画像パッチはフロー一致/拡散損失を得る。両目標は同じ重みを最適化。基本アーキテクチャはStable Diffusion 3（MMDiT）は従兄弟に近い。このレッスンはTransfusion論文を読み、おもちゃの2損失トレーナーを構築、1つのトランスフォーマーが両仕事をするのを可能にする注意マスクをトレース。

**タイプ:** ビルド
**言語:** Python (stdlib、MNIST規模のおもちゃでの2損失トレーナー)
**前提条件:** Phase 12 · 11 (Chameleon)、Phase 8 (生成AI)
**所要時間:** 約180分

## 学習目標

- 1つのバックボーンで2つの損失（テキストトークンでのNTP、画像パッチでの拡散MSE）を実行するトランスフォーマーをワイヤ。
- 画像パッチを横切る双方向注意と加えてテキストトークンを超える因果注意がなぜ正しいマスク選択か説明できる。
- Transfusionスタイル（連続画像、拡散損失）をChameleonスタイル（離散画像、NTP）と計算、品質、コード複雑性で比較できる。
- MMDiTの貢献を名前付け：各ブロックでのモダリティ固有重み、残差ストリームでの関節注意。

## 問題

離散対連続画像トークンの議論はLLMより古い。連続表現（生ピクセル、VAE隠れ）は詳細を保持。離散トークン（VQインデックス）はトランスフォーマーのネイティブ語彙に合わせるが、量子化ステップで詳細を失う。

Chameleon / Emu3は離散に：1つの損失、1つのアーキテクチャ、しかし画像忠実度はトークナイザー品質でキャップ。

拡散モデルは連続に：例外的画像品質、しかし LLMから別のモデル、複雑なノイズスケジュールエンジニアリング、テキスト生成との簡潔統合なし。

Transfusionは聞く：両方を持つことができるか？画像を連続に保つ、依然として1つのモデルを訓練、2つの損失を1つの勾配ステップに縫い付け。

## コンセプト

### 2損失アーキテクチャ

単一のデコーダオンリートランスフォーマーは以下を含むシーケンスを処理：

- テキストトークン（離散、BPE語彙から）。
- 画像パッチ（連続、16x16ピクセルブロック隠れ暗に投影、ViTエンコーダの入力と同じことで線形埋め込みを通して）。
- `<image>`と`</image>`タグ連続パッチが住む所をマーク。

前方パスは一度実行。損失はトークンあたり2つのヘッドの1つを選ぶ：

- テキストトークン：ボキャブラリロジットヘッドで標準交差エントロピー。
- 画像パッチ：連続パッチでの拡散損失 — 各パッチに追加されたノイズを予測。

勾配は共有トランスフォーマーボディを通して流れ。両損失は同時に共有重みを改善。

### 注意マスク：因果テキスト+双方向画像

テキストトークンは因果でなければならない — テキストトークンが将来テキストに注意することをあなたは許すことができない、または教師強制は壊れ。画像パッチ、しかし、1つのスナップショットを表現；それらは同じ画像ブロック内で双方向に注意すべき。

マスク：

```
M[i, j] = 1 if:
  (i is text and j is text and j <= i)   # causal for text
  OR (i is image and j is image and same_image_block(i, j))   # bidirectional within image
  OR (i is text and j is image and j < i_image_end)   # text attends to previous images
  OR (i is image and j is text and j < i_image_start)   # image attends to preceding text
```

訓練と推論でブロック三角マスクとして実装。

### トランスフォーマー内拡散損失

拡散損失は標準：画像パッチにノイズを追加、モデルにノイズを予測するように聞く（または正潔パッチ、等価）。Transfusionのバージョンはフロー一致を使用 — ノイズから正潔への速度場を予測。

訓練中：
1. 各画像パッチ x0で、ランダムなタイムステップ tをサンプル。
2. ノイズ εをサンプル、xt = (1-t) * x0 + t * ε（フロー一致のための線形補間）を計算。
3. トランスフォーマー v_theta(xt, t)を予測；損失 = MSE(v_theta(xt, t), ε - x0)。
4. 同じシーケンスからのテキスト NTP損失に並行バックプロップ。

推論で、生成は：
- テキストトークン：標準自動回帰サンプリング。
- 画像パッチ：拡散サンプリングループ（10-30ステップ典型）過去のテキストトークンに条件付き。

### MMDiT: Stable Diffusion 3のバリアント

Stable Diffusion 3（Esser et al.、2024年3月）はTransfusionと同じ時代周辺のMMDiT（Multimodal Diffusion Transformer）を出荷。アーキテクチャはシブリング。

MMDiTのキー差異：

- ブロックあたりのモダリティ固有重み。各トランスフォーマーブロックはテキストトークン対画像パッチのための別のQ、K、V、MLP重みを持つ。注意は関節（クロスモダリティ）；他は全てモダリティ固有。
- Rectified フロー訓練。既知のサンプリングとDDPMより単純数学が具体的フロー一致バリアント。
- スケール。MMDiTはSD3（2Bと8Bパラメータバリアント）のバックボーン。Transfusionの論文は7Bにスケール。

両方は同じコアアイデアに収束：1つのトランスフォーマー実行テキストでのNTPと連続画像表現での拡散。

### なぜこれがChameleonスタイルを打ち負かすか

連続拡散と離散NTP間の品質ギャップ画像生成での測定可能。Transfusion論文レポート：

- 7Bパラメータで、同じサイズChameleonスタイルモデルをFIDで3-5ポイント打ち負かし。
- トークナイザー訓練なし必要 — 画像エンコーダはより単純（隠れへの線形投影、ViTの入力層と同じ）。
- 推論は画像パッチデノイズを並列化、自動回帰画像トークン対。

不利：Transfusionは双損失モデル、訓練力学をトリッキーにする。損失重みは調整が必要。NTP対拡散間スケジュール不一致は1つのヘッドが支配するように引き起こすことができる。

### 何が下流に座るか

Janus-Pro（レッスン12.15）はTransfusionのアイデアを改善理解と生成のためのビジョンエンコーダをデカップル — SigLIPのための1つ、VQ別のもの — トランスフォーマーボディを共有している。Show-o（レッスン12.14）は拡散を離散拡散に交換（マスク予測）。統一生成ファミリーはTransfusionの後急速に枝。

2026本番VLMが画像を出力 — Gemini 3 Pro、GPT-5、Claude Opus 4.7の画像生成パス — ほぼ確実にこのファミリーのいくつかの子孫を使用。詳細は所有権。

## 使う

`code/main.py`は小さいMNISTのような問題の上のおもちゃ Transfusionを構築：

- テキストキャプションは数字を説明する短整数シーケンス（0-9）。
- 画像は4x4バイトグリッド。
- 共有重み線形投影のペアがトランスフォーマースタンドイン機能；テキストでのNTP損失、ノイズパッチでのMSE損失。
- 訓練ループは2つの損失を交互、注意マスク明示的。
- 生成は1つの前方パスで、テキストキャプションと4x4画像を作成。

トランスフォーマーはおもちゃ。2損失配管、注意マスク構築、推論ループは実アーティファクト。

## デプロイ

このレッスンは`outputs/skill-two-loss-trainer-designer.md`を作成。新しいマルチモーダル訓練タスク（テキスト+画像、テキスト+オーディオ、テキスト+ビデオ）を与えられて、それは2損失スケジュール設計（損失重み、マスク形、共有対モダリティ固有ブロック）フラグ実装リスク。

## 演習

1. Transfusionスタイルモデルは70%テキストトークンと30%画像パッチを訓練。画像拡散損失は大きさでテキストNTP損失の約10倍。何損失重みが彼らをバランス？

2. シーケンスのためのブロック三角マスクを実装：`[T, T, <image>, P, P, P, P, </image>, T]`。各エントリ0または1をマーク。

3. MMDiTはモダリティ固有QKV重みを持つ。これはTransfusionの完全に共有されたトランスフォーマー対どのくらいのパラメータ数オーバーヘッドを追加？7Bパラメータでは、それは価値がある？

4. 生成：テキストプロンプト与えられて、モデルは50トークン NTPを実行、その後`<image>`をヒット、その後20デノイズステップで256パッチの上で拡散を実行。合計何前方パス？

5. SD3論文セクション3を読む。Rectified フローを説明し、なぜそれはDDPMより少ないステップで推論に収束。

## キー用語

| 用語 | 人が言うこと | それが実際に意味すること |
|------|-----------------|------------------------|
| 2損失訓練 | 「NTP+拡散」 | 単一トランスフォーマー最適化テキストトークン上の交差エントロピーと同じ勾配ステップで連続画像パッチ上のMSEの両方 |
| フロー一致 | 「Rectified フロー」 | 拡散バリアントはノイズから正潔データへの速度場を予測；DDPMより単純数学 |
| MMDiT | 「Multimodal DiT」 | Stable Diffusion 3のアーキテクチャ：関節注意、モダリティ固有MLP及びノルム |
| ブロック三角マスク | 「因果テキスト+双方向画像」 | テキスト全体で因果しかし画像領域内双方向注意マスク |
| 連続画像表現 | 「VQなし」 | 画像パッチ実値ベクトルとして、整数コードブックインデックスではなく |
| 速度予測 | 「v-パラメータ化」 | ネットワーク出力はノイズ自体ではなく、ノイズとデータ間速度場 |
