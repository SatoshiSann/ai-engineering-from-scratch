# マルチモーダル エージェント とコンピュータ使用（Capstone）

> 2026年先端製品はマルチモーダル エージェント がスクリーンショットを読み、ボタンをクリック、ウェブUIをナビゲート、フォームを埋め、エンドツーエンド でワークフロー を完了。SeeClick とCogAgent（2024）はGUI根拠プリミティブを証明。Ferret-UI はモバイルを加え。ChartAgent はビジュアルツール使用を導入。VisualWebArena とAgentVista（2026）はベンチマーク先端が追うと — そしてGemini 3 Pro とClaude Opus 4.7 でも難しいAgentVista タスク上で~30%スコア。本capstone はPhase 12のすべてのスレッドを引き：知覚（高解像度VLM）、推論（ツール使用LLM）、根拠（座標出力）、ロング地平線メモリ、評価。

**タイプ:** Capstone
**言語:** Python（stdlib、アクション スキーマ + エージェント ループ スケルトン）
**前提条件:** Phase 12 · 05（LLaVA）、Phase 12 · 09（Qwen-VL JSON）、Phase 14（Agent Engineering）
**所要時間:** 約240分

## 学習目標

- マルチモーダル エージェント ループを設計：知覚 -> 推論 -> 行為 -> 観察 -> 繰り返し。
- GUI根拠出力スキーマを構築（クリック座標、タイプ テキスト、スクロール、ドラッグ）VLM が JSON として発行できる。
- スクリーンショットオンリー エージェント 対 アクセシビリティツリー エージェント 対 ハイブリッド エージェント を比較。
- 小さい VisualWebArena スライスでマルチモーダル エージェント ベンチマーク評価をセットアップ。

## 問題

予約サイト ワークフロー：「4月15日にトウキョウへのフライトを探して、通路シート 800ドル以下、予約。」

マルチモーダル エージェント は必要：

1. ブラウザのスクリーンショットを取得。
2. スクリーンショット + URL + ゴール を計画に解析。
3. 構造化アクション を発行：クリック（x,y で）、「トウキョウ」をタイプ（要素E で）、スクロール ダウン、選択（ラジオボタン）。
4. アクション をブラウザに適用。
5. 新しい状態を観察（次のスクリーンショット）。
6. タスクが完了するまで繰り返し。

各ステップはマルチモーダル VLM呼び出し。VLM出力は解析可能 JSON でなければならない。エラーはステップ全体にわたって複合するため、リカバリが重要。

## コンセプト

### GUI根拠 — プリミティブ

GUI根拠は：スクリーンショットと自然言語指示が与えられた場合、（x、y）座標をクリック（または別のアクション）に出力。

SeeClick（arXiv:2401.10935）は最初の開き 結果 スケール：VLM を合成+実 GUI データで細調整、座標を平文トークンとして出力。動作。

CogAgent（arXiv:2312.08914）は1120x1120高解像度エンコーディング を加えた密なUI のため。スコア：ウェブ ナビゲーション上で~84%。

Ferret-UI（arXiv:2404.05719）はモバイル UI に焦点、iOS アクセシビリティ データと統合。

出力フォーマット は通常JSON：

```json
{"action": "click", "x": 384, "y": 220, "element_desc": "Search button"}
```

`element_desc`はリカバリを助け：座標がスクリーンショット間でドリフトする場合、セマンティック ヒント はシステムに再根拠を許す。

### アクション スキーマ

典型的なアクション スキーマは6-10アクション タイプを持つ：

- `click`：（x、y）
- `type`：（テキスト、x?, y?）
- `scroll`：（方向、量）
- `drag`：（x0、y0、x1、y1）
- `select`：（option_index）
- `hover`：（x、y）
- `navigate`：（url）
- `wait`：（ミリ秒）
- `done`：（成功、説明）

エージェント は1ステップあたり1アクションを発行。ブラウザ ラッパーは実行し新しい状態を返す。

### スクリーンショットのみ対 アクセシビリティツリー

2つの入力モード：

- スクリーンショットのみ：完全イメージ、構造情報なし。最も汎用；すべてのアプリで動作。
- アクセシビリティツリー：構造化 DOM / iOS アクセシビリティ 情報。根拠のためにずっとより信頼できる；ツリーが使用可能なところで動作。
- ハイブリッド：両方、アトミック アクション のための信頼できるgrounder としてのツリーとセマンティック コンテキストのためのスクリーンショット。

本番 エージェント はハイブリッド を可能な時に使用。ブラウザ自動化（Selenium + アクセシビリティ）は常にツリーを持つ；デスクトップ アプリ 時々。

### ロング地平線メモリ

20ステップ ワークフロー は20スクリーンショットを生成。VLM のコンテキスト は速く詰まる。3つの圧縮戦略：

- サマリ チェーン：5ステップ毎に、何が起きたかをまとめる、古いスクリーンショットを削除。
- スキップフレーム：最初、最後、3番目のスクリーンショット毎を保つ。
- ツール記録ログ：アクション を実行、テキスト ログを保つ、古いスクリーンショットを再見ない。

Claude のコンピュータ使用API はログ パターンを使用。より単純、より信頼できる。

### ビジュアル ツール 使用

ChartAgent（arXiv:2510.04514）はビジュアル ツール使用をチャート理解のため導入：クロップ、ズーム、OCR、外部検知を呼ぶ。エージェント は出力可能「領域（100、200、300、400）にクロップその後OCRを呼ぶ」ツール呼ぶとして。ツール はテキストを返す；VLM は推論を続ける。

このパターン は一般化：設定のマーク プロンプト、領域注釈、外部検知 ツール は同じフィット「ツール呼ぶを出力、構造化応答を受け取る」スキーマ。

### 2026ベンチマーク

- ScreenSpot-Pro。GUI根拠 ~1k ウェブ スクリーンショット上。開き SOTA Qwen2.5-VL-72B ~85%。先端 ~90%。
- VisualWebArena。エンドツーエンド ウェブ タスク（ショップ、フォーラム、分類。開き SOTA ~20%。Gemini 3 Pro ~27%。
- AgentVista（arXiv:2602.23166）。最難しい2026ベンチマーク。12ドメイン全体のリアリスティック ワークフロー。先端モデル スコア27-40%；開いたモデル 10-20%。
- WebArena / WebShop。より古いベンチマーク；先端で飽和。

### なぜそれはまだ難しい

エージェント パフォーマンス ボトルネック：

1. ファインスケールでのビジュアル根拠。「小さい X をクリック」はモバイル 解像度でしばしば失敗。
2. ロング地平線 計画。10アクション後、エージェント はゴールからドリフト。
3. エラー リカバリ。クリックが失敗（間違ったボタン）、検出 + リカバリはまれに訓練データ。
4. クロスページ コンテキスト。タブ間ジャンプ または長フォーム は状態を失う。

研究方向：メモリ アーキテクチャ、明確なリプランニング、マルチモーダル 検証（アクション成功のためのスクリーンショット マッチ）。

### capstone ビルド・イット

capstone タスク：コンピュータ使用 エージェント を構築する：

1. 予約サイト モック ページのHTML + スクリーンショット を読む。
2. マルチステップ シーケンス の計画：検索 -> 選択 -> フォーム埋め -> 提出。
3. アクション スキーマ にマッチする JSON アクション を発行。
4. 固定10タスク スライスで評価。

レッスン はリアル ブラウザに拡張しやすい scaffold コードを提供。

## 使用方法

`code/main.py` はcapstone scaffold：

- アクション スキーマ JSON 定義（10アクション）。
- モック ブラウザ 状態 dict として。
- エージェント ループ スケルトン：状態を受け取る、アクション を発行、適用、ループ。
- 10タスク ミニベンチマーク（合成 ページ）エンドツーエンド 成功率を測定するため。
- アクション が失敗する時のエラー リカバリ フック。

## 提出

本レッスン は`outputs/skill-multimodal-agent-designer.md`を生成。コンピュータ使用 製品（ドメイン、アクション セット、評価ターゲット）が与えられた場合、完全なエージェント ループ、メモリ 戦略、根拠 モード、期待ベンチマーク スコアを設計。

1. `screenshot_region`ツール（クロップ + ズーム）でアクション スキーマ を拡張。何のタスク が利点？

## 主要な用語

| 用語 | 人々が言う | 実際の意味 |
|------|------------|---------|
| GUI根拠 | 「クリック座標」 | モデル はスクリーンショット上の指示のターゲットのため（x,y）を出力 |
| アクション スキーマ | 「ツール 定義」 | 有効なアクション（クリック、タイプ、スクロール、ドラッグ）の JSON 説明 |
| アクセシビリティツリー | 「構造化 DOM」 | ブラウザ/iOS API からの機械読み取り可能UI 階層 |
| ハイブリッド エージェント | 「スクリーンショット + ツリー」 | イメージと構造化 情報を両方使用；どちらより単独より信頼できる |
| ビジュアル ツール 使用 | 「ズーム/クロップ/検知」 | エージェント は外部ビジョン ツール（OCR、検知）を計画中に呼ぶ |
| サマリ チェーン | 「メモリ 圧縮」 | 定期的なテキスト サマリ は長いスクリーンショット 履歴を置き換える |
| VisualWebArena | 「E2E ウェブ ベンチ」 | エンドツーエンド ウェブ タスク用2024ベンチマーク |
| AgentVista | 「2026難しい ベンチ」 | 12ドメイン リアリスティック ワークフロー；Gemini 3 Pro でも~30%スコア |
