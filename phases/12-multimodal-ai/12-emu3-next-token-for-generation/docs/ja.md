# Emu3: 画像とビデオ生成のための次トークン予測

> BAAIのEmu3（Wang et al., 2024年9月）は拡散対自動回帰の議論を終わらせるべき2024年の結果。単一のLlamaスタイルのデコーダオンリートランスフォーマー、次トークン予測目標のみで訓練、テキスト+VQ画像トークン+3D VQビデオトークンの統一語彙を横切って、SDXLを画像生成で打ち負かし、LLaVA-1.6を知覚で。CLIPロスなし。拡散スケジュールなし。分類器フリーガイダンスは推論で品質のために使用されるが、コア訓練目標は教師強制で次トークン予測。自然に出版。このレッスンはEmu3論文 — なぜより良いトークナイザーとスケールは全てあなたが必要か — を読み、拡散アプローチと対比。

**タイプ:** 学習
**言語:** Python (stdlib、3Dビデオトークナイザー数学+自動回帰サンプラースケルトン)
**前提条件:** Phase 12 · 11 (Chameleon)
**所要時間:** 約120分

## 学習目標

- Emu3の単一損失次トークン目標が拡散が画像品質のために必要という長く持たれた仮定にもかかわらず機能する理由を説明できる。
- 3Dビデオトークナイザーを説明できる：時空VQコードブックが何に見えるか、なぜパッチが時間を横切るか。
- Emu3対Stable Diffusion XLを（訓練計算、推論コスト、品質上限）で比較できる。
- 同じEmu3モデルが演じる3つのロール：Emu3-Gen（画像生成）、Emu3-Chat（知覚）、Emu3-Stage2（ビデオ生成）の名前を挙げられる。

## 問題

2024年を通した習慣的知識：画像生成は拡散が必要。議論：離散画像トークンは詳細を再構成するために多すぎる情報を失い、自動回帰サンプリングは数千のトークン全体にエラーを蓄積。Stable Diffusion、DALL-E 3、Imagen、Midjourneyはすべて拡散の何かの形を使用。Chameleon（レッスン12.11）は小さいスケールでこれを部分的に反証したが、品質でSDXLとマッチしなかった。

Emu3は議論に頭突き攻撃。主張：より良いビジュアルトークナイザー+十分なスケール+次トークン損失 = 知覚も行う同じモデルで拡散打ち負かし画像生成。

賭けが出版したときに論争。2年後、オープンソース統一生成ファミリー（Emu3、Show-o、Janus-Pro、Transfusion）はリサーチのデフォルトパス；本番最前端モデルはいくつかのバリアントを使用するかもしれない。

## コンセプト

### Emu3トークナイザー

重要な成分はビジュアルトークナイザー。Emu3はトークンあたり8x8解像度削減でカスタムIBQクラストークナイザー（Inverse Bottleneck Quantizer、SBER-MoVQGANファミリー）を訓練。512x512画像はコードブックサイズ32768で64x64 = 4096トークンになる。

これはChameleonの8192でK=8192での512x512あたり1024トークンより大きいがトークンあたり安い（小さいコードブック検索、より単純なコーデック）。重要メトリック：再構成PSNR 30.5 dB、Stable Diffusionの連続隠れスペースで32 dBと競争。

ビデオについて：3D VQトークナイザーは時空パッチ（4x4x4ピクセル）を1つの整数にエンコード。8 FPSで4秒のクリップ32フレーム；256x256で4x空間と4x時間削減で、トークン数は(256/4) * (256/4) * (32/4) = 64 * 64 * 8 = 32,768トークン。

トークナイザー品質は上限。Emu3の貢献は部分的に「私たちは非常に良いトークナイザーを訓練した」。

### 単一損失訓練

Emu3は1つの目標を使用：テキストトークン、2D画像トークン、3Dビデオトークンを横切った共有語彙での次トークン予測。重みは訓練中にモダリティ固有係数で掛け算されて貢献を平衡、しかし損失関数は同じ。

ミックスで訓練：
- 画像生成：`<text caption> <image> image_tokens </image>`
- 画像知覚：`<image> image_tokens </image> <question> text_tokens`
- ビデオ生成：`<text caption> <video> video_tokens </video>`
- ビデオ知覚：類推。
- テキストのみ：標準NTP。

モデルはデータ分布から画像トークン対テキストトークンを出力するときを学ぶ。生成は`<image>`タグ後の画像トークンを予測することからモデルから現れる。

### 分類器フリーガイダンスと温度

自動回帰画像生成は推論で分類器フリーガイダンス（CFG）とずっと良い。Emu3はそれを使用：2回生成、完全キャプション付き1回、空のキャプション付き1回、ガイダンス重み（典型的3.0-7.0）でロジットをミックス。これは拡散が使用する同じCFGトリック、自動回帰設定に借用。

温度は重要：高すぎると、成果物；低すぎると、モード崩壊。Emu3の推奨温度は知覚で1.0、画像生成で0.8。

### 3つのロール、1つのモデル

Emu3は3つの機能的に異なるAPI但し1つの基本重みセットとして配付：

- Emu3-Gen。画像生成。入力テキスト、出力画像トークン。
- Emu3-Chat。VQAとキャプショニング。入力画像（トークン）、出力テキスト。
- Emu3-Stage2。ビデオ生成とビデオVQA。入力テキストまたはビデオ、出力テキストまたはビデオ。

タスク固有ヘッドなし。異なるプロンプトテンプレートのみ。同じチェックポイント。

### ベンチマーク

Emu3論文から（2024年9月）：

- 画像生成：MJHQ-30K FIDでSDXLを打ち負かし（5.4対5.6）、GenEval全体（0.54対0.55 — 統計結合）、Deep-Evalの合成でオンパー。
- 画像知覚：VQAv2でLLaVA-1.6を打ち負かし（75.1対72.4）MMMUでおおよそマッチ。
- ビデオ生成：4秒クリップ品質でSoraエラ公的ベンチマークされたモデルと競争FVD。

数値は常に勝つわけではない — Emu3は1ポイントをここでトレードして1ポイントそこで — しかし主張「次トークン予測は全てあなたが必要」はモダリティを横切って防衛可能。

### 計算コスト

Emu3は7Bパラメータモデルで約3000億マルチモーダルトークンで訓練。GPU時間はおおよそLlama-2-7B事前学習に比較可能（A100クラスシリコンで2k-4k GPU年）。Stable Diffusion 3のような拡散モデルは類似予算で訓練するが、別のテキストエンコーダと複雑なパイプラインが必要。

推論で、Emu3は画像あたりSDXLより遅い：30 tok/sで4096画像トークンは512x512画像あたり約2分対SDXL 2-5秒。推測デコーディングとKVキャッシュ最適化はギャップを狭めるが閉じない。自動回帰画像生成は計算ヘビー；これは立つトレードオフ。

### なぜそれが重要か

Emu3の深い貢献は概念的。次トークン予測が画像生成での拡散にマッチするためにスケールする場合、統一モデルパス（1つの損失、1つのバックボーン、どんなモダリティ）は実行可能。将来のモデルは別のテキストエンコーダ、別の拡散スケジューラ、別のVAEが必要ではない。1つのトランスフォーマー、モダリティあたり1つのトークナイザー、スケール。

Show-o、Janus-Pro、InternVL-Uはすべてこの論文に構築またはチャレンジ。中国ラボ（BAAI、DeepSeek）はこの方向にとってより積極的に2025年を通してUS ラボより出版。

## 使う

`code/main.py`は2つのおもちゃの部分を構築：

- 2D対3D VQトークナイザーカウント計算器：（解像度、パッチ、clip_length、FPS）を与えられて、画像対ビデオのトークン数を計算。
- 温度での分類器フリーガイダンス付きの自動回帰画像トークンサンプラー。

CFG実装はEmu3のレシピにマッチ — ガイダンス重みで条件と無条件ロジットをミックス。

## デプロイ

このレッスンは`outputs/skill-token-gen-cost-analyzer.md`を作成。生成製品仕様（画像またはビデオ、ターゲット解像度、品質ティア、レイテンシ予算）を与えられて、それはトークン数、推論コストを計算、Emu3ファミリー対拡散を選ぶ。

## 演習

1. Emu3は8x8削減で512x512画像あたり4096トークン作成。1024x1024と2048x2048の等価を計算。推論レイテンシにはどうなる？

2. Emu3セクション3.3をビデオトークナイザーについて読む。3D VQパッチ形となぜそれは8x8x1ではなく4x4x4。

3. 分類器フリーガイダンス重み5.0対3.0：どんなビジュアル効果？`code/main.py`の数学をトレース。

4. Emu3-7Bで300Bトークンの訓練FLOPSを計算し、Stable Diffusion 3と比較。どちらが訓練するのでより高価？

5. Emu3はFIDでSDXLを打ち負かすが、VQAv2対特化VLMで。統一損失アプローチが異なるベンチマークで特化者対異なる強さを示す理由を説明。

## キー用語

| 用語 | 人が言うこと | それが実際に意味すること |
|------|-----------------|------------------------|
| 次トークン予測 | 「NTP」 | 標準自動回帰損失：token[0..i]与えられてtoken[i+1]を予測；トークン化されたときすべてのモダリティで機能 |
| IBQトークナイザー | 「逆ボトルネック量子化器」 | VQ-VAEのクラスはより大きいコードブック（32768以上）と再構成良い Chameleonより |
| 3D VQ | 「時空量子化器」 | コードブック（時間、行、列）でインデックス；1つのトークンは4x4x4ピクセルキューブをカバー |
| 分類器フリーガイダンス | 「CFG」 | 重み gamma;でロジット条件と無条件をミックス推論で画像品質をブースト |
| 統一語彙 | 「共有トークン」 | テキスト+画像+ビデオはすべて同じ整数スペースから引き出す；モデルはどんなモダリティが次に来るかを予測 |
| MJHQ-30K | 「画像生成ベンチマーク」 | 30kプロンプトでのMidjourney品質ベンチマーク；Emu3はここでFIDをレポート |
