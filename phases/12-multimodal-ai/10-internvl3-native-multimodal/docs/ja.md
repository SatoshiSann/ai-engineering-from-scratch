# InternVL3: ネイティブマルチモーダル事前学習

> InternVL3以前の全てのオープンVLMは同じ3段階のレシピに従っていた：数兆のテキストトークンで訓練されたテキストLLMを取得し、ビジョンエンコーダをボルトオンし、その後シームをファインチューニングする。これは機能するが「アライメント負債」がある — テキストLLMは完全な事前学習予算をテキストのみに使い、ビジュアルトークンをネイティブに理解しない。事後的にビジョンを追加すると、LLMはテキストを忘れずにビジュアル入力をテキスト推論に関連付ける方法を再学習しなければならない。InternVL3（Zhu et al., 2025年4月）は事後的アプローチを否定する：1回の事前学習実行、テキストとマルチモーダルが最初から交互に混在。結果は78Bパラメータのオープンモデルで、MMMU-ProについてGemini 2.5 Proと同等。このレッスンはネイティブ事前学習の根拠と、それを実装した場合の変更点を読む。

**タイプ:** 学習
**言語:** Python (stdlib、訓練コーパスミキサー)
**前提条件:** Phase 12 · 05、Phase 12 · 07 (レシピ)
**所要時間:** 約120分

## 学習目標

- 事後的VLM訓練がアライメント負債を蓄積する理由を説明し、3つの測定可能な症状（破滅的忘却、回答ドリフト、ビジュアル-テキスト矛盾）を引用できる。
- InternVL3のネイティブ事前学習コーパスミックスとなぜテキスト:交互:キャプションの比率が重要なのかを説明できる。
- V2PE（可変ビジュアルポジションエンコーディング）をQwen2-VLのM-RoPEと比較できる。
- Visual Resolution Router（ViR）とDecoupled Vision-Language（DvD）デプロイメント最適化の名前を挙げられる。

## 問題

事後的VLM訓練はデフォルトである。LLaVA、BLIP-2、Qwen-VL、Idefics — すべて既に事前学習されたLLM（Llama、Vicuna、Qwen、Mistral）を取得してビジョンを追加する。訓練段階は通常以下のようになる：

1. 固定LLM + 固定ビジョンエンコーダ + 訓練可能なプロジェクター、キャプションペアで訓練され埋め込みを整列。
2. LLMをアンフリーズ、命令データで訓練（LLaVA-Instruct、ShareGPT4V）。
3. 任意のタスク固有のファインチューン。

アライメント負債の3つの症状が現れる：

- 破滅的忘却。事後的VLMはテキストのみのスキルを忘れる。GSM8Kスコアが5-10ポイント低下。Hellaswagスコアが低下。テキストのみのエージェントが回帰。
- 回答ドリフト。同じビジュアル質問の小さな言い回しが異なる回答を得る。ビジョンエンコーダはLLMに弱い結合でLLM自体のトークンより接続される。
- ビジュアル-テキスト矛盾。VLMは画像を正しく説明できるが、その後、自分の説明と矛盾する質問に答える。ビジュアルトークンはテキストと同じ方法でLLMの内部一貫性チェックに参加しない。

これらの症状はよく文書化されている。MM1.5セクション4はそれらを定量化する。LLaVA-OneVisionのアブレーションはそれらをほのめかしている。ネイティブ事前学習がその答え。

## コンセプト

### ネイティブマルチモーダル事前学習

InternVL3は最初からネイティブマルチモーダルなコーパスで訓練する。ミックスは：

- 40%テキストのみデータ（FineWeb、Proof-Pile-2など）
- 35%交互画像-テキストデータ（OBELICS、MMC4スタイル）
- 20%ペアの画像-キャプションデータ
- 5%ビデオ-テキストデータ

ビジョントークン、テキストトークン、クロスモーダル相互作用すべてが最初の勾配ステップから同じ損失に参加する。アライメント事前学習なし、プロジェクター固定段階なし、回復する破滅的忘却なし。

訓練はベースモデルの単一段階である。命令チューニングが続くが、ベースモデルはビジュアルトークンを一級市民として既に理解している。

### V2PE（可変ビジュアルポジションエンコーディング）

Qwen2-VLは固定軸割り当てでM-RoPEを使用する。InternVL3はV2PEを導入する：ポジションエンコーディングはモダリティタイプ（テキスト、画像、ビデオ）ごとに変わり、学習可能なスケーリングがある。実際には：

- テキストトークンは1Dポジション（テキストインデックス）を取得する。
- 画像パッチは2Dポジション（行、列）を取得する。
- ビデオフレームは3Dポジション（時間、行、列）を取得する。

3つは同じRoPE周波数ベースを共有するが、バンドあたりの隠れた次元割り当ては固定分割ではなく学習可能なパラメータである。事前学習中に時間周波数と空間周波数解像度をトレードオフする自由度。

V2PEのアブレーション主張：同じ計算でのビデオベンチマークではM-RoPEより1-2ポイント。革命ではないが、よりクリーン。

### Visual Resolution Router（ViR）

デプロイメント最適化。すべての画像が完全解像度エンコーディングが必要ではない。低いディテールの1つのオブジェクトを持つ写真は、1280pxネイティブでエンコードされるとトークンを無駄にする。ViRはエンコーディング前に質問に答えるために必要な最小解像度を予測する小さいクラシファイア。

ルーティングは3つのティアがある：低解像度（256トークン）、中（576）、高（2048以上）。本番トラフィックの60%のクエリに対して、低または中で十分。ネット効果：同じ品質で2-3倍のスループット。

### Decoupled Vision-Language デプロイメント（DvD）

大規模なVLMを提供する場合、ビジョンエンコーダは画像ごとに1回実行されますが、LLMはすべての出力トークンに対して自動回帰的に実行されます。2つのコンポーネントには異なるボトルネック（ビジョン = conv + attentionのGPUメモリ帯域幅; LLM = KVキャッシュ）があります。DvDは異なるGPUに分割し、ストリーミング間で接続されます。

8B + 400Mエンコーダモデルの場合、DvDは共存配置と比較して、ノードあたりのスループットをおおよそ2倍にします。

### 単一段階対マルチ段階品質

InternVL3の主要ベンチマーク主張：78Bパラメータで、Gemini 2.5 ProのMMMU-Proに一致。38Bで、GPT-4oに一致。8Bで、オープン8Bリーダーボードを主導。すべて単一段階のプリトレイン+命令チューンレシピで。

アライメント負債仮説は測定可能：InternVL3-8BはビジョンベンチマークゲインのユニットあたりQwen2.5-VL-7Bより少ないテキストベンチマークポイント（MMLU、GSM8K）を失う。訓練が2つの部分ではなく1つであるため、モデルはより汎用性が高い。

### InternVL3.5とInternVL-U

InternVL3.5（2025年8月）レシピをスケール。同じネイティブ事前学習アプローチ、より多くのデータ、より多くのパラメータ。MMMUの改善は段階的。

InternVL-U（2026）統一生成を追加 — 同じバックボーンの上のMMDiTヘッドを介した画像出力。「U」は「Understanding + generation」を表し、Transfusionスタイルの統一モデル（レッスン12.13）を追求している。同じネイティブ事前学習バックボーンは理解と生成ヘッドの両方をサポートする。

### ネイティブ事前学習のトレードオフ

ネイティブ事前学習は無料ではない：

- 計算。ゼロからVLMを訓練することはテキストLLMを訓練するのと同じコスト — 何百万ものGPU時間。事後的適応は既存のLLM重みを再利用し、ほとんどのコストを節約。
- データ。スケールでの交互画像-テキストコーパスは稀。OBELICSは141M文書、MMC4は571M。テキスト単独は15Tトークンで出荷。マルチモーダル事前学習データの希少性はハード制約。
- ベースLLM再利用。ネイティブ事前学習は後で新しいLLMをドロップする選択肢を放棄する。事後的は適応器のみを再訓練することでLlama-3.1をLlama-4に交換できる。

InternVL3が行う賭け：アライメント負債は再利用損失より悪い。ベンチマークは主張を支持。生産コストはバーを上げ、将来のラボが安価に複製することを防ぐ。事後的VLMはほとんどのプロジェクトでも安いため、存在し続ける。

## 使う

`code/main.py`は訓練コーパスミキサーとViRルーターシミュレーター。それは：

- ターゲットコーパスミックス（%text、%interleaved、%caption、%video）を取り、モダリティあたりの予想ステップを計算。
- クエリのバッチでViRルーティングをシミュレート（分布：50%低詳細、30%中、20%高詳細）して平均トークン数をレポート。
- エンコーダ対LLM FLOPSを与えるDvDスループット推定をレポート。
- 事後的対ネイティブ事前学習の並行を印刷パラメータ、計算、データ、および予想されるアライメント負債症状。

## デプロイ

このレッスンは`outputs/skill-native-vs-posthoc-auditor.md`を作成する。提案されたVLM訓練計画を与えられて、それはネイティブか事後的かを監査し、アライメント負債リスクをフラグ、コーパスミックスを推奨。新しいオープンVLMプロジェクトのサイズを決定する必要があり、訓練戦略を選ぶ必要があるときそれを使用。

## 演習

1. InternVL3-8B（ネイティブプリトレイン）とLLaVA-OneVision-7B（事後的）間の計算デルタを推定。GPU時間の比率おおよそ？ギャップを説明するもの？

2. InternVL3は40%テキスト/35%交互/20%キャプション/5%ビデオをレポート。ターゲットタスクがビデオヘビーの場合、新しい比率を提案し、なぜベースモデルが依然として実質的なテキストとキャプションデータが必要なのか議論する。

3. MM1.5セクション4を忘却について読む。事後的訓練が最大回帰を示した正確なベンチマークを名前付け。回帰はどのくらいのコストがかかった？

4. ViRは60%のトラフィックを低解像度エンコーディングにルーティング。どんなクエリがそれをミスルーティング（高解像度が必要であった場合低解像度に送信）？3つのルーター失敗モードを提案。

5. DvDはビジョンとLLMを異なるGPUに分割。どんなトラフィックパターンの下でDvDはヘルプの代わりにスループットを傷つける？

## キー用語

| 用語 | 人が言うこと | それが実際に意味すること |
|------|-----------------|------------------------|
| ネイティブマルチモーダル事前学習 | 「一緒に最初から」 | テキスト+画像+ビデオトークンは後でボルトオンではなくステップ1から損失に参加 |
| アライメント負債 | 「事後的ペナルティ」 | 測定可能な回帰ビジョンを固定LLMにボルトオンすることから来るテキストスキルと回答一貫性 |
| V2PE | 「可変ビジュアルポジションエンコーディング」 | モダリティごとの学習可能なポジションエンコーディング割り当て；InternVL3のM-RoPE後継 |
| ViR | 「解像度ルーター」 | エンコーディング前にクエリごとに必要な最小解像度を選び、推論トークンを節約する小さいクラシファイア |
| DvD | 「デカップリングされたデプロイメント」 | ビジョンエンコーダを1つのGPU、LLMを別のGPU、ストリームハンドオフ；大規模VLMのスループットを2倍 |
| InternVL-U | 「統一理解+生成」 | 2026フォローアップはネイティブ事前学習バックボーンに画像生成ヘッドを追加 |
| 交互コーパス | 「OBELICS/MMC4」 | 自然な読み順でテキストと画像を含む文書；ネイティブ事前学習の原材料 |
