# LoRA と QLoRA によるファインチューニング

> 完全なファインチューニングの7Bモデルは56GBのVRAMが必要です。あなたはそれを持っていません。ほとんどの企業もそうではありません。LoRAを使用すると、パラメータの1%未満を訓練することで、同じモデルを6GBでファインチューニングできます。これは妥協ではありません - ほとんどのタスクで完全なファインチューニング品質に匹敵します。オープンソース ファインチューニング エコシステム全体がこの1つのトリックで実行します。

**タイプ:** ビルド
**言語:** Python
**前提条件:** Phase 10, Lesson 06（命令チューニング/ SFT）
**所要時間:** 約75分
**関連:** Phase 10はSFT/DPOループをゼロから対象にします。このレッスンは、2026年のPEFTツールキット（PEFT、TRL、Unsloth、Axolotl、LLaMA-Factory）にそれらを接続します。

## 学習目標

- 事前訓練済みモデルのアテンション層に低ランク アダプター行列（AとB）を注入することで、LoRAを実装します
- LoRA対完全なファインチューニングのパラメータ節約を計算します：ランク r、d_model寸法列車 2*r*d パラメータ。d^2ではなく
- QLoRA（4ビット量子化ベース+LoRAアダプター）を使用してモデルをファインチューニングし、コンシューマーGPUメモリ内に収まる
- LoRA重みを基本モデルにマージし直して展開し、アダプターを使用した推論速度と比較します

## 問題

基本モデルがあります。Llama 3 8B。会社の声で顧客サポートチケットに答えるようにしたい。SFTはその答えです。しかし、SFTにはコスト問題があります。

完全なファインチューニングはモデルのすべてのパラメータを更新します。Llama 3 8Bは80億パラメータを持っています。fp16では、各パラメータは2バイトです。これは重みをロードするためだけに16GBです。訓練中に、勾配（16GB）、Adam用のオプティマイザ状態（運動+分散で32GB）、アクティベーション。合計：単一8Bモデルのためにおよそ56GBのVRAM。

A100 80GBはこれに装着することができます。2つのA100は、クラウドプロバイダーで毎時3-4ドルの費用がかかります。50,000の例に対して3つのエポックで訓練は6-10時間かかります。それは$30-40実験。ハイパーパラメータを正しく取得するために10個の実験を実行し、展開前に$400を費やしました。

Llama 3 70Bにこれをスケーリングし、数字が不合理になります。単独の重みに140GB。クラスターが必要。実験$100以上。

よりディープ問題もあります。完全なファインチューニング。モデル内のすべての重みを変更します。顧客サポート データをファインチューニング場合、モデルの一般的な能力を低下させるかもしれません。これは、破壊的な忘却と呼ばれます。モデルはタスクで良くなり、すべての他で悪くなります。

より少ないパラメータを訓練し、より少ないメモリを使用し、モデルの既存の知識を破壊しないメソッドが必要です。

## 概念

### LoRA：低ランク適応

Edward HuおよびMicrosoftの同僚は2021年6月にLoRAを公開しました。論文の洞察：ファインチューニング中の重みの更新は、低い固有ランクを持っています。4096x4096重みマトリックスのすべての16.7百万パラメータを更新する必要はありません。更新の有用な情報はランク16または32のマトリックスでキャプチャできます。

数学です。標準的な線形層は計算します：

```
y = Wx
```

Wは d_out x d_in マトリックス。4096x4096アテンション投影の場合、それは16,777,216パラメータです。

LoRAはWを凍結し、低ランク分解を追加します：

```
y = Wx + BAx
```

Bは（d_out x r）、Aは（r x d_in）です。ランク r はd より遥かに小さい - 通常8、16、または32。

r=16で4096x4096層：
- オリジナルパラメータ：4096 x 4096 = 16,777,216
- LoRAパラメータ：（4096 x 16）+（16 x 4096）= 65,536 + 65,536 = 131,072
- 削減：131,072 / 16,777,216 = 0.78%

パラメータの0.78%を訓練し、95-100%品質を得ています。

### スケーリング係数：Alpha

LoRAはスケーリング係数アルファを導入し、低ランク更新が出力に影響する程度を制御します：

```
y = Wx + （alpha / r） * BAx
```

Alpha = rの場合、スケーリングは1x。Alpha = 2r（一般的なデフォルト）の場合、スケーリングは2x。このハイパーパラメータは、LoRAパスの学習レートを基本学習レートとは独立に制御します。

実用的なガイダンス：
- alpha = 2 * ランクは一般的なコミュニティ慣例（オリジナルペーパーはほとんどの実験でalpha = rankを使用）
- alpha = ランク保守的だが安定。1xスケーリングを与えます
- 高いアルファは、より大きい更新/段階あたりを意味し、収束を加速させたり、不安定性を引き起こす可能性があります

### どこにLoRAを適用するか

トランスフォーマーには多くの線形層があります。すべてにLoRAを追加する必要はありません。オリジナルペーパーは異なる組み合わせをテストしました：

| ターゲット層 | 訓練可能パラメータ（7B） | 品質 |
|------------|-------------------------|------|
| q_projのみ | 4.7M | 良 |
| q_proj + v_proj | 9.4M | より良い |
| q_proj + k_proj + v_proj + o_proj | 18.9M | アテンション用最高 |
| すべての線形（アテンション + MLP） | 37.7M | 限界利益、2xパラメータ |

ほとんどのタスク用のスイートスポット：q_proj + v_proj。これはクエリーおよび値投影をターゲットにします。どちらが、モデルが何に注意し、どの情報を抽出するかを制御します。MLPレイヤーを追加してコード生成のような複雑なタスク用に役立つが、シンプルなタスクの利益の削減のために2倍のパラメータ。

## キーターム

| 用語 | 簡潔な定義 |
|------|---------|
| LoRA | 基本重みをフリーズし、小さなAとBマトリックスを訓練する低ランク適応 |
| QLoRA | 4ビットNF4で基本モデルを量子化し、fp16でLoRAアダプターを訓練 |
| ランク（r） | 低ランク分解の内部寸法。表現力対パラメータ数を制御 |
| Alpha | LoRA出力のスケーリング係数。訓練中の適応の強度を制御 |
| NF4 | 4ビット量子化。ニューラルネットワーク重みの分布に最適化 |
| アダプター | 保存されたLoRA AとBマトリックス。完全なモデルなしで任意のベースモデルコピーに読み込める |
| マージ | W + （alpha/r） * BAを計算し、アダプター オーバーヘッドなしで、新しい完全モデルを保存します |

詳細については、英語ドキュメントを参照してください。
