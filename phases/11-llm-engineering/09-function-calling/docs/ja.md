# 関数呼び出し及びツール使用

> LLMは何もできません。彼らはテキストを生成します。それが全体的な能力です。天気をチェックできない、データベースをクエリできない、メールを送信できない、コードを実行できない、ファイルを読むことができない。見たことがあるすべての「AIエージェント」は、どの関数を呼び出すかを言うJSONを生成するLLMです - その後、実際にコードがそれを呼び出します。モデルは脳です。ツールは手です。関数呼び出しはそれらを接続する神経系です。

**タイプ:** ビルド
**言語:** Python
**前提条件:** Phase 11 Lesson 03（構造化出力）
**所要時間:** 約75分
**関連:** Phase 11 · 14（モデルコンテキストプロトコル）- ツールがホスト間で共有されるとき、インラインの関数呼び出しからMCPサーバーにアップグレードしてください。このレッスンはインラインケースを対象にしています。MCPはプロトコルケースを対象にしています。

## 学習目標

- ツール スキーマを定義し、モデルのツール呼び出しJSONを解析し、関数を実行し、結果を返すという関数呼び出しループを実装します
- モデルが確実に呼び出すことができるように、明確な説明と型付きパラメータを持つツール スキーマを設計します
- 複雑なクエリーに答えるために複数の関数呼び出しを連鎖させるマルチターン エージェント ループを構築します
- 関数呼び出しのエッジケースを処理します：並列ツール呼び出し、エラー伝播、無限ツール ループを防止

## 問題

チャットボットを構築します。ユーザーが尋ねます：「今、東京の天気は何ですか？」

モデルは応答します：「私はリアルタイム天気データにアクセスできませんが、季節に基づいて、東京はおそらく15度摂氏の周辺です...」

これは免責事項に着た幻覚です。モデルは天気を知っていません。決してそうなりません。天気は毎時間変わります。モデルのトレーニング データは数ヶ月前です。

正しい答えはOpenWeatherMap APIを呼び出し、現在の温度を取得し、実数を返す必要があります。モデルはAPIを呼び出すことができません。あなたのコードができます。不足している部分：モデルに「I必要ですこれらの引数でこの関数を呼び出す」を言わせ、アプリケーションが実行するのを許可する構造化プロトコル。

これが関数呼び出しです。モデルは、呼び出す関数と引数をJSON構造で出力します。アプリケーションが関数を実行し、結果をキャプチャします。結果は会話に戻り、モデルは実データを使用して最終的な答えを生成します。

モデルが何も実行しません。呼び出す内容と引数のみ決定します。あなたのコードが実行者です。

## キーターム

| 用語 | 簡潔な定義 |
|------|---------|
| 関数呼び出し | モデルが特定の引数を使用してどの関数を呼び出すかを説明する構造化JSON。コードが実行します |
| ツール定義 | 機能の内容、パラメータ、型を記述するJSONスキーマ |
| ツール選択 | モデルが必須（required）、自動（auto）、または特定（named）ツールを呼び出すかどうかを制御 |
| 並列呼び出し | モデルが単一のターンで複数のツール呼び出しを出力。ラウンドトリップを削減 |
| ツール結果 | 実行結果。モデルが実データを使用できるように会話に戻ります |
| MCP | モデルコンテキストプロトコル。あらゆる互換クライアント間で移植可能ツール共有 |

詳細については、英語ドキュメントを参照してください。
