## 問題

テキストを分類する必要がある。メールをスパムと非スパムに。顧客レビューをポジティブとネガティブに。サポートチケットをカテゴリーに。何千もの特徴量（単語ごとに1つ）と限られた訓練データがある。

ほとんどの分類器はここで詰まる。ロジスティック回帰は何千もの重みを確実に推定するのに十分なサンプルが必要だ。決定木は1単語ずつ分割して激しく過学習する。10,000次元でのKNNは無意味だ。なぜならすべての点が他のすべての点と同じくらい遠いからだ。

ナイーブベイズはこれを処理する。数学的に間違った仮定をして（クラスが与えられたとき、すべての特徴量が他のすべての特徴量から独立しているという仮定）、それでもテキスト分類で「よりスマートな」モデルを上回る。特に小さい訓練セットで。単一のデータパスで訓練する。数百万の特徴量にスケールする。確率の推定を生成する（ただし独立仮定のため多くの場合較正が不十分だ）。

間違った仮定が良い予測につながる理由を理解することは、機械学習についての根本的なことを教えてくれる：最良のモデルは最も正しいものではなく、データに対して最良のバイアスと分散のトレードオフを持つものだ。

## コンセプト

### ベイズの定理（簡単なレビュー）

ベイズの定理は条件付き確率を反転させる：

```
P(クラス | 特徴量) = P(特徴量 | クラス) * P(クラス) / P(特徴量)
```

`P(クラス | 特徴量)` が必要だ -- 文書内の単語が与えられたとき、その文書がクラスに属する確率。これは以下から計算できる：
- `P(特徴量 | クラス)` -- このクラスの文書でこれらの単語を見る尤度
- `P(クラス)` -- クラスの事前確率（一般的にスパムはどれくらい一般的か？）
- `P(特徴量)` -- 証拠、すべてのクラスで同じなので比較するときは無視できる

`P(クラス | 特徴量)` が最も高いクラスが勝つ。

### ナイーブな独立仮定

`P(特徴量 | クラス)` を正確に計算するには、すべての特徴量の同時確率を推定する必要がある。10,000語の語彙では、2^10,000の可能な組み合わせに対する分布を推定する必要がある。不可能だ。

ナイーブな仮定：クラスが与えられたとき、すべての特徴量は条件付き独立だ。

```
P(w1, w2, ..., wn | クラス) = P(w1 | クラス) * P(w2 | クラス) * ... * P(wn | クラス)
```

1つの不可能な同時分布の代わりに、n個の簡単な特徴量ごとの分布を推定する。各分布にはカウントだけが必要だ。

この仮定は明らかに間違いだ。「機械」と「学習」という単語はいかなる文書でも独立ではない。しかし分類器は正確な確率の推定を必要としない。正確なランキングを必要とする -- どのクラスが最も高い確率を持つか。独立仮定は系統的な誤差を導入するが、その誤差はすべてのクラスに同様に影響するので、ランキングは正しいままだ。

### なぜそれでも機能するか

3つの理由：

1. **較正よりランキング。** 分類は上位ランクのクラスが正しいことだけが必要だ。P(スパム)=0.99999でも真の確率が0.7でも、分類器はスパムを正しく選ぶ。正確な確率が必要ではない。正しい勝者が必要だ。

2. **高バイアス、低分散。** 独立仮定は強い事前分布だ。モデルを強く制約し、過学習を防ぐ。限られた訓練データでは、わずかに間違っているが安定したモデルが、理論的には正しいが激しく不安定なモデルを上回る。これがバイアスと分散のトレードオフだ。

3. **特徴量の冗長性が打ち消し合う。** 相関した特徴量は冗長な証拠を提供する。分類器はこの証拠を二重に数えるが、正しいクラスに対しても二重に数える。「機械」と「学習」が常に一緒に現れる場合、両方が「テック」クラスの証拠を提供する。NBはそれらを2回数えるが、正しいクラスに対して2回数える。

4番目の実践的な理由：ナイーブベイズは非常に高速だ。訓練は単一のデータパスで頻度を数えるだけだ。予測は行列乗算だ。数百万の文書を秒で訓練できる。この速さにより、より速く反復し、より多くの特徴量セットを試し、遅いモデルよりも多くの実験を実行できる。

### 数学のステップバイステップ

具体的な例を通じて追いかけてみよう。2つのクラスがあるとする：スパムと非スパム。語彙には3つの単語がある：「free」、「money」、「meeting」。

訓練データ：
- スパムメール：「free」が80回、「money」が60回、「meeting」が10回（合計150単語）
- 非スパムメール：「free」が5回、「money」が10回、「meeting」が100回（合計115単語）
- メールの40%がスパム、60%が非スパム

ラプラス平滑化（alpha=1）で：

```
P(free | spam)    = (80 + 1) / (150 + 3) = 81/153 = 0.529
P(money | spam)   = (60 + 1) / (150 + 3) = 61/153 = 0.399
P(meeting | spam) = (10 + 1) / (150 + 3) = 11/153 = 0.072

P(free | not-spam)    = (5 + 1) / (115 + 3) = 6/118 = 0.051
P(money | not-spam)   = (10 + 1) / (115 + 3) = 11/118 = 0.093
P(meeting | not-spam) = (100 + 1) / (115 + 3) = 101/118 = 0.856
```

新しいメールには：「free」（2回）、「money」（1回）、「meeting」（0回）。

```
log P(spam | email) = log(0.4) + 2*log(0.529) + 1*log(0.399) + 0*log(0.072)
                    = -0.916 + 2*(-0.637) + (-0.919) + 0
                    = -3.109

log P(not-spam | email) = log(0.6) + 2*log(0.051) + 1*log(0.093) + 0*log(0.856)
                        = -0.511 + 2*(-2.976) + (-2.375) + 0
                        = -8.838
```

スパムが大差で勝つ。「free」が2回現れることはスパムの強い証拠だ。「meeting」が現れないことは両方のログ和にゼロを寄与することに注意する（0 * log(P)）-- 多項式NBでは、存在しない単語は効果がない。ベルヌーイNBが単語の不在を明示的にモデル化する。

### 3つの変種

ナイーブベイズには3つのフレーバーがある。各フレーバーは `P(特徴量 | クラス)` を異なる方法でモデル化する。

#### 多項式ナイーブベイズ

各特徴量をカウントとしてモデル化する。特徴量が単語頻度またはTF-IDF値であるテキストデータに最適だ。

```
P(word_i | class) = (クラス内のword_iのカウント + alpha) / (クラス内の総単語数 + alpha * 語彙サイズ)
```

`alpha` はラプラス平滑化（以下で説明）だ。この変種はテキスト分類のワークホースだ。

#### ガウスナイーブベイズ

各特徴量を正規分布としてモデル化する。連続特徴量に最適だ。

```
P(x_i | class) = (1 / sqrt(2 * pi * var)) * exp(-(x_i - mean)^2 / (2 * var))
```

各クラスは特徴量ごとに独自の平均と分散を持つ。特徴量が各クラス内で本当にベルカーブに従う場合によく機能する。

#### ベルヌーイナイーブベイズ

各特徴量をバイナリ（存在または不在）としてモデル化する。短いテキストまたはバイナリ特徴量ベクトルに最適だ。

```
P(word_i | class) = (word_iを含むクラス内の文書数 + alpha) / (クラス内の総文書数 + 2 * alpha)
```

多項式とは異なり、ベルヌーイは単語の不在を明示的にペナルティ化する。「free」が通常スパムに現れるがこのメールには不在の場合、ベルヌーイはそれをスパムに対する証拠としてカウントする。

### 各変種をいつ使うか

| 変種 | 特徴量タイプ | 最適な用途 | 例 |
|------|------------|-----------|---|
| 多項式 | カウントまたは頻度 | テキスト分類、bag-of-words | メールスパム、トピック分類 |
| ガウス | 連続値 | 正規分布に近い特徴量を持つ表形式データ | アイリス分類、センサーデータ |
| ベルヌーイ | バイナリ（0/1） | 短いテキスト、バイナリ特徴量ベクトル | SMSスパム、存在/不在特徴量 |

### ラプラス平滑化

テストデータに特定のクラスの訓練データには一度も現れなかった単語が現れた場合はどうなるか？

平滑化なし：`P(word | class) = 0/N = 0`。積全体に掛けられた1つのゼロが `P(class | features) = 0` にし、他のすべての証拠に関わらず。単一の未見の単語が他のすべての証拠がそれをサポートするにもかかわらず予測全体を破壊する。

ラプラス平滑化はすべての特徴量カウントに小さなカウント `alpha`（通常は1）を追加する：

```
P(word_i | class) = (count(word_i, class) + alpha) / (total_words_in_class + alpha * vocab_size)
```

alpha=1で、すべての単語が少なくとも小さな確率を得る。テストメールに「discombobulate」という単語が現れてもスパム確率を殺さない。平滑化はベイズ的解釈を持つ：単語分布に一様ディリクレ事前分布を置くことと等価だ。

alphaが高いほど平滑化が強い（より一様な分布）。alphaが低いほどモデルがデータをより信頼する。alphaは調整するハイパーパラメータだ。

alphaの効果：

| Alpha | 効果 | 使用する場合 |
|-------|------|------------|
| 0.001 | ほぼ平滑化なし、データを信頼する | 非常に大きい訓練セット、未見の特徴量が予期されない |
| 0.1 | 軽い平滑化 | 大きい訓練セット |
| 1.0 | 標準的なラプラス平滑化 | デフォルトの出発点 |
| 10.0 | 重い平滑化、分布を平坦化する | 非常に小さい訓練セット、多くの未見の特徴量が予期される |

### 対数空間での計算

多数の確率（各1未満）を掛け合わせると、浮動小数点のアンダーフローが発生する。真の値が非常に小さな正の数であっても、浮動小数点ではゼロになる。

解決策：対数空間で作業する。確率を掛け合わせる代わりに、その対数を足す：

```
log P(class | x1, x2, ..., xn) = log P(class) + sum_i log P(xi | class)
```

これにより予測がドット積になる：

```
log_scores = X @ log_feature_probs.T + log_class_priors
prediction = argmax(log_scores)
```

行列乗算だ。それがナイーブベイズ予測が非常に速い理由だ -- 単層線形モデルと同じ操作だ。

### ナイーブベイズ対ロジスティック回帰

両方ともテキストの線形分類器だ。違いはモデル化するものにある。

| 側面 | ナイーブベイズ | ロジスティック回帰 |
|------|-------------|-----------------|
| タイプ | 生成的（P(X\|Y)をモデル化） | 識別的（P(Y\|X)をモデル化） |
| 訓練 | 頻度を数える | 損失関数を最適化する |
| 少ないデータ | より良い（強い事前分布が役立つ） | より悪い（重みを推定するのに不十分） |
| 多いデータ | より悪い（間違った仮定が傷つける） | より良い（柔軟な境界） |
| 特徴量 | 独立を仮定する | 相関を処理する |
| 速度 | 単一パス、非常に速い | 反復最適化 |
| 較正 | 確率が不良 | より良い確率 |

経験則：ナイーブベイズから始める。十分なデータがあってNBが頭打ちになったら、ロジスティック回帰に切り替える。

## 構築

`code/naive_bayes.py` のコードはMultinomialNBとGaussianNBの両方をスクラッチで実装する。

### MultinomialNB

スクラッチからの実装：

1. **fit(X, y)**: 各クラスに対して、各特徴量の頻度を数える。ラプラス平滑化を追加する。対数確率を計算する。クラス事前分布（クラス頻度の対数）を保存する。

2. **predict_log_proba(X)**: 各サンプルに対して、すべてのクラスのlog P(クラス) + sum log P(特徴量_i | クラス) を計算する。これは行列乗算だ：X @ log_probs.T + log_priors。

3. **predict(X)**: 最高の対数確率を持つクラスを返す。

```python
class MultinomialNB:
    def __init__(self, alpha=1.0):
        self.alpha = alpha

    def fit(self, X, y):
        classes = np.unique(y)
        n_classes = len(classes)
        n_features = X.shape[1]

        self.classes_ = classes
        self.class_log_prior_ = np.zeros(n_classes)
        self.feature_log_prob_ = np.zeros((n_classes, n_features))

        for i, c in enumerate(classes):
            X_c = X[y == c]
            self.class_log_prior_[i] = np.log(X_c.shape[0] / X.shape[0])
            counts = X_c.sum(axis=0) + self.alpha
            self.feature_log_prob_[i] = np.log(counts / counts.sum())

        return self
```

重要な洞察：適合後、予測は行列乗算にバイアスを加えるだけだ。ナイーブベイズが非常に速い理由だ。

### GaussianNB

連続特徴量の場合、クラスごと特徴量ごとに平均と分散を推定する：

```python
class GaussianNB:
    def __init__(self):
        pass

    def fit(self, X, y):
        classes = np.unique(y)
        self.classes_ = classes
        self.means_ = np.zeros((len(classes), X.shape[1]))
        self.vars_ = np.zeros((len(classes), X.shape[1]))
        self.priors_ = np.zeros(len(classes))

        for i, c in enumerate(classes):
            X_c = X[y == c]
            self.means_[i] = X_c.mean(axis=0)
            self.vars_[i] = X_c.var(axis=0) + 1e-9
            self.priors_[i] = X_c.shape[0] / X.shape[0]

        return self
```

予測は特徴量ごとのガウスPDFを使い、特徴量にわたって（対数空間で加算して）掛け合わせる。

### デモ：テキスト分類

コードは2つのクラス（技術記事とスポーツ記事）をシミュレートする合成bag-of-wordsデータを生成する。各クラスは異なる単語頻度分布を持つ。MultinomialNBは単語カウントを使ってそれらを分類する。

合成データはこのように動作する：200個の「単語」（特徴量の列）を作成する。単語0〜39は技術記事で高頻度、スポーツで低頻度だ。単語80〜119はスポーツで高頻度、技術で低頻度だ。単語40〜79は両方で中程度の頻度だ。これにより一部の単語が強いクラス指標で他はノイズである現実的なシナリオが作られる。

### デモ：連続特徴量

コードはアイリスのようなデータ（3クラス、4特徴量、ガウスクラスター）を生成する。GaussianNBはクラスごとの平均と分散を使って分類する。各クラスは異なる中心（平均ベクトル）と異なる広がり（分散）を持ち、カテゴリー間で測定が系統的に異なる現実世界のデータを模倣する。

コードはまた以下も示す：
- **平滑化比較:** 異なるアルファ値でMultinomialNBを訓練し、平滑化強度が精度に与える効果を示す。
- **訓練サイズ実験:** NB精度が訓練データが20から1600サンプルに増えるにつれてどう改善するか。NB は非常に少ないサンプルでも適切な精度に達する -- これが主な利点だ。
- **混同行列:** クラスごとの適合率、再現率、F1スコアでNBがどこで間違いを犯すかを示す。

### 予測速度

ナイーブベイズの予測は行列乗算だ。n個のサンプル、d個の特徴量、kクラスの場合：
- MultinomialNB：1回の行列乗算 (n x d) @ (d x k) = O(n * d * k)
- GaussianNB：n * k個のガウスPDF評価、各々d特徴量にわたって = O(n * d * k)

両方ともすべての次元で線形だ。すべての訓練点への距離計算が必要なKNN、またはすべてのサポートベクトルに対するカーネル評価が必要なRBFカーネルのSVMと比較する。NBは予測時に桁違いに速い。

## 活用

sklearnでは、両方の変種が1行で書ける：

```python
from sklearn.naive_bayes import GaussianNB, MultinomialNB

gnb = GaussianNB()
gnb.fit(X_train, y_train)
print(f"GaussianNB精度: {gnb.score(X_test, y_test):.3f}")

mnb = MultinomialNB(alpha=1.0)
mnb.fit(X_train_counts, y_train)
print(f"MultinomialNB精度: {mnb.score(X_test_counts, y_test):.3f}")
```

sklearnでのテキスト分類：

```python
from sklearn.feature_extraction.text import CountVectorizer
from sklearn.naive_bayes import MultinomialNB
from sklearn.pipeline import Pipeline

text_clf = Pipeline([
    ("vectorizer", CountVectorizer()),

text_clf.fit(train_texts, train_labels)
accuracy = text_clf.score(test_texts, test_labels)
```

### ナイーブベイズとTF-IDF

生の単語カウントは出現ごとにすべての単語に同じ重みを与える。しかし「the」や「is」のような一般的な単語はすべてのクラスで頻繁に現れる -- 情報を持たない。TF-IDF（Term Frequency - Inverse Document Frequency）は一般的な単語を低く評価し、稀で識別的な単語を高く評価する。

```python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.naive_bayes import MultinomialNB
from sklearn.pipeline import Pipeline

text_clf = Pipeline([
    ("tfidf", TfidfVectorizer()),
    ("classifier", MultinomialNB(alpha=0.1)),
])
```

TF-IDF値は非負なので、MultinomialNBで機能する。TF-IDF + MultinomialNBの組み合わせは、テキスト分類の最も強力なベースラインの1つだ。10,000未満の訓練サンプルを持つデータセットでは頻繁により複雑なモデルを上回る。

### 短いテキストのBernoulliNB

短いテキスト（ツイート、SMS、チャットメッセージ）の場合、BernoulliNBはMultinomialNBを上回ることができる。短いテキストは単語カウントが少ないので、MultinomialNBが依存する頻度情報がノイジーだ。BernoulliNBは存在または不在のみを気にするので、短いテキストでより信頼性が高い。

```python
from sklearn.naive_bayes import BernoulliNB
from sklearn.feature_extraction.text import CountVectorizer

text_clf = Pipeline([
    ("vectorizer", CountVectorizer(binary=True)),
    ("classifier", BernoulliNB(alpha=1.0)),
])
```

CountVectorizerの `binary=True` フラグはすべてのカウントを0/1に変換する。なしでもBernoulliNBは機能するが、設計されていないカウントを見ることになる。

### NB確率の較正

NB確率は較正が不十分だ。NBがP(スパム)=0.95と言うとき、真の確率は0.7かもしれない。確率の推定が信頼できる必要がある場合（例：閾値を設定したり、他のモデルと組み合わせたりする場合）、sklearnのCalibratedClassifierCVを使う：

```python
from sklearn.calibration import CalibratedClassifierCV

calibrated_nb = CalibratedClassifierCV(MultinomialNB(), cv=5, method="sigmoid")
calibrated_nb.fit(X_train, y_train)
proba = calibrated_nb.predict_proba(X_test)
```

これは交差検証を使ってNBの生スコアの上にロジスティック回帰を適合させる。結果の確率は真のクラス頻度にはるかに近い。

### よくある落とし穴

1. **負の特徴量値。** MultinomialNBは非負の特徴量を必要とする。負の値がある場合（特定の設定のTF-IDFや標準化された特徴量など）、代わりにGaussianNBを使うか、特徴量を正にシフトする。

2. **ゼロ分散の特徴量。** GaussianNBは分散で割る。特徴量があるクラスでゼロ分散を持つ場合（すべての値が同一）、確率計算が壊れる。コードはこれを防ぐためにすべての分散に小さな平滑化項（1e-9）を追加する。

3. **クラス不均衡。** メールの99%が非スパムの場合、事前分布P(非スパム)=0.99が非常に強く、尤度証拠を圧倒する。クラス事前分布を手動で設定するか、sklearnのclass_priorパラメータを使うことができる。

4. **特徴量スケーリング。** MultinomialNBはスケーリングを必要としない（カウントで動作する）。GaussianNBもスケーリングを必要としない（特徴量ごとの統計を推定する）。これはスケールに敏感なロジスティック回帰やSVMに対する利点だ。

### ナイーブベイズが失敗するとき

NBは独立仮定が誤ったランキングを引き起こすとき（単に誤った確率ではなく）に失敗する。これは以下の場合に発生する：

1. **強い特徴量の相互作用。** クラスが2つの特徴量の組み合わせに依存するが、どちらか単独ではない場合（XORのようなパターン）、NBはそれを完全に見逃す。各特徴量単独では証拠を提供せず、NBはそれらを非線形に組み合わせることができない。

2. **相反する証拠を持つ高度に相関した特徴量。** 特徴量Aが「スパム」を示し、特徴量Bが「非スパム」を示すが、AとBが完全に相関している（現実では常に一致する）場合、NBは実際にはないところに矛盾する証拠を見る。

3. **非常に大きい訓練セット。** 十分なデータがあれば、ロジスティック回帰などの識別モデルは真の決定境界を学習し、NBを上回る。少ないデータで役立った独立仮定が今はモデルを抑制する。

実際には、これらの失敗モードはテキスト分類では稀だ。テキストの特徴量は多数で個々に弱く、独立仮定の誤差は打ち消し合う傾向がある。強く相関した特徴量が少ない表形式データでは、最初にロジスティック回帰または木ベースのモデルを検討する。

## 演習

## 用語集

| 用語 | よく言われること | 実際の意味 |
|------|----------------|----------------------|
| ナイーブベイズ | 「シンプルな確率分類器」 | 特徴量がクラスが与えられた条件付き独立であるという仮定でベイズの定理を適用する分類器 |
| 条件付き独立 | 「特徴量は互いに影響しない」 | P(A, B \| C) = P(A \| C) * P(B \| C) -- Cを知っていれば、BはAについて新しい情報を教えない |
| ラプラス平滑化 | 「加算1平滑化」 | ゼロ確率が予測を支配するのを防ぐために、すべての特徴量に小さなカウントを追加する |
| 事前分布 | 「データを見る前に信じていたこと」 | P(クラス) -- 特徴量を観測する前の各クラスの確率 |
| 尤度 | 「データがどれだけ当てはまるか」 | P(特徴量 \| クラス) -- クラスが既知のとき、これらの特徴量を観測する確率 |
| 事後分布 | 「データを見た後に信じること」 | P(クラス \| 特徴量) -- 特徴量を観測した後のクラスの更新された確率 |
| 生成モデル | 「データがどのように生成されるかをモデル化する」 | P(X \| Y) と P(Y) を学習し、ベイズの定理を使ってP(Y \| X) を得るモデル |
| 識別モデル | 「決定境界をモデル化する」 | Xがどのように生成されるかをモデル化せずに、直接P(Y \| X) を学習するモデル |
| 対数確率 | 「アンダーフローを避ける」 | 多数の小さな数の積が浮動小数点でゼロになるのを防ぐために、Pの代わりにlog Pで作業する |
