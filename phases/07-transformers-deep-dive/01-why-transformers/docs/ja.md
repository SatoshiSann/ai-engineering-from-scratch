# なぜトランスフォーマーか — RNNの問題

> RNNは1つずつトークンを処理する。トランスフォーマーはすべてのトークンを一度に処理する。2017年以降の深層学習のあらゆるスケーリング曲線を変更した単一のアーキテクチャの賭け。

**タイプ:** 学習
**言語:** Python
**前提条件:** Phase 3 （深層学習コア）, Phase 5 · 09 （Sequence-to-Sequence）, Phase 5 · 10 （アテンション機構）
**所要時間:** 約45分

## 問題

2017年以前、惑星上のあらゆる最先端シーケンスモデル — 言語、翻訳、音声 — は再帰ニューラルネットワークだった。LSTMとGRUは半世紀のImageNet等価翻訳ベンチマークで優勝した。誰もが持っていた唯一のツール。

3つの致命的な弱点を持っていた。シーケンシャル計算は、時間軸に沿った並列化ができないことを意味した: トークン `t+1` はトークン `t` から隠れ状態が必要だ。1,024トークンシーケンスは1つのGPUで1,024シリアルステップを意味し、サイクル当たり1,000,000浮動小数点演算ができる。訓練壁時計時間は、並列化向けに設計されたハードウェア上でシーケンス長で直線的にスケール。

消失勾配は、50トークン前の情報がすでに50の非線形を通じて圧縮されていることを意味した。ゲート付きリカレントユニット（LSTM, GRU）は圧力を和らげたが、決して排除しなかった。長範囲依存 — 「京都への飛行機上の去年の夏に読んだ本は…」 — は日常的に失敗した。

固定幅隠れ状態は、エンコーダが全ソースシーケンスを単一ベクトルにしぼり、デコーダが何も見る前にボトルネックが同じ形状であった。ソースが5トークンまたは500かは問題ではない。

2017年論文「Attention Is All You Need」は急進的なことを提案した: 再帰性を完全に破棄。すべての位置が並列で他のあらゆる位置に注目できるようにする。1,024のシリアルステップではなく1つの大きな行列乗算で訓練。

結果は2026年までにすべてのモダリティを支配する。言語（GPT-5、Claude 4、Llama 4）、ビジョン（ViT、DINOv2、SAM 3）、オーディオ（Whisper）、生物学（AlphaFold 3）、ロボティクス（RT-2）。同じブロック、異なる入力。

## コンセプト

![RNNシーケンシャル計算 vs トランスフォーマー並列アテンション](../assets/rnn-vs-transformer.svg)

**ボトルネックとしての再帰性。** RNNは `h_t = f(h_{t-1}, x_t)` を計算。各ステップが前の段階に依存。`h_5` を `h_4` の前に計算できない。10,000+並列コアを持つ最新GPUでは、長いシーケンスでシリコンの99%を無駄にする。

**ブロードキャストとしてのアテンション。** 自己注意は `output_i = sum_j(a_ij * v_j)` をすべてのペア `(i, j)` 同時に計算。全N×Nアテンションマトリックスはバッチ処理matmulで一度に充てん。どのステップも別に依存しない。GPUはそれが大好き。

**スピードアップは定数ではない。** `O(N)` シリアル深さと `O(1)` シリアル深さの違い。実際には、トランスフォーマーはマッチしたハードウェアで1エポックあたり5–10倍高速訓練N=512で、ギャップはシーケンス長で拡大し、 `O(N²)` アテンションメモリ壁に当たるまで（Flash AttentionがレッスンI2で後で修正）。

**トランスフォーマーが何を代わりに。** アテンション メモリは `O(N²)` でスケール。2K コンテキスト、それでいい。128K コンテキストで、スライディングウィンドウ、RoPE外推、Flash Attention タイル、線形アテンション変種が必要。再帰性は時間とメモリの両方で `O(N)` だった; トランスフォーマーは時間をメモリと交換してから並列処理を通じて時間を取り戻す。

**帰納的バイアスシフト。** RNNは局所性と最近性を仮定。トランスフォーマーは何も仮定しない — あらゆるペアが アテンション候補。これがトランスフォーマーが十分なデータで訓練する必要があるが、それがあるとさらにスケール理由だ。Chinchilla（2022）は公式化: 十分なトークンを考えると、トランスフォーマーは常に等パラメータ数のRNNを打つ。

## ビルド

ニューラルネットワークなし — 最初のボトルネック数値をシミュレートし、ラップトップで隙間を感じるようにする。

### ステップ1: シリアル深さを測定

`code/main.py`を見よ。2つの関数を構築する。1つはシーケンスを添付チェーンとして符号化（RNNのようにシリアル）。1つはそれを並列削減として符号化（アテンションのようなブロードキャスト）。同じ数学、異なる依存グラフ。

```python
def rnn_style(xs):
    h = 0.0
    for x in xs:
        h = 0.9 * h + x   # 並列化不可: h前の h に依存
    return h

def attention_style(xs):
    return sum(xs) / len(xs)  # あらゆる x は独立
```

シーケンス100,000要素まで両方で時間を計測。RNNバージョンは O(N) で単一CPU パイプライン。純粋Python でも、attention-style削減は長さ >= 1,000で それを打つなぜなら Python の `sum()` はCで実装され、ステップあたりインタープリタオーバーヘッドなく反復。

### ステップ2: 理論的操作をカウント

両アルゴリズムはN adds をする。違いは *依存性深さ*: 次が始まる前に順序に実行される必要がある操作の数。RNN深さ = N。アテンション深さ = log(N) ツリー削減で、または並列スキャンで1。GPU時間を決めるのは深さ、operationカウント。

### ステップ3: 長いシーケンス上の実験的スケーリング

タイミング表を印刷し、O(N)ギャップを見え。2026年Macラップトップで、1,000要素以下シーケンスは測定する速すぎるらしい。100,000の長さはクリーンな線形スキャンを示す。16,384トークントランスフォーマーを持つ12レイヤーLSTM等価にスケール、訓練壁時計が2016年のブロッカーだったなぜ見る。

## 使用

2026年でもRNNを選ぶときはいつ:

| 状況 | ピック |
|-----------|------|
| ストリーミング推論、1つずつトークン、定常メモリ | RNN または state-space モデル（Mamba, RWKV） |
| 非常に長いシーケンス（>1M トークン）アテンションメモリ爆発する | Linear attention, Mamba 2, Hyena |
| エッジデバイス matmul accelerator なし | Depthwise-separable RNN スティルFLOPs/watt 勝つ |
| 他の何か（訓練、バッチ推論、コンテキスト 128K まで） | トランスフォーマー |

State-space モデル（SSM）Mamba のような本質的には再帰RNNで構造化パラメータ化、両ベスト与える: `O(N)` スキャンメモリ、並列訓練 selective scan でそれら。90%のトランスフォーマー品質を回復する長いコンテキストスケーリング好ましい。2026年ほとんどのフロンティアラボ ハイブリッド SSM+transformer モデル訓練する（例 Jamba, Samba） — 再帰性は死んでいない、それはコンポーネント。

## 演習

1. **簡単。** `code/main.py`から `rnn_style` を取得し、スカラー隠れ状態を長さ-64隠れ状態ベクトルで置換。再測定。隠れ状態次元でシリアルオーバーヘッドはどの程度増える。
2. **中程度。** 並列プレフィックスサム（Hillis-Steele スキャン）を純Python で実装。純シリアルスキャンでは長さ1024で同じ数値出力それが生成します確認。深さをカウント。
3. **難しい。** アテンション-style削減をPyTorch on GPU にポート。シーケンス長64から65,536にスイープしながら時間。プロットし曲線形状を説明。

## 主要用語

| 用語 | 人々は何を言うか | 実際には何を意味するか |
|------|-----------------|-----------------------|
| 再帰性 | 「RNNはシーケンシャル」 | 計算ステップ `t` がステップ `t-1` に依存、時間軸上でシリアル実行を強制。 |
| シリアル深さ | 「グラフがどの程度深いか」 | 依存operationの最長チェーン; 無限ハードウェアでも壁時計を限定。 |
| アテンション | 「トークンを互いに見える」 | 加重和 `sum_j a_ij v_j` 位置iとj間の類似度スコアから来る `a_ij`。 |
| コンテキストウィンドウ | 「モデルがどの程度見る」 | アテンションレイヤーが入力として取ることができる位置の数; 二次メモリコストはここでスケール。 |
| 帰納的バイアス | 「アーキテクチャに焼き込まれた仮定」 | データがどのように見えるかについて先行; CNNは平行移動不変性を仮定、RNNは最近性を仮定。 |
| State-space モデル | 「RNN でそれの後ろの代数」 | 構造化 state-space マトリックスを通じた並列訓練で再帰性パラメータ化。 |
| 二次ボトルネック | 「なぜコンテキストがそれほど多く代わりに」 | アテンション メモリ = `O(N²)` シーケンス長で; Flash Attention 定数隠さない、スケーリング。 |
