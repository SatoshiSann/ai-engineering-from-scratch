# スケーリング則

> 2020年のKaplan論文は言いました：より大きいモデル、より低い損失。2022年のHoffmann論文は言いました：あなたは過小訓練していました。計算は2つのバケット — パラメータとトークンに入り、スプリットは明らかではありません。

**タイプ:** 学習
**言語:** Python
**前提条件:** Phase 7 · 05 (完全なトランスフォーマー), Phase 7 · 07 (GPT)
**所要時間:** 約45分

## 問題

Cの FLOPs の訓練計算があり、最高のモデルが必要な場合、2つのノブに直面します：

1. **何個のパラメータ(N)？** より大きいモデル、より高い容量。
2. **何個の訓練トークン(D)？** より多くのデータ、より良い容量使用。

FLOPはおおよそ`6 × N × D`でスケール。Nを上げD を下げ、またはDを上げNを下げることができます。どちらがより良い？

2022年の前に、答えは「Nを難しく押す」でした。GPT-3(2020)は175Bパラメータで~300Bトークンで訓練されました。パラメータごと約1.7トークンの比率。Kaplanスケーリング則はこれをサポートしました。

Hoffmann et al. (2022)、Chinchillaと呼ばれた小さなモデルファミリーを訓練、何か異なるを見つけた：最適な比率はより近い**パラメータごと20トークン**。GPT-3は10倍過小訓練されました。Chinchilla(70Bパラメータ、1.4Tトークン)はGPT-3(175B、300Bトークン)をすべてのベンチマークで2.5倍少ない推論費用で上回りました。

2026年はChinchillaの世界 — 1つの重要なねじれで。Llama 3 8Bは15兆トークンで訓練されました、パラメータごと1,875トークンの比率。Chinchilla最適の94倍。推論コストはスケールで使用されるモデルの訓練コストより重要です。訓練コスト(より小さい展開フットプリント)を減らす目的で訓練の過剰(Chinchilla過去)は2026年デフォルト。

### Hoffmann則

Chinchilla論文から、損失は以下を従う：

```
L(N, D) = A / N^α + B / D^β + E
```

- `N` = パラメータ(非埋め込み)。
- `D` = 訓練トークン。
- `α ≈ 0.34`、`β ≈ 0.28`(ほぼ対称)。
- `E ≈ 1.69`、削減できない損失天井。
- `A ≈ 406`、`B ≈ 411`。

2つの項はスケールする際に互いに取引。固定計算(C = 6ND)で`N`に関して導関数を取得し解く：

```
N_opt ≈ 0.6 × (C/6)^0.5
D_opt ≈ 0.6 × (C/6)^0.5
D_opt / N_opt ≈ 20
```

計算最適：パラメータごと20トークン。

### なぜとにかく過訓練

Chinchilla最適は訓練FLOP単位訓練損失を最小化。しかし訓練費用を支払う一度；推論費用永遠に。

月に1兆トークンを提供するチャットボット用、推論が総費用を支配。Llamaのアプローチ：小さいスケール、長スケール。8B 15Tトークンで深い推論最適化：

- 消費者GPUに適合。
- レイテンシーはChinchilla最適の70Bの分数。
- 品質はほとんどのタスク十分に近い。

DeepMindの2024論文(「Over-trainingは新しい最適」)公式化この。推論支配ワークロード用、右の比率は提供音量により約100-500トークンパラメータ。

### 出現平滑性

請求：特定の能力(演算、複数ステップ推論、思考チェーン追従)は「出現」いくつかのスケール。

Schaeffer et al. (2023)議論このは測定成果物：出現メトリクス不連続スコアリング(正確一致、閾値精度)を使用し基礎ロジット平滑改善を隠す。継続的メトリクス(交差エントロピー)平滑曲線を表示。

2026年コンセンサス：継続的損失経由の予測は信頼できる。ベンチマークジャンプはしばしばスコアラー成果物。継続的メトリクスに対する予算計画。

### 2026年画像

スケーリング則は仍然機能しますが：

| 係数 | 変更方法 |
|--------|-------------|
| データ品質 | 「良い」トークン(Phi-スタイル)を選別、曲線を>2倍有効計算シフト |
| MoE | 総パラメータはアクティブFLOPから分離；スケーリング則/アクティブFLOP |
| 事後学習 | いくつかの能力(命令追従、コード)は事前訓練より SFT+RLHF とシフト |
| マルチモダル | 画像+テキストトークンは一緒にスケール；モダリティあたり個別曲線 |
| 合成データ | モデルは訓練データを生成；有効計算は化合物 |

Muon オプティマイザ(Kimi Moonlight、2024)は一致データでAdamWよりも~2倍有効計算利得を示しました。いくつかの2026年訓練ラン、デフォルトでMuonを使用。スケーリング則の形ではなく、絶対定数を変更。

## ビルド

`code/main.py`を見てください。Chinchilla損失方程式を実装し、各々複数計算予算計算最適(N、D)を解く。

### ステップ1：Chinchilla損失

```python
def chinchilla_loss(N, D, A=406.4, B=410.7, alpha=0.34, beta=0.28, E=1.69):
    return A / N ** alpha + B / D ** beta + E
```

固定Cで(N、D)上の輪郭`L`をプロット。最小を見つける。

### ステップ2：計算最適フロンティア

計算予算1e17から1e25 FLOPs、`6ND = C`従う損失を最小化する(N、D)を見つける。比率`D/N ≈ 20`を検証。

### ステップ3：過訓練費用

10倍小さいモデル訓練の追加損失を計算(最適Nの1/10、最適Dの10倍)。推論FLOP節約(Nに比例)をレポート交換。

### ステップ4：実モデルと比較

GPT-3、Chinchilla、Llama 3 8B、DeepSeek-V3(アクティブパラメータ)、既知の(N、D)ペア削除し、予測対レポート損失を比較。

## 使用

フロンティアモデルを訓練するのはほぼありません。しかしスケーリング則は言う：

1. **あなたのファインチューンは十分なデータを持つかどうか。** あなたのタスク固有データがベースモデルのパラメータごと20トークン未満なら、いくつかの損失床に飽和を期待。
2. **より大きいベースモデルをピックするかどうか。** あなたの予算のすべてを推論に費やしている場合、より小さい、より長訓練モデルを優先。
3. **返品が減少する場所。** 1000倍Chinchilla最適を超えて、ログ損失変更はノイズ。

**2026年研究軌跡：**

- **データ制約レジーム。** ウェブは有限高品質トークン数(フィルタリング後の~5-10兆英語)。フロンティア事前訓練はこの天井に接近。合成データ、多言語、マルチモダル、RLHF規模ファインチューニングは次のレバー。
- **計算乗算トリック。** Muon オプティマイザ、MoE、より良いデータ選別 — それぞれ絶対定数をシフト、漸近ではなく。
- **RLのスケーリング則。** オープン質問。初期証拠はRLサンプルの電力則を示唆しますが事前訓練ら非常に異なる指数で。

## キーワード

| 用語 | 人々が言うこと | 実際の意味 |
|------|-----------------|-----------------------|
| パラメータ(N) | 「モデルサイズ」 | 非埋め込み重みカウント；容量を決定。 |
| トークン(D) | 「訓練データ」 | 見た訓練トークン数；パラメータがどのように利用されるかを決定。 |
| 計算(C) | 「FLOPs費やし」 | 約6 × N × D標準トランスフォーマー。 |
| Chinchilla最適 | 「D/N ≈ 20」 | 事前訓練のFLOP単位損失を最小化する比率。 |
| 過訓練 | 「Chinchilla過去」 | 余分訓練FLOPsを費やし推論FLOPsを保存；D/N >> 20。 |
| 削減不可能な損失 | 「床」 | スケーリング則のEタームは、データ自体のエントロピー。 |
| 出現能力 | 「スケールで急激にジャンプ」 | しばしばスコアラー成果物；継続的な損失は滑らか。 |
| 有効計算 | 「訓練効率乗算」 | より良いデータ / オプティマイザ / アーキテクチャはFLOPがどのくらい行くかを乗算。 |
