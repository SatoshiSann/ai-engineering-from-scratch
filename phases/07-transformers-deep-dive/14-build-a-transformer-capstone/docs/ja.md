## 問題

あなたはすべての論文を読みました。あなたはアテンション、マルチヘッドスプリット、位置エンコーディング、エンコーダとデコーダブロック、BERTとGPT損失、MoE、KVキャッシュを実装しました。今実トークンでそれらをラップしゲーション。

カプストーン：小さいデコーダのみトランスフォーマーを文字レベル言語モデリング。それはShakespeareを読みます。それは新しいShakespeareを生成します。それはラップトップで10分未満で訓練するのに十分小さい。それは本当のLMを得るために、より大きいデータセットと訓練に長くスワップするのに十分正しい。

これはコースの「nanoGPT」。これは元ではない — Karpathy の2023 nanoGPT チュートリアルはすべての学生が少なくとも1回書く参照実装。私たちはシェイプを持ち上げ、私たちがカバーしたものの周りに再道具。

### 何を出荷するか

- `GPTConfig` — すべてのハイパーパラメータを設定する1つの場所。
- `MultiHeadAttention` — 因果、バッチ、オプションFlash様経路(PyTorch `scaled_dot_product_attention`)付き。
- `SwiGLUFFN` — 現代的なFFN。
- `Block` — 事前正規化、残差ラップ注意 + FFN。
- `GPT` — 埋め込み、積み重ねブロック、LMヘッド、generate()。
- AdamW、コサインLR、勾配クリッピングを持つ訓練ループ。
- Shakespeareテキスト上のキャラクターレベルトークナイザ。

### 何を出荷しないか

- RoPE — レッスン04で概念的に実装。ここで私たちは簡単のために学習可能な位置埋め込みを使用。演習はあなたにRoPEをスワップするよう求める。
- 生成中のKVキャッシュ — 各生成ステップ完全なプレフィックスの注意を再計算。遅いですがシンプル。演習はあなたにKVキャッシュを追加するよう求める。
- Flash Attention — PyTorch 2.0+は自動的に入力がマッチする場合にディスパッチ；私たちは`F.scaled_dot_product_attention`を使用。
- MoE — ブロックあたり単一FFN。あなたはレッスン11でMoEを見ました。

### ターゲットメトリック

Mac M2ラップトップ、`tinyshakespeare.txt` 上 2,000 ステップで訓練された 4層、4ヘッド、d_model=128 GPT：

- 訓練損失は~4.2(ランダム)から~1.5に収束、約6分。
- サンプル出力はShakespeare形：古風な単語、行ブレーク、「ROMEO:」のような適切な名前が出現。
- Val損失(モデルが見たことのないホールドアウト最終10%)は訓練損失をずっと追跡；この規模/予算オーバーフィッティングなし。

## ビルド

このレッスンはPyTorchを使用。`torch`をインストール(CPUビルドは良い)。`code/main.py`を見てください。スクリプトは処理：

- `tinyshakespeare.txt` をダウンロード欠落している場合(またはローカルコピーを読む)。
- バイトレベルキャラクタートークナイザ。
- 90/10トレインバル分割。
- サポートされたハードウェアでのbf16自動キャストの訓練ループ。
- 訓練完了後のサンプリング。

### ステップ1：データ

```python
text = open("tinyshakespeare.txt").read()
chars = sorted(set(text))
stoi = {c: i for i, c in enumerate(chars)}
itos = {i: c for c, i in stoi.items()}
encode = lambda s: [stoi[c] for c in s]
decode = lambda xs: "".join(itos[x] for x in xs)
```

65ユニークキャラクタ。小さい語彙。4バイト vocab_size に適合。BPEなし、トークナイザ劇なし。

### ステップ2：モデル

`code/main.py`を見てください。ブロックはレッスン05から教科書 — 事前正規化、RMSNorm、SwiGLU、因果MHA。パラメータ数4/4/128：~800K。

### ステップ3：訓練ループ

長さ256トークンウィンドウのランダムバッチを取得。前向き。シフトバイワン交差エントロピー。逆向き。AdamWステップ。ログ。繰り返す。

```python
for step in range(max_steps):
    x, y = get_batch("train")
    logits = model(x)
    loss = F.cross_entropy(logits.view(-1, vocab_size), y.view(-1))
    loss.backward()
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    opt.step()
    opt.zero_grad()
```

### ステップ4：サンプル

プロンプトが与えられた場合、繰り返し前向き、top-pロジットからサンプル、追加、継続。500トークン後に停止。

### ステップ5：出力を読む

2,000ステップ後：

```
ROMEO:
Away and mild will not thy friend, that thou shalt wit:
The chief that well shame and hath been his friends,
...
```

Shakespeareではなく。しかしShakespeare形。~800Kパラメータとラップトップで6分にはっきりした勝利。

## 使用

このカプストーンは参照アーキテクチャ。実物に出荷する3つの拡張：

1. **トークナイザをスワップ。** BPE を使用(例`tiktoken.get_encoding("cl100k_base")`)。語彙サイズは65から~50,000にジャンプ。モデル容量が補償する必要があります。
2. **より大きなコーパスを訓練。** `OpenWebText`または`fineweb-edu`を使用(HuggingFace)。単一A100で10Bトークン は125M-paramGPTで~24時間。
3. **RoPE + KV キャッシュ + Flash Attention を追加。** 下の演習は各を通し歩く。

これは125M-パラメータGPTで流暢な英語を生成で終了。フロンティアモデルではない。しかし同じコードパス — ただしより大きい — はKarpathy、EleutherAI、Allen Institute は2026年研究チェックポイント訓練に使用。

## 演習

1. **簡単。** `code/main.py`を実行。訓練済みモデルの最終ステップの検証損失は2.0を下回ることを検証。`max_steps`を2,000から5,000に変更 — val損失は改善し続けますか？
2. **中程度。** 学習可能な位置埋め込みをRoPEで置換。`MultiHeadAttention`内のQ とK に回転を適用。訓練し、val損失が少なくとも同じくらい低いことを検証。
3. **中程度。** サンプリングループでKVキャッシュを実装。キャッシュで500トークンを生成し、キャッシュなし。壁時計はラップトップで5-20倍改善。
4. **難しい。** 次プラス1トークン予測モデルに2番目のヘッドを追加(MTP — DeepSeek-V3からのマルチトークン予測)。共同訓練。val損失が助ける場合？
5. **難しい。** ブロックあたり単一FFNをMoE 4専門家で置換。ルータ + top-2ルーティング。val損失がマッチしたアクティブパラメータでどのように変化するかを見る。

| 用語 | 人々が言うこと | 実際の意味 |
|------|-----------------|-----------------------|
| nanoGPT | 「Karpathyチュートリアルリポ」 | 最小デコーダのみトランスフォーマー訓練コード、~300 LOC；正規の参照。 |
