# GPT — 因果言語モデリング

> BERTは両側を見る。GPTは過去だけを見る。三角形マスクは現代的なAIにおいて最も重要な単一行のコードである。

**タイプ:** ビルド
**言語:** Python
**前提条件:** Phase 7 · 02 (自己注意), Phase 7 · 05 (完全なトランスフォーマー), Phase 7 · 06 (BERT)
**所要時間:** 約75分

## 問題

言語モデルは1つの質問に答えます。最初の`t-1`個のトークンが与えられたとき、トークン`t`に対する確率分布は何か？次トークン予測というシグナルで学習すれば、一度に1トークンずつ任意のテキストを生成できるモデルが得られます。

シーケンス全体を並列に学習するには、各位置の予測が前の位置のみに依存する必要があります。そうしないと、モデルが答えを見ることで簡単に不正を行います。

因果マスクはこれを実現します。ソフトマックスの前のアテンション値に加算される`-inf`値の上三角行列です。ソフトマックスの後、これらの位置は0になります。各位置は自分自身と前の位置にのみアテンションできます。全シーケンスに一度だけ適用するので、N個の並列次トークン予測が1回の前向きパスで得られます。

GPT-1 (2018)、GPT-2 (2019)、GPT-3 (2020)、GPT-4 (2023)、GPT-5 (2024)、Claude、Llama、Qwen、Mistral、DeepSeek、Kimi — これらはすべて同じコアループを持つデコーダのみの因果トランスフォーマーです。ただしサイズが大きく、データが優れ、RLHFが優れているだけです。

## コンセプト

![因果マスクが三角形アテンション行列を作成する](../assets/causal-attention.svg)

### マスク

長さ`N`のシーケンスが与えられた場合、`N × N`行列を構築します。

```
M[i, j] = 0       if j <= i
M[i, j] = -inf    if j > i
```

ソフトマックスの前の生のアテンション値に`M`を加算します。`exp(-inf) = 0`なので、マスクされた位置は重みゼロで寄与します。アテンション行列の各行は前の位置のみの確率分布です。

実装コスト：1つの`torch.tril()`呼び出し。計算時間：ナノ秒。分野への影響：すべて。

### 並列学習、順序付き推論

学習：全体の`(N, d_model)`シーケンスを一度前向きパスします。N個の交差エントロピー損失(位置あたり1つ)を計算します。合計します。バックプロパゲーション。シーケンスに沿って並列。このためGPT学習はスケールします — 1つのGPUパスでバッチ内の100万トークンを処理します。

推論：トークンごとに生成します。`[t1, t2, t3]`をフィード、`t4`を取得。`[t1, t2, t3, t4]`をフィード、`t5`を取得。`[t1, t2, t3, t4, t5]`をフィード、`t6`を取得。KVキャッシュ(レッスン12)は`t1…tn`の隠れ状態を保存するので、ステップごとに再計算する必要がありません。しかし推論での順序付き深さ=出力長です。これが自己回帰税であり、すべてのLLMのデコード遅延がボトルネックである理由です。

### 損失 — 1つずつシフト

トークン`[t1, t2, t3, t4]`が与えられた場合：

- 入力：`[t1, t2, t3]`
- ターゲット：`[t2, t3, t4]`

すべての位置`i`について、`-log P(target_i | inputs[:i+1])`を計算します。合計します。これはシーケンス全体の交差エントロピーです。

あなたが聞いたことのあるすべてのトランスフォーマーLMはこの損失で学習します。事前学習、ファインチューニング、SFT — 同じ損失、異なるデータ。

### デコード戦略

学習後、サンプリングの選択は人々が考えるより重要です。

| 方法 | 何をするか | いつ使用するか |
|--------|--------------|-------------|
| 貪欲 | 各ステップでアルグマックス | 決定的なタスク、コード補完 |
| 温度 | ロジットをTで割る、サンプル | 創造的なタスク、より高いT=より多くの多様性 |
| Top-k | 上位kトークンのみからサンプル | 低確率の裾を削除 |
| Top-p (nucleus) | 累積確率≥pの最小セットからサンプル | 2020年以降のデフォルト；分布形状に適応 |
| Min-p | `p > min_p * max_p`のトークンを保持 | 2024年以降；top-pよりも長い裾を拒否するのに優れている |
| 推論的デコーディング | ドラフトモデルが N トークンを提案、大きいモデルが検証 | 同じ品質で2-3倍の遅延削減 |

2026年では、min-p +温度0.7はオープンウェイトモデルの合理的なデフォルトです。推論的デコーディングは本番推論スタックのすべてにとって必須です。

### 「GPTレシピ」が機能した理由

1. **デコーダのみ。** エンコーダのオーバーヘッドなし。レイヤーあたりアテンション + FFNの1回のパス。
2. **スケーリング。** 124M → 1.5B → 175B → 兆。Chinchillaスケーリング則(レッスン13)はコンピュートの費やし方を教えます。
3. **コンテキスト内学習。** 約6B～13Bの周辺で出現。モデルはファインチューニングなしで少数例に従うことができます。
4. **RLHF。** 人間の選好に対する事後学習により、生の事前学習済みテキストがチャットアシスタントに変換されました。
5. **事前正規化 + RoPE + SwiGLU。** スケールでの安定した学習。

コアアーキテクチャはGPT-2以来あまり変わっていません。興味深いことはすべてデータ、スケール、事後学習で起こりました。

## ビルド

### ステップ1：因果マスク

`code/main.py`を見てください。1行：

```python
def causal_mask(n):
    return [[0.0 if j <= i else float("-inf") for j in range(n)] for i in range(n)]
```

ソフトマックスの前のアテンション値に加算します。これが全体のメカニズムです。

### ステップ2：2層のGPTish モデル

2つのデコーダブロックをスタック(マスク済み自己注意 + FFN、交差注意なし)。トークン埋め込み、位置エンコーディング、非埋め込み(トークン埋め込み行列に結合 — GPT-2以来の標準トリック)を追加します。

### ステップ3：次トークン予測、エンドツーエンド

20トークンのおもちゃ語彙では、すべての位置でロジットを生成します。シフトバイワンターゲットに対する交差エントロピー損失を計算します。勾配なし — これは前向きパスの健全性チェックです。

### ステップ4：サンプリング

貪欲、温度、top-k、top-p、min-pを実装します。固定プロンプトで各実行し、出力を比較します。サンプリング関数は10行です。

## 使用

PyTorch、2026年の使用法：

```python
from transformers import AutoModelForCausalLM, AutoTokenizer
model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-3.2-3B-Instruct")
tok = AutoTokenizer.from_pretrained("meta-llama/Llama-3.2-3B-Instruct")

prompt = "Attention is all you need because"
inputs = tok(prompt, return_tensors="pt")
out = model.generate(
    **inputs,
    max_new_tokens=64,
    temperature=0.7,
    top_p=0.9,
    do_sample=True,
)
print(tok.decode(out[0]))
```

カバーの下では、`generate()`は前向きパスを実行し、最終位置のロジットを取得し、次のトークンをサンプリングし、追加し、繰り返します。すべての本番LLM推論スタック(vLLM、TensorRT-LLM、llama.cpp、Ollama、MLX)は同じループを重い最適化で実装します — バッチ化プリフィル、継続的バッチ処理、KVキャッシュペーシング、推論的デコーディング。

**GPT vs BERT、各1行：** GPTは`P(x_t | x_{<t})`を予測します。BERTは`P(x_masked | x_unmasked)`を予測します。損失はモデルが生成できるかどうかを決定します。

1. **簡単。** `code/main.py`を実行し、因果アテンション行列がソフトマックス後に下三角であることを確認します。スポットチェック：行3は列0-3のみに重みを持つべきです。
2. **中程度。** 幅4のビーム検索を実装します。10個の短いプロンプトでビーム4対貪欲のパープレキシティを比較します。ビームは常に勝ちますか？(ヒント：通常、翻訳の場合は勝ちますが、オープンエンドのチャットではありません。)

## キーワード

| 用語 | 人々が言うこと | 実際の意味 |
|------|-----------------|-----------------------|
| 因果マスク | 「三角形」 | 位置`i`が位置`≤ i`のみを見るようにアテンション値に追加される上三角`-inf`行列。 |
| 次トークン予測 | 「損失」 | すべての位置でモデルの分布に対する真の次トークンの交差エントロピー。 |
| 自己回帰 | 「一度に1つずつ生成」 | 出力をフィードバックとして入力に供給；並列性は学習中のみ、生成中ではない。 |
| ロジット | 「事前ソフトマックススコア」 | ソフトマックス前のLM頭の生出力；サンプリングはこれで起こります。 |
| 温度 | 「創造性ノブ」 | ロジットをTで割る；T→0=貪欲、T→∞=均一。 |
| Top-p | 「nucleus sampling」 | 分布を≥pの合計の最小セットに切り詰める；残りからサンプル。 |
| Min-p | 「top-pより良い」 | `p ≥ min_p × max_p`のトークンを保持；分布の鋭さに応じて切り詰めを適応させる。 |
| 推論的デコーディング | 「ドラフト + 検証」 | 安価なモデルがNトークンを提案；大きいモデルが並列で検証。 |
| 教師強制 | 「学習トリック」 | 学習中、モデルの予測ではなく真の前のトークンをフィード。すべてのseq2seq LMで標準。 |
