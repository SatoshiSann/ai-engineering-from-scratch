# T5、BART — エンコーダ・デコーダモデル

> エンコーダが理解します。デコーダが生成します。それらを一緒に戻すと、入力 → 出力タスク用に構築されたモデルが得られます：翻訳、要約、書き直し、トランスクリプション。

**タイプ:** 学習
**言語:** Python
**前提条件:** Phase 7 · 05 (完全なトランスフォーマー), Phase 7 · 06 (BERT), Phase 7 · 07 (GPT)
**所要時間:** 約45分

## 問題

デコーダのみのGPTとエンコーダのみのBERTはそれぞれ2017年のアーキテクチャを異なる目標のために削減します。しかし多くのタスクは本質的に入力-出力です：

- 翻訳：英語 → フランス語。
- 要約：5,000トークン記事 → 200トークン要約。
- 音声認識：オーディオトークン → テキストトークン。
- 構造化抽出：散文 → JSON。

これらでは、エンコーダ・デコーダが最もクリーンな適合を提供します。エンコーダがソースの密な表現を生成します。デコーダが出力を生成し、すべてのステップでその表現にクロスアテンションします。学習は出力側のシフトバイワンです。GPTと同じ損失ですが、エンコーダ出力に条件付けられています。

2つの論文が現代的なプレイブックを定義しました：

1. **T5** (Raffel et al. 2019)。「Text-to-Text Transfer Transformer。」すべてのNLPタスクがテキスト入力、テキスト出力として再構築されました。単一アーキテクチャ、単一語彙、単一損失。マスク済みスパン予測で事前学習(入力のスパンを破損、出力でデコード)。
2. **BART** (Lewis et al. 2019)。「Bidirectional and Auto-Regressive Transformer。」デノイジングオートエンコーダ：入力を複数の方法で破損(シャッフル、マスク、削除、回転)し、デコーダに元のを再構成するよう求める。

2026年ではエンコーダ・デコーダ形式は入力構造が重要な場所で生きています：

- Whisper(音声 → テキスト)。
- Googleの翻訳スタック。
- 明確なコンテキストと編集構造を持つコード補完/修復モデル。
- 構造化推論タスク用のFlan-T5とバリアント。

デコーダのみがスポットライトで勝ちました。しかしエンコーダ・デコーダは消えませんでした。

### 前向きループ

```
ソーストークン ─▶ エンコーダ ─▶ (N_src, d_model)  ──┐
                                                 │
ターゲットトークン ─▶ デコーダブロック              │
                 ├─▶ マスク済み自己注意           │
                 ├─▶ 交差注意 ◀───────────┘
                 └─▶ FFN
                ↓
              次トークンロジット
```

重要なことに、エンコーダは入力ごとに1回実行されます。デコーダは自己回帰的に実行されますが、すべてのステップで同じエンコーダ出力にクロスアテンションします。エンコーダ出力をキャッシュすることは、長い入力の無料の高速化です。

### T5事前学習 — スパン破損

入力のランダムスパンを選択(平均長3トークン、合計15%)。各スパンを一意なセンチネルで置換：`<extra_id_0>`、`<extra_id_1>`など。デコーダがセンチネルプレフィックスを持つ破損したスパンのみを出力：

```
ソース: The quick <extra_id_0> fox jumps <extra_id_1> dog
ターゲット: <extra_id_0> brown <extra_id_1> over the lazy
```

全シーケンス予測よりも安価なシグナル。T5論文のアブレーション研究ではMLM (BERT)およびプリフィックスLM (UniLM)と競争的です。

### BART事前学習 — マルチノイズデノイジング

BARTは5つのノイジング関数を試します：

1. トークンマスキング。
2. トークン削除。
3. テキスト充填(スパンをマスク、デコーダが正しい長さを挿入)。
4. 文の順列。
5. ドキュメント回転。

テキスト充填 + 文の順列の組み合わせが最良のダウンストリーム結果を生成しました。デコーダは常に元のを再構成します。BARTの出力は破損したスパンのみではなく、完全なシーケンス — したがって事前学習計算はT5より高いです。

### 推論

GPTと同じ自己回帰生成。貪欲/ビーム/top-p サンプリングが適用されます。ビーム検索(幅4-5)は出力分布がチャットより狭いため、翻訳と要約の標準です。

### 2026年で各バリアントをいつ選択するか

| タスク | エンコーダ・デコーダ？ | なぜ |
|------|------------------|-----|
| 翻訳 | はい、通常 | クリアなソースシーケンス；固定出力分布；ビーム検索が機能 |
| 音声テキスト | はい(Whisper) | 入力モダリティは出力と異なる；エンコーダがオーディオ特徴を形成 |
| チャット/推論 | いいえ、デコーダのみ | 永続的な「入力」なし — 会話がシーケンス |
| コード補完 | 通常いいえ | デコーダのみが長いコンテキストで勝つ；Qwen 2.5 Coderのようなコードモデルはデコーダのみ |
| 要約 | どちらでも機能 | BART、PEGASUSが以前のデコーダのみベースラインを上回る；現代的なデコーダのみLLMが同等 |
| 構造化抽出 | どちら | T5は「テキスト → テキスト」が任意の出力形式を吸収するのでクリーン |

~2022以降の傾向：デコーダのみがエンコーダ・デコーダが所有していたタスクを引き継ぐ。(a)命令調整されたデコーダのみLLMがプロンピングで何でも一般化、(b)1つのアーキテクチャがスケールしやすい2つより、(c)RLHFはデコーダを想定。エンコーダ・デコーダは入力モダリティが異なる(音声、画像)か、ビーム検索品質が重要な場所で持ちこたえます。

## ビルド

`code/main.py`を見てください。おもちゃコーパス用のT5スタイルのスパン破損を実装します — 最も有用な単一のレッスン片です。この以来、すべてのエンコーダ・デコーダ事前学習レシピで表示されるためです。

### ステップ1：スパン破損

```python
def corrupt_spans(tokens, mask_rate=0.15, mean_span=3.0, rng=None):
    """~mask_rateのトークンに合計するスパンを選択。(corrupted_input, target)を返す。"""
    n = len(tokens)
    n_mask = max(1, int(n * mask_rate))
    n_spans = max(1, int(round(n_mask / mean_span)))
    ...
```

ターゲット形式はT5規約：`<sent0> span0 <sent1> span1 ...`。破損した入力はスパン位置でセンチネルトークンとともに変わらないトークンをインターリーブします。

### ステップ2：ラウンドトリップの検証

破損した入力とターゲットが与えられた場合、元のセンテンスを再構成します。あなたの破損が可逆的であれば、前向きパスは適切に定義されています。これは健全性チェック — 実際の学習はこれを決して行いませんが、テストは安価で、スパン記録のオフバイワンバグをキャッチします。

### ステップ3：BARTノイジング

5つの関数：`token_mask`、`token_delete`、`text_infill`、`sentence_permute`、`document_rotate`。2つを組成し、結果を表示します。

## 使用

HuggingFace参照：

```python
from transformers import T5ForConditionalGeneration, T5Tokenizer
tok = T5Tokenizer.from_pretrained("google/flan-t5-base")
model = T5ForConditionalGeneration.from_pretrained("google/flan-t5-base")

inputs = tok("translate English to French: Attention is all you need.", return_tensors="pt")
out = model.generate(**inputs, max_new_tokens=32)
print(tok.decode(out[0], skip_special_tokens=True))
```

T5トリック：タスク名が入力テキストに含まれます。同じモデルが数十のタスクを処理します。各タスクはテキスト入力、テキスト出力だからです。2026年ではこのパターンが命令調整されたデコーダのみモデルで一般化されていますが、T5が最初にそれを成文化しました。

## 演習

1. **簡単。** `code/main.py`を実行、30トークンセンテンスにスパン破損を適用、非センチネルソーストークンと解読ターゲットスパンの連結が元のを再現することを検証します。
2. **中程度。** BARTの`text_infill`ノイズを実装：ランダムスパンを単一の`<mask>`トークンで置換、デコーダが正しいスパン長と内容を推測する必要があります。1つの例を表示します。
3. **難しい。** `flan-t5-small`を小さい英語 → pig-Latin コーパス(200ペア)でファインチューン。ホールドアウト50ペアセットでBLEUを測定。同じデータと同じコンピュートで`Llama-3.2-1B`をファインチューンするのと比較します。

## キーワード

| 用語 | 人々が言うこと | 実際の意味 |
|------|-----------------|-----------------------|
| エンコーダ・デコーダ | 「Seq2seqトランスフォーマー」 | 2つのスタック：入力用の双方向エンコーダ、出力用の交差注意を持つ因果デコーダ。 |
| 交差注意 | 「ソースがターゲットと対話する場所」 | デコーダのQ × エンコーダのK/V。エンコーダ情報がデコーダに入る唯一の場所。 |
| スパン破損 | 「T5の事前学習トリック」 | ランダムスパンをセンチネルトークンで置換；デコーダがスパンを出力。 |
| デノイジング目的 | 「BARTのゲーム」 | ノイジング関数を入力に適用、デコーダをクリーンなシーケンスを再構成するよう訓練。 |
| センチネルトークン | 「`<extra_id_N>`プレースホルダ」 | 破損したスパンをソースでタグ付け、ターゲットで再タグ付けする特殊トークン。 |
| Flan | 「命令調整されたT5」 | >1,800のタスクで微調整されたT5；エンコーダ・デコーダを命令追従で競争的にしました。 |
| ビーム検索 | 「デコード戦略」 | 各ステップで上位k部分シーケンスを保持；翻訳/要約の標準。 |
| 教師強制 | 「訓練時入力」 | 学習中、サンプリングされたものではなく真の前の出力トークンをデコーダにフィード。 |
