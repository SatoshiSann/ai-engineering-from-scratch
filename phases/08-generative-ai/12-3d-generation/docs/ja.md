# 3D生成

> 3Dは2D-3Dレバレッジが最強なモダリティ。2023年の突破口は3D ガウシアン スプラッティング。2024-2026生成プッシュマルチビュー拡散+3D再構成を層の上に、単一プロンプトまたはフォトからオブジェクトおよびシーンを生成。

**タイプ:** Learn
**言語:** Python
**前提条件:** Phase 4 (ビジョン)、Phase 8 · 07 (潜在拡散)
**所要時間:** 約45分

## 問題

3Dコンテンツは痛い：

- **表現。** メッシュ、点クラウド、ボクセルグリッド、符号付き距離フィールド（SDF）、ニューラルラジアンスフィールド（NeRF）、3Dガウシアン。各トレードオフがある。
- **データ希少性。** ImageNetは14Mイメージを持つ。最大クリーン3Dデータセット（Objaverse-XL、2023）は約10Mオブジェクトを持つ、ほとんど低品質。
- **メモリ。** 512³ボクセルグリッドは128Mボクセル；有用なシーンNeRFは1M サンプル/ray。生成は再構成より難しい。
- **監督。** 2D画像に対して、ピクセルを持つ。3Dに対して、通常少数の2Dビューを持つ、3Dまで持ち上げる必要。

2026スタックは2つの問題を分ける。まず、拡散モデルで*2Dマルチビュー画像*を生成。次に、*3D表現*（通常ガウシアン スプラッティング）をこれらの画像にフィット。

### 表現：3D ガウシアン スプラッティング (Kerbl et al., 2023)

シーンを約1M 3Dガウシアンのクラウドとして表現。各59パラメータ：位置（3）、共分散（6、またはクォターニオン4+スケール3）、不透明度（1）、球面調和色（度3で48、度0で3）。

レンダリング = 投影+アルファ合成。高速（4090で1080p約100 fps）。微分可能。地面の真実写真に対して勾配降下でフィット。シーンは消費者GPUで5-30分でフィット。

上部の2つの2023-2024イノベーション：
- **生成ガウシアン スプラット。** LGM、LRM、InstantMeshのようなモデル1つまたは少数画像から直接ガウシアン クラウドを予測。
- **4D ガウシアン スプラッティング。** 動的シーン用フレームあたり相殺を持つガウシアン。

### マルチビュー拡散

事前訓練された画像拡散モデルをファインチューンしてテキストプロンプトまたは単一画像から同じオブジェクトの複数一貫したビューを生成。Zero123 (Liu et al., 2023)、MVDream (Shi et al., 2023)、SV3D (Stability, 2024)、CAT3D (Google, 2024)。通常オブジェクト周囲4-16ビューを出力、ガウシアン スプラッティング対たたたた3DをNeRFで持ち上げ。

### NeRF（文脈のために）

ニューラルラジアンスフィールド (Mildenhall et al., 2020)。小さいMLP`(x、y、z、ビュー方向)`を受け取り、`(色、密度)`を出力。レイに沿って統合することで描画。メッシュベース新規ビュー合成より品質で打ち勝つが100-1000倍遅いレンダリング。ほとんどリアルタイム用ガウシアン スプラッティングで置き換えられたが、研究で依然支配的。

## ビルドしよう

`code/main.py`小さい2D「ガウシアン スプラッティング」フィット実装：合成ターゲット画像（滑らかなグラデーション）を2Dガウシアン スプラットの合計として表現。位置、色、および共分散を勾配降下でターゲットマッチに最適化。2つのコア操作を見る：フォワード描画（スプラット+アルファ合成）およびフィット勾配降下で。

### ステップ1：2D ガウシアン スプラット

```python
def gaussian_at(x, y, gaussian):
    px, py = gaussian["pos"]
    sigma = gaussian["sigma"]
    d2 = (x - px) ** 2 + (y - py) ** 2
    return math.exp(-d2 / (2 * sigma * sigma))
```

### ステップ2：スプラットを合計することで描画

```python
def render(image_size, gaussians):
    img = [[0.0] * image_size for _ in range(image_size)]
    for g in gaussians:
        for y in range(image_size):
            for x in range(image_size):
                img[y][x] += g["color"] * gaussian_at(x, y, g)
    return img
```

実3D ガウシアン スプラッティング深さでソートガウシアンおよび順序のアルファ合成。わたしたちの2D トイただ合計。

### ステップ3：勾配降下でフィット

```python
for step in range(steps):
    pred = render(size, gaussians)
    loss = mse(pred, target)
    gradients = compute_grads(pred, target, gaussians)
    update(gaussians, gradients, lr)
```

## ピットフォール

- **ビューの不一貫性。** 4つのビューを独立して生成し、オブジェクト構造について異なるが、3Dフィットはぼやける。修正：共有アテンションを持つマルチビュー拡散。
- **裏側ハルシネーション。** 単一画像→3Dは目に見えない側を発明する必要。品質は大きく変わる。
- **ガウシアン スプラット爆発。** 制約なし訓練は10M スプラットに成長し、過度に適合。密化+刈り込みヒューリスティック（3D-GS元の論文から）必須。
- **トポロジー問題。** 陰的フィールド（SDF）からメッシュはしばしば穴または自己交差を持つ。シッパーの前にリメッシャー（例えばblenderのボクセルリメッシュ）を実行。
- **訓練データのライセンス。** Objaverseは混合ライセンスを持つ；商用使用はモデルごと変わる。

## 本番注：3Dはまだ共有基盤を持たない

イメージ（潜在拡散+DiT）およびビデオ（時空DiT）と異なり、3Dは2026年で支配的な単一実行時を持たない。本番決定木は表現フォークする：

- **NeRF / 三平面。** 推論はレイマーチング+サンプルごとのMLP前進。512² 描画は数百万MLPフォワードが必要。サンプルレイ積極的にバッチ；SDPA/xformersが適用。
- **マルチビュー拡散+LRM再構成。** 2段階パイプライン。段階1（マルチビュー DiT）は拡散サーバーはレッスン07のよう。段階2（LRM トランスフォーマー）はビュー全体の1ショット前進パス。全体レイテンシプロファイルは「拡散+1ショット」 — ステージごとの配信プリミティブ選ぶ。
- **SDS / DreamFusion。** アセット最適化ごと、推論ではない。ビルドジョブ、リクエストハンドラではない。

ほとんどの2026製品のため、右答えは「リクエスト時マルチビュー拡散モデルを実行、非同期に3DGSに再構成、リアルタイム表示のために3DGSを配布」。これはワークロードを清潔に分割GPU推論サーバー（高速）と非同期オプティマイザー（遅い）。
