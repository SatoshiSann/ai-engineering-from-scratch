# 流マッチング & 直線化流

> 拡散モデルはノイズからデータへの曲がった経路を歩くため、20-50サンプリングステップを取る。流マッチング (Lipman et al., 2023)と直線化流（Liu et al., 2022）訓練直線経路。直線経路はより少ないステップを意味する、より少ないステップは、より高速推論を意味。Stable Diffusion 3、Flux.1、およびAudioCraft 2はすべて2024年で流マッチングに切り替えた。

**タイプ:** Build
**言語:** Python
**前提条件:** Phase 8 · 06 (DDPM)、Phase 1 · 計算
**所要時間:** 約45分

## 問題

DDPMの逆プロセスは`N(0, I)`からデータ分布バック1000ステップの確率的歩行。IDIMはそれを20-50決定論的ステップに折りたたんだ。より少ないステップが欲しい — 理想的に1つ。ブロッカーは、逆プロセスを解くODEがスティフ；経路は曲がったもの。

モデルを訓練することができたら、ノイズからデータへの経路が*直線*であるようにしたら、`t=1`から`t=0`への単一Eulerステップは機能するだろう。流マッチングは直接これを構築する：`x_1 ∼ N(0, I)`からデータへの`x_0 ∼`直線補間を定義、訓練ベクトル場`v_θ(x、t)`その時間微分マッチ、推論で統合。

直線化流 (Liu 2022)さらに行く：反復的に再流手順で経路をまっすぐにしてより直線的なODEをプロデューサー。2つの再流反復後、2ステップサンプラーは50ステップDDPM品質と一致。

### 直線流

定義：

```
x_t = t · x_1 + (1 - t) · x_0,   t ∈ [0, 1]
```

ここで`x_0 ~ データ`および`x_1 ~ N(0、I)`。この直線に沿った時間微分は定数：

```
dx_t / dt = x_1 - x_0
```

ニューラルベクトル場`v_θ(x_t、t)`を定義し、この微分にマッチするよう訓練：

```
L = E_{x_0, x_1, t} || v_θ(x_t, t) - (x_1 - x_0) ||²
```

これは**条件付き流マッチング**損失（Lipman 2023）。訓練は無シミュレーション：ODEをアンロール決して。単に`(x_0、x_1、t)`をサンプルして、回帰。

### サンプリング

推論では、学習されたベクトル場を*時間をさかのぼって*統合：

```
x_{t-Δt} = x_t - Δt · v_θ(x_t, t)
```

`x_1 ~ N(0、I)`から開始、Euler-ステップ下`t=0`に。

### 直線化流 (Liu 2022)

直線流は機能するが、学習経路は*実際に直線ではない* — 多くの`x_0`が同じ`x_1`にマップ可能なため、曲がる。直線化流の再流ステップ：

1. ランダムペアリングで流モデル v_1 を訓練。
2. v_1 から統合することで N ペア`(x_1、x_0)`をサンプル、ランディング`x_0`。
3. ペアの例で v_2 を訓練。ペアが「ODE マッチ」されたため、それらの間の直線補間は本当にフラット。
4. 反復。

実践では、2つの再流反復は近線形に到達、2-4ステップ推論を有効化。SDXL-Turbo、SD3-Turbo、LCMはすべて流マッチング蒸留モデル。

### なぜこれは2024年イメージで勝ったのか

3つの理由：

1. **無シミュレーション訓練** — 訓練中ODEアンロール、実装が簡単。
2. **より良い損失ジオメトリ** — 直線経路は一貫した信号ノイズ、一方DDPMε-損失は悪いSNR スケジュールの端。
3. **より速い推論** — SDXL-Turbo品質で4-8ステップ；一貫性蒸留で1ステップ。

## 流マッチング vs DDPM — 正確な接続

ガウシアン条件付きパスでの流マッチングは拡散*特定ノイズスケジュール*。`x_t = α(t) x_0 + σ(t) x_1`スケジュール選び、流マッチングは`v = α'·x_0 - σ'·x_1`とStratonovich再配置拡散を回復。2つは、ガウシアン経路を代数的に等価。

流マッチングが追加したもの：目標の*明確さ*（明白な速度）、より清潔な損失、および非ガウシアン補間を試す許可。

```python
def sample(net, num_steps):
    x = rng.gauss(0, 1)
    for i in range(num_steps):
        t = 1.0 - i / num_steps
        dt = 1.0 / num_steps
        x -= dt * net_forward(x, t)
    return x
```

## ピットフォール

- **時間パラメータ化。** 流マッチングは`t ∈ [0, 1]`を使用します`t=0`データで、`t=1`ノイズで。DDPMは`t ∈ [0, T]`を使用`t=0`データで、`t=T`ノイズで。同じ方向、異なるスケール。論文はこれを定数間違う。
- **スケジュール選択。** 直線化流の直線は「その」流マッチングスケジュール、しかしコサインまたはlogit-normal t-サンプリング（SD3はこれを実行）をより良いスケールカバレッジに使用できる。
- **再流コスト。** 再流用のペアダータセット生成は完全な推論パス/サンプル。1-2ステップ推論が本当に必要な場合のみ再流を実行。
- **Classifier-free guidanceはまだ適用。** ε for v をその線形結合にスワップ：`v_cfg = (1+w) v_cond - w v_uncond`。

## 演習

1. **簡単。** `code/main.py`を実行し、1ステップ vs 20ステップ MSE vs 真のデータ分布を比較。
2. **中程度。** 一様`t`サンプリングからlogit-normal（mid-tでサンプリング集中）に切り替え。モデル品質は改善するか？
3. **難しい。** 1つの再流反復を実装：最初のモデルを統合することで`(x_0、x_1)`ペアを生成、ペアの2番目モデルを訓練、1ステップサンプル品質を比較。
