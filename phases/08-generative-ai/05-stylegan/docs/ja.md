# StyleGAN

> ほとんどのジェネレータは `z` をすべてのレイヤーの同じ時間にかき混ぜる。StyleGANはそれを分離: まず `z` を中程度 `w` にマップ、その後*注入* `w` すべての解像度レベル以上を通じて AdaIN。その単一の変更は潜在空間を解き放ち、写真実在的フェイスは7年連続で解決された問題を作った。

**タイプ:** ビルド
**言語:** Python
**前提条件:** Phase 8 · 03 (GANs)、Phase 4 · 08 (Normalization)、Phase 3 · 07 (CNNs)
**所要時間:** ~45分

## 問題

DCGANマップ `z` トランスポーズ畳み込みのスタックを通じてイメージへ。問題: `z` はすべてを制御 — ポーズ、ライティング、識別、背景 — 纏った。`z` の1つの軸に沿って移動、4つ全ては移動。あなたはモデルに「同じ人、異なるポーズ」を尋ねることができない、なぜなら表現はその方法ファクタしない。

Karras et al. (2019、NVIDIA)提案: 止める `z` を直接conv層への給食。給食定数 `4×4×512` テンサーとしてネットワーク入力へ。8層MLP learn that maps `z ∈ Z → w ∈ W`。注入 `w` すべての解像度によって *適応インスタンス正規化* (AdaIN): 正規化各conv特性マップ、その後スケール及びシフト `w` の活性化射影による。追加層あたりノイズ確率的詳細のため(皮膚毛穴、髪ストランド)。

結果: `W` はおおよそ直交軸を「高レベルスタイル」(ポーズ、識別)対「微調整スタイル」(ライティング、色)。スワップスタイルは2つのイメージ間により、イメージAの `w` を低解像度レベルのために使用、イメージBの `w` をハイ-。これをロック解除編集、クロス-ドメイン様式化、および全体的「StyleGAN-反転」研究ラインを編集。

## コンセプト

![StyleGAN: マッピング ネットワーク + AdaIN + 層ノイズあたり](../assets/stylegan.svg)

**マッピング ネットワーク。** `f: Z → W`、8層MLP。`Z = N(0, I)^512`。`W` はガウスに強制されない — データ-適応形状を学習。

**合成ネットワーク。** 学習定数 `4×4×512` から開始。各解像度ブロック: `アップサンプル → conv → AdaIN(w_i) → ノイズ → conv → AdaIN(w_i) → ノイズ`。解像度の倍: 4, 8, 16, 32, 64, 128, 256, 512, 1024。

**AdaIN。**

```
AdaIN(x, y) = y_scale · (x - mean(x)) / std(x) + y_bias
```

ここで `y_scale` および `y_bias` は `w` の活性化射影から来る。正規化特性マップあたり、その後リスタイル。「スタイル」ここでは特性マップの最初および二次統計。

**層ノイズあたり。** 単一チャネルガウスノイズは各特性マップへ追加、学習層ごとチャネル係数でスケール。グローバル構造を影響しない確率的詳細を制御。

**切詰めトリック。** 推論で、サンプル `z`、計算 `w = mapping(z)`、その後 `w' = ŵ + ψ·(w - ŵ)` ここで `ŵ` は多くのサンプルの上での平均 `w`。`ψ < 1` 多様性のためのトレード品質。ほぼすべてのStyleGANデモは `ψ ≈ 0.7` を使用。

## StyleGAN 1 → 2 → 3

| バージョン | 年 | イノベーション |
|---------|------|------------|
| StyleGAN | 2019 | マッピング ネットワーク + AdaIN + ノイズ + 段階的に成長。 |
| StyleGAN2 | 2020 | 重量除調変は AdaIN を置き換え(しずくアーティファクトを修正); スキップ/残余アーキテクチャ; パス-長さ正則化。 |
| StyleGAN3 | 2021 | エイリアス-フリー畳み込み + 同変カーネル; テクスチャがピクセルグリッドに粘る除去。 |
| StyleGAN-XL | 2022 | クラス-条件付け、1024²、ImageNet。 |
| R3GAN | 2024 | より強い正則化でリブランド; FFHQ-1024のパラメータで拡散へのギャップを閉じる20x。 |

2026年では StyleGAN3 は(a)狭いドメイン写真実在性で高FPS、(b)少数ショットドメイン適応の既定値はパラメータ保ったままの新しいデータセットで訓練、100イメージ、凍結マッピング)、(c)反転ベース編集(見つける `w` real フォトを再構成してから、その `w` を編集)。オープンドメインテキスト-to-イメージのため、それはツール — 拡散はない。

## ビルドする

`code/main.py` はおもちゃ「スタイル-GAN軽い」を1-Dで実装: マッピングMLP、合成関数、学習定数ベクトルを取り、`w`-導出スケール/バイアスでモジュレート、および層ノイズあたり。これは `w` を通じて注入は、またはジェネレータの入力への `z` に連結する、及び/または打つ示す。

### ステップ 1: マッピング ネットワーク

```python
def mapping(z, M):
    h = z
    for i in range(num_layers):
        h = leaky_relu(add(matmul(M[f"W{i}"], h), M[f"b{i}"]))
    return h
```

### ステップ 2: 適応インスタンス正規化

```python
def adain(x, w_scale, w_bias):
    mu = mean(x)
    sd = std(x)
    x_norm = [(xi - mu) / (sd + 1e-8) for xi in x]
    return [w_scale * xi + w_bias for xi in x_norm]
```

特性マップあたりスケール及びバイアスは `w` を通じて線形射影から来る。

### ステップ 3: 層ノイズあたり

```python
def add_noise(x, sigma, rng):
    return [xi + sigma * rng.gauss(0, 1) for xi in x]
```

シグマ層ごとチャネルは学習可能。

## 落とし穴

- **しずくアーティファクト。** StyleGAN 1はしずくが生成特性マップで、なぜなら AdaIN は平均をゼロにした。StyleGAN 2の重量除調変は、代わりに畳み込み重みをスケール、これを修正。
- **テクスチャ粘着。** StyleGAN 1及び2テクスチャはオブジェクト座標でないピクセル座標に続く(補間の際に見えるとき)。StyleGAN 3のエイリアス-フリー畳み込みは窓sinc filtres でこれを修正。
- **モード補償。** 切詰め `ψ < 0.7` はクリーンなサンプル見える狭いコーンから; `ψ = 1.0` を使用多様性が必要に場合。
- **反転は損失。** リアル フォトを `W` へ反転することは通常最適化またはエンコーダ(e4e、ReStyle、HyperStyle)を通じて行われ。結果は多くの反復の上ドリフト。

## 使用

| ユースケース | アプローチ |
|----------|----------|
| 写真実在的人間フェイス(アニメ、製品、ナロー) | StyleGAN3 FFHQ / カスタム微調整 |
| フェイス フォトからの編集 | e4e反転 + StyleSpace / InterFaceGAN方向 |
| フェイス スワップ / reenactment | StyleGAN + エンコーダ + ブレンド |
| アバター パイプライン | ADAのための低-データ微調整を持つStyleGAN3 |
| ドメイン適応少数イメージから | マッピング ネットワークを凍結、合成を微調整 |
| マルチ-モーダルまたはテキスト-条件付け生成 | しない — 拡散を使用 |

製品グレードデモため「フォト人の顔の」が答え、StyleGAN は推論コスト(単一順伝播、<10ms 4090上)および品質のためのシャープネスで拡散を打つ同じ品質バー。

`outputs/skill-stylegan-inversion.md` で保存。スキルは本物フォトを取り、出力: 反転メソッド(e4e / ReStyle / HyperStyle)、予想潜在損失、編集予算(どのくらい遠く `W` でアーティファクトを前に移動できる)、および既知-良い編集方向(年齢、表現、ポーズ)のリスト。

## 演習

1. **簡単。** `code/main.py` を実行 `adain_on=True` および `adain_on=False` を持つ。出力の拡散をスプレッド比較固定潜在対摂動潜在のため。
2. **中程度。** ミックス正則化を実装: 訓練バッチのため、計算 `w_a`、`w_b`、適用 `w_a` 合成の最初の半分のため及び `w_b` 2番目のもの。デコーダはlearn分解スタイルか?
3. **難しい。** 事前訓練StyleGAN3 FFHQ モデルを取る(ffhq-1024.pkl)。見つける `w` 方向がラベル付きサンプルでSVMを訓練により「スマイル」を制御; アイデンティティがドリフトする前にどのくらい遠く押すかをレポート。

## 重要な用語

| 用語 | 人々が言う | 実際の意味 |
|------|-----------------|-----------------------|
| マッピング ネットワーク | 「MLP」 | `f: Z → W`、8層、潜在幾何学をデータ統計から分離。 |
| W空間 | 「スタイル空間」 | マッピング ネットワークの出力; おおよそ分解。 |
| AdaIN | 「適応インスタンス正規化」 | 正規化特性マップ、その後スケール + シフト `w`-射影による。 |
| 切詰めトリック | 「Psi」 | `w = mean + ψ·(w - mean)`、ψ<1 多様性を品質と交換。 |
| パス-長さ正則化 | 「PL reg」 | `w` の単位変更あたりの大きい変更をイメージのペナルティ; `W` をなめらか。 |
| 重量除調変 | 「StyleGAN2 修正」 | 活性化の代わりに正規化conv重み; しずくアーティファクトを殺す。 |
| エイリアス-フリー | 「StyleGAN3のトリック」 | 窓sinc フィルタ; ピクセルグリッドへのテクスチャ粘着を除去。 |
| 反転 | 「本物イメージのためにw見つける」 | 最適化またはエンコード `x → w` ので `G(w) ≈ x`。 |

## 本番ノート: なぜStyleGAN依然として2026年に配布

StyleGAN3 on 4090 1024² FFHQフェイスを10 ms以下で生成 — `num_steps = 1`、VAEデコードなし、クロス-アテンション処理なし。本番項では、これはすべてのイメージジェネレータのフロアレイテンシ。50-ステップSDXL + VAE-デコード同じ解像度でのパイプラインは~3秒。それは**300×ギャップ**、および狭いドメイン製品(アバターサービス、IDドキュメント パイプライン、ストック フェイス生成)のため、それはTCOで勝つ。

2つの運用上の結果:

- **スケジューラなし、バッチャーなし。** 静的バッチはターゲット占有率で最適。継続バッチ(LLMおよび拡散に必須)はゼロ利点を提供してから各リクエストは同じFLOPsを取る。
- **切詰め `ψ` はセーフティノブ。** `ψ < 0.7` マッピング ネットワークの範囲の狭いコーンからサンプル。これは配置層が見本分散の上にレバーを持つ唯一のもの。ピークロードで下 `ψ`、プレミアムユーザーのために上げる。
