# 潜在拡散 & Stable Diffusion

> 512×512画像でのピクセル空間拡散は計算上の悪行だ。Rombach et al. (2022)は気づいた — セマンティック構造を捉えるだけで十分で、残りは別のデコーダで処理できる、ということに。VAEの潜在空間内で拡散を実行する。その1つのアイデアがStable Diffusionだ。

**タイプ:** Build
**言語:** Python
**前提条件:** Phase 8 · 02 (VAE)、Phase 8 · 06 (DDPM)、Phase 7 · 09 (ViT)
**所要時間:** 約75分

## 問題

512²でのピクセル空間拡散は、U-Netが形状`[B, 3, 512, 512]`のテンソルで実行されることを意味する。各サンプリングステップは500Mパラメータ U-Netで約100 GFLOPSである。50ステップは画像あたり5 TFLOPSだ。10億枚の画像で訓練すると、計算コストは莫大になる。

これらのFLOPSのほとんどは、ネットを通じて知覚的に重要でない詳細を押し流すのに費やされている — ロッシーVAEが圧縮できる高周波テクスチャである。Rombach のアイデア：VAEを1回訓練して（*第1段階*）、それを固定し、拡散を4チャネル64×64潜在空間内で完全に実行する（*第2段階*）。同じU-Net。ピクセルの1/16。同等の品質で約64倍少ないFLOPS。

これはStable Diffusionのレシピである。SD 1.x / 2.xは`64×64×4`潜在に対して860M U-Netを使用し、SDXLは`128×128×4`に対して2.6B U-Netを使用した、SD3はU-Netを拡散変換器（DiT）と流マッチングに置き換えた。Flux.1-dev (Black Forest Labs, 2024)は12Bパラメータ DiT-MMDiTを搭載している。すべて同じ2段階基盤で実行される。

## コンセプト

![潜在拡散: VAE圧縮+潜在空間での拡散](../assets/latent-diffusion.svg)

**2つの段階、別々に訓練される。**

1. **段階1 — VAE。** エンコーダ`E(x) → z`、デコーダ`D(z) → x`。ターゲット圧縮：各空間軸で8倍ダウンサンプル+チャネルを調整してピクセル数の約1/16になるように。損失=再構成（L1 + LPIPS知覚的）+ KL（小さい重みなので`z`がガウシアンに強制されないため、`z`から正確なサンプリングが必要ないので）。敵対的損失で訓練されることが多いため、デコードされた画像は鮮明だ。

2. **段階2 — `z`上の拡散。** `z = E(x_real)`をデータとして扱う。U-Net（またはDiT）を訓練して`z_t`をノイズ除去する。推論時：`z_0`を拡散経由でサンプルし、`x = D(z_0)`を実行する。

**テキスト条件付け。** 2つの追加コンポーネント。凍結されたテキストエンコーダ（SD 1.xのCLIP-L、SD 2/XLのCLIP-L+OpenCLIP-G、SD3およびFluxのT5-XXL）。クロスアテンション注入：すべてのU-Netブロックが`[Q = 画像特徴、K = V = テキストトークン]`を受け取り、それらを混在させる。トークンはテキストが画像に影響を与える唯一の方法だ。

**損失関数はレッスン06と同じだ。** DDPMと拡散の同じMSE / 流マッチング。データドメインを入れ替えるだけだ。

## ビルドしよう

`code/main.py`は、レッスン06のDDPMの上にトイ1-D「VAE」（デモ用のアイデンティティエンコーダ+デコーダ、実VAEは畳み込みネットワーク）をスタックし、classifier-free guidanceでクラス条件付けを追加する。同じ拡散損失は、生の1-D値でも符号化された値でも実行されることを示している — これが重要な洞察である。

### ステップ1：エンコーダ/デコーダ

```python
def encode(x):    return x * 0.5          # トイ「圧縮」をより小さいスケールに
def decode(z):    return z * 2.0
```

実VAEは訓練されたウェイトを持つ。教育的に、この線形マップは拡散が`z`で元データ空間を気にしないことを示すのに十分だ。

### ステップ2：`z`-空間での拡散

レッスン06と同じDDPM。ネットが見るデータは`z = E(x)`。`z_0`をサンプルした後、`D(z_0)`でデコードする。

### ステップ3：classifier-free guidance

訓練中、クラスラベルを10%の時間ドロップ（ヌルトークンで置き換え）。推論で、`ε_cond`と`ε_uncond`の両方を計算してから：

```python
eps_cfg = (1 + w) * eps_cond - w * eps_uncond
```

`w = 0` =ガイダンスなし（完全多様性）、`w = 3` =デフォルト、`w = 7+` =飽和/過シャープ。

### ステップ4：テキスト条件付け（コンセプト、コードではない）

クラスラベルを凍結されたテキストエンコーダ出力で置き換える。テキスト埋め込みをクロスアテンション経由でU-Netに供給する：

```python
h = h + CrossAttention(Q=h, K=text_embed, V=text_embed)
```

これはクラス条件付き拡散モデルとStable Diffusionの間の唯一の実質的な違いだ。

## ピットフォール

- **VAE スケール不一致。** SD 1.x VAEはスケーリング定数（`scaling_factor ≈ 0.18215`）をエンコード後に適用している。これを忘れるとU-Netは完全に間違った分散を持つ潜在で訓練される。すべてのチェックポイントは1つを搭載している。
- **テキストエンコーダが静かに間違っている。** SD3はT5-XXL with >=128トークンが必要で、CLIP-onlyへのフォールバックはロッシーだ。常に`use_t5=True`をチェックするか、プロンプト忠実性は崩壊する。
- **潜在空間を混在させる。** SDXL、SD3、Fluxはすべて異なるVAEを使用する。SDXL潜在で訓練されたLoRAはSD3では機能しない。Hugging Face diffusers 0.30+は不一致チェックポイントの読み込みを拒否する。
- **CFGが高すぎる。** `w > 10`は飽和した、油っぽい画像を生成し、多様性の代償としてプロンプトに過度に適合する。甘いスポットは`w = 3-7`である。
- **負のプロンプトが漏れ出している。** 空の負プロンプトはヌルトークンになり、満たされた負プロンプトは`ε_uncond`になる。これらは同じではなく、一部のパイプラインはヌルに静かにデフォルト化する。

## 演習

1. **簡単。** `code/main.py`をガイダンス`w ∈ {0, 1, 3, 7, 15}`で実行する。クラスごとの平均サンプルを記録する。実データ平均を超えてクラス平均が分岐する`w`は何か？
2. **中程度。** トイ線形エンコーダをtanh-MLPエンコーダ/デコーダペアと再構成損失で交換する。新しい潜在での拡散を再訓練する。サンプル品質は変わるか？
3. **難しい。** diffusersで実際のStable Diffusion推論をセットアップ：`sdxl-base`をロード、CFG=7で30 Eulerステップを実行し、タイムする。次に`sdxl-turbo`を4ステップとCFG=0に切り替える。同じサブジェクト、異なる品質 — 何が変わったか、なぜか説明する。

## 本番注：8GB消費者GPUでFlux-12Bを実行する

参照Flux統合は標準的な「消費者GPUがあり、これを配布できるか？」レシピの標準である。トリックは本番推論文献に記載されている同じ3ノブレシピを拡散DiTに適用するのと同じだ：

1. **段階的読み込み。** Fluxには、VRAMで共存する必要がない3つのネットワークがある：T5-XXLテキストエンコーダ（約10 GB fp32）、CLIP-L（小さい）、12B MMDiT、およびVAE。最初にプロンプトをエンコード、エンコーダを*削除*、DiTをロード、ノイズ除去、DiTを*削除*、VAEをロード、デコード。消費者8GB GPUは一度に1つの段階だけ収まる。
2. **bitsandbytesの4ビット量子化。** `BitsAndBytesConfig(load_in_4bit=True, bnb_4bit_compute_dtype=torch.bfloat16)`をT5エンコーダとDiTの両方に。メモリを8倍削減、品質低下はAritaのベンチマーク（ノートブックにリンク）によるテキスト画像に対して認識不可。
3. **CPU オフロード。** `pipe.enable_model_cpu_offload()`は、各前進パスが進むにつれてモジュールをCPUとGPU間で自動交換する。レイテンシを10-20%追加するが、パイプラインがすべて実行されるようにする。

メモリ会計：`10 GB T5 / 8 = 1.25 GB`量子化、`12 Bパラメータ × 0.5バイト = 約6 GB`量子化DiT、プラス活性化。stas00の用語では、これはTP=1推論の極端な端 — モデル並列化なし、最大量子化。本番ではH100でTP=2またはTP=4を実行します；単一の開発ラップトップの場合、これはレシピだ。
