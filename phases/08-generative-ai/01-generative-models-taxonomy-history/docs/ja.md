# 生成モデル — 分類 & 歴史

> すべてのイメージモデル、テキストモデル、ビデオモデル、および3Dモデルは5つのバケットの1つに適合する。間違ったバケットを選ぶと、数週間数学と戦う。正しいものを選ぶと、分野の過去12年間の進歩があなたの頭にきれいに積み重なる。

**タイプ:** 学習
**言語:** Python
**前提条件:** Phase 2 (ML Fundamentals)、Phase 3 (Deep Learning Core)、Phase 7 · 14 (Transformers)
**所要時間:** ~45分

## 問題

生成モデルは1つの仕事をする: 未知の分布 `p_data(x)` から抽出された訓練サンプルを与えられて、同じ分布から来たように見える新しいサンプルを出力する。顔、文、MIDIファイル、タンパク質構造 — 全て目をつぶるなら同じ問題。

問題は `p_data` がおよそ数百万の次元を持つ空間に住ん(512x512 RGBイメージは ~786k 次元)でいることであり、サンプルはその空間内の細い多様体に座り、そしてあなたはおそらく約10Mの例のみを持つ。密度をブルートフォースするのは絶望的。すべての生成モデルは1つの難しい問題を少しやさしい問題と交換する妥協案である。

5つの家族が過去12年間を生き残った。各家族がどんな妥協をするかを知ることは、それがいくつかのタスクで勝ち、他のタスクで壊れるかに関わる理由を教えてくれる。

## コンセプト

![5つの家族の生成モデル — 何をモデル化するかでの分類](../assets/taxonomy.svg)

**1. 明示的密度、扱いやすい。** `log p(x)` を実際に評価できる合計として書く。自己回帰モデル(PixelCNN、WaveNet、GPT)は `p(x) = ∏ p(x_i | x_<i)` を因数分解。正規化フロー(RealNVP、Glow)は単純なベースの可逆変換として `p(x)` を構築。長所: 正確な尤度、クリーン訓練損失。短所: 自己回帰推論はシリアル(長いシーケンスで遅い)、フローは可逆アーキテクチャが必要(アーキテクチャ的に限定的)。

**2. 明示的密度、近似。** `log p(x)` を下から(ELBO)バウンドして、バウンドを最適化。VAEs (Kingma 2013)はエンコーダ-デコーダと変分事後。拡散モデル(DDPM、Ho 2020)は暗黙的に加重ELBOを最適化する除ノイザーを訓練。拡散は2026年のイメージ、ビデオ、および3Dバックボーン。

**3. 暗黙的密度。** 密度を全くスキップ; サンプルを生成する生成器 `G(z)` と本物を偽物から区別する判別器 `D(x)` を学習。GANs (Goodfellow 2014)。推論時に高速(1つの順伝播)だが、訓練中に悪名高く不安定。StyleGAN 1/2/3は2026年でも固定ドメイン写真実在(顔、寝室)に対してSOTAのままである。

**4. スコアベース/連続時間。** ログ密度の勾配 `∇_x log p(x)` (スコア)を直接学習。Song & Ermon (2019)はスコアマッチングが拡散をSDEに一般化することを示した。フロー マッチング(Lipman 2023)は2024-2026のホットさ: シミュレーションフリー訓練、より直線的なパス、DDPM より4-10倍高速なサンプリング。Stable Diffusion 3、Flux、AudioCraft 2はすべてフロー マッチングを使用。

**5. トークンベース離散コード上の自己回帰。** VQ-VAEまたは剰余量子化を使用して高次元データを短いシーケンスの離散トークンに圧縮してから、トランスフォーマーを使用してトークンシーケンスをモデル化。Parti、MuseNet、AudioLM、VALL-E、Soraのパッチトークナイザーはすべてこれを使用。これはバケット1プラス学習トークナイザー。

## 5-質問トリアージ

新しい生成モデルペーパーが落ちるとき、方法セクションを読む前にこれらの5つの質問に回答。

1. **何がモデル化されているか?** ピクセル、潜在、離散トークン、3Dガウス、メッシュ、波形?
2. **密度は明示的か暗黙的か?** `log p(x)` を書き下す?
3. **サンプリング: ワンショットまたは反復?** 反復は推論が遅いことを意味; ワンショットは通常敵対的または蒸留。
4. **条件: 無条件、クラス、テキスト、イメージ、ポーズ?** これは損失とアーキテクチャスキャフォルディングを決定。
5. **評価: FID、CLIPスコア、IS、人間嗜好、タスク精度?** 各は既知の失敗モード(Lesson 14参照)。

このフェーズのすべてのレッスン内でこれら5つを再回答する。終了するまで、それらは反射。

## ビルドする

このレッスンのコードは軽量な可視化: 3つのおもちゃのアプローチ(カーネル密度、離散ヒストグラム、および最近隣サンプル「GAN的」生成器)を使用して1-Dガウス混合をサンプルからフィット、そのため明示的対暗黙密度の差を1画面に印字できる問題で見ることができる。

`code/main.py` を実行。2000サンプルを2モード-ガウス混合から抽出して、その後出力:

```
明示的密度 (ヒストグラム): p(x in [-0.5, 0.5]) ≈ 0.38
近似密度 (KDE):     p(x in [-0.5, 0.5]) ≈ 0.41
暗黙的 (最近隣サンプル生成): 20個の新しいサンプル出力、p(x) なし
```

注意: 最初の2つは「このポイントはどのくらい可能性があるか」と尋ねることができる。3番目はできない。これは今後のすべてのレッスン*明示的対暗黙的*区別である。

## 演習

1. **簡単。** これら5つの製品のそれぞれについて、家族とバックボーンを特定: ChatGPTイメージ、Midjourney v7、Sora、Runway Gen-3、ElevenLabs。証拠は公開技術レポートからのべき。
2. **中程度。** 明日読むつもりのペーパーは拡散より100倍高速なサンプリングを主張。条件付けと高解像度上でスピードアップが生き延びるかどうかを確認するために3つの質問を書き下す。
3. **難しい。** あなたが気にする1つのドメイン(例: タンパク質構造、CAD、分子、軌跡)。その領域の現在のSOTAモデルのための5質問トリアージに回答して、より良いモデルが何を変えるかスケッチ。

## 重要な用語

| 用語 | 人々が言う | 実際の意味 |
|------|-----------------|-----------------------|
| 生成モデル | 「新しいものを作ります」 | `p_data(x)` のサンプラーを学習、オプションで `log p(x)` を公開。 |
| 明示的密度 | 「評価できます」 | モデルは閉鎖形または扱いやすい `log p(x)` を提供。 |
| 暗黙的密度 | 「GAN風」 | サンプラーのみ — 与えられたポイントの `p(x)` を評価する方法なし。 |
| ELBO | 「証拠下限」 | `log p(x)` 上の扱いやすい下限; VAEsおよび拡散が最適化。 |
| スコア | 「ログ密度の勾配」 | `∇_x log p(x)`; 拡散およびSDEモデルはこのフィールドを学習。 |
| 多様体仮説 | 「データは表面に住む」 | 高次元データは低次元多様体に集中; なぜ次元削減が作動するか。 |
| 自己回帰 | 「次のピースを予測」 | 条件のの製品として結合因数分解。 |
| 潜在 | 「圧縮コード」 | デコーダが入力を再構成できる低次元表現。 |

## 本番ノート: 5つの家族、5つの推論形状

各家族は異なる推論-サーバーコスト曲線にマップ。本番-推論文献はLLM推論をプリフィル + デコードとしてフレーム; 同じ分解がここに適用:

- **自己回帰 (バケット1および5)。** シリアルデコードはレイテンシを支配; KV-キャッシュ、継続バッチ、および推測デコーディング直接適用。
- **VAE / 拡散 / フロー-マッチング (バケット2および4)。** LLM感覚でのデコードなし。コスト = `num_steps × step_cost`、および `step_cost` は完全潜在解像度での変圧器またはU-Netの順伝播。本番ノブはステップ数(DDIM / DPM-Solver / 蒸留)、バッチサイズ、および精度(bf16 / fp8 / int4)。
- **GAN (バケット3)。** 1つの順伝播。スケジュール、KV-キャッシュなし。TTFT ≈ 総レイテンシ。これがStyleGANがなお狭いドメインUXで勝つ理由。

ペーパー抄録で「拡散より高速」を見るとき、「より少ないステップ × 同じステップコスト」または「同じステップ × より安いステップコスト」に翻訳。その他すべてはマーケティング。
