# インペイント、アウトペイント & 画像編集

> テキスト画像は新しいものを作る。インペイントは古いものを修正する。本番では、請求可能な画像作業の70%は編集である — 背景を交換、ロゴを削除、キャンバスを拡張、手を再生成。インペイントは拡散が生活費を稼ぐ場所だ。

**タイプ:** Build
**言語:** Python
**前提条件:** Phase 8 · 07 (潜在拡散)、Phase 8 · 08 (ControlNet & LoRA)
**所要時間:** 約75分

## 問題

クライアントが背景に気を散らす看板がある完璧な製品写真を送る。看板を削除して、他のすべてをピクセル同一に残したい。テキスト画像をゼロから実行できない — 結果は異なる色、異なる照明、異なる製品角度を持つだろう。*マスク領域のみ*を再生成したい、再生成は周囲のコンテキストを尊重してもらいたい。

それがインペイントだ。バリアント：

- **インペイント。** マスク内を再生成、外側のピクセルを保持。
- **アウトペイント。** マスク外（またはキャンバスの外）を再生成、内側を保持。
- **画像編集。** 全体像を再生成するが、元のセマンティックまたは構造忠実性を維持（SDEdit、InstructPix2Pix）。

2026年のあらゆる拡散パイプラインはインペイントモードを配布。Flux.1-Fill、Stable Diffusion Inpaint、SDXL-Inpaint、DALL-E 3 Edit。彼らは同じ原理で動作。

### 素朴なアプローチ（そしてなぜ間違っているのか）

マスク付きでテキスト画像を実行。各サンプリングステップで、ノイズの多いレイテント内でマスクされていない領域を前進拡散クリーン画像と置き換える。機能...悪く。マスク領域の中にあることについてモデルに情報がないため、バウンダリアーティファクトが出血する。

### 適切なインペイントモデル

9入力チャネルの代わりに入力を受け取るために修正されたU-Netを訓練：

```
input = concat([ ノイズの多いレイテント (4ch), エンコードされた画像 (4ch), マスク (1ch) ], dim=channel)
```

追加チャネルはVAEエンコードされたソース画像とシングルチャネルマスクのコピー。訓練時、ランダムにマスク領域をマスクし、マスク領域のみのノイズを除去するようモデルを訓練、マスクされていない領域は条件付けシグナルとしてクリーンが与えられる。推論では、モデルはマスク領域を囲むものを「見」、首尾一貫した完成を生成できる。

SD-Inpaint、SDXL-Inpaint、Flux-Fillはすべてこの9チャネル（またはアナログ）入力を使用。Diffusers `StableDiffusionInpaintPipeline`、`FluxFillPipeline`。

### SDEdit (Meng et al., 2022) — 無料編集

いくつかの中間`t`までソース画像にノイズを追加し、新しいプロンプトで`t`から0まで逆チェーンを実行。再訓練なし。開始`t`の選択は忠実性と創造的自由を取引する：

- `t/T = 0.3` → ソースとほぼ同じ、小さいスタイル変化
- `t/T = 0.6` → 中程度の編集、粗い構造を保持
- `t/T = 0.9` → 近ノイズから生成、最小ソース保持

### InstructPix2Pix (Brooks et al., 2023)

拡散モデルを`(入力画像、指示、出力画像)`トリプルでファインチューン。推論では、入力画像と「サンセット化」、「ドラゴン追加」などのテキスト指示の両方で条件付け。2つのCFGスケール：画像スケールとテキストスケール。

### RePaint (Lugmayr et al., 2022)

標準無条件拡散モデルを保持。各逆ステップで、リサンプル — 時々ノイジーな状態に戻りジャンプして再生成。バウンダリアーティファクトを回避。訓練されたインペイントモデルがないときに使用。

## ビルドしよう

`code/main.py`は5次元データで小さい1-Dインペイント方式を実装。2つのクラスタの1つから5フロート毎のサンプル混合データでDDPMを訓練。推論では、5次元の2つを「マスク」、マスクされていない3つのノイズの多いフォワード版を各ステップに注入、マスク領域のみを再生成。

### ステップ1：5-D DDPMデータ

```python
def sample_data(rng):
    cluster = rng.choice([0, 1])
    center = [-1.0] * 5 if cluster == 0 else [1.0] * 5
    return [c + rng.gauss(0, 0.2) for c in center], cluster
```

### ステップ2：すべての5次元でデノイザーを訓練

標準DDPM。ネット出力は5-Dノイジーな入力に対して5-Dノイズ予測。

### ステップ3：推論時、マスク認識逆

```python
def inpaint_step(x_t, mask, clean_image, alpha_bars, t, rng):
    # マスクされていない薄暗いものをクリーンソースの新しくノイズを出された版と置き換える
    a_bar = alpha_bars[t]
    for i in range(len(x_t)):
        if not mask[i]:
            x_t[i] = math.sqrt(a_bar) * clean_image[i] + math.sqrt(1 - a_bar) * rng.gauss(0, 1)
    # ...その後x_tで通常の逆ステップを実行する
```

これは素朴なアプローチでトイ1-Dデータで機能する。実画像インペイントはテクスチャコヒーレンスが重要なので、9チャネル入力を使用する。

### ステップ4：アウトペイント

アウトペイントはインペイントでマスク反転：新しい（以前存在しなかった）キャンバスをマスク、残りを元で埋める。同一訓練目的。

## ピットフォール

- **シーム。** 素朴なアプローチはマスク全体で勾配情報が流れないため、目に見える境界を残す。修正：マスクを8-16ピクセル拡張、または適切なインペイントモデルを使用。
- **マスク漏れ。** 条件付け画像のマスクされていない領域が低品質またはノイズの多い場合、マスク内生成を汚す。わずかにデノイズまたはぼかし。
- **CFGはマスクサイズと相互作用。** 小さいマスク上の高CFG = 飽和パッチ。小さい編集のCFGを低減。
- **SDEdit忠実性断崖。** `t/T = 0.5`から`t/T = 0.6`に行くと、サブジェクトの身元が失われる可能性。スイープしてチェックポイント。
- **プロンプト不一致。** プロンプトは*新しいコンテンツのみ*ではなく、*全体*画像を説明する必要。「犬の上に座っている猫」ではなく「椅子」。

## 演習

1. **簡単。** `code/main.py`で、マスクされた次元の分数を0.2から0.8に変動させる。インペイント品質（マスク薄暗い残差）が無条件生成と等しい時点の分数は何か？
2. **中程度。** RePaintを実装：10ステップの逆ごとに、5ステップバック（ノイズ追加）ジャンプし再デノイズ。マスク端でバウンダリ残差を低減するかどうか測定。
3. **難しい。** Hugging Face diffusersを使用して比較：SD 1.5 Inpaint + ControlNet-Openpose vs Flux.1-Fill 20顔再生成タスク。ポーズ準拠と身元保持を別々にスコア。

## 本番注：編集パイプラインはレイテンシ敏感

ユーザーが画像を編集するのは、サブ5秒ラウンドトリップを期待。1024²で30ステップSDXL-InpaintはL4で3-4秒、プラスSAM マスク生成（約200 ms）とVAE符号化/デコード（合計約500 ms）。本番フレーミングでは、これはスループット境界ではなくTTFT境界 — バッチ1、低コンカレンシー、すべてのステージを最小化：

- **SAM-Hは遅いものだ。** 1024²でのSAM-Hは約200 ms；SAM-ViT-Bは軽い品質損失で約40 ms。SAM 2（ビデオ）時間オーバーヘッドを追加；単一画像編集に使用しない。
- **可能なときエンコードをスキップ。** `pipe.image_processor.preprocess(img)`潜在にエンコード。前の生成から潜在を持っている場合（反復編集UIで典型的）、`latents=...`経由で直接渡し1つのVAEエンコードをスキップ。
- **マスク拡張はスループットでも重要。** 小さいマスクはU-Netフォワードパスほとんどが無駄（マスクされていないピクセルはとにかく固定）。`diffusers`の`StableDiffusionInpaintPipeline`は完全U-Netを実行；9チャネル適切インペイントバリアントのみマスクされた計算を利用。
- **Flux-Kontextは2025年の答え。** `(ソース画像、指示)`上単一フォワードパス — 別別マスク、SDEdit ノイズスイープなし。H100では、約1.5秒でエディタを配布。アーキテクチャレッスン：ステージを折りたたむ。
