# 拡散モデル — DDPM from Scratch

> Ho、Jain、Abbeel (2020) 分野がやめることができないレシピを与えた。ノイズを持つデータを破壊、数千の小さいステップの上。1つのニューラルネットをトレーニングしてノイズを予測。推論でプロセスを逆転。今日のすべての主流イメージ、ビデオ、3D、および音楽モデルはこのループの上で走り、おそらくフロー マッチングまたは一貫性トリックが上のとき。

**タイプ:** ビルド
**言語:** Python
**前提条件:** Phase 3 · 02 (Backprop)、Phase 8 · 02 (VAE)
**所要時間:** ~75分

## 問題

あなたはサンプラーのためにを欲する `p_data(x)`。GANs minimax ゲームを遊び、それはしばしば発散。VAEsはガウスデコーダからぼやけたサンプルを生成。あなたが実際に欲するものは訓練目的、それは(a)単一安定損失(鞍ポイント、minimax なし)、(b) `log p(x)` への下限(ので、あなたは尤度)、および(c)サンプル、それはMark品質と一致。

Sohl-Dickstein et al. (2015) 理論的な回答を持った: マルコフ鎖 `q(x_t | x_{t-1})` をガウスノイズを徐々に追加で定義、そして逆チェーン `p_θ(x_{t-1} | x_t)` をdenoiseをトレーニング。Ho、Jain、Abbeel (2020) 損失を1行に単純化できることを示した — ノイズを予測 — そしてマスを片付けた。2020では、これは好奇。2021では、イメージを生成-最先端。2022では、Stable Diffusionになった。2026では、これは基質。

## コンセプト

![DDPM: ノイズを前に、逆にdenoiseを後に](../assets/ddpm.svg)

**前方プロセス `q`。** `T` 小さいステップの上でガウスノイズを追加。閉鎖-形 — 理由マスが扱いやすい — 累積ステップはまたガウス:

```
q(x_t | x_0) = N( sqrt(α̅_t) · x_0,  (1 - α̅_t) · I )
```

ここで `α̅_t = ∏_{s=1..t} (1 - β_s)` `β_t` のスケジュール。ピック `β_t` 1e-4から0.02への線形以上T=1000ステップおよび `x_T` はおおよそ `N(0, I)`。

**逆プロセス `p_θ`。** ニューラルネットをトレーニング `ε_θ(x_t, t)` ノイズを予測されました追加。与えられた `x_t`、denoiseによって:

```
x_{t-1} = (1 / sqrt(α_t)) · ( x_t - (β_t / sqrt(1 - α̅_t)) · ε_θ(x_t, t) )  +  σ_t · z
```

ここで `σ_t` は `sqrt(β_t)` または学習分散。式は醜いがそれは単に代数 — ポスト `x_{t-1}` を与える `q(x_{t-1} | x_t, x_0)` および代用 `x_0` そのノイズ-予想推定で解く。

**訓練損失。**

```
L_simple = E_{x_0, t, ε} [ || ε - ε_θ( sqrt(α̅_t) · x_0 + sqrt(1 - α̅_t) · ε,  t ) ||² ]
```

サンプル `x_0` データから、ピック、ランダム `t`、サンプル `ε ~ N(0, I)`、noisy `x_t` を計算、1ショットを通じた閉鎖-形、及びノイズの上に帰す。1つの損失、minimax なし、KL なし、再パラメータ化トリックなし。

**サンプリング。** スタート `x_T ~ N(0, I)`。 `t = T` から `1` へ逆ステップを反復。完了。

## なぜ作動するか

3つ直観:

1. **Denoising は簡単; 生成は難しい。** `t=T` では、データは純粋ノイズ — ネットは自明の問題を解く。`t=0` では、ネットはいくつかピクセルだけをきれいにする必要。中間 `t` では、問題は難しいがネットはすべてのノイズレベル同じ重みを通じた多くの勾配。

2. **スコア マッチング変装で。** Vincent (2011)証明ノイズを予測することは推定のと同等 `∇_x log q(x_t | x_0)`、*スコア*。逆SDE はこのスコアを使用、密度勾配の上る歩行 — 高-確率地域への有向ランダムウォーク。

3. **ELBO単純MSEに削減。** フル変分下限はKLタームを持つタイムステップあたり。DDPM のパラメータ化と、それらKLタームはノイズ予想特定係数にMSEに単純化; Ho係数をドロップしました(「シンプル」損失を呼び)、そして品質*改良*。

## ビルドする

`code/main.py` は1-D DDPMを実装。データはつ-モード混合。「ネット」は小さいMLPを取る `(x_t, t)` ノイズを予想する出力。訓練は1-行損失。サンプリングは逆チェーンを反復。

### ステップ 1: 前方スケジュール(閉鎖-形)

```python
betas = [1e-4 + (0.02 - 1e-4) * t / (T - 1) for t in range(T)]
alphas = [1 - b for b in betas]
alpha_bars = []
cum = 1.0
for a in alphas:
    cum *= a
    alpha_bars.append(cum)
```

### ステップ 2: サンプル `x_t` 1ショット

```python
def forward_sample(x0, t, alpha_bars, rng):
    a_bar = alpha_bars[t]
    eps = rng.gauss(0, 1)
    x_t = math.sqrt(a_bar) * x0 + math.sqrt(1 - a_bar) * eps
    return x_t, eps
```

### ステップ 3: 1つ訓練ステップ

```python
def train_step(x0, model, alpha_bars, rng):
    t = rng.randrange(T)
    x_t, eps = forward_sample(x0, t, alpha_bars, rng)
    eps_hat = model_forward(model, x_t, t)
    loss = (eps - eps_hat) ** 2
    return loss, gradient_step(model, ...)
```

### ステップ 4: 逆サンプリング

```python
def sample(model, alpha_bars, T, rng):
    x = rng.gauss(0, 1)
    for t in range(T - 1, -1, -1):
        eps_hat = model_forward(model, x, t)
        beta_t = 1 - alphas[t]
        x = (x - beta_t / math.sqrt(1 - alpha_bars[t]) * eps_hat) / math.sqrt(alphas[t])
        if t > 0:
            x += math.sqrt(beta_t) * rng.gauss(0, 1)
    return x
```

1-D問題のために、40タイムステップ及び24ユニットMLPで、これはつ-モード混合を~200エポック上で学習。

## タイム条件付け

ネットはどのタイムステップがdenoiseするかを知る必要。2つ標準オプション:

- **Sinusoidal埋め込み。** トランスフォーマー位置符号化のような。`embed(t) = [sin(t/ω_0), cos(t/ω_0), sin(t/ω_1), ...]`。MLPを通じて、ネットに放送。
- **Film / グループ-正規化条件付け。** 埋め込み投影層ごとチャネル-ごとスケール/バイアスへ(FiLM)。

おもちゃコードはsinusoidal → concat。本番U-NetsはFiLM。

## 落とし穴

- **スケジュールは多くを大事。** 線形 `β` は DDPMデフォルトであるがコサイン スケジュール(Nichol & Dhariwal、2021)は同じ計算のためのより良いFIDを与える。品質がプラトーの場合、スケジュール切り替え。
- **Timestep埋め込みは脆い。** 生ラッシュで許さ `t` 浮動小数点のためのおもちゃ1-D動作するが失敗イメージ; 常に適切な埋め込みを使用。
- **V-予想対ε-予想。** 狭いレジームのため(非常に小さいまたは非常に大きい t)、`ε` は悪い信号-to-ノイズ。V-予想(`v = α·ε - σ·x`)はより安定; SDXL、SD3、およびFluxはそれを使用。
- **分類器-フリーガイダンス。** 推論で、計算両条件付けおよび無条件 `ε`、その後 `ε_cfg = (1 + w) · ε_cond - w · ε_uncond` `w ≈ 3-7` を持つ。Lesson 08でカバー。
- **1000ステップはたくさんである。** 本番 DDIM を使用(20-50ステップ)、DPM-Solver (10-20ステップ)、またはド蒸留(1-4ステップ)。Lesson 12を見よ。

## 演習

1. **簡単。** チェンジ T から40 まで `code/main.py` で10に。サンプル品質(出力の視覚的ヒストグラム)はどのくらい低下するか? `T` どのカウント2つ-モード構造は崩壊するか?
2. **中程度。** ε-予想からV-予想に切り替え。逆ステップを再-導出。最終的なサンプル品質を比較。
3. **難しい。** 分類器-フリーガイダンスを追加。クラスラベル `c ∈ {0, 1}` で条件付け、訓練の間に10%のそれを削除、そしてサンプリング時間で `ε = (1+w)·ε_cond - w·ε_uncond` を使用。条件付け-モード-ヒット率を測定 `w = 0, 1, 3, 7` で。

## 重要な用語

| 用語 | 人々が言う | 実際の意味 |
|------|-----------------|-----------------------|
| 前方プロセス | 「ノイズを追加」 | 固定マルコフ鎖 `q(x_t \| x_{t-1})` データを破壊。 |
| 逆プロセス | 「Denoising」 | 学習チェーン `p_θ(x_{t-1} \| x_t)` データを再構成。 |
| β スケジュール | 「ノイズはしご」 | ステップ毎分散; 線形、コサイン、またはシグモイド。 |
| α̅ | 「アルファバー」 | 累積製品 `∏(1 - β)`; 閉鎖-形を与える `x_t` から `x_0`。 |
| シンプル損失 | 「ノイズでのMSE」 | `\|\|ε - ε_θ(x_t, t)\|\|²`; すべての変分導出は崩壊これにする。 |
| ε-予想 | 「ノイズを予測」 | 出力はノイズ追加; スタンダードDDPM。 |
| V-予想 | 「速度を予測」 | 出力は `α·ε - σ·x`; より優れた条件付けアクロス t。 |
| DDPM | 「紙」 | Ho et al. 2020; 線形β、1000ステップ、U-Net。 |
| DDIM | 「決定論的サンプラー」 | 非-マルコフサンプラー、20-50ステップ、同じ訓練目的。 |
| 分類器-フリーガイダンス | 「CFG」 | 条件付けを増幅するために混ぜる条件付けおよび無条件ノイズ予想。 |

## 本番ノート: 拡散推論はステップ-数問題

DDPMペーパーはT=1000逆ステップを走らせ。誰もそれを本番に配布しない。本番推論スタックすべてはピック1つ、3つ戦略のうち — そして各は本番フレーミングの「レイテンシはどこから来るか」に清潔にマップ:

1. **より速いサンプラー、同じモデル。** DDIM (20-50ステップ)、DPM-Solver++ (10-20)、UniPC (8-16)。逆ループのドロップ-置き換え; 訓練された `ε_θ` 重みは手つかず。レイテンシ 20-50倍をドロップ。
2. **蒸留。** 学習学生がより少ないステップで教師にマッチするか: 段階的蒸留(2 → 1)、一貫性モデル(恣意的 → 1-4)、LCM、SDXL-Turbo、SD3-Turbo。別から別にレイテンシ5-10倍、訓練を必要とする。
3. **キャッシュおよびコンパイル。** `torch.compile(unet, mode="reduce-overhead")`、TensorRT-LLM の拡散バックエンド、`xformers`/SDPAアテンション、bf16重み。ステップあたりをドロップレイテンシ~2倍。積み重なる(1)および(2)。

本番拡散サーバーのため、予算対話は本番文献がLLMs のために述べるもののと同じ: レイテンシは `num_steps × step_cost + VAE_decode` であり、スループットは `batch_size × (num_steps × step_cost)^-1`。TTFTは小さい(1ステップ); TPOT-同等は完全応答時間、イメージ生成は「全-at-当たり」ユーザーパースペクティブ。
