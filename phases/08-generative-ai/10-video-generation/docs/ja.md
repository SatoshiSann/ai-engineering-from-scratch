# ビデオ生成

> 画像は2-Dテンソル。ビデオは3-Dテンソル。理論は同じ；計算は10-100倍難しい。OpenAIのSora（2月2024）は可能であることを証明。2026年にVeo 2、Kling 1.5、Runway Gen-3、Pika 2.0、およびWAN 2.2は1080pテキストから本番ビデオを配布 — またオープンウェイト スタック（CogVideoX、HunyuanVideo、Mochi-1、WAN 2.2）は12か月遅れている。

**タイプ:** Build
**言語:** Python
**前提条件:** Phase 8 · 07 (潜在拡散)、Phase 7 · 09 (ViT)、Phase 8 · 06 (DDPM)
**所要時間:** 約45分

## 問題

24fpsで10秒1080pビデオは1920×1080×3ピクセルの240フレーム。それは約1.5 GBクリップあたり生データ。ピクセル空間拡散は実行不可能。必要：

1. **時空圧縮。** フレーム内にビデオをエンコードするVAE、空間時間パッチの序列に。
2. **時間的コヒーレンス。** フレームはコンテンツ、照明、秒以上のオブジェクト身元を共有する必要。ネットは動き を モデル化する必要がある。
3. **計算予算。** ビデオ訓練は同じモデルサイズで画像より10-100倍高い。
4. **条件付け。** テキスト、画像（第1フレーム）、オーディオ、またはビデオ。ほとんどの本番モデルはすべて4つを受け入れる。

このアーキテクチャを解く **拡散トランスフォーマー（DiT）** 時空パッチに適用、大規模（プロンプト、キャプション、ビデオ）データセットで訓練。レッスン06と同じ拡散損失。

### パッチ化

ビデオを3D VAE（学習時空圧縮）でエンコード。潜在は形状`[T_潜在, H_潜在, W_潜在, C_潜在]`。`[t_p, h_p, w_p]`サイズのパッチに分割。Sora-スタイルモデルの場合、`t_p = 1`（フレームあたりパッチ）または`t_p = 2`（2フレームごと）。10秒1080p ビデオは約20,000-100,000パッチに圧縮。

### 時空DiT

トランスフォーマーはパッチの平坦序列を処理。各パッチは3D位置埋め込み（時間+y+x）。アテンションは通常因子分解：

- **空間アテンション**各フレームのパッチ内。
- **時間アテンション**同じ空間位置でフレーム全体。
- **完全3Dアテンション**16-100倍高い；低解像度または研究でのみ使用。

### テキスト条件付け

大規模テキストエンコーダとのクロスアテンション（SoraのT5-XXL、CogVideoX-5BはT5-XXLを使用）。長いプロンプトが重要 — Soraの訓練セットはGPT生成密再キャプション平均クリップあたり200トークンを持っていた。

### 訓練

標準拡散損失（ε または v 予測）時空潜在以上。データ：ウェブビデオ+約100Mキュレートされたクリップ+合成テキストキャプション。計算：さらに小さい研究実行でも10,000+ GPU時間；Sora-スケールは100,000+。

## ビルドしよう

`code/main.py`コア時空DiT思想をシミュレート：小さい合成ビデオをパッチ化、フレームごとポジション埋め込み追加、パッチ上で変換器スタイル アテンションで全体序列をデノイズ。NumPy なし；純粋Python。隣接フレーム パッチが共有デノイザーおよびポジション埋め込みを共有する時、1-Dでも時間的コヒーレンスが出現することを示す。

### ステップ1：小さい合成1-D「ビデオ」をパッチ化

```python
def make_video(T_frames=8, rng=None):
    # 「ビデオ」は滑らかな軌跡に従う1-D値の序列
    base = rng.gauss(0, 1)
    return [base + 0.3 * t + rng.gauss(0, 0.1) for t in range(T_frames)]
```

### ステップ2：フレームあたりポジション埋め込み


### ステップ3：デノイザーが全体序列を見る

各フレーム独立をデノイズの代わりに、その小さいネット連結すべてフレーム値+それらのポジション埋め込み予測ノイズすべてのフレーム共同。

### ステップ4：時間的コヒーレンステスト

訓練後、ビデオをサンプル。フレーム間デルタを測定。モデルが時間構造を学習している場合、デルタは各フレーム独立サンプリングより小さいまま。

## ピットフォール

- **フレームごと独立サンプリング = フリッカー。** 各フレーム別に画像拡散を実行する場合、ノイズが各フレーム独立なので、出力フリッカー。ビデオ拡散はフレーム通じアテンション対またはノイズ共有結合して修正。
- **素朴な3Dアテンション = OOM。** 10秒1080pレイテント全体3Dアテンションは、数百億操作。空間+時間に因子分解。
- **データキャプション、サイズより重要。** Soraの主なアップグレード以前の作業は訓練約10倍詳細キャプション（GPT-4再ラベルクリップ）。OpenAIの技術報告はこれについて明示的。
- **第1フレーム条件付け。** ほとんどの本番モデルはまた画像を第1フレームとして受け入れる。これは「画像ビデオ」モード；訓練このバリアント含む。
- **物理ドリフト。** 長いクリップ(>10秒)微妙な不一貫性蓄積。スライディングウィンドウ生成+キーフレームアンカリング助け。

## 演習

1. **簡単。** `code/main.py`で、フレーム間デルタを比較（a）独立フレームサンプリング、（b）共同序列サンプリング。デルタの平均と分散を報告。
2. **中程度。** 第1フレーム条件を追加：フレーム0を所与値にピン、残りをサンプル。ピン値がどう伝搬するか測定。
3. **難しい。** HuggingFace diffusersを使用してローカルGPUでCogVideoX-2Bを実行。720p6秒クリップで20推論ステップをプロファイル。時空アテンションをプロファイル、ボトルネック識別。

## 本番注：ビデオレイテントはメモリ帯域幅問題

24fpsで10秒1080pクリップは240フレーム×1920×1080×3 ≈ 1.5GB生ピクセル。4倍ビデオVAE圧縮（`2 × 空間 × 2 × 時間`）後、レイテントはリクエストあたり約100 MB。バッチ1で30ステップ通じ時空DiT実行、HBMで約3 GB/ステップを移動 — メモリ帯域幅、FLOPSでない、ボトルネック。

3本番ノブ、すべて直接本番推論文献推論章からリストされたレシピ：

- **DiT全体TP。** テキストからビデオモデルは常に≥10Bパラメータ。4つのH100全体TP=4は標準；405Bクラスモデル PP=2 × TP=2。ステップあたりレイテンシはTP までほぼ線形降下、all-reduce壁まで。
- **フレームバッチング=継続的バッチ処理。** 生成時、ビデオはアテンション結合されたフレームの概念的バッチ。継続的バッチング（飛行中スケジューリング）適用：フレーム`t+1`レンダリング開始、フレーム`t-1`返却中、モデルアーキテクチャスライディング窓生成許す場合。
- **クリップレベル prefill キャッシュ。** 画像からビデオの場合、第1フレーム条件付けはLLMのプロンプト prefill に類似：1度計算、時間デコーダパス全体を再利用。これは効果的にビデオのKV-キャッシュ。
