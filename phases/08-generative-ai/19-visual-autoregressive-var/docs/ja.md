# ビジュアル自動回帰モデリング (VAR): 次スケール予測

> 拡散モデルは時間的に反復的にサンプリングします (ノイズ除去ステップ)。VAR はスケール的に反復的にサンプリングします — 1x1 トークンを予測し、次に 2x2、その後 4x4、最終解像度まで、各スケールが前のスケールに条件付きます。2024年のペーパーは、VAR が画像生成の GPT スタイルのスケーリング則にマッチし、同じ計算予算で DiT を上回ることを示しました。このレッスンではコアメカニズムを構築します。

**タイプ:** ビルド
**言語:** Python (PyTorch 付き)
**前提条件:** Phase 7 Lesson 03 (マルチヘッドアテンション)、Phase 8 Lesson 06 (DDPM)
**所要時間:** 約 90 分

## 問題

自動回帰生成は言語モデリングを支配してきました。スケーリングが予測可能だからです。より多くの計算、より多くパラメータ、より低いパープレキシティ、より良い出力。2024 年より前の画像生成には、主に 2 つの AR 試みがありました: PixelRNN/PixelCNN (ピクセルごと) と DALL-E 1 / Parti / MuseGAN (VQ-VAE コードのトークンごと)。

どちらも生成順序の問題に苦しんでいました。ピクセルとトークンは 2D グリッドに配置されていますが、AR モデルは 1D ラスター順でそれらを訪問する必要があります。早い隅のピクセルは、画像が最終的にどのようになるかについて何も知りません。生成品質は GPT-on-text よりも悪くスケーリングし、決して一致した計算での拡散モデル品質に達しませんでした。

VAR は、生成されるものを変えることで生成順序の問題を修正します。画像トークンを空間的に 1 つずつ予測する代わりに、VAR は増加する解像度で画像全体を予測します。ステップ 1: 1x1 トークン (画像全体の「要約」) を予測します。ステップ 2: トークンの 2x2 グリッド (粗い特徴) を予測します。ステップ 3: 4x4 グリッドを予測します。ステップ K: 最終 (H/8)x(W/8) グリッドを予測します。

各スケールは、すべての前スケール (「スケール順」で因果的) に注意を払い、それ自身のスケール内では並列に注意を払います。順序の問題は消えます: スケール k での完全な画像は、1 つのトランスフォーマー パスで生成されます。

## コンセプト

### VQ-VAE マルチスケール トークナイザー

VAR は **マルチスケール離散トークナイザー** が必要です。画像 x に対して、段階的に高解像度のトークングリッドのシーケンスを生成します:

```
x -> encoder -> latent f
f -> tokenize at 1x1: token grid z_1 of shape (1, 1)
f -> tokenize at 2x2: token grid z_2 of shape (2, 2)
...
f -> tokenize at (H/p)x(W/p): token grid z_K of shape (H/p, W/p)
```

各 z_k は同じコードブック (典型的なサイズ 4096-16384) を使用します。各スケールでのトークン化は独立していません — 各スケールの残差を合計すると f を再構成するように学習されます:

```
f ≈ upsample(embed(z_1), target_size) + ... + upsample(embed(z_K), target_size)
```

これは **残差 VQ** バリアントです。スケール k は、スケール 1..k-1 が見落とした部分をキャプチャします。デコーダはすべてのスケール埋め込みの合計を取り、画像を生成します。

マルチスケール VQ トークナイザーは一度学習され (VQGAN のように)、その後凍結されます。すべての生成作業は上部の自動回帰モデルによって行われます。

### 次スケール予測

生成モデルは、すべての前スケールからのトークンを見て、次のスケールでのトークンを予測するトランスフォーマーです。

入力シーケンス構造:
```
[START, z_1 tokens, z_2 tokens, z_3 tokens, ..., z_K tokens]
```

位置埋め込みは、スケール インデックスとスケール内の空間位置の両方をエンコードします。アテンションはスケール順で因果的です: スケール k、位置 (i, j) のトークンは、スケール 1..k のすべてのトークンに注意を払うことができ、スケール k 自体のトークンで使用されるスケール内順序で以前に来たトークンに注意を払うことができます (VAR はスケール内因果性なしで固定位置アテンションを使用します — スケール内のすべての位置は並列に予測されます)。

訓練損失: 各スケール k で、すべての前スケール トークンが与えられた場合、トークン z_k を予測します。離散 VQ コード上の交差エントロピー損失。GPT と同じ構造ですが、「シーケンス」がスケール構造です。

### 生成

推論時:
```
generate z_1 = sample from p(z_1)                    # 1 token
generate z_2 = sample from p(z_2 | z_1)              # 4 tokens in parallel
generate z_3 = sample from p(z_3 | z_1, z_2)         # 16 tokens in parallel
...
decode: f = sum of embed-and-upsample scales 1..K
image = VAE_decoder(f)
```

K = 10 スケールの場合、生成は 10 トランスフォーマー フォワード パスです。各パスは全スケールを並列で生成します — スケール内でのトークンごとの自動回帰はありません。256x256 画像の場合、これは DiT の 28-50 に対して大約 10 パスです。

### なぜ次スケールが次トークンより勝つのか

3 つの構造的勝利:

1. **粗から細へが自然な画像統計に整合する。** 人間の視覚知覚と画像データセットはどちらも、スケール依存の正則性を示します: 低周波構造は安定で予測可能です。高周波の詳細は低周波の内容に条件付きです。次スケール予測はこれを利用します。
2. **スケール内の並列生成。** GPT スタイルのトークン AR とは異なり、VAR はスケール内のすべてのトークンを 1 ステップで生成します。有効生成長は線形ではなく対数スケールです。
3. **生成順序バイアスなし。** スケール k のトークンはスケール k-1 のすべてを見ます。後期コンテキストが利用可能になる前にコミットするように早期トークンを強制する「左」または「上」バイアスはありません。

### スケーリング則

Tian ら は、VAR が ImageNet での FID のべき法則スケーリング曲線に従うことを示しました — GPT がパープレキシティに対して行うのと同じです。パラメータまたは計算を 2 倍にすると、エラーが確実に半分になります。これは言語モデルと同じくらい洗練されてスケーリング動作を示す最初の画像生成モデルでした。結果は、VAR スケール予測が計算から予測可能になり、アーキテクチャごとの経験的推測ではなくなることです。

### 拡散との関係

VAR と拡散は同じデータ圧縮の物語を共有します: どちらも生成問題をより簡単な部分問題のシーケンスに分割します。

- 拡散: 段階的にノイズを追加し、1 つのステップを元に戻すことを学びます。
- VAR: 段階的に解像度を追加し、次のスケールを予測することを学びます。

それらは問題を通じた異なる軸です。どちらも扱いやすい条件付き分布を生成します。経験的に VAR は推論で高速です (より少ないパス、スケール内すべて並列) し、DiT on class-conditional ImageNet にマッチするかそれを上回ります。テキスト条件付き VAR (VARclip、HART) はアクティブな研究方向です。

## ビルド

`code/main.py` では:
1. 合成「画像」データ (2D ガウス環) 上に小さな **マルチスケール VQ トークナイザー** を構築します。
2. **VAR スタイルのトランスフォーマー** を次スケール予測にトレーニングします。
3. トランスフォーマーを 4 回呼び出す (4 スケール) ことでサンプリングしてデコードします。
4. スケール順訓練がスケール内で生成を並列にすることを確認します。

これはおもちゃの実装です。ポイントは、スケール構造化アテンションマスクと実際に動作する並列スケール内生成を見ることです。

## シップ

このレッスンは `outputs/skill-var-tokenizer-designer.md` を生成します — マルチスケール トークナイザーを設計するためのスキル: スケール数、スケール比、コードブック サイズ、残差共有、デコーダ アーキテクチャ。

## 演習

1. **スケール数アブレーション。** 4、6、8、10 スケールで VAR をトレーニングします。自動回帰パスの数と比較した再構成品質を測定します。より多くのスケール = より細い残差 = より良い品質だがより多くのパス。

2. **コードブック サイズ。** コードブック サイズ 512、4096、16384 でトークナイザーをトレーニングします。より大きなコードブックはより良い再構成を提供しますが、予測がより難しくなります。膝を見つけます。

3. **スケール内並列チェック。** トレーニングされた VAR に対して、アテンション パターンを明示的に測定します。スケール k 内では、モデルがスケール内位置に注意を払わずにクロススケール位置に注意を払いますか? マスク実装を確認します。

4. **VAR vs DiT スケーリング。** 同じ ImageNet クラス条件付きタスクに対して、一致したパラメータ予算 (例: 33M、130M、458M) で VAR と DiT をトレーニングします。FID 対計算をプロットします。VAR は各サイズで DiT を上回る必要があります — 小規模でペーパーの結果を再現します。

5. **テキスト条件付け。** テキスト埋め込み (CLIP プール) を取得するように VAR を拡張し、adaLN を介して追加の条件付き入力として使用します。これは HART レシピです。テキスト整列サンプリングで FID がどのくらい改善されますか?

## キー用語

| 用語 | 人が言うこと | 実際の意味 |
|------|------------|--------|
| VAR | "Visual AutoRegressive" | VQ トークングリッドのピラミッド上で次スケール予測による画像生成 |
| Next-scale prediction | "粗を予測してから細を予測" | モデルは増加する解像度スケールでトークンを予測し、すべての前スケールに条件付き |
| Multi-scale VQ tokenizer | "Residual VQ" | 増加する解像度の K トークングリッドを生成し、デコーダがすべてのスケールを合計する VQ-VAE |
| Scale k | "Pyramid level k" | K 解像度レベルの 1 つで、k=1 で 1x1 から k=K で (H/p)x(W/p) まで |
| Parallel-within-scale | "1 フォワード/スケール" | スケール k のすべてのトークンは 1 つのトランスフォーマー パスで予測され、自動回帰ではありません |
| Causal-across-scales | "Scale-ordered attention" | スケール k のトークンはスケール 1..k のすべてに注意を払うことができますがスケール k+1..K には払えません |
| Residual VQ | "Additive tokenization" | 各スケールのトークンは下位スケールで残された残差をエンコードします。デコーダはすべてのスケール埋め込みを合計 |
| VAR scaling law | "Image GPT scaling" | FID は計算の予測可能なべき法則に従い、言語モデルのパープレキシティのように |
| HART | "Hybrid VAR + text" | MaskGIT スタイルの反復デコードと VAR のスケール構造を組み合わせるテキスト条件付き VAR バリアント |
| Scale position embedding | "(scale, row, col) triple" | 位置エンコーディングはスケール インデックスと空間座標の両方をスケール内で実行 |
