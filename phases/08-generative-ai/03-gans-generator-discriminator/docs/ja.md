# GAN — 生成器対判別器

> Goodfellowの2014年のトリックは密度を全くスキップするしていた。2つのネットワーク。1つは偽物を作る。1つはそれらをキャッチ。それら偽物が本物と区別不可能になるまで戦う。それは作動しない。それはしばしば作動しない。それが作動する場合、サンプルは依然として狭いドメインの最先端である。

**タイプ:** ビルド
**言語:** Python
**前提条件:** Phase 3 · 02 (Backprop)、Phase 3 · 08 (Optimizers)、Phase 8 · 02 (VAE)
**所要時間:** ~75分

## 問題

VAEはサンプルをぼやかしている、なぜならそれらのMSEデコーダ損失は*平均*イメージに対してベイズ最適 — そして多くの尤もらしいイメージの平均は模糊たイメージ。あなたは*尤もらしさ*を報酬する損失を欲する、ピクセル単位の接近性ではなく、どのターゲットをも。尤もらしさのために閉鎖形がない。あなたはそれを学習する必要。

Goodfellowのアイデア: 分類ツール `D(x)` をビジュアル本物から偽物を区別するように訓練、生成器 `G(z)` を `D` を愚かす。`G` の損失信号は何 `D` がまさに本物のように見えるもの考えるもの。この信号は `G` が改良するとき更新、移動ターゲットを追跡。両ネットワークが収束する場合、`G` は `log p(x)` を決して書き下さないまま、データ分布を学習している。

これは敵対的訓練。数学は最小-最大ゲーム:

```
min_G max_D  E_real[log D(x)] + E_fake[log(1 - D(G(z)))]
```

2026年GANはもはや SOTA 生成ツール(拡散とフロー マッチングはその王冠を食べた)でない。しかしStyleGAN 2/3は依然として最も鋭いフェイスモデルを配布した、GAN判別ツールは*知覚的損失*として拡散訓練で使用される、そして敵対的訓練は高速1-ステップ蒸留(SDXL-Turbo、SD3-Turbo、LCM)のパワーは実時間拡散を配布できるあなた。

## コンセプト

![GAN訓練: 最小-最大の生成器と判別器](../assets/gan.svg)

**生成器 `G(z)`。** ノイズベクトル `z ~ N(0, I)` をサンプルにマップ。デコーダ形状ネットワーク(密または転置conv)。

**判別器 `D(x)`。** サンプルをスカラー確率(またはスコア)にマップ。本物 → 1、偽物 → 0。

**損失。** 2つ交互の更新:

- **訓練 `D`:** `loss_D = -[ log D(x) + log(1 - D(G(z))) ]`。本物=1、偽物=0に関するバイナリ・クロス-エントロピー。
- **訓練 `G`:** `loss_G = -log D(G(z))`。これは*非飽和*形式Goodfellowは使用した(オリジナル `log(1 - D(G(z)))` は飽和し、`D` 確信時に勾配を殺す)。

**訓練ループ。** `D` の1つのステップ、`G` の1つのステップ。繰り返し。

**なぜ作動するか。** `G` が完全に `p_data` にマッチする場合、`D` はチャンスより優れることができず、0.5すべてで出力; `G` はもはや勾配を取得。平衡。

**なぜ破れるか。** モード崩壊(`G` 1つモード `D` 分類できないを見つけて永遠にそれを鋳造)、消滅勾配(`D` 学習が速すぎて `log D` は飽和)、訓練不安定(学習率、バッチサイズ、すべて)。

## GANが作動させた変異

| 年 | イノベーション | 修正 |
|------|------------|-----|
| 2015 | DCGAN | Conv/deconv、バッチ正規化、LeakyReLU — 最初の安定アーキテクチャ。 |
| 2017 | WGAN、WGAN-GP | BCE をWasserstein距離 + 勾配ペナルティで置き換え。消滅勾配を修正。 |
| 2017 | スペクトラル正規化 | 判別ツールをLipschitz-バウンド。依然として2026年の判別ツールで使用。 |
| 2018 | Progressive GAN | 低解像度で訓練、層を追加。最初のメガピクセル結果。 |
| 2019 | StyleGAN / StyleGAN2 | マッピング ネットワーク + 適応インスタンス正規化。固定-ドメイン写真実在性 SOTA。 |
| 2021 | StyleGAN3 | エイリアス-フリー、翻訳-同変 — 依然として2026年フェイスゴールドスタンダード。 |
| 2022 | StyleGAN-XL | 条件付け、クラス-認識、大きいスケール。 |
| 2024 | R3GAN | より強い正則化でリブランド; 1024² でトリックなし作動。 |

## ビルドする

`code/main.py` は1-Dデータ上の小さいGANを訓練: 2つのガウス混合。生成器と判別ツールは単一の隠し層MLP。順伝播、逆伝播、および最小-最大ループを手書き。目標は2つのキー失敗モード(モード崩壊 + 消滅勾配)が生じるのを見る。

### ステップ 1: 非飽和損失

Vanillaグッドフェローの損失 `log(1 - D(G(z)))` はD が高確信でG の偽物を分類する場合0に行く。その時点で `G` のための勾配は基本的にゼロ — `G` は改良できない。非飽和形式 `-log D(G(z))` は相対的な漸近: これが`D` 確信する場合吹く、`G` 強い信号を与える。

```python
def g_loss(d_fake):
    # maximize log D(G(z))  <=>  minimize -log D(G(z))
    return -sum(math.log(max(p, 1e-8)) for p in d_fake) / len(d_fake)
```

### ステップ 2: 1つの判別ツール ステップ毎生成器ステップ

```python
for step in range(steps):
    # train D
    real_batch = sample_real(batch_size)
    fake_batch = [G(z) for z in sample_noise(batch_size)]
    update_D(real_batch, fake_batch)

    # train G
    fake_batch = [G(z) for z in sample_noise(batch_size)]  # fresh fakes
    update_G(fake_batch)
```

`G` のための新しい偽物、さもなければ勾配は古い。

### ステップ 3: モード崩壊を監視

```python
if step % 200 == 0:
    samples = [G(z) for z in sample_noise(500)]
    mode_a = sum(1 for s in samples if s < 0)
    mode_b = 500 - mode_a
    if min(mode_a, mode_b) < 50:
        print("  [!] mode collapse: one mode is starved")
```

標準的な症状: リアルモードの1つが生成を停止。判別ツールはそれを偽物として見ることはないため、停止訂正。

## 落とし穴

- **判別ツール太強。** `D` の学習率を2-5x削減、またはインスタンス/層ノイズを追加。`D` が >95% 精度に到達する場合、`G` は死ぬ。
- **生成器はモードを暗記。** ノイズを `D` 入力に追加、minibatch-判別層を使用、またはWGAN-GPに切り替え。
- **バッチ正規化統計を漏らす。** 本物バッチ + 偽物バッチが同じBN層を流れるが、それらの統計を混ぜる。代わりにインスタンス正規化またはスペクトラル正規化を使用。
- **Inception-スコア ゲーミング。** FIDおよびISは低サンプル数でノイズ。≥10k サンプルで評価で使用。
- **ワンショット サンプリングは条件付けタスクのための嘘。** あなたは依然としてCFGスケール、切詰めトリック、および使用可能出力の再-サンプリング必要。

## 使用

2026年GANスタック:

| 状況 | ピック |
|-----------|------|
| 写真実在的人間フェイス、固定ポーズ | StyleGAN3 (最も鋭い、最小) |
| アニメ / 様式化されたフェイス | StyleGAN-XLまたはStable Diffusion LoRA |
| イメージ-to-イメージ翻訳 | Pix2Pix / CycleGAN (Phase 8 · 04) またはControlNet (Phase 8 · 08) |
| 高速1-ステップテキスト-to-イメージ | 拡散の敵対的蒸留(SDXL-Turbo、SD3-Turbo) |
| 拡散訓練内部での知覚的損失 | イメージクロップでの小さいGAN判別ツール |
| すべてマルチ-モーダル、オープンエンド | しない — 拡散またはフロー マッチングを使用 |

GANは鋭いがナロー。ドメインがオープンになる場合 — フォト、恣意的テキストプロンプト、ビデオ — 拡散に切り替え。敵対的トリックはコンポーネントで生きる(知覚的損失、蒸留)、スタンドアロン生成ツール。

## 演習

1. **簡単。** `code/main.py` をストック設定で実行。それからsetで `D_LR = 5 * G_LR` となりつつ再実行。`G` の損失は多くどのくらい速く崩壊するか?
2. **中程度。** Goodfellow BCE損失をWGAN損失で置き換え: `loss_D = E[D(fake)] - E[D(real)]`、`loss_G = -E[D(fake)]`、およびクリップ `D` の重み `-0.01, 0.01` に。訓練はより安定か? ウォールクロック収束を比較。
3. **難しい。** 1-Dの例を2-Dデータに拡張(リングの8ガウス混合)。ステップ1k、5k、10kで生成ツールが捕捉する8つのモードの多くを追跡。Minibatch判別を実装して再-測定。

## 重要な用語

| 用語 | 人々が言う | 実際の意味 |
|------|-----------------|-----------------------|
| 生成器 | 「G」 | ノイズ-to-サンプル ネットワーク、`G: z → x̂`。 |
| 判別器 | 「D」 | 分類ツール `D: x → [0, 1]`、本物対偽物。 |
| 最小-最大 | 「ゲーム」 | 結合目的の `min_G max_D`。 |
| 非飽和損失 | 「修正」 | `log(1 - D(G(z)))` の代わりに `G` のために `-log D(G(z))` を使用。 |
| モード崩壊 | 「G は1つのもの暗記」 | 生成ツールは少数の独別出力を生成、多様なデータ。 |
| WGAN | 「Wasserstein」 | BCEをEarth-Mover距離 + 勾配ペナルティで置き換え; より滑らかな勾配。 |
| スペクトラル正規化 | 「Lipschitzトリック」 | 傾き結合 `D` の重み正規ノルムを制約; 訓練を安定。 |
| StyleGAN | 「作動するもの」 | マッピング ネットワーク + AdaIN; フェイスに関しては最高-級、依然として2026年。 |

## 本番ノート: ワンショット推論はGANの永続的な利点

GANはもはやオープンドメイン生成で品質に勝ない、しかしそれらは推論コスト上依然として勝つ。本番-推論文献語彙で、GANは:

- **プリフィル、デコードステージなし。** 単一の `G(z)` 順伝播。TTFT ≈ 総レイテンシ。
- **KVキャッシュプレッシャーなし。** 唯一の状態は重み。バッチサイズは活性化メモリ、キャッシュでバウンド。
- **自明な継続バッチ。** すべてのリクエストは同じ固定FLOPsを取るため、サーバーのターゲット占有率での静的バッチは通常最適。飛行中スケジューラが不要。

これがGAN蒸留(SDXL-Turbo、SD3-Turbo、ADD、LCM)が2026年でテキスト-to-イメージ高速の支配的テクニックある理由: 20-50-ステップ拡散パイプラインを1-4つのGAN-スタイル順伝播に崩壊、拡散ベースの分布を保つ。敵対的損失は訓練時間ノブとして生き残る、遅い生成ツールをハイスピード1つに変える。
