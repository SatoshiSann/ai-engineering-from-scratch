# 生成エージェントと顕現シミュレーション

> Park et al。2023年（UIST '23、arXiv:2304.03442）は**Smallville**に、サンドボックス25のエージェント、3部構造を入植：**メモリストリーム**（自然言語ログ）、**反映**（エージェントがそれ自身のストリームについて生成する高次合成）、**計画**（日レベル動作、その後サブ計画）。ランドマーク結果はバレンタインデー党出現：1つのエージェントは「バレンタインデー党を投げたいさらなるスクリプティング24個エージェント始まった無知と出現 — 招待は招待は招待人口を広がる、コーディネート日付、党は起こった — 知識、 from 24エージェント始まった知識。アブレーション全部3つのコンポーネント信頼できる必要。ドキュメント失敗は空間ノルムエラー（入力閉じた店、シェア単一ユーザーバスルーム）。これは2026年エージェントシミュレーション参照アーキテクチャであり、マルチエージェント社会評価。

**タイプ:** Learn + Build  
**言語:** Python (stdlib)  
**前提条件:** Phase 16 · 04 (Primitive Model), Phase 16 · 13 (Shared Memory)  
**所要時間:** 約75分

## 問題

大部分のマルチエージェントシステムは厳密スクリプト化チーム：計画計画、コーダーコード、レビュアーレビュー。これは定義されたタスクに機能。それはメモリがあるとき浮かぶ顕現、無スクリプト動作をキャプチャしない、優先順位、オープンワールド。研究、社会シミュレーション、ますますゲームAIこれ第2タイプを必要とします。

Smallvilleアーキテクチャはそれのベンチマークです。Park 2023までは、最高のエージェントシミュレーションは浅い書き直し従者でした。それ以後、パターンはオープンワールド生成エージェント用の標準デフォルト。2026年でエージェントシミュレーションを構築するならば、Smallvilleの3つのコンポーネントを使用するか、明示的にあなたはなぜではないかを正当化。

## コンセプト

### 3つのコンポーネント

**メモリストリーム。** 観察、行動、反映、計画の追記のみログ。各エントリはタイムスタンプ、タイプ、説明（自然言語）、派生メタデータ：**最新性**、**重要度**（エージェント自己評価1-10）、**関連性**（現在のクエリへの余弦類似性）。

```
[2026-02-14 09:12:03] observation: Isabella Rodriguez asked me if I like jazz
[2026-02-14 09:14:22] reflection:   I enjoy long conversations about music
[2026-02-14 10:05:00] plan:         Attend Isabella's Valentine's Day party tonight
```

メモリ取得は3つのスコアを結合：`score = w_recency * e^(-decay * age) + w_importance * importance + w_relevance * cos_sim`。トップ-kエントリが現在のプロンプトに入る。

**反映。** 定期的に（Nメモリか重要イベント毎）、エージェント生成最近のメモリから高次合成。反映エントリはストリームに戻りおよび任意他のメモリのように取得可能。これはエージェント「理解」をビルド方法です — 建築の長期信念の等価物。

**計画。** トップダウン分解。最初、日レベル計画大きいストローク（「仕事に行く、Klausと夜食」）。その後時間レベル計画。その後アクションレベル計画。計画は修正可能：観察が計画と矛盾する場合、エージェントは影響されるセグメント再計画。

### なぜすべての3つが重要か（アブレーション）

Park et al。観察、反映、計画を落とすアブレーションを実行。各アブレーション信頼性を傷つけ：

- **観察なし**、エージェントはコンテキストを逃し、古い信念に行為。
- **反映なし**、エージェントはより高次信念を形式化できない。相互作用は浅い止まる。
- **計画なし**、行動は反応的ノイズになる。目標は散乱。

人間評価者から信頼性スコアはすべての3つで最高。1つ任意を落とすことは測定可能な後退を生産。

### バレンタインデー出現

1つのエージェント、Isabella Rodriguez、目標でシード「バレンタインデー党をHobbs Cafe2月14日5pm投げたいが」。24個他のエージェントそのようなシードなしに受け取ります。シミュレート日上で：

1. Isabellaの計画は人々を招待するを含む。
2. 各招待は隣人のメモリストリーム内の観察になる。
3. その隣人の反映生成信念：「Isabellaは党を投げる。」
4. 隣人の計画「2月14日党に出席」を組む。
5. 隣人は他の隣人に言う。招待は中心調整なく広がる。
6. 2月14日5pmで、いくつかのエージェントはHobbs Cafeで收束。

これは顕現は技術感度：システムレベル行動（党）がローカル相互作用（二国間招待+個人計画）から中心オーケストレータなし浮かぶ。

### ドキュメント化失敗モード

Park et al。明示的にドキュメント：

- **空間ノルムエラー。** エージェント閉じた店に歩く。エージェント同じシングルユーザーバスルームを使用試みる。エージェント食べる食べないために意図した部屋。モデルはシングル環境からソーシャル物理ノルムを推論しない。
- **メモリオーバーフロー。** ディープシミュレーション実行はメモリ取得コストを増加させ。実用救救：定期メモリコンパクション（要約とプルーン）低重要度エントリに減衰。
- **反映幻覚。** 反映はメモリストリームに存在しない関係を発明できる。軽減：反映プロンプトにソースメモリIDを含める、取得時に検証。

これらは本番関連失敗モード：すべての2026年エージェントシミュレーションはそれらを相続。

### 3部分実装ルール

1. **メモリは追記のみ。** メモリエントリを決して変異させる。修正は新しいエントリ。
2. **重要度スコアは安い。** LLMを呼び出して重要度レート1-10で書き時。スコア。
3. **取得はランク、フィルタされず。** トップ-kで結合スコア。ハードフィルタ（コンテキストを失う）を使用しない。
4. **反映は定期的に実行。** 未処理メモリの重要度合計が閾値（例えば150）を超える場合、トリガー。
5. **計画は修正可能。** 新しい観察が計画と矛盾する場合、セグメント全体でなく再生成受ける。

### Smallville超える生成エージェント

2024-2026年追従文献アーキテクチャを拡張：

- **マルチエージェント社会シミュレーション方針/市場研究。** Smallvilleのような人口はフィーチャー応答にユーザー動作をシミュレート。A/Bテストより速い。精度は論争。
- **ゲーム用NPC AI。** RPGスクリプト化クエストの代わりに顕現物語を生成する。
- **生成エージェント評価ベンチマーク。** タスク精度より、メトリックは信頼できるさらに長時間実行する行動の一貫性になる。

アーキテクチャは参照。拡張は成分交換（メモリ用ベクトルストア、取得増強反映、神経シンボリック計画）しかし3部分構造を保つ。

### なぜこれマルチエージェントエンジニアリングに重要か

Smallvilleはマルチエージェント出現が安い証拠が部品が正しい。アーキテクチャは今オープンソースモデルに複製（小さいLLMは信頼できるを失う鋭くない）。すべての本番システム**顕現社会行動**を必要とするこの形を使用。あらゆるシステムが**タイトなタスク実行**を必要とする初期のスーパーバイザー/ロール/プリミティブパターンを使用。

## ビルド

`code/main.py`は3つのコンポーネントをstdlib Pythonスクリプト化エージェントポリシー実装（本当のLLMなし）。デモはミニチュアでバレンタイン党出現を再現：

- `MemoryStream` — 追記のみログ最新性/重要度/関連性取得。
- `reflect(stream)` — 最近高重要度メモリ上スクリプト化反映。
- `plan(agent_state)` — 現在信念に基づいて日レベルと時間レベル計画。
- シナリオ：5個エージェント。エージェント1「5pm党を投げる」から始まる。シミュレート刻み全体で、招待が広がり、エージェント収束。

実行：

```
python3 code/main.py
```

期待出力：刻みバー刻み追跡。最終刻みで、少なくとも3個の5個エージェントは党を彼ら計画に表示、そしてそれらは党位置に収束。シングルシードはどのオーケストレータなしで調整到着を生産。

## 使用

`outputs/skill-simulation-designer.md`生成エージェントシミュレーションを設計：エージェント数、メモリスキーマ、反映間隔、計画地平、評価メトリック。

## 出荷

本番シミュレーションルール：

- **メモリはデータベース。** スケールで本物ストア選ぶ（ベクトルDB、Postgres）。インメモリstdlibはプロトタイプ。
- **取得追跡をログ。** すべてのアクション、ログトップ-kメモリはそれを駆動。これはあなたのデバッグ能力。
- **エージェント毎トークン予算。** 各エージェント取得+反映+計画刻みあたりはO（k）LLM呼び出し。N個エージェント × T刻み × 呼び出し-毎-刻みは予算を矮小化できる。
- **メモリを定期的にコンパクト。** 低重重要度エントリを要約するプルーン。保持ポリシーは設計決定、詳細でない。
- **空間/社会ノルム違反を明示的に検出。** アーキテクチャはそれらを学ばない。

## 演習

1. `code/main.py`を実行。3+エージェント党に収束することを確認。10にエージェント増やす — 出現はまだ起こるか?
2. 反映ステップを取り除く。行動何のように見えるか? Park 2023アブレーション検出に対応。
3. 競う種シード目標を導入（「Klaus5pmで研究談話を与えたい」）。エージェント分割、またはどちらかの目標がドミナント? それを決定するもの?
4. 空間制約を追加：Hobbs Cafeは最多4つのエージェント保つ。シミュレーションはグレースフリーにオーバーフロー処理か、「シングルユーザーバスルーム」失敗パターンにヒットするか?
5. Park et al。を読む（arXiv:2304.03442）セクション6（顕現動作実験）。このミニチュアで再現できない1つの行動を特定。アーキテクチャのどのコンポーネントを拡張するだろうか?

| 用語 | 人々が言うこと | 実際の意味 |
|------|---|---|
| メモリストリーム | 「エージェントの日記」 | 観察、行動、反映、計画のログを追記のみ。 |
