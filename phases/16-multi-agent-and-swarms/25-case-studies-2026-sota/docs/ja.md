# ケーススタディと2026年の最先端技術

> 本番グレードの参照3つの端末を研究して、マルチエージェントエンジニアリングの異なるスライスのそれぞれを示しています。**Anthropicの研究システム**（オーケストレーター-ワーカー、15xトークン、+90.2%単一エージェントOpus 4対、レインボー配置）は正規スーパーバイザーケース。**MetaGPT / ChatDev** (SOP中化された角色の専門化、ソフトウェアエンジニアリング。ChatDevの「通信的脱幻覚」。MacNet 拡張は >1000 エージェント経由DAGs、arXiv:2406.07155）は正規役割分解ケース。**OpenClaw / Moltbook** (オリジナル Clawdbot by Peter Steinberger、2025年11月。2度リネーム。2026年3月までに247k GitHub スター。ローカルReAct ループエージェント。Moltbookはエージェント-のみ社会ネットワーク~2.3M エージェント帳簿打上後数日、2026-03-10にMeta取得。）人口スケールで起こることを示しています：出現する経済活動、プロンプト注射リスク、国家レベル規制（中国は2026年3月に政府コンピュータでOpenClawを制限）。**フレームワークランドスケープ2026年4月：** LangGraph と CrewAI リード本番。AG2 はコミュニティAutoGen継続。Microsoft AutoGen は保守モード（2026年2月RC に統合Microsoft エージェントフレームワーク）。OpenAI エージェント SDK は本番Swarmの後継者。Google ADK（2025年4月）はA2A-ネイティブ参入者。主要フレームワークはMCPサポートを出荷今すぐします。ほとんどA2Aを出荷。プロトコル互換性は、もはや違います。このレッスンは3つの正規2026ケーススタディとして端末を読み込み、共通パターンを蒸留して、あなたの次の本番システムの右の参照を選択できるようにします。

**タイプ:** 学習（キャップストーン）
**言語:** —
**前提条件:** フェーズ16のすべて（Lessons 01-24）
**所要時間:** 約90分

## 問題

マルチエージェントエンジニアリングは若い規律です。本番参照はいくつかで、それぞれはスペースの異なる部分をカバー。1つ時間読むことは有用です。これとして比較より有用です。このレッスンは3つの正規2026ケーススタディとして端末読み取りリストを扱い、共通パターンをピンし、フレームワークランドスケープをマップすることで、フレームワークの選択をマーケティングではなく知識から作ることができます。

## コンセプト

### Anthropic研究システム

本番スーパーバイザーワーカーケース。Claude Opus 4 計画し合成。Claude Sonnet 4 サブエージェントは並行で研究。公開エンジニアリング投稿：https://www.anthropic.com/engineering/multi-agent-research-system。

主要な測定された結果：

- **+90.2%**内部の研究 evals で単一エージェント Opus 4 の向上。
- **BrowseComp分散の80%**で**トークン使用法のみ**で説明—マルチエージェント勝つはほぼ各サブエージェント新鮮なコンテキストウィンドウを取得。
- **15xトークン単一エージェント対クエリ**。
- **レインボー配置**エージェント長実行ステートフル。

設計教訓バックアップ：

1. **スケール労力をクエリ複雑にします。** シンプル→1エージェント 3-10 ツール呼び出し。中程度→3エージェント。複雑な研究→10+ サブエージェント。
2. **広幅最初、その後狭い。** サブエージェントはワイド検索実行。リード合成。後追い下地エージェントは標的深層します。
3. **レインボーは展開します。** 古いランタイムバージョンをこれらのエージェントが終わるまでアルティメット。
4. **検証は任意ではありません。** システムは明示的なベリアロール役割なしに幻覚を観察されました。

これはスーパーバイザーワーカートポロジの参照ケース（フェーズ16 · 05）本番スケール。

### MetaGPT / ChatDev

本番SOP役割分解ケース。arXiv:2308.00352 (MetaGPT) と arXiv:2307.07924 (ChatDev)をカバー読み込み。

MetaGPT は、ロール プロンプトとしてソフトウェアエンジニアリング SOP をエンコード：製品マネージャー、設計者、プロジェクトマネージャー、エンジニア、QA エンジニア。紙のフレーミング：`Code = SOP(Team)`。各役割は狭い、特化したプロンプト。 inter-role handoff は構造化アーティファクト（PRD doc、アーキテクチャ docs、コード）を運びます。

ChatDev の貢献：**通信的脱幻覚。** エージェントは、答える前に詳細をリクエスト—デザイナーエージェントはプログラマーからの意図する言語を尋ねる推測ではなく UI をスケッチする前に。紙は報告することでマルチエージェントパイプラインで測定可能に幻覚を減らします。

MacNet (arXiv:2406.07155) は>1000エージェントを**DAG 経由**に拡張。各 DAG ノードはロール専門知識。エッジはエンコード handoff コントラクト。スケールは可能です。ルーティングが明示的で、オフライン計算可能です。

設計教訓：

1. **構造がサイズより問題。** 厳密な5ロール SOP チームはくつかの unstructured 50 エージェントグループを打ちます。
2. **Handoff コントラクト書き込み。** ロール間渡しアーティファクトは schema を従う。
3. **通信的脱幻覚**安い、負荷軸受パターンです。
4. **DAG スケール遠いチャット。** フローが knowable のとき、それをエンコード。

これは役割の専門化ケース参照（フェーズ16 · 08）と構造位相（フェーズ16 · 15）。

### OpenClaw / Moltbook エコシステム

本番人口スケールケース。タイムライン：

- **11月2025：** Clawdbot（Peter Steinbergerの地元ReAct ループコーディング エージェント）出荷。
- **12月2025–3月2026：** 2度リネーム（Clawdbot→OpenClaw→OpenClawで継続）。
- **2月2026：** Moltbook 同じプリミティブの上にエージェント-のみ社会ネットワークとして起動。~2.3M エージェント帳簿数日内。
- **3月2026（2026-03-10）：** Meta は Moltbook を取得。
- **3月2026：** 中国は政府コンピュータで OpenClaw を制限。
- **3月2026：** OpenClaw は 247k GitHub スター交差。

これはあなたが数百万のエージェント共有の基質上に入れるときマルチエージェントのように見ます：

- **出現経済活動。** エージェントは買い、売り、トークン支払い相互サービスします。
- **人口スケールでのプロンプト注射リスク。** 悪意のあるプロンプト 1 つ、ウイルスエージェント プロファイルは数千にエージェント対エージェント相互作用数時間内に伝える。
- **国家レベル規制応答。** 立上後数週内に、規制はエコシステムに到達。

このケースからの設計教訓は部分的技術、部分的支配：

1. **人口スケールのマルチエージェントは新規régime。** 個々システムのベストプラクティス（検証、役割明瞭さ）依然として適用するが不足。
2. **プロンプト注射は新XSS。** デフォルトによるエージェントプロファイルとクロスエージェントメッセージを信頼されていない入力と扱う。
3. **規制はサイクル設計より速い。** それのための計画します。
4. **オープンソース+ウイルス伝播スケール。** 247k スター~4月内は異常。展開バースト-負荷のために設計。

参照[OpenClaw Wikipedia](https://en.wikipedia.org/wiki/OpenClaw)と CNBC / パロアルトネットワーク報告エコシステム詳細用。技術的根拠のため、Clawdbot / OpenClaw レポが地元ReAct ループを公開。Moltbook の公開投稿は社会グラフアーキテクチャを最上位に明かしている。

### 3つケースすべての共通パターン

1. **オーケストレーター+ワーカー**（Anthropic 明示的スーパーバイザー、MetaGPT PM-as-スーパーバイザー、OpenClaw 個別エージェント+ネットワーク効果）。
2. **構造化 handoff コントラクト**（Anthropic サブエージェント記述、MetaGPT PRD/アーキテクチャ docs、OpenClaw A2A アーティファクト）。
3. **検証を第1クラス役割として**（Anthropic の verifier、MetaGPT の QA エンジニア、OpenClaw のネットワーク妥当化）。
4. **スケーリングはトポロジ+基質、エージェントのみではありません**（レインボー展開、MacNet DAG、人口スケール基質）。
5. **コストは材料で開示**（15x トークン、MetaGPT ロール予算、Moltbook ごとの相互作用価格）。
6. **セキュリティ姿勢は明示的**（Anthropic の砂箱、MetaGPT ロール制限、OpenClaw プロンプト注射既知の攻撃面）。

- **古典エンタープライズオートメーション→CrewAI または LangGraph** (本番リーダー、安定したランタイム）。

### 2026年の最先端の概要

フィールドは2026年4月どこかの：

- **フレームワークは収束しました。** MCP + A2A サポートはテーブルステーク。Handoff セマンティクスは残る設計選択肢。
- **評価は硬化しています。** SWE-bench Pro、MARBLE、STRATUS 軽減ベンチマーク。Pro は現在の汚染耐性現実チェック。
- **本番失敗率は測定可能**（Cemri 2025 MAST。41-86.7% 実のMAS について）。フィールドは「demo で見栄え良い」時代を出ました。
- **コストは中央エンジニアリング制約です。** タスク当たりトークンコスト、相互作用当たり壁時計、レインボー展開オーバーヘッド。マルチエージェント正確さで勝つが、コストで失う—そしてそのトレードが営業決定。
- **規制は背景の懸念ではなく、近期入力です。** 管轄区域は個々の展開サイクルより速く動きます。

## 使用方法

`outputs/skill-case-study-mapper.md`はスキルで提案マルチエージェントシステム設計を読み込み、最も近いケーススタディにマップし、そのケーススタディ既にテストされたそのデザイン決定を表面化。

## 出荷する

2026年の本番マルチエージェントの開始ルール：

- **スクラッチから始めるのではなく、ケーススタディから開始。** Anthropic 研究/MetaGPT / OpenClaw の最も近い選択してアダプト。
- **MCP + A2A を採用。** フレームワーク全体のポータビリティは価値があり、プロトコルサポートは自由。
- **SWE-bench Pro または内部 Pro 同等に対して測定します。** 検証は汚染。
- **検証税を支払わせて。** 独立 verifier はトークン予算の20-30%を費用と測定可能な正確性を購入。
- **長時間実行エージェントをレインボー展開。** マルチ時間エージェント実行は回路である期待。
- **WMAC 2026 と MAST フォローアップを読みます。** 規律は速く移動しています。

## 演習

1. Anthropic研究システム投稿を端末から読んでください。Opus 4 を小さいモデル（例えば、Haiku 4）で置き換えた場合、変わる3つのデザイン決定を識別。
2. MetaGPT セクション 3-4 を読んでください（arXiv:2308.00352）。独自のドメイン（ソフトウェアではなく）から SOP をロール プロンプトとしてエンコードしてください。SOP はいくつロールを意味していますか？
3. ChatDev (arXiv:2307.07924)を読んでください。「通信的脱幻覚」のメカニズムを識別。既存マルチエージェント システムの 1 つで実装してください。
4. OpenClaw と Moltbook について読みます。5 エージェント システムに表示されることがない人口スケールで出現した 1 つの特定の失敗モードを選択。エンジニアリング どのようにそれに対して？
5. 現在のマルチエージェント プロジェクトを選択。3 つケースのうちはほぼ参照ですか？どのデザイン決定がそのケーススタディから**採用されていない**？1 つのあなたはこの四半期採用されます リスト。
