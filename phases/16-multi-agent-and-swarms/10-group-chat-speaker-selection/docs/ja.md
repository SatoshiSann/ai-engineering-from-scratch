# グループチャットとスピーカー選択

> AutoGenのGroupChatとAG2のGroupChatは、N個のエージェント間で1つの会話を共有します。セレクタ関数（LLM、ラウンドロビン、またはカスタム）が次に誰が話すかを選びます。これは、エージェント型マルチエージェント会話の原型です。エージェントは静的グラフ内での役割を知らず、単に共有プールに反応します。AutoGen v0.2のGroupChatセマンティクスはAG2フォークで保持されました。AutoGen v0.4はイベント駆動型アクターモデルとして書き直されました。Microsoftは2026年2月にAutoGenをメンテナンスモードにし、それをSemantic Kernelとマイクロソフトエージェントフレームワーク（RC 2026年2月）にマージしました。GroupChatプリミティブは、AG2とMicrosoft Agent Frameworkの両方に残されています。1度学び、どこででも使うことができます。

**タイプ:** Learn + Build  
**言語:** Python (stdlib)  
**前提条件:** Phase 16 · 04 (Primitive Model)  
**所要時間:** 約60分

## 問題

静的グラフ（LangGraph）は、ワークフローが既知の場合に優れています。実際の会話は静的ではありません。時にはコーダーがレビュアーに、時には研究者に、時には執筆者に尋ねます。あらゆる可能なハンドオフをハードコーディングすると、エッジの爆発が起こります。*共有プールに反応するエージェント*が必要で、誰が次に話すかを決める何らかの関数が必要です。

それはまさにAutoGen GroupChatが行うことです。

## コンセプト

### 形状

```
              ┌─── shared pool ────┐
              │   m1  m2  m3  ...  │
              └─────────┬──────────┘
                        │ (everyone reads all)
      ┌───────┬─────────┼─────────┬───────┐
      ▼       ▼         ▼         ▼       ▼
    Agent A  Agent B  Agent C  Agent D  Selector
                                           │
                                           ▼
                                  "next speaker = C"
```

すべてのエージェントがすべてのメッセージを見ます。セレクタ関数は、各ターンで次に誰が話すかを選ぶために呼び出されます。

### 3つのセレクタのフレーバー

**ラウンドロビン。** 固定サイクル。決定論的。Nで線形にスケーリングしますが、コンテキストを無視します。コーダーは、トピックが法的レビューであっても、ターンを得ます。

**LLM選択。** 最近のプールを読み、最適な次のスピーカーを返すLLMへの呼び出し。コンテキスト認識ですが遅い。毎ターン、LLM呼び出しが追加されます。AutoGenのデフォルトです。

**カスタム。** 任意のロジックを持つPython関数。一般的：フォールバックルール付きLLM選択（例："常にコーダーの後にベリファイアにターンを与える"）。

### ConversableAgent API

```
agent = ConversableAgent(
    name="coder",
    system_message="You write Python.",
    llm_config={...},
)
chat = GroupChat(agents=[coder, reviewer, tester], messages=[])
manager = GroupChatManager(groupchat=chat, llm_config={...})
```

`GroupChatManager`がセレクタを保持します。エージェントがターンを完了すると、マネージャーはセレクタを呼び出し、次のエージェントを返します。ループは終了条件まで続きます。

### 終了

3つの一般的なパターン：

- **最大ラウンド。** 総ターン数のハードキャップ。
- **「TERMINATE」トークン。** エージェントは標識メッセージを出力でき、1つが表示されるとマネージャーは停止します。
- **ゴール到達チェック。** 軽量ベリファイアが各ターンを実行し、完了したら会話を停止します。

### AutoGen → AG2分割とMicrosoft Agent Frameworkマージ

2025年初頭、Microsoftはイベント駆動型アクターモデルを中心にAutoGen（v0.4）の大規模な書き直しを開始しました。コミュニティはAutoGen v0.2のGroupChatセマンティクスをAG2としてフォークし、早期採用者が統合していたAPIを保持しました。

2026年2月、Microsoftはイベント駆動型アクターモデルをマージしてAutoGenがメンテナンスモードになることを発表し、**Microsoft Agent Framework**（RC 2026年2月、現在Semantic Kernelとマージされた）に統合されました。GroupChatコンセプトは両方のトラックに残っています。実装の詳細が異なります。AG2はv0.2互換コード用の推奨される上流です。

### GroupChatが適合する場合

- **突現的な会話。** あらゆる可能な次のスピーカーを事前に配線したくない。
- **役割混合タスク。** コーダーが研究者に尋ねる、研究者がアーキビストに尋ねる、アーキビストがコーダーに戻す。フローはDAGではありません。
- **探索的問題解決。** 「ブレーンストーミング会議」のように考える。「組立ライン」ではなく。

### 失敗するとき

- **厳密な決定論。** LLMセレクタは矛盾する可能性があります。同じプロンプト、異なる実行、異なる次のスピーカー。
- **シコファンティー滝。** エージェントは最も自信を持って話した者に従う。明示的に反対のプロンプトを指定してください。
- **コンテキスト肥大化。** すべてのエージェントがすべてのメッセージを読む。10ターン後、コンテキストは膨大です。投影（レッスン15）を使用して、ビューをスコープします。
- **ホットスピーカー。** 1つのエージェントがセレクタがその専門を好むため会話を支配します。スピーカーバランスをセレクタフィーチャーとして導入します。

### グループチャット対スーパーバイザー

同じプリミティブ、異なるデフォルト：

- スーパーバイザー：1つのエージェントが計画し、他が実行します。セレクタは「プランナーに何をするか尋ねる」。
- グループチャット：すべてのエージェントはピア。セレクタは共有プール上の関数。

どちらもレッスン04から4つのプリミティブを使用します。グループチャットはLLM選択オーケストレーションとフルプール共有状態をデフォルトにします。

## ビルド

`code/main.py`は、stdlibから最初からGroupChatを実装します。3つのエージェント（コーダー、レビュアー、マネージャー）、ラウンドロビンとLLM選択の変種、`TERMINATE`トークンでの終了。

デモは、会話トランスクリプトに加えて、両方の変種のセレクタ決定トレースを出力します。

実行：

```
python3 code/main.py
```

## 使用

`outputs/skill-groupchat-selector.md`は、与えられたタスクのGroupChatセレクタを構成します。ラウンドロビン対LLM選択対カスタム、およびセレクタ入力（最近のメッセージ、エージェント専門、ターン数）を使用するもの。

## 出荷

チェックリスト：

- **最大ラウンドキャップ。** 常に。一般的なタスクでは10-20。
- **スピーカーバランスメトリック。** エージェントごとのターン数を追跡。不均衡が閾値を超えた場合にアラート。
- **終了トークン。** `TERMINATE`または専用ベリファイアエージェント。
- **投影またはスコープメモリ。** 約10メッセージ後、各エージェントにスコープビューのみを与えることを検討して、コンテキスト肥大化を防ぎます。
- **セレクタロギング。** LLM選択の変種については、セレクタの入力と選択の両方をログします。そうでなければデバッグは不可能です。

## 演習

1. `code/main.py`を実行します。ラウンドロビン対LLM選択の会話を比較します。各条件下ではどのエージェントが支配しますか?
2. セレクタに「max-speaks-per-agent」ルールを追加します。トランスクリプトへの影響は?
3. ゴール到達終了を実装：レビュアーが「承認」を返すと停止します。ラウンドキャップの前にどのくらいの頻度でトリガーされますか?
4. AutoGen安定ドキュメントを読む（https://microsoft.github.io/autogen/stable/user-guide/core-user-guide/design-patterns/group-chat.html）。`GroupChatManager`で使用されるデフォルトセレクタを特定します。
5. AG2リポジトリを読む（https://github.com/ag2ai/ag2）そしてそのv0.2 GroupChatをv0.4イベント駆動版と比較します。v0.4が追加する具体的なプロパティ（スループット、フォルトトレランス、コンポーザビリティ）は?

## キーターム

| 用語 | 人々が言うこと | 実際の意味 |
|------|---|---|
| GroupChat | 「1つのチャットルームのエージェント」 | 共有メッセージプール+セレクタ関数。AutoGen / AG2プリミティブ。 |
| スピーカー選択 | 「誰が次に話すか」 | 次のエージェントを選ぶ関数。ラウンドロビン、LLM選択、またはカスタム。 |
| GroupChatManager | 「会議のホスト」 | AutoGenコンポーネント。セレクタを所有し、ターンをループする。 |
| ConversableAgent | 「ベースエージェント」 | AutoGenベースクラス。メッセージを送受信できるエージェント。 |
| 終了トークン | 「「停止」の言葉」 | 標識文字列（通常`TERMINATE`）がチャットを終了します。 |
| ホットスピーカー | 「1つのエージェントが支配」 | セレクタが同じエージェントを選び続けるフェイルモード。 |
| コンテキスト肥大化 | 「プールが無制限に増える」 | 各エージェントがすべての先行メッセージを読む。コンテキストはターンで増える。 |
| 投影 | 「スコープビュー」 | 共有プールへの役割固有のビュー。コンテキスト肥大化を防ぎます。 |
