# エージェント経済、トークンインセンティブ、評判

> 長期計画の自律エージェント（METRの1時間から8時間の労働曲線）は経済的エージェンシーが必要です。出現する**5層スタック**は、**DePIN**（物理的なコンピュート）→**アイデンティティ**（W3C DIDs +評判資本）→**認知**（RAG + MCP）→**決済**（アカウント抽象化）→**ガバナンス**（Agentic DAOs）。本番エージェントインセンティブネットワークには**Bittensor**（TAOサブネットはタスク固有モデルに報酬）、**Fetch.ai / ASI Alliance**（ASI-1 Mini LLM + FETトークン）、および**Gonka**（トランスフォーマーベースのPoWが生産的なAIタスク計算を再割り当て）が含まれます。学術作業：AAMAS 2025の分散LaMASは**Shapley値クレジット属性**を使用して、寄稿エージェントに公正に報酬を与えます。Google Researchは「大規模言語モデルのメカニズム設計」は単調な集約下で秒間価格支払いでの**トークンオークション**を提案しています。このレッスンは最小限のエージェント市場を構築し、Shapley値クレジット属性をマルチエージェントパイプラインに適用し、ゲーム理論機械が具体的に着地するように2番目の価格トークンオークションを実行します。

**タイプ:** 学習
**言語:** Python (stdlib)
**前提条件:** フェーズ16 · 16 (交渉と交渉)、フェーズ16 · 09 (並列群ネットワーク)
**所要時間:** 約75分

## 問題

マルチエージェントシステムは、エージェントが共同で価値を生成するが、個別に報酬を受ける必要がある場合が複雑になります。古典的なメカニズム—等しい分割、最後の寄稿者は全て—不公正または gameable です。Shapleyの値経由のコーリション基ベースの報酬は構成で公正ですが、計算に費用がかかります。2025-2026の文献は有用な近似を押します：Shapley サンプリング、単調な集約オークション、および確認された寄付から蓄積するオンチェーン評判。

クレジット属性を超えて、フィールドは実際の経済的エージェントに変わりました：Bittensor TAOはサブネット固有のモデルを微調整するために採掘計算に報酬、Fetch.ai/ASIはASI-1 Mini LLM使用法にFETトークンを報酬、Gonkaは変換プルーフオブワークの生産的なAIタスクを再割り当てします。自律的にトランザクトエージェントは今日存在します。問題はインセンティブをどのように整列させるかです。

このレッスンはエージェント経済を特定の問題の家族—クレジット属性、メカニズム設計、評判—として扱い、各を最小限の数学で構築して、アイデアが着地させるため。

## コンセプト

### 5層のエージェント経済スタック

1. **DePIN（物理的なコンピュート）。** GPU、ストレージ、帯域幅をリースする分散インフラ。Bittensorサブネット、Renderネットワーク、Akash。エージェント固有ではなく。エージェントはそれを使用します。
2. **アイデンティティ。** W3C Decentralized Identifiers (DIDs) は各エージェントに任何のプラットフォームに独立した耐久IDを与えます。評判はDIDに蓄積されます。エージェント ネットワーク プロトコル (ANP) は DIDs を発見層として使用します。
3. **認知。** エージェントの推論ループ：LLM + RAG + MCP。これは他のフェーズが構築するものです。
4. **決済。** アカウント抽象化（ERC-4337）は、ETHを保持せずに独自の残高からガスを支払えるようにエージェントを使用させます。エージェントはサービス、互いに、または計算に支払うことができます。
5. **ガバナンス。** Agentic DAOs：ガバナンス構造で、人間**と**エージェントが投票評判にリンク投票力のプロトコル変更。

すべての5を使用しないすべての本番システム。Bittensorは1、2、部分的に3、部分的に4、5なしを使用します。OpenAI エージェントは、3を除き、誰もが使用しません。スタックはリファレンスマップであり、要件ではありません。

### Bittensor、Fetch.ai、Gonka — 実行内容

**Bittensor (TAO)。** サブネットは専門タスク（言語モデリング、画像生成、予測）です。マイナーはモデル出力を提出します。バリデーターはそれらをランク付け。スタック重み付けスコアはTAO報酬を配布します。各サブネットには独自の評価があります。経済的な教訓：使用されたコンピュートではなく、タスク固有の出力品質に対して支払いを与えます。

**Fetch.ai / ASI Alliance。** ASI-1 Mini LLMは Fetch.ai ネットワークで実行されます。ユーザーは推論にFETトークンを支払います。エージェント-as-ピアの物語はここでより強いです：Fetch上のエージェントはタスクの別のものを呼び出し、FETで支払うことができます。

**Gonka。** 変換証拠-オブ-ワーク：「作業」は変換の順方向パスです。マイナーは既知の正しい出力（訓練データから）で推論タスクを実行して稼ぎます。ハッシュベースのPoWではなくリソース生産PoW。

すべて3つは2026年4月の本番グレードです。支払配布が異なります。Bittensor はサブネット検証者に対する品質に報酬。Fetch はユーザー支払いで測定されたユーティリティに報酬。Gonka は検証可能な推論作業に報酬。

### Shapley値クレジット属性

3つのエージェントはタスクで協力します。出力は0.8にスコアリング。誰が何に貢献しましたか？

Shapley値：4つの公理（効率、対称性、直線性、null）を満たす一意のクレジット割り当て。エージェント`i`の場合：

```
shapley(i) = (1/N!) * sum over all orderings O of (v(S_i_O ∪ {i}) - v(S_i_O))
```

ここで`S_i_O`は順序付けOのiの前のエージェントのセットです。実際には、すべての順列を列挙し、各順列の各エージェントの限界貢献を記録し、平均。

N=3エージェントの場合、6つの順列があります。N=10の場合、3.6M—したがって実際には、列挙ではなく順序付けをサンプルします。

### 集約用の2番目の価格オークション

Google Research（「大規模言語モデルのメカニズム設計」）は、LLMの出力を集約するための2番目の価格トークンオークションを提案しています。セットアップ：Nエージェント各提案完成；各自は選択されたためのプライベート値があります。競売人は最高値の提案を選択し、**第二番目に高い**値を支払います。単調な集約下（値は提案されたもの、多くが入札されたもの、どのように依存）、これは真実です—エージェントは真の価値を入札します。

LLMシステムについてなぜこれが重要です：複数のエージェントで完成タスクをアウトソースでき、異なる価格。オークションは最高を選択+公正に支払い、エージェントは報告の誤りのインセンティブ持ちません。

### 評判資本

DID束の評判スコアは確認された貢献から蓄積します。シンプルな更新ルール：

```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

減衰因数`alpha`1に近い。評判：

- はルーティング決定（「高リプエージェントに困難なタスクを送信」）の読み取りが安いです。
- 鍛造に費用がかかります（時間の経過に蓄積、DIBにバインド）。
- スラッシュできます：確認に失敗した貢献減算。

### AAMAS 2025分散LaMAS

LaMAS提案（AAMAS 2025）は、DIBアイデンティティ、Shapley値クレジット属性、簡単なオークションメカニズムを組み合わせます。重要な主張：クレジット属性ステップを分散させることで、システムは監査可能で単一のポイント操作に耐性を持つようにしています。

### 経済が崩壊する場所

- **価格オラクル操作。** クレジット関数をゲームできる場合、エージェントはそれをゲームします。すべてのメカニズムは敵対的テストが必要です。
- **Sybil攻撃。** 1つのオペレーターはN個の偽のエージェントを紡ぎ、独自の貢献をインフレさせます。DIBsスローしますがこれを停止しません。評判費用は軽減です。
- **検証コスト。** クレジット属性は検証者と同じくらい公正です。検証が安い場合（小さなLLM）、ゲームすることができます。高価な場合（人間パネル）、システムはスケールしません。
- **規制の懸念。** エージェント経済は金融規制と交差します。Bittensor、Fetch、Gonkaはすべて2026年の一部の管轄地で法的なグレーゾーンで運営されています。

### エージェント経済が意味するとき

- **異種のオペレーターを持つオープンネットワーク。** 単一のチームはすべてのエージェントを制御しません。
- **検証可能な出力。** 検証なしで、クレジット属性は推測です。
- **長期計画ワークフロー。** ワンショットタスクは評判蓄積から利益を得ません。
- **トークン化された支払いは法的に実行可能です**あなたの管轄で。

閉じた企業システムでは、経済は簡単な割り当て（マネージャーは仕事を割り当て、メトリクスは内部的）を与える。経済学文献はほとんどのオープンネットワークに適用されます。

## 構築する

`code/main.py`を実装：

- `shapley(value_fn, agents)` — 小さなNのための列挙による正確なShapley計算。
- `second_price_auction(bids)` — 真実のメカニズム。勝者は第二番目に高い支払い。
- `Reputation` — 指数減衰とスラッシングを持つDIB結ばれた評判。
- デモ1：3つのエージェント協力、正確なShapleyがクレジットを属性。
- デモ2：5つのエージェントがタスクスロットを入札します。2番目の価格オークションは勝者+支払いを選択します。
- デモ3：100ラウンドは異種のレップ持つエージェントのタスク割り当て。レップ重み付けルーティングはランダムを打ちます。

実行：

```
python3 code/main.py
```

予想される出力：各エージェントのShapley値。オークション結果は真実のビッドの均衡を示します。レップ重み付けルーティングはウォームアップ後にランダムの上で10-20%品質ゲインを示します。

## 出荷する

2026年にエージェント経済を実行する場合：

- **トークンではなく、評判で開始してください。** 評判は実装が安く、個別で価値があります。トークンは法的および経済的な複雑さを追加します。
- **報酬する前に検証してください。** 独立した検証ステップなしにクレジットを配布しないこと。自己報告品質Sybilゲームを蓄積します。
- **Shapley正確ではなく、Shapleyサンプル。** 100-1000の順序付けをサンプル。正確な列挙はスケールしません。
- **減衰因数と床の評判をキャップします。** 無限減衰は正当な寄稿者をワイプします。遅すぎる減衰は古い高リプエージェントに報酬します。
- **敵対的にメカニズムを監査します。** レッドチームシナリオを開く前に実行します。すべてのメカニズムはゲーム理論を持ちます。穴を見つけたいのは、攻撃者ではなく。

## 演習

1. `code/main.py`を実行します。Shapley値が総値に合計することを確認します（効率公理）。価値関数を変更します。Shapley配分は予想される方向で変更されますか？
2. Shapley **サンプリング**を実装（K順序付けにわたるモンテカルロ）。Kは近似精度にどのように影響しますか？N=4の正確さと比較します。
3. オークション前の連合形成ステップを実装してください：エージェントはユニットにマージし、ユニットとして入札できます。どのコーリションフォーム？結果はパレートより個別の入札ですか？

## 主要用語

| 用語 | 人々が言うこと | 実際に意味すること |
|------|----------------|------------------------|
| DePIN | 「分散物理インフラ」 | トークンインセンティブ計算/ストレージ/帯域幅。Bittensor、Akash、Render。 |
| DID | 「分散識別子」 | ポータブルIDのための W3C 仕様。エージェント評判は DID に、プラットフォームに結ばれます。 |
| ERC-4337 | 「アカウント抽象化」 | ガスをスポンサーできる契約アカウント、エージェントの支払いを有効に。 |
| Shapley値 | 「公正なクレジット属性」 | 効率、対称性、直線性、null を満たす一意の配分。 |
| 2番目の価格オークション | 「Vickrey オークション」 | 真実のメカニズム：勝者は第二番目に高い入札を支払います。単調な集約互換。 |
| 評判資本 | 「累積品質スコア」 | 確認された貢献から蓄積した DID 結ばれたスコア。時間の経過とともに減衰。 |
| Agentic DAO | 「エージェント +人間統治」 | 第1クラスと評判に投票力をリンクするエージェント投票手段の DAO。 |
| TAO / FET / GPUクレジット | 「トークン指標」 | Bittensor TAO、Fetch.ai FET、様々なDePINトークン。 |
