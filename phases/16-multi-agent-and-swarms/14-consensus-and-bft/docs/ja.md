# コンセンサスとビザンチン障害耐性エージェント

> 古典分散システムBFTは確率的LLMに合致します。2025-2026年に3つの研究方向が浮上：**CP-WBFT**（arXiv:2511.10400）は信頼プローブで各投票を重み付け。**DecentLLMs**（arXiv:2507.14928）はリーダーレスで並列ワーカー提案と幾何中央値集約。**WBFT**（arXiv:2505.05103）は加重投票と階層構造クラスタリングを結合してコアとエッジノードを分割。誠実な経験的結果「AI エージェント が同意できるか?」（arXiv:2603.01213）は、スカラー同意さえ脆弱 — 1つの欺瞞的エージェントが混合エージェントを危険にさらす。BFTは必要だが十分でない。このレッスンは最小BFTプロトコルを構築し、3つのエージェント特定攻撃（ビザンチン嘘、シコファント適合、相関エラー単一文化）を注入し、各コンセンサス変種がどう対処するかを測定。

**タイプ:** Learn + Build  
**言語:** Python (stdlib)  
**前提条件:** Phase 16 · 07 (Society of Mind and Debate), Phase 16 · 13 (Shared Memory)  
**所要時間:** 約75分

## 問題

N個のLLMエージェント各々が答えを生成。彼ら不一致。多数決が間違ったものを選ぶ。なぜなら2つのエージェントは相関（同じベースモデル、同じ訓練データ、同じ失敗モード）。3つ目のエージェントは偶然異なる方法で間違っている。だから多数は偽りの多数。

ここで欺瞞的エージェントを追加：それは意図的に嘘をつく。またはシコファントエージェント：それは最後に話した人と同意。古典BFTでは、仮定はビザンチンノードが分数`f < n/3`であり、任意に行動します。2026年の現実は、LLMノードは誠実でも確率的で、モデル全体で相関があり、互いの出力に影響される。独立ベルヌーイ投票者として扱う代わりに。

古典BFT（PBFT、1999）は間違っていない。それは不完全。それはビット反転を扱う。それは「3つの誠実なエージェントは訓練データを共有するから幻覚を共有」を扱わない。このレッスンはPBFTの基礎から構築し、3つの2025-2026年適応を層状にする。

## コンセプト

### 古典BFTが与えるもの

実用ビザンチン障害耐性（Castro & Liskov、OSDI 1999）は`f < n/3`ビザンチンノードを許容。プロトコルは3つのフェーズ（事前準備、準備、確定）と2つのプリミティブ（署名メッセージ、クォーラム証明書）。`n >= 3f + 1`誠実または悪意あるノード間の単一値への同意。

保証は強力ですが仮定：

1. **独立した障害。** ビザンチンは協調しない。
2. **誠実なノードは真に誠実。** 誠実な出力の正確さは問題外。プロトコルのみが不一致を揃える。
3. **質問は確定的答えを持つ。** 間違った事実への同意はまだ同意。

LLMエージェントはすべて3つを違反。同じベースモデルを実行する2つのエージェント共有障害。「誠実な」LLMはまだ幻覚。曖昧な質問では、「真実」はエージェント決定 — 外部神託はなし。

### 3つのLLM特定攻撃

**ビザンチン嘘。** 1つのエージェント意図的間違い答え出力。古典BFTはこれを処理。`f < n/3`。

**シコファント適合。** 1つのエージェントは他の答え読む投票前に、最後に話した者と揃える。悪意なし、だが大きい音の声と相関。古典BFTはこれを防がない。エージェントはすべての署名チェックを通過。

**相関エラー単一文化。** 3つのエージェント共有ベースモデル。彼ら同じ間違い答え幻覚。多数は間違い。古典BFTは助けない。すべて3つは「誠実に」同意。

### 2025-2026年応答

**CP-WBFT**（arXiv:2511.10400）— 信頼プローブ加重BFT。各投票は答えに信頼プローブを附属（自己報告確率、または別の較正モデル予測）。投票重みは信頼でスケール。報告+85.71% BFT完全グラフ改善。軽減：シコファント適合（適合エージェントは自発立場で信頼度低いほう持つ傾向）。

**DecentLLMs**（arXiv:2507.14928）— リーダーレス。ワーカーエージェント並列提案、評価者エージェントスコア提案、最終答えはスコア位置の幾何中央値。堅牢`f < n/2`。軽減：ビザンチン嘘と相関エラー（幾何中央値は外れ値に堅牢で、モデル偏向平均へなく密クラスタに引く）。

**WBFT**（arXiv:2505.05103）— 加重BFT階層構造クラスタリング。投票重みはレスポンス品質プラス履歴学習信頼スコアで割り当て。エージェント分割コア対エッジ。コアエージェント最初同意を達成する必要。エッジエージェント従う。軽減：スケーラビリティ（コアコンセンサス小さく高速）と部分的に単一文化（コアは多様性で選択可能）。

### 経験的：「AI エージェント が同意できるか?」（arXiv:2603.01213）

論文はスカラー同意を測定（LLMエージェント単一数値値への同意）複数フロンティアモデル。検出は不快：

- 敵対者なしでさえ、LLMエージェント多くのベンチマークのスカラー質問で30%以上の率で不一致。
- 単一欺瞞ペルソナを採用するエージェント混合エージェントコンセンサスを誠実ベースラインから40+パーセントポイント引く。
- 不一致率は多様性と相関 — 異種アンサンブル不一致が多い同質より（良い：相関なしエラー）だが速度で遅い（悪い：より長い時間対一致）。

テイク：BFTは出力を揃える機械を与える。それは揃い出力が正しいか教えない。検証（Phase 16 · 08役割専門化）、多様性（Phase 16 · 15議論変種）、評価者エージェント（Phase 16 · 24ベンチマーク）と結合。

### コアプロトコル、ストリップダウン

LLMエージェント最小BFTラウンド：

```
1. task arrives; each agent i produces answer a_i
2. each agent attaches confidence probe c_i in [0, 1]
3. aggregator collects (a_i, c_i) from all n agents
4. aggregator groups by semantic cluster (equivalent answers)
5. aggregator computes weight for each cluster C:
     w(C) = sum_{i in C} c_i
6. winner = cluster with max weight, if max > threshold * sum(c_i)
   else: retry or escalate
7. minority clusters logged with provenance for post-hoc audit
```

セマンティッククラスタリングステップはLLM特定のツイスト。2つの答え「研究は4.2%報告」と「4.2%改善」は同じクラスタ。ナイーブな文字列等質チェックはこれを逃す。本番では、安いな埋め込みモデルまたは明示的正規化を使用。

### 閾値チューニング

`threshold`パラメーター何時に受け入れるか、何時に再試行するか決定。低すぎる：弱い多数を受け入れる。高すぎる：何も受け入れない。経験的範囲：0.5-0.67 `n=5-7`エージェント、小さい`n`に高い。閾値以下、人間または異なるアンサンブルにエスカレート。

### コンセンサスが助けない場所

- **曖昧な質問。** 質問が確実な真実がなければ、コンセンサスは意見。そう呼ぶ。
- **複合質問。** 「コード書いて説明」 — 2つの答え。各独立で投票。
- **敵対マルチラウンド。** エージェント先ラウンドを観察し模倣できる（Du 2023討論）、彼ら関係なく互いと同意し始める。ラウンドを制限（通常2-3）。

## ビルド

`code/main.py`実装：

- `AgentVoter` — スクリプト化ポリシー（答え、信頼）。
- `MajorityVote` — 古典複数。
- `CPWBFT` — 信頼加重投票セマンティッククラスタリング。
- `DecentLLMs` — スコア提案上の幾何中央値集約。
- `Scenario` — 3つの攻撃パターン下でそれぞれの集約を実行。

実装攻撃パターン：

1. `byzantine`：1つのエージェント高信頼で嘘。
2. `sycophancy`：1つのエージェント最初の答え見たコピー、対応信頼。
3. `monoculture`：3つのエージェント間違い答え共有（相関エラー）中位信頼。

実行：

```
python3 code/main.py
```

期待出力：テーブル（攻撃、集約） -> 最終答え、正しい答えハイライト。複数失敗単一文化ケース。CPWBFT信頼重みシコファント緩和。DecentLLMs幾何中央値誠実クラスタ方向単一文化が人口未満半分のとき引く。

## 出荷

何らかのコンセンサスメカニズム出荷前：

- **3つ以上のパターンで攻撃テスト。** プロトコル予測的に失敗するべき、静かでなく。
- **各マイノリティクラスタをログ検証可能付き。** マイノリティクラスタは早期警告システム相関エラー。
- **制限ラウンド施行。** 「同意まで議論を続ける」なし。それはシコファント報酬。
- **同意を正確さから分離。** コンセンサス出力検証者に。検証者はアンサンブルから独立。
- **同意率を監視。** 急上昇は適合性バイアス。急落はモデル漂流。

## 演習

1. `code/main.py`を実行。複数失敗単一文化攻撃を確認。CPWBFTは部分的に軽和するとき単一文化信頼度0.7以下。
2. 4つ目の攻撃パターン追加：**静かな棄権** — 1つのエージェント答えを拒否（「わからない」）。各集約が棄権をどのように扱うべきか? 実装選択。
3. セマンティッククラスタリングを文字列正規化から埋め込み類似性にスワップ（任意のオープンソース埋め込みモデル使用）。シコファント攻撃何が起こるか?
4. CP-WBFT（arXiv:2511.10400）を読む。信頼プローブ較正ステップを実装（別の較正モデル各エージェント自己報告信頼をチェック）。単一文化シナリオで精度ゲイン測定。
5. 「AI エージェント が同意できるか?」（arXiv:2603.01213）を読む。簡略スカラー一致実験を再現：3つのエージェント、1つのスカラー質問、欺瞞ペルソナプロンプト。CPWBFTまたはDecentLLMsはそれをキャッチするか?

## キーターム

| 用語 | 人々が言うこと | 実際の意味 |
|------|---|---|
| BFT | 「ビザンチン障害耐性」 | Castro-Liskov 1999プロトコル。`f < n/3`任意障害でのコンセンサス。 |
| ビザンチン | 「任意の悪い動作」 | メッセージドロップ、静かに失敗できるノード — クラッシュセーフではなく何もできない。 |
| 信頼プローブ | 「どのくらい確かですか?」 | 投票に附属自己報告または較正予測確率。 |
| セマンティッククラスタリング | 「同じ答え、異なる言葉」 | 投票カウント前に同等答えをグループ。 |
| 幾何中央値 | 「堅牢な中心」 | サンプルポイントへの距離合計を最小化するポイント。外れ値に堅牢、平均と異なり。 |
| 単一文化 | 「同じモデル、同じ失敗」 | エージェント共有訓練データまたはベースモデル時の相関エラー。 |
| シコファント適合 | 「大きい音の声と同意」 | エージェント投票は誰が最初/最も大きいと話すか方向バイアス。 |
| コア/エッジ | 「階層BFT」 | WBFT分割：小さいコアコンセンサス最初、エッジノード従う。遅延制限。 |
