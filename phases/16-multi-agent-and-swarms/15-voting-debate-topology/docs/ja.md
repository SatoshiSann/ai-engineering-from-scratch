# 投票、自己一貫性、議論トポロジー

> 最安集約：NサンプルN個独立エージェント、多数決。Wang et al。2022自己一貫性は1つのモデルをNサンプル。マルチエージェント拡張異種で単一文化から逃げます。異なるモデル、異なるプロンプト、異なる温度、異なるコンテキスト。多数決を超えて、議論トポロジーが重要：MultiAgentBench（arXiv:2503.01935、ACL 2025）評価スター/チェーン/ツリー/グラフ調整、**グラフが研究に最適**見つけた。~4以上のエージェント「調整税」。AgentVerse（ICLR 2024）2つの顕現パターンドキュメント — 志願行動と適合行動。適合は両方フィーチャー（コンセンサスを見つける）とリスク（グループシンク、レッスン24）。このレッスンはトポロジースペースを地図化、各変種を構築、調整税を測定。

**タイプ:** Learn + Build  
**言語:** Python (stdlib)  
**前提条件:** Phase 16 · 07 (Society of Mind and Debate), Phase 16 · 14 (Consensus and BFT)  
**所要時間:** 約75分

## 問題

議論は精度を改善できます（Du et al。、arXiv:2305.14325）。それはまた低下させることができます。議論が助けるかは4つの構造上の選択に依存：

1. 誰が誰と話すか（トポロジー）。
2. 何ラウンド（Du 2023：ラウンド両方とエージェント独立重要）。
3. エージェントが異種であるか（異なるベースモデル単一文化破壊）。
4. 敵対する声が存在するか（スチールマニング対ストローマニング）。

「5つのエージェント実行と投票」をタスクに付けるチーム単一エージェント対してしばしば後退。失敗はランダムではなく。彼ら追跡トポロジーと異種性。このレッスンはトポロジーマップ。

## コンセプト

### 自己一貫性、シングルモデルベースライン

Wang et al。2022（「自己一貫性は鎖の思考推論を改善」）サンプル同じモデルNタイムズ温度 > 0で多数投票推論パスの答え。結果GSM8K：N=40サンプルで単一欲張り解読上の実質的ゲイン。自己一貫性は単一エージェント先駆者マルチエージェント投票。

制限：自己一貫性は1つのベースモデルを使用。エラーは構成上相関。モデルが系統的バイアスを持つ場合、すべてのNサンプルそれを共有。

### マルチエージェント投票、異種拡張

Nサンプルを*異なる* Nエージェントに置き換え。異なるベースモデル（Claude、GPT、Llama）、異なるプロンプト、異なるツールアクセス。ベネフィット：相関のないエラー。コスト：異なるエージェント異なる額。調整オーバーヘッドを追加。

標準2026年名称異種議論は**A-HMAD** — 敵対異種マルチエージェント議論。普遍的に採用されていない。論文は「異なるモデル議論、単一文化倒壊から相関エラーを減らす」のため用語を使用。

### 4つのトポロジー

```
star                chain               tree                graph

    ┌─A─┐           A─B─C─D         ┌──A──┐              A───B
    │   │                           │     │              │ × │
    B   C                           B     C              D───C
    │   │                          / \   / \
    D   E                         D   E F   G           (fully connected)
```

スター：1つのハブ、全て他はハブのみと話す。スーパーバイザーワーカーに等価。バックチャネルなし。
チェーン：線形、各エージェント先のを見る。パイプラインのような。
ツリー：階層、階層的エージェント（レッスン06）で使用。
グラフ：任意-任意。完全結合最大公約数と任意DAG含む。

### 調整税（MultiAgentBench）

MultiAgentBench（MARBLE、ACL 2025、arXiv:2503.01935）ベンチマークスター、チェーン、ツリー、グラフ。タスクスイート研究、コーディング、計画を含む。主要測定結果：

- **グラフ**トポロジーは研究タスクに勝つ。情報は任意-任意流れる。エージェント互いを批評できる。
- **スター**は高速答え事実タスクに勝つ。ハブフィルター統合。
- **チェーン**は段階パイプライン（段階洗練）に勝つ。
- **調整税**グラフトポロジーで~4エージェント超えて現れ。壁時計とトークンコスト品質より速く成長。

4エージェント上限は経験的でなく基本。2026年LLMコンテキスト容量を反映：各エージェントのコンテキストはピア出力で埋められる。エージェントN+1の限界値はN+1を追加するとき、全員が全員を見たら低下。

### マルチエージェント議論戦略（「MADに行くべきか?」）

arXiv:2311.17371は2023年MAD戦略調査。主要検出レプリケート：自己一貫性（独立サンプリング+集約）に構造的に似ているMAD変種しばしば自己一貫性を同じ予算で使用するとき低下。MADは最もエージェント真に異種で議論に敵対的構造（1つのエージェント引数に対して）があるとき助ける。

### AgentVerse顕現パターン

AgentVerse（ICLR 2024、https://proceedings.iclr.cc/paper_files/paper/2024/file/578e65cdee35d00c708d4c64bce32971-Paper-Conference.pdf）明示的設計なしマルチエージェント議論から浮上する2つの行動ドキュメント：

- **志願。** エージェントは助けを提供（「次のステップをとることができます」）不快迫。有用：最も能力あるエージェントサブタスク仕事に割り当て。
- **適合。** エージェントは批評者と立場を調整、批評者が間違っていても。これは議論等価シコファント（レッスン14）。

適合は何故議論まで一致報酬がいじめ。制限ラウンド別のジャッジは軽減。

### 異種性：精度移動実際のノブ

実用文献2024-2026年パターン：N個あるエージェント1つをあるベースモデルに交換はNを1増やすより大きい精度バンプを与える。直感は単一文化 — 新しい独立エラーソースは追加相関サンプルより価値。

極限で、異種性は数を打つ。3つ異なるモデルは5つのコピー1つのモデル上の大部分のタスク清潔な確定真実を持つを打つ。

### 陪審方法

Sibyl フレームワーク（Minsky-LLM文献に引用）「陪審」を形式化 — 小さい専門エージェントセット答えを各ステージで投票して洗練。平均投票より、陪審は役割を持つ：1つのエージェント交差試験、1つのコンテキストを供給、1つのもっともらしさをスコア。陪審方法は平均投票（安い、単一文化傾向）と完全MAD（高コスト、適合傾向）の中点。

### 投票と議論ドミナント場合

- 質問は確定真実を持つ（事実、数学、コード行動）。投票コンセンサスは意味。
- エージェントが異なるソースまたはツール（異種は利用可能）にアクセス可能。
- ラウンドは制限（通常2-3）と別のジャッジまたはベリファイアがある。
- 予算は3-5エージェント許す。グラフトポロジー上で5-7を超えて、調整税はドミナント。

### 投票と議論が傷つける場合

- 質問は意見形状。エージェント最も自信ありのような答えに転換。正確ではなく。
- すべてのエージェントはベースモデルを共有。単一文化はコンセンサスを無意味にする。
- ラウンドは無制限。適合は毎回勝つ。
- タスクは簡単。単一エージェント自己一貫性N=5は安い正確性と同じ。

## ビルド

`code/main.py`実装：

- `run_star(agents, hub, question)` — ハブは各ワーカーをポール、集約。
- `run_chain(agents, question)` — シーケンシャル洗練。
- `run_tree(root, children, question)` — 階層。深さ2集約。
- `run_graph(agents, question, rounds)` — 全-全議論、制限ラウンド。
- スクリプト化異種ダイアル：各エージェントは`error_bias`その系統的悪さを示す。
- 測定ハーネスは各トポロジーを実行N=3, 5, 7リポート（精度、total_tokens、wallclock_simulated）。

実行：

```
python3 code/main.py
```

期待出力：テーブルトポロジー × N → (精度、トークン、遅延)。グラフはN=3-5研究スタイルタスクで勝つ。スターは高速事実タスクで勝つ。グラフN=7は調整税を示す（遅延精度より速く膨張）。

## 使用

`outputs/skill-topology-picker.md`はスキル。タスク説明を読み、トポロジー推奨（スター/チェーン/ツリー/グラフ）、N（エージェント数）、異種プロフィール（使用ベースモデル）、ラウンド制限。

## 出荷

任意のアンサンブル：

- **自己一貫性から始めるN=5**1つの強いベースモデル使用。それは安いベースライン。
- **異種投票にアップグレードN=3**精度が重要な場合。デルタを測定。
- **議論トポロジーへのアップグレードのみ**タスクが構造を持つ（研究、マルチステップ）と制限ラウンドが可能な場合。
- **常にマイノリティクラスタをログ。** マイノリティが永続的に正しい場合、多様性信号。
- **精度と並行に壁時計とトークンをベンチマーク。** 「10倍コストの精度改善」はビジネス決定。

## 演習

1. `code/main.py`を実行。プロットグラフトポロジー調整税曲線：精度対N、トークン対N。どのNで曲線は膨張しますか?
2. A-HMADを実装：3つのエージェント意図的に異なるバイアス。すべて-同じバイアスベースラインはA-HMADとどう比較するか?
3. グラフトポロジーに「ジャッジ」役割を追加。投票、のみ最終コンセンサスをスコア。これは顕現適合行動を変化させるか?
4. AgentVerseペーパーを読む（ICLR 2024）。実装は最も強くどの顕現行動を展示するか特定。プロンプト変更で反対の行動を引き出せるか?
5. MultiAgentBench（arXiv:2503.01935）セクション4（トポロジー実験）を読む。「グラフが研究を勝つ」結果をハーネス使用からペーパータスク1を再現。
