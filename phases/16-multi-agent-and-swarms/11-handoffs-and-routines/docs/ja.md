# ハンドオフとルーチン — ステートレスオーケストレーション

> OpenAIのSwarm（2024年10月）は、マルチエージェントオーケストレーションを2つのプリミティブに蒸留しました。**ルーチン**（命令+ツールをシステムプロンプトとして）と**ハンドオフ**（別のエージェントを返すツール）。状態マシンなし、分岐DSLなし。LLMはツール呼び出しをして適切なハンドオフツールを呼び出すことでルーティングします。OpenAI Agents SDK（2025年3月）は本番後継者です。Swarmそのものは、最もクリーンな概念的参照のままです。その全ソースが数百行に適合します。パターンはウイルスのように広がっています。APIサーフェスはざっと「agent = prompt + tools; handoff = function returning agent」だからです。制限事項：ステートレス、つまりメモリは呼び出し元の問題です。

**タイプ:** Learn + Build  
**言語:** Python (stdlib)  
**前提条件:** Phase 16 · 04 (Primitive Model)  
**所要時間:** 約60分

## 問題

すべてのマルチエージェントフレームワークはそのDSLを学ぶことを望んでいます。LangGraphノードとエッジ、CrewAIクルーとタスク、AutoGenGroupChatとマネージャー。DSLは現実的な抽象化ですが、それはその物をそれが必要とするよりも重く感じさせます。

Swarmは反対方向に押します。モデルが既に持っているツール呼び出し機能を使用します。ハンドオフはツール呼び出しになります。オーケストレーターは現在会話を保持しているどのエージェントでもあります。状態マシンはエージェントのシステムプロンプト内に含まれています。

## コンセプト

### 2つのプリミティブ

**ルーチン。** エージェントの役割と利用可能なツールを定義するシステムプロンプト。スコープされた指示のセットのようにそれを考える。「あなたはトリアージエージェント。ユーザーが払い戻しについて尋ねたら、払い戻しエージェントに渡す。」

**ハンドオフ。** エージェントが呼び出せるツール。新しいエージェントオブジェクトを返します。Swarmランタイムはエージェント戻り値を検出し、次のターンのアクティブエージェントを切り替えます。

それが全体の抽象化です。

```
def transfer_to_refunds():
    return refund_agent  # Swarm sees Agent return → switch active agent

triage_agent = Agent(
    name="triage",
    instructions="Route the user to the right specialist.",
    functions=[transfer_to_refunds, transfer_to_sales, transfer_to_support],
)
```

トリアージエージェントのシステムプロンプトは、ユーザーメッセージに基づいて適切なハンドオフを選ぶようにしています。LLMのツール呼び出しがルーティングを行います。

### なぜそれはウイルスのように広がるのか

- **小さいAPI。** 学ぶ2つの概念。
- **モデルが既に行うことを使用。** ツール呼び出しは既にプロバイダー全体で本番品質です。
- **状態マシンの負担がない。** グラフを説明しない。エージェントのプロンプトが誰に渡すかを説明します。

### ステートレストレード

Swarmは実行間で明示的にステートレスです。フレームワークは実行中のメッセージ履歴を保持しますが、何も永続化しません。メモリ、継続性、長時間実行タスク—全て呼び出し元の問題。

本番（OpenAI Agents SDK、2025年3月）ではこれが変わった主なものの1つでした。SDKはハンドオフプリミティブを保持しながら、内蔵セッション管理、ガードレール、トレースを追加します。

### Swarm/ハンドオフが適合する場合

- **トリアージパターン。** フロントラインエージェントはユーザーをスペシャリストにルーティングします。
- **スキルベースハンドオフ。** 「タスクがコードを必要とする場合、コーダーを呼び出す。研究を必要とする場合、研究者を呼び出す。」
- **短い、制限された会話。** カスタマーサポート、FAQ-to-チケット、シンプルなワークフロー。

### Swarmが苦労する場合

- **共有メモリとの長いセッション。** ハンドオフは会話状態を新しいエージェントのプロンプトとプラス履歴にリセットします。呼び出し元管理メモリなしでエージェント間の永続状態なし。
- **並列実行。** ハンドオフは1つずつ。アクティブエージェントが切り替わります。並列性は呼び出し元が複数のSwarm実行をオーケストレートする必要があります。
- **監査とリプレイ。** ステートレス実行は正確にリプレイするのが難しい。LLMのハンドオフ選択は決定論的ではありません。

### OpenAI Agents SDK（2025年3月）

本番後継は以下を追加します：

- **セッション状態。** 実行間の永続スレッド。
- **ガードレール。** 入出力検証フック。
- **トレース。** すべてのツール呼び出しとハンドオフはログされます。
- **ハンドオフフィルター。** ハンドオフで何のコンテキストが転送されるかを制御します。

ハンドオフプリミティブが残ります。本番エルゴノミクスはそれの周りに追加されます。

### SwarmはGroupChat

両方ともLLM駆動ルーティングを使用しますが、**次を選ぶ誰**で異なります：

- GroupChat：セレクタ（関数またはLLM）が外部から次のスピーカーを選びます。
- Swarm：現在のエージェントがハンドオフツールを呼び出して後継者を選びます。

Swarmは「エージェントが次に何があるかを決める」。GroupChatは「マネージャーが次に何があるかを決める」。Swarmの決定はアクティブなエージェントのツール呼び出しに住む。GroupChatのは`GroupChatManager`に住む。

## ビルド

`code/main.py`はSwarmをstdlibから実装します。エージェントデータクラス、ハンドオフメカニズム（ツールはエージェントを返す）、エージェント切り替えを検出する実行ループ。

デモ：トリアージエージェントは払い戻し、営業、またはサポートスペシャリストにルーティングします。各スペシャリストは独自のツールを持っています。実行ループが各ハンドオフを出力します。

実行：

```
python3 code/main.py
```

## 使用

`outputs/skill-handoff-designer.md`は与えられたタスクのハンドオフトポロジーを設計します。どのエージェントが存在するか、どのハンドオフを呼び出すか、どのコンテキストを転送するか。

## 出荷

チェックリスト：

- **ハンドオフロギング。** すべてのハンドオフ、from-agent、to-agent、コンテキストスナップショットでトレースイベントを記述します。
- **コンテキスト転送ルール。** ハンドオフで何が移動するかを決定：完全履歴（高コスト）、最後のNメッセージ、または概要。
- **ハンドオフ上のガードレール。** 異なるツールパーミッションを持つスペシャリストへのハンドオフは認証される必要があります。そうでなければプロンプト注入は望ましくないハンドオフを強制できます。
- **ループ検出。** 2つのエージェントが前後にハンドオフするのは一般的な失敗。シンプルな最後のK環チェックで検出します。
- **フォールバックエージェント。** ハンドオフターゲットが存在しない場合、安全なデフォルトにフォールバックします。

## 演習

1. `code/main.py`を実行、払い戻しエージェントにトリアージします。2番目のターンのアクティブエージェントが払い戻しであることを確認します。
2. ループ検出ルールを追加：同じ2つのエージェントが3回連続でハンドオフした場合、強制終了します。フォールバックを設計します。
3. OpenAI Agents SDKドキュメントでハンドオフフィルターを読む。「ハンドオフ上の概要」版を実装：発信エージェントはコンテキストを箇条書き概要に圧縮してから、着信エージェントが引き継ぎます。
4. SwarmハンドオフとGroupChatManagerセレクタを比較します。どのパターンがプロンプト注入をより悪くし、なぜ?
5. Swarmコックブック（https://developers.openai.com/cookbook/examples/orchestrating_agents）を読む。Swarmが行う1つの明示的な設計決定を特定し、OpenAI Agents SDKが変更または保持したもの。

## キーターム

| 用語 | 人々が言うこと | 実際の意味 |
|------|---|---|
| ルーチン | 「エージェントプロンプト」 | システムプロンプト+ツールリスト。役割と利用可能なハンドオフを定義します。 |
| ハンドオフ | 「別のエージェントに転送」 | アクティブエージェントが呼び出せるツール。新しいエージェントを返す。ランタイムはアクティブエージェントを切り替える。 |
| ステートレス | 「実行間のメモリなし」 | Swarmは何も永続化しない。メモリは呼び出し元の責任。 |
| アクティブエージェント | 「今誰が話しているか」 | 現在会話を保持しているエージェント。ハンドオフはこれを変更します。 |
| コンテキスト転送 | 「ハンドオフで何が移動するか」 | 履歴の転送ポリシー：完全、最後のN、または概要。 |
| ハンドオフループ | 「エージェントがピンポン」 | 2つのエージェントが互いに前後にハンドオフし続けるフェイルモード。 |
| OpenAI Agents SDK | 「本番Swarm」 | 2025年3月後継。ハンドオフプリミティブを保持しながら、セッション、ガードレール、トレースを追加。 |
| ハンドオフフィルター | 「転送で制御」 | SDKフィーチャー。ハンドオフ境界でコンテキストを検査・修正します。 |
