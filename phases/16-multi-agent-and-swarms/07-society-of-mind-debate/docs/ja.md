# マインドの社会とマルチエージェント・デベート

> Minsky 1986 前提 — 知能は専門家の社会 — 毎十年再発見される。2023年にDu et al.はそれを具体的なアルゴリズムに変えた。複数のLLMインスタンスは答えを提案し、互いの答えを読み、批判、更新。N ラウンド上で彼らはゼロショット CoT と6つの推理と正確性タスクの上の反射を勝つコンセンサスに収束。2つの結果が問題。**複数エージェント**と**複数ラウンド**は独立に貢献。社会は単一エージェント独白を勝つ。マルチラウンド・エクスチェンジはワン・ショット・投票を勝つ。

**タイプ:** Learn + Build
**言語:** Python (stdlib)
**前提条件:** Phase 16 · 04 (Primitive Model)
**所要時間:** ~60 minutes

## 問題

自己一貫性 — 1つのモデルを何度もサンプル、多数答えを取る — あなたが推理改善に取り付けることができる最も安いもの。それは機能するが、速く飽和。2倍サンプルして、別の意味のあるジャンプを見ない。

デベートは飽和を壊す。N の1つのモデルから独立サンプルの代わりに、N エージェントは互いの推理を読み、改める。サンプル間の相関は落ちる（彼らはもはや i.i.d. ではない）、収束ポイントはしばしば i.i.d. 投票が信頼して誤ったところで正しい。

## コンセプト

### Du et al. 2023 アルゴリズム

arXiv:2305.14325 から（ICML 2024）。

1. N エージェント単位は質問に初期答えを生成。
2. ラウンド r = 2..R の場合。各エージェントは他のエージェント r-1 答えを示され「これらを考えて、更新答えを与える」と言われ。
3. R ラウンド後、最終答えの多数投票。

論文はMMUL、GSM8K、伝記、MATH、正確性ベンチマークでテスト。デベートはシステマチック CoT と自己反射を勝つ。

### 2つの独立なノブ

同一論文からアブレーション。

- **エージェント・カウント単位**（1ラウンド、N の多数投票）は単一エージェント上のほとんどのタスクで勝つが、プラトー。
- **ラウンド・カウント単位**（1つのエージェント独自の前の推理を見る）ほぼ助けない — 反射の既知の弱点。
- **一緒に両方**大きなジャンプを生成。マルチ・エージェント・エクスチェンジは複数ラウンド間でドライブゲイン。

### なぜそれは機能するか

2つのメカニズム。

1. **意見相違への露出。** エージェントが別のエージェントの推理チェインと異なる結論を見るとき、それはいずれかを正当化するか更新する必要がある。どちらの方法も、ラウンド r+1 のコンテキストはラウンド r より豊か。
2. **相関エラー削減。** 自己一貫性では、すべてのサンプルは同じモデルからくるため、エラーは相関 — あなたはの自信自信をもって間違った答えに平均。異なるモデルまたは異なるシード・デコレート。異なる*デベート・ビュー*さらに分散。

### ヘテロジーニアス・デベート

A-HMAD と関連フォローアップ*異なるベース・モデル*異なるエージェント用に使用。Llama+Claude+GPT デベート1つのモデル・ファミリーの相関エラーは他では共有されないため単一文化崩壊（Lesson 26）を削減。

ダウンサイド。弱いモデルがデベートに参加は合意をその間違った答えに向けてドラッグできる（「MAD に行くべきか?」、arXiv:2311.17371 を参照）。

### NLSOM — 129エージェント・エクステンション

Zhuge et al.（「自然言語ベースのマインドの社会」、arXiv:2305.17066）このアイデアを129メンバー社会にスケール。結果。特化と自己組織化は規模で出現、システムは視覚質問応答のような タスクを単一エージェント上で外出フォーム。

### 失敗モード

- **イエスマンカスケード。** すべてのエージェント最も自信のあるエージェント・デファー。デベートは最もうるさい声に崩壊。プロンプト対立的なロール（「1つのエージェント一定逆位置を主張」）がヘルプ。
- **トピック・ドリフト。** 多くのラウンド上のデベート・ドリフト・オリジナル・クエリから。軽減。すべてのラウンド・クエリを再注入。
- **計算ブロウ・アップ。** N エージェント × R ラウンド = N·R LLM 呼び出し、コンテキストが大きくなるそれぞれ。5エージェント、5ラウンド・デベートは成長コンテキストで25呼び出し。クエリ当たりのコストはシングル CoT 呼び出し 10× を超過できる。

## 作成する

`code/main.py`は3エージェント × 3ラウンド・デベートを各エージェントがわずかに異なる（可能性のある間違い）答えで始まる数学質問で実行。エージェントはスクリプト化 — それぞれ「更新」隣接メディアンをスクリプト化自信をもって重み付け平均化。収束はログを通じてラウンド・バイ・ラウンド見える。

デモは2つのキー効果を示す。

- 単一ラウンド・エクスチェンジはエージェント正答に近い。
- ラウンド2過去の余分なラウンド頭打ち収益を示す（Du et al. プラトーとマッチ）。

実行する。

```
python3 code/main.py
```

## 使用する

`outputs/skill-debate-configurator.md`新しいタスク用のデベートを設定。エージェント数、ラウンド数、異質性（同じモデル vs 混合）、ロール割り当て（対称 vs 1つ対立的）。これはトークン・コストを実行する前に推定。

## 出荷する

デベートをシップするとき。

- **ラウンドをキャップ3。** Du et al. はゲインの大部分を3ラウンドが捕捉を示す。ここより多い費用、質ではない。
- **エージェントをキャップ5。** 5を超え、コンテキスト肥大化と費用支配。
- **異質性がデフォルトで。** プール内の少なくとも2つの異なるベース・モデル。
- **対立スロット。** 1つのエージェントは関わらず同意を指示。イエスマンを壊す。
- **ログすべてのラウンド。** 中央のラウンドを隠すデベート・システムをデバッグまたは監査できない。

## 演習

1. `code/main.py`を実行してから、ラウンド・カウントを5に設定してヘッドがあるなことを見。追加の収束が止まるのはどのラウンド?
2. 4番目のエージェントを対立的なロールで追加。これは収束を壊すか改善するか?
3. プロット（印刷）合意スコア/ラウンドあたり（多数答えの分数上のエージェント）。それはいつ1.0を打つかおよびそれは「正しい」に相当するか?
4. Du et al.セクション4アブレーションを読む。「エージェント・単位」vs「ラウンド・単位」vs「両方」結果をこのコードを使用して複製。
5. 「MAD に行くべきか?」（arXiv:2311.17371）を読んで、ラウンド・ロビンを超えたデバート変種の2つをリスト — 例えば、裁判官が率い、デベートの鎖、対立的。

## キーターム

| ターム | 人々が言うこと | 実際には何を意味するか |
|------|----------------|------------------------|
| マインドの社会 | 「Minsky のアイデア」 | インタアクティング・スペシャリストとしての知能。1986フレーミング今 LLM デベート経由で運用。 |
| マルチエージェント・デベート | 「エージェントは議論」 | N エージェントが提案、互いに批判、R ラウンド上の改めて、多数投票。 |
| コンセンサス | 「彼らは同意」 | 認識的真実ではない — 多数答えの分数。自信をもって誤っているかもしれない。 |
| ラウンド | 「エクスチェンジ・ステップ」 | 1ラウンド = 各エージェント・オーザーを読み、1度更新。 |
| ヘテロジーニアス・デベート | 「ミックス・モデル・ファミリー」 | 異なるベース・モデル使用エラーを分散。 |
| イエスマン・カスケード | 「すべてが最もうるさい1つに同意」 | デベート失敗どこエージェント正確さに関わらず最も自信のあるエージェントに延期。 |
| NLSOM | 「129エージェント社会」 | 自然言語マインドの社会。Zhuge et al. スケール・バージョン。 |
| 相関エラー | 「同じモデル、同じバグ」 | 自己一貫性が飽和する理由。異なるビュー・アクロス・デベート分散。 |
