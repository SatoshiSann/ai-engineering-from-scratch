# LLMのための群れ最適化（PSO、ACO）

> バイオインスパイアされた最適化はLLMの返品を作成しています。**LMPSO** (arXiv:2504.09247) は、各粒子の速度がプロンプトであり、LLMが次の候補を生成するPSOを使用します。構造化されたシーケンス出力（数式、プログラム）でよく機能します。**モデル群**（arXiv:2410.11163）各LLMエキスパートをモデル重みマニフォールド上のPSO粒子として扱い、9つのデータセットで**平均13.3%のゲイン**を報告します。わずか200のインスタンス。**SwarmPrompt** (ICAART 2025) はプロンプト最適化のためにPSO +グレーウルフをハイブリッド化します。**AMRO-S** (arXiv:2603.12933) はマルチエージェントLLMルーティング用のACOインスパイア香水専門家です—**4.7倍のスピードアップ**、解釈可能なルーティング証拠、非同期更新をゲートで品質低下させてキルを解凍し、推論学習から学習。このレッスンはプロンプトパラメータスペースでPSOを実装し、エージェントルーティングでACOを実装し、なぜこれらの古典的なアルゴリズムがLLM時代に適しているのか、そしていつそうでないかを測定します。

**タイプ:** 学習 + 構築
**言語:** Python (stdlib)
**前提条件:** フェーズ16 · 09 (並列群ネットワーク)、フェーズ16 · 14 (合意とBFT)
**所要時間:** 約75分

## 問題

タスク評価で62%のスコアを取得するプロンプトがあります。改善したいです。素朴な動きは勾配のない手動の調整で、悪くスケーリングします。強化学習は報酬信号と十分なロールアウトを訓練する必要があります。プロンプトを通した逆プロプは本当に可能ではありません—プロンプトは差別化可能なパラメータではなく、離散的な文字列です。

古典的なバイオインスパイア最適化—連続的な検索空間のためのPSO、パス選択のためのACO—はこのレジームのために正確に設計されました：勾配のない、人口ベース、評価ごとに安価。LLMをペアリングして勾配のない検索ステップを取得し、驚くほど実用的なオプティマイザを取得します。

同じパターンがマルチエージェントシステムの**ルーティング**エージェントに適用されます。ACOスタイルのフェロモン軌跡は、どのエージェントがどのタスク型で最もよく機能したかを記録し、ルーターが軌跡を悪用でき、フェロモンが減衰してルートを再発見できます。

## コンセプト

### PSOリフレッシャー（Kennedy＆Eberhart 1995）

粒子群最適化：連続的な検索空間内の粒子の集団。各粒子には位置`x_i`と速度`v_i`があります。各イテレーション：

```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
evaluate fitness(x_i)
update p_best_i if improved
update g_best if global best
```

ここで`p_best`は粒子の独自の最高、`g_best`は群の最高、`w, c1, c2`は慣性+認知+社会的重み、`r1, r2`はランダム係数です。

### LLM出力のPSO—LMPSO

arXiv:2504.09247 はLLMが生成した構造化出力（数式、プログラム）のためにPSOを適応させます。各粒子は候補出力です。速度は、現在の出力を個人的/グローバルな最高に向けて修正する方法を説明する**プロンプト**です。LLMは速度プロンプトから新しい出力を生成します。速度の「慣性」は「小さな増分の変更を加える」のようなプロンプトです。

これは次の場合にうまく機能します：
- 出力は構造化されています（解析可能、評価可能）。
- フィットネスは自動的です（テスト実行、算術評価）。
- 集団は小さいです（約10〜30粒子）ため、総LLM呼び出しは管理可能なままです。

フィットネスが人間のレビューを必要とする場合、機能しません—繰り返されたコストが禁止的になります。

### モデル群

arXiv:2410.11163 はPSOを出力層から**モデル**層にします。各「粒子」はエキスパートLLM（パラメータ）です。群は勾配のないアップデート経由でパラメータを集合的な最高に移動します。報告：9つのデータセット上の12ベースラインの平均13.3%ゲイン、反復ごとにわずか200のインスタンス。

重要な洞察は、LLMエキスパートモデルが共有パラメータマニフォール（アダプタの重み、LoRA deltas）ですでに近いことです。この低次元部分空間でのPSOは安価で効果的です。

### ACOリフレッシャー（Dorigo 1992）

アリのコロニー最適化：蟻はグラフを横断します。各パスにはフェロモン軌跡があります。アリの移動確率はフェロモンの強さによって重み付けされます。タスクを完了するアリはソリューション品質に比例してフェロモンを堆積します。フェロモンは時間の経過とともに減衰します。

### AMRO-S—エージェントルーティング用のACO

arXiv:2603.12933 はマルチエージェントルーティングにACOを使用します。各タスク型は「宛先」です。各エージェントは可能なルートです。フェロモンは良い出力を生成するルートを強化します。主な貢献：

- **解釈可能なルーティング証拠。** フェロモンの強さは人間が読める信号です。
- **品質ゲート非同期更新。** フェロモンは品質チェックが通った後にのみ更新され、推論から学習を分離します。
- **マルチエージェントルーティングベンチマークで4.7倍のスピードアップ**。

品質ゲートの問題：なしでは、高速で誤りがあるエージェントはフェロモンを蓄積し、システムは悪いルートにロックインします。

### LLMにPSO/ACOを使用する場合

**PSO を使用する場合：**
- 検索スペースは連続的または連続パラメーター（プロンプト埋め込み、LoRA重み、数値生成パラメーター）にマップします。
- フィットネスは安価で自動的です。
- 人口は小さい（10-30）することができます。

**ACO を使用する場合：**
- ルーティングまたはパス選択の問題があります。
- 決定は時間の経過とともに強化されます（同じタスク型が戻ってきます）。
- ルーティング決定の解釈可能な証拠が必要です。

**どちらも使用しないでください。**
- フィットネスには人間のレビューが必要です（繰り返されたコストが多すぎます）。
- 検索スペースは離散的で組み合わせ的な方法でPSOがカバーしない（代わりに遺伝的アルゴリズムを使用）。
- リアルタイム決定は厳密なレイテンシが必要です（PSO/ACOは単一パス適切なものに対して遅く収束します）。

### なぜバイオインスパイアはまだ勝つ

勾配ベースの方法は差別化可能な信号が必要です。LLM出力とルーティング決定は自明に差別化可能ではありません。疑似勾配法（強化学習ルーター、DPOスタイルのプロンプトチューナー）は機能しますが、高価な訓練が必要です。

PSO と ACO は評価者関数のみが必要です。候補出力またはルーティング決定をスコアリングできる場合、スペース上で最適化できます。それは適用可能性のバーをはるかに低くします。

### 実用的な制限

- **集団予算。** N粒子×T反復×評価ごとのコスト。LLM評価で約$0.02/呼び出しで、20粒子PSO実行50反復は約$20の費用がかかります。したがって計画します。
- **探索対搾取。** フェロモン減衰率とPSOの慣性トレードオフ。減衰が速すぎる→ソリューションを忘れる。遅すぎる→早い局所的最適値にスタック。
- **破壊的なドリフト。** フィットネスランドスケープが移動する場合、アルゴリズムは両方とも収束して分岐する可能性があります（新しいデータ分布）。最高のフィットネスの安定性を監視します。

## 構築する

`code/main.py`を実装：

- `LMPSO` — 数値プロンプトパラメーター（温度、top_kの重み）上のPSO。各粒子の「LLM生成」はスクリプト化されたフィットネス関数として シミュレートされます。30回の反復を実行し、g_best収束を表示します。
- `AMRO_S` — ACOスタイルのルーティング。3エージェント、4タスク型、フェロモンマトリックス、ルーティングされた100タスク。（task_type→エージェント選択）時間の経過に伴う分布を印刷してトレイル形成を表示します。
- 比較：同じタスクストリームでのランダムルーティング対ACOルーティング。品質とレイテンシを測定します。

実行：

```
python3 code/main.py
```

予想される出力：
- LMPSO：g_best フィットネスは30回の反復で最適近くにランダムから改善します。
- AMRO-S：フェロモンテーブルはタスク型ごとに正しいエージェントで安定しています。ACOルーティングは品質で約30～40%とレイテンシでランダムを打ちます（より少ないリトライ）。

## 使用方法

`outputs/skill-swarm-optimizer.md` は、LLM/エージェント最適化の問題のためにPSO、ACO、遺伝的アルゴリズム、および勾配ベースのオプティマイザーの間で選択するのに役立ちます。

## 出荷する

- **小さくスタート。** 10-20粒子、20-50回の反復。収束曲線が明らかなゲインを示す場合のみスケール。
- **反復ごとにフェロモンまたはg_bestをログします。** トレイルなしで群れのオプティマイザーをデバッグするのは痛いです。
- **品質ゲート更新。** 特にACOルーティング：高速で誤りのあるエージェントはフェロモンを蓄積してはいけません。
- **分布シフトで減衰をリセットします。** 評価分布が変わる場合、古いフェロモンは古くなっています。一時的に減衰率をリセットまたは2倍にします。
- **反復ごとのコストをキャップします。** 反復ごとのコストメトリクスを出力します。コスト$500/反復で0.5%を獲得するPSOは出荷できません。

## 演習

1. `code/main.py`を実行します。LMPSOの収束を観察します。集団サイズを5、10、20、50に変更します。時間から収束が飽和するサイズは何ですか？
2. **破壊的ドリフト**の実験を実装します：反復30後、フィットネス関数を変更します。PSO はどのくらい速く適応しますか？`p_best`をリセットすると役に立ちますか？
3. AMRO-S に品質ゲートを追加します：フェロモン堆積のみ評価スコア> 0.7の実行で。これはゲートなしバージョン対収束をどのように変えますか？
4. LMPSO (arXiv:2504.09247)を読んでください。紙の「速度をプロンプトとして」をあなたの数値速度に戻します。シミュレーションで何が失われ、何が保存されますか？
5. AMRO-S (arXiv:2603.12933)を読んでください。分離された「推論高速パス」を非同期フェロモンアップデートで実装します。これはサスティンド負荷下でシステムレイテンシをどのように変えますか？

## 主要用語

| 用語 | 人々が言うこと | 実際に意味すること |
|------|----------------|------------------------|
| PSO | 「粒子群最適化」 | Kennedy-Eberhart 1995。集団ベースの勾配のないオプティマイザー。 |
| ACO | 「アリ式コロニー最適化」 | Dorigo 1992。フェロモン軌跡経由のパス/ルート最適化。 |
| LMPSO | 「LLM生成を使用したPSO」 | arXiv:2504.09247。速度はプロンプト;LLMは候補を生成します。 |
| モデル群 | 「エキスパートの重みでPSO」 | arXiv:2410.11163。モデルパラメーター部分空間での勾配のないアップデート。 |
| AMRO-S | 「エージェントルーティング用ACO」 | arXiv:2603.12933。タスク型×エージェント上のフェロモンマトリックス。 |
| p_best / g_best | 「個人的/グローバルな最高」 | 粒子ごとおよび群全体の最高のソリューション。 |
| フェロモン | 「ルーティングメモリ」 | エッジの強さ。時間の経過とともに減衰。品質で堆積します。 |
| 品質ゲート更新 | 「良い実行からのみ学ぶ」 | 品質チェックで条件付けフェロモン堆積。 |
| 破壊的ドリフト | 「分布シフト」 | フィットネスランドスケープの変更。古いp_bestとフェロモンは古くなります。 |
