# ネイティブスパース注意(DeepSeek NSA)

> 64kトークン、注意は70-80%デコード レイテンシを食べる。すべてのオープン モデル ラボは修正計画を持つ。DeepSeek NSA(ACL 2025ベストペーパー)粘着1つ:3つ並行注意枝--圧縮粗粒度トークン、選択的に保存微細粒度トークン、スライディング ウィンドウ ローカル コンテキスト--学習ゲートを通じ組み合わせ。それは ハードウェア配列(カーネルフレンドリー)、ネイティブ訓練可能(事前訓練で機能、推論でボルトオン、またいいえ)、64k デコード上 FlashAttention 実行しながらもフル注意品質に一致またはビートより高速。このレッスンは3つ枝エンドツーエンドを構築し、スパース性がエンドツーエンド微分可能である理由を示す。

**タイプ:** ビルド
**言語:** Python (標準ライブラリ)
**前提条件:** フェーズ7·12(KVキャッシュ、フラッシュアテンション)、フェーズ7·15(注意亜種)、フェーズ10·16(微分注意)
**所要時間:** 約60分

## 学習目標

- 短い合成シーケンスで3枝組み合わせを実装stdlib Pythonおよびゲート重みを確認する。

## 問題

シーケンス長N 完全注意費用`O(N^2)`時間および`O(N)` KVキャッシュ毎レイヤー。64kトークンで、計算およびメモリ帯域幅の数は大惨事。NSAペーパーから測定理論見積:注意は64kで総デコード レイテンシの70-80%をアカウント。

スパース注意は明白な答え。前の試みは2つバケツ落ちる。固定パターン スパース(スライディング ウィンドウ、ストライド、ブロック ローカル)情報を投げ捨て、長い範囲 リコールタスクで失敗。推論時 スパース(KVキャッシュ刈り込み、H2O、StreamingLLM)密な注意で事前訓練モデルに適用され、潜在スピードアップの分数だけ回復モデル決してスパース パターンを通じて情報をルートするよう求めかもしれないため。

ネイティブ スパース注意(Yuan et al.、DeepSeek+PKU+UW、ACL 2025ベストペーパー、arXiv:2502.11089)は両方:スパース性パターン モデル事前訓練中学習、実装カーネル配列アルゴリズム実際推論で計算貯蓄を配信。2年前、NSA またはダイレクト 子孫はすべてのフロンティア長いコンテキスト モデルで既定注意。

## コンセプト

### 3つ並行枝

各クエリのために、NSA 3回注意を実行、KVキャッシュの3つ異なるビューに対:

1. **圧縮枝。** トークンはブロック`l`(通常32または64)にグループ化。各ブロックは小さい学習MLPを通じ単一要約トークンに圧縮。クエリは圧縮トークン、全シーケンスの粗粒度ビューを取得して出席。

2. **選択枝。** 圧縮枝から注意スコアを使用、トップkブロック最も関連電流クエリ特定。微細粒度(非圧縮)トークンそれらのブロックから読まれおよびクエリは彼らすべてに出席。圧縮枝注意 ルーティング シグナル選択として思う。

3. **スライディング ウィンドウ枝。** クエリ最新`W`トークンに出席(通常512)ローカル コンテキスト用。この枝は構造重いショート レンジ パターンをキャプチャ(構文、ローカル coreference)他が見落とすかもしれないそれ。

3つ枝出力は学習毎位置ゲート通じ組み合わせ:

```
out = g_cmp * out_cmp + g_sel * out_sel + g_win * out_win
```

`g_cmp、g_sel、g_win`はクエリ上小さいMLPから ゲート重み。それら1に合計する必要はない--彼らは枝を独立に重くできる。

### なぜこれが"ネイティブ訓練可能"

選択ステップ(トップk ブロック)は離散。離散操作勾配フロー壊す。前 スパース注意選択を通じてバックプロップをスキップした(訓練制限)またはで連続緩和を使用しなかった推論でリアル スパース性与え。

NSAはこれを迂回:圧縮枝注意は全シーケンスの微分可能粗粒度注意。トップk操作だけ再利用最上注意スコア圧縮枝から決定どのブロック ロード メモリから。勾配圧縮枝スコアを流れ込む(影響両方圧縮出力AND選択ロジック)、および選択ブロック貢献最終出力も微分可能。非微分可能`top_k`操作は計算グラフでいいえ op--それのみ制御ロード メモリからどのブロック。

これはなぜNSA事前訓練エンド エンドで使用できるのか。モデルは学習情報3つ枝を通じルート共同、スパース パターンを生成推論実際スピードアップ約束を配信。

### ハードウェア配列カーネル

NSAのカーネルは最新のGPU メモリ階層のために設計。カーネルはロードクエリGQA グループ(外部 ループ)、そのグループ(内部ループ)当たりに対応するスパース KVブロック取得、SRAMで注意実行。各クエリ グループがそれら同じ選択ブロックを見るため(選択は毎クエリ グループ、いいえ毎クエリ ヘッド)、KVロードはグループ全体で償却。演算強度高いまま。

ペーパーは 9倍 Triton カーネルが64k デコード上 FlashAttention、スピードアップ 比率ですより高速実行シーケンス長を報告。フォワード および逆カーネルの両方与えられた。

### 計算バジェット

`N` = シーケンス長、`l` = 圧縮ブロック サイズ、`k` = トップk選択 カウント、`w` = スライディング ウィンドウ、`b` = 選択ブロック サイズ(通常`l`に等しい)。

- 圧縮枝:`O(N/l)`キー 毎 クエリ、そのため`O(N * N / l)`総。
- 選択枝:`O(k * b)`キー毎クエリ、そのため`O(N * k * b)`。
- スライディング枝:`O(w)`キー毎クエリ、そのため`O(N * w)`。

総:`O(N * (N/l + k*b + w))`。

`N = 64k、l = 64、k = 16、b = 64、w = 512`で:毎クエリ費用は`1000 + 1024 + 512 = 2536`キー。完全注意は`64000`キー。25倍計算削減。

`N = 128k、l = 64、k = 16、b = 64、w = 512`で:毎クエリ 費用は`2000 + 1024 + 512 = 3536`キー。完全注意は`128000`キー。36倍削減。利益は シーケンス長、それはポイント全体で成長。

## 構築それ

`code/main.py`3つ枝を短い合成シーケンスで実装し、表示:

- 圧縮MLP(シンプル平均 プール ベースラインは教育学的明確用に使用される;リアルNSA学習MLPを使用)。
- トップkブロック選択圧縮枝スコアによって駆動。
- スライディング ウィンドウ注意最後`w`トークン上。

- フル注意へのコンピュート カウント印刷比較。

### ステップ1:圧縮トークンを ブロック

```python
def compress(K, l):
    n = len(K)
    n_blocks = (n + l - 1) // l
    out = []
    for b in range(n_blocks):
        start, end = b * l, min((b + 1) * l, n)
        block = K[start:end]
        summary = [sum(row[d] for row in block) / len(block) for d in range(len(K[0]))]
        out.append(summary)
    return out
```

### ステップ2:圧縮枝注意

クエリ圧縮キーを対する実行ソフトマックス注意。圧縮枝スコアトップk選択ため二重シグナル。

### ステップ3:トップkブロック選択

最高スコア圧縮ブロック`k`のインデックスを選択。元の圧縮 トークンからロード、その上出席実行。

### ステップ4:スライディング ウィンドウ注意

最後を取る`w`トークン、標準注意実行。

### ステップ5:ゲート+組み合わせ

クエリ上小さいMLP 3つゲート重み生成。最終出力は3つ枝出力の重み付き合計。

### ステップ6:コンピュート カウント

毎クエリ各枝から、総デコンプ キーの数を印刷。完全注意`N`(に比較。1024トークン合成`l = 32、k = 4、w = 128`、NSA見て`32 + 128 + 128 = 288`キー 毎クエリ対1024フル注意--3.5倍少なくキー毎クエリ。

## 使う

NSA DeepSeekの独自長いコンテキスト事前訓練パイプラインで出荷。統合状態公開推論スタック2026年4月の現在:

- **DeepSeek内部**:ネイティブ、発表 重み NSA またはその後継DSA(Deepseek スパース注意)を使用。
- **vLLM**:実験NSAサポート DeepSeek-V3.x重み 開発。
- **SGLang**:NSA ベンチマーク発表;本番パス vLLMフォロー。
- **llama.cpp/CPU**:サポート いいえ;カーネル分解 オーバーヘッドが CPU スループット価値なし。

いつNSA到達するのか:

- 事前訓練またはコンテッドトレーニング実行64k+コンテキストをターゲット。
- 長いコンテキスト チェックポイント DeepSeekの推論。重みは NSA ネイティブ。

ないとき:

- 既存密な注意事前訓練モデル提供。継続訓練なしNSA改造できない。
- コンテキスト下16k。3枝オーバーヘッドが貯蓄支配。
- インタラクティブ チャットでバッチ1。レイテンシ 敏感 デコード 利益、のみ長いコンテキスト。
