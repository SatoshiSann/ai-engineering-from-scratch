# 推測的デコーディングおよびEAGLE

> フロンティアLLM生成トークン1つは数十億のパラメータ全体フォワード パスが必要。フォワード パス大幅に過剰供給される:ほとんどの時間ずっと小さいモデルが正しく次の3-5トークンを推測でき、ビッグ モデル だけが*検証*推測必要。推測が正しい時、5トークンを1の価格でした。推測的デコーディング(Leviathan et al.2023)このは正確作成、およびEAGLE-3(2025)受け入れレート4.5トークン毎検証へ--4-5倍スピードアップが一致した出力分布。

**タイプ:** ビルド
**言語:** Python (numpy)付き
**前提条件:** フェーズ10 レッスン12(推論最適化)、フェーズ10 レッスン04(ミニGPT事前訓練)
**所要時間:** 約75分

## 問題

H100上の70B級モデルのデコード スループット 通常40-80トークン/秒。各トークンはHBMから すべてのモデル重みを読み込む完全フォワード パスが必要。モデル同じ出力変更なしより小さくできない。バッチ サイズをメモリを超えて増加させることできない。固い--推測的デコーディングを除き、モデル フォワード パスあたり複数トークン出力できるなら。

オートリグレッシブ 生成外部 シリアルに見える:`x_{t+1} = sample(p(· | x_{1:t}))`。しかしそこに並行性 機会。もし"次の4トークンは多分 [a、b、c、d]"と言う安い予測子を持ったなら、検証できる5つポジション すべての単一フォワード パスビッグ モデルのと受け入れ最長一致プレフィックス。

Leviathan、Kalai、Matias(2023、"推測的デコーディングを通じトランスフォーマーから高速推論")これは賢い受け入れ/拒否ルール通じ正確ターゲット モデルのサンプリング分布を保つ。同一出力分布、2-4×高速化。

## コンセプト

### 2つモデル セットアップ

- **ターゲット モデル`M_p`**:大きい、遅い、高品質モデル サンプル欲しい。分布:`p(x)`。
- **ドラフト モデル`M_q`**:小さい、高速、低品質モデル。分布:`q(x)`。5-30×小さい。

毎ステップ:

1. ドラフト モデル提案`K`トークン オートリグレッシブ:`x_1、x_2、...、x_K ~ q`。
2. ターゲット モデル実行1つフォワード パス`K+1`位置 すべてを通じ並行で、製作`p(x_k)`提案 すべてのトークンのため。
3. 受け入れ/拒否 各トークン左 右へ下の改良拒否 サンプリング ルール。受け入れ 最長一致 プレフィックス。
4. いくつか トークン拒否されるなら、補正分布 から補正をサンプル、停止。さもなければ、ボーナス トークンを`p(· | x_1...x_K)`からサンプル。

ドラフトが正確ターゲット場合合致、あなた取得K+1トークン毎ターゲット フォワード。ドラフト位置1で間違った場合、1トークンのみを取得。

### 正確ルール

推測的デコーディング**は正確 分布上 サンプリングと同等**p。拒否ルール:

```
各 ドラフトされた トークン x_t:
    r ~ Uniform(0、1)
    if r < p(x_t) / q(x_t):
        accept x_t
    else:
        sample 置換 残差から:(p - q)+ / ||(p - q)+||_1
        stop
```

ここに`(p - q)+` pointwise 差の正部分を示します。ドラフトおよびターゲット同意する場合(`p ≈ q`)受け入れが 近いゼロに1。彼ら不同意する、残差分布はそのため建設全体のサンプル仍然`p`。

**グリーディー ケース。** 温度=0サンプリング ただ`argmax(p) == x_t`チェック。はい、受け入れ;いいえ、出力`argmax(p)`止。

### 予期スピードアップ

ドラフト モデルの毎トークン受け入れレートは`α`ターゲット フォワード毎生成されたトークンの期待は:

```
E[tokens] = (1 - α^{K+1}) / (1 - α)        # K = ドラフト長、α in [0、1]
```

`α = 0.8、K = 4`で:`(1 - 0.8^5)/(1 - 0.8) = 3.36`トークン毎フォワード。単一ターゲット フォワード費用大体`cost_q * K + cost_p`(K ドラフト ステップ プラス1つターゲット 検証)。`cost_p >> cost_q * K`場合スピードアップ比率は`3.36× / 1 = 3.36×`スループット上。

唯一の リアル パラメータは`α`、どれ全く依存ドラフト ターゲット 配置。良いドラフト がすべて。

### 訓練ドラフト:蒸留

ランダム 小さいモデルはドラフトを悪い作る。標準レシピは蒸留:

1. 小さいアーキテクチャを選ぶ(~1B 70B ターゲット、~500M 7B ターゲット)。
2. 実行 ターゲット モデル大きいテキスト コーパス;保存 その 次トークン分布。
3. 訓練ドラフトKL 分岐ターゲット分布に対して(トークン 地上に対して いいえ)。

結果:`α`通常チャット 自然言語上0.6-0.8コーディング、0.7-0.85上。スピードアップ本番で2-3×。

### EAGLE:ツリー ドラフティング + 特徴 再利用

Li、Wei、Zhang、Zhang(2024、"EAGLE:推測的サンプリング 機能不確定 リシンク")観察2つ非効率標準推測的デコーディングで:

1. ドラフトK 連続ステップ、各フル スタック。しかしドラフト再利用できる ターゲット特徴(隠し状態)最新検証から--ターゲット既に豊かな表現計算している ドラフトが スクラッチから再導出。
2. ドラフト出力線形チェーン。場合ドラフト出力できた*ツリー* 候補(各ノード複数推測)、ターゲットの単一フォワード パス検証複数 候補パス並行ツリー注意マスクを通じ、最長受け入れ枝ピック。

EAGLE-1の変更:
- ドラフト 入力 = ターゲットの最後層隠し状態位置 t、生トークンなし。
- ドラフトアーキテクチャ = 1つトランスフォーマー デコーダ層(別の小さいモデルなし)。
- 出力 = ツリーK = 4-8候補毎深、深4-6。

EAGLE-2(2024)動的ツリー トポロジーを追加:ツリー ドラフト不確実 広く成長およびそこにそれ留まる自信 。引き上げ`α_effective`検証費用増加なし。

EAGLE-3(Li et al.2025、"EAGLE-3:訓練時テスト通じ大型言語モデル推論加速スケール")固定トップ層 機能依存削除およびドラフト 訓練で新しい"テスト時シミュレーション"損失 -- ドラフト訓練の出力 マッチしたターゲットのテスト時分布ではなく 教師 強制 訓練分布。受け入れレート0.75(EAGLE-2)からは0.82(EAGLE-3)の上昇、平均トークン/検証 3.0から4.5に。

### ツリー注意検証

ドラフトが出力 ツリーの場合、ターゲット モデル検証使用**ツリー注意マスク** --因果マスク純粋線の代わりにツリー トポロジーエンコード。各トークン出席のみに ツリー内その祖先へ。検証パス 仍然1つフォワード、1つmatmul;トポロジー マスク費用のみいくつかエキストラKV エントリ。

```
        根
       /    \
      a      b
     / \    / \
    c  d   e   f
```

場合`a、b`競う最初トークン候補および`c、d、e、f`は2番目トークン候補、すべて 6つポジション 検証される1つフォワード パス。出力は最長プレフィックス どの受け入れられた パス。

### いつ それ 勝つ、いつ いいえ勝つ

**勝つ:**
- チャット/完了予測可能テキストと(コード、一般的な英語、構造化 出力)。`α`高い。
- 設定 GPU未使用コンピュート ルーティング デコード(メモリ境界フェーズ)。ツリー ドラフティング使用 利用可能 FLOP。

**失う / なし勝つ:**
- 非常にストキャスティック 出力(高温度ですべてクリエイティブ書き込み)。`α`ドロップ 方向 `1/|vocab|`へ。
- バッチ サービング 非常に高い同時処理 -- バッチング 既に FLOP埋めため、ツリー検証用のルーム わずか。
- 非常に小さいターゲット モデル ドラフト そのほぼ小さい ここに ウサギ。

本番店 通常報告 2-3×ウォール 時計スピードアップチャット上、3-5×コード生成、クリエイティブ執筆でほぼゼロ。

## ビルドそれ

`code/main.py`:

- リファレンス`speculative_decode(ターゲット、ドラフト、プロンプト、K、温度)`実装 正確拒否 ルール、検証保存 ターゲット分布(経験的KL < 0.01対簡潔 ターゲット サンプリング)。
- EAGLEスタイル ツリー ドラフター 構築 深さK ツリー トップp分岐。
- ツリー注意マスク ビルダー ツリー トポロジー正しい因果パターンを生成 検証者 のため。
- 受け入れレート チーフ 実行 両方 小さいLM(蒸留1つGPT-2 小さい GPT-2 媒体 ターゲットから)。

```python
def speculative_step(p_target、q_draft、K、temperature=1.0):
    """推測的デコーディング1つラウンド。受け入れられたトークン リストを返す。"""
    # 1. ドラフト K トークン
    draft_tokens = []
    q_probs = []
    state = draft_state_init()
    for _ in range(K):
        probs = softmax(q_draft(state) / temperature)
        t = np.random.choice(len(probs)、p=probs)
        draft_tokens.append(t)
        q_probs.append(probs[t])
        state = draft_step(state、t)

    # 2. ターゲット計算 p 毎 ドラフトされた位置 + 1 エキストラ
    p_probs_all = target_forward_batched(p_target、draft_tokens、temperature)

    # 3. 受け入れ/拒否 左 右へ
    accepted = []
    for k、tok in enumerate(draft_tokens):
        r = np.random.uniform()
        if r < p_probs_all[k][tok] / q_probs[k]:
            accepted.append(tok)
        else:
            residual = np.maximum(p_probs_all[k] - q_probs[k]、0)
            residual /= residual.sum()
            accepted.append(np.random.choice(len(residual)、p=residual))
            return accepted
    # 4. すべて K 受け入れられた → ターゲットから ボーナス トークン サンプル
    accepted.append(np.random.choice(len(p_probs_all[-1])、p=p_probs_all[-1]))
    return accepted
```

## 使う

- **vLLM**および**SGLang**出荷 最初クラス推測的デコーディング。フラグ:`--speculative_model`、`--num_speculative_tokens`。EAGLE-2/3サポート`--spec_decoding_algorithm eagle`フラグを通じ。
- **NVIDIATensorRT-LLM**サポート Medusa およびEAGLE ツリー ネイティブ。
- **リファレンス ドラフト モデル**:`Qwen/Qwen3-0.6B-spec`(ドラフト Qwen3-32B)、`meta-llama/Llama-3.2-1B-Instruct-spec`(ドラフト 70B)。
- **Medusa ヘッド**(Cai et al.2024、"Medusa:シンプル LLM推論加速 フレームワーク 複数デコーディング ヘッド"):ドラフト モデル代わりに、K平行 予測ヘッドをターゲット自身に追加。デプロイをシンプル化、EAGLE よりわずかに低い受け入れ。

## 演習

1. 正確 拒否 ルール実装し、経験的検証。実行 10K サンプル 通じ`speculative_decode`および通じ簡潔 ターゲット サンプリング;計算テレビ 距離 2つ出力分布。~ただし0.01。

2. スピードアップ公式を計算。与えられた固定`α`および`K`、プロット予期 トークン毎ターゲット フォワード。見つけ最適K α ∈ {0.5、0.7、0.9}のため。

3. 小さい ドラフトを訓練 取る124M GPT-2ターゲット およぶ蒸留30M GPT-2 ドラフト100M トークン KL損失で。測定`α`ホールドアウト テキスト。予期:0.6-0.7。

4. EAGLEスタイル ツリー ドラフティングを実装。チェーン代わり、ドラフト 出力トップ3各深度の枝。構築ツリー 注意マスク。検証ターゲット受け入れ 最長訂正枝。

5. 失敗モード を測定。実行推測的デコーディング温度=1.5(高 確率)。表示α 崩壊およびアルゴリズムはドラフト オーバーヘッド由来簡潔 デコーディングより遅い。

## キーターム

| 用語 | 人々が言う | 実際に意味するもの |
|------|-----------------|------------------------|
| ターゲット モデル | "ビッグ モデル" | 遅い、高品質 モデル サンプル欲しい(p分布) |
| ドラフト モデル | "スペキュレータ" | 小さい、高速 予測機(q分布);5-30倍小さい |
| K / ドラフト 長 | "先読み" | 毎検証パス推測されたトークン数 |
| α / 受け入れレート | "ヒット レート" | トークンごと ベルヌーイ確率ドラフトの提案受け入れされた |
| 正確 拒否 ルール | "受け入れテスト" | r < p/q 比較ターゲット分布を保つ |
| 残差 分布 | "補正 p-q" | (p - q)+ / ||(p - q)+||_1、分布 拒否 上からサンプル |
| ツリー ドラフティング | "枝推測" | ドラフト 出力 ツリー候補、ツリー構造 注意 マスク付きで検証1つパスで |
