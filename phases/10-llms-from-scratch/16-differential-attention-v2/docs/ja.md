# 微分注意(V2)

> ソフトマックス注意は一致していない全トークンに少量の確率を広がる。100kトークン上でノイズは信号をおぼれ加算する。微分トランスフォーマー(Ye et al.、ICLR 2025)は2つのソフトマックスの差を計算することで固定、共有ノイズ床を減算。DIFF V2(Microsoft、2026年1月)は本番スタック書き直し:マッチベースラインデコード レイテンシ、カスタムカーネルなし、FlashAttention互換。このレッスンはV1からV2エンドツーエンド、stdlib Pythonで実行できるの違い操作の作業おもちゃの実装。

**タイプ:** ビルド
**言語:** Python (標準ライブラリ)
**前提条件:** フェーズ7·02(自己注意)、フェーズ7·15(注意亜種)、フェーズ10·14(アーキテクチャ ウォークスルー)
**所要時間:** 約60分

## 学習目標

- ソフトマックス注意がノイズ床を持ち、なぜコンテキスト長で成長する理由を正確に述べ。
- 微分注意公式を導き、なぜ減算が信号を保持しながら共有ノイズ成分をキャンセルする理由を説明。
- V1からV2差を歩く:何が高速化、何がより簡単、何がより安定、そしてなぜ各変更が本番事前訓練のために必要だったか。
- 純粋Pythonからの微分注意を実装し、スクラッチ、およびのノイズキャンセル特性を経験的に検証合成信号プラス ノイズクエリ。

## 問題

標準ソフトマックス注意はスケールで運用頭痛に変わる数学特性を持っています。クエリ`q`のために、注意の重みは`softmax(qK^T / sqrt(d))`。ソフトマックスは決してドロップ出力できない正ゼロ--一致していないすべてのトークンは肯定質量をいくつか取得。その残留質量はノイズ、コンテキスト長とスケール。128kトークンで、非一致トークンそれぞれがわずか0.001%確率得の場合、127999合計の約12%に貢献。モデルはノイズ床でいくらか を学ぶ必要がある搭乗コンテキスト成長。

経験的これは注意ヘッド干渉として示す:長いコンテキスト RAG中の幻想的引用、100kトークン検索タスク の紛失中部障害、針干し針ベンチマーク上の微細精度低下32k超。微分トランスフォーマー紙(arXiv:2410.05258、ICLR 2025)差を測定:DIFF トランスフォーマーは同じサイズベースラインより低いパープレキシティ、より高い長いコンテキスト精度、より少ない幻想を作る。

DIFF V1は3つの問題フロンティア事前訓練パイプラインの外保つ。その値キャッシュは毎デコード ステップ2回ロードされて、カスタムCUDAカーネルそれFlashAttention互換性壊した、その毎ヘッド RMSNorm不安定化70B+スケール長い実行訓練。DIFF V2(Microsoft unilm blog、2026年1月20日)すべて3つ固定。このレッスンは両方バージョンを歩く、差分演算子を構築、おもちゃクエリ上ノイズキャンセルをベンチマーク。

## コンセプト

### ソフトマックスのノイズ床

クエリ`q`およびキー`K = [k_1、...、k_N]`のために、注意の重みは:

```
w_i = exp(q . k_i / sqrt(d)) / sum_j exp(q . k_j / sqrt(d))
```

いいえ`w_i`決してゼロ。`k_i`が`q`と完全に関係ないなら、スコア`q . k_i`は0なし--変動ゼロ周りで分散`||q||^2 / d`。ソフトマックス正規化後、関係ないトークン毎まだ貢献`O(1/N)`加重合計。関係ないトークンの総貢献は`O((N-1)/N) = O(1)`--小さい量なし。

モデルは何かハードトップkのようなものを望んでいる:一致トークン上重み高い、他の場所近いゼロ。ソフトマックスは直接それをする滑らかすぎる。

### 微分思想

各ヘッドのQ およびK投影を2つに分割:Q = (Q_1、Q_2)およびK = (K_1、K_2)。2つの注意マップを計算:

```
A_1 = softmax(Q_1 K_1^T / sqrt(d))
A_2 = softmax(Q_2 K_2^T / sqrt(d))
```

出力:

```
DiffAttn = (A_1 - lambda * A_2) V
```

減算は2つのマップが共有するどれほどのノイズ分布をキャンセル。両方マップはランダム初期化で約127k一致していないトークン上の均一の重みを持つなら、キャンセル。シグナル--少数の実際に関連トークン上のピークウェイト--のみもし両方マップで同じ大きさで表示どれほどかんセル、訓練後そのしない。

`lambda`は毎ヘッド学習可能スカラー、パラメータ化`lambda = exp(lambda_q1 dot lambda_k1) - exp(lambda_q2 dot lambda_k2) + lambda_init`。それは負である可能。`lambda_init`デフォルト小さい正の数0.8のような。

### なぜこれはヘッド ノイズキャンセリングを一致

2つのノイジー マイクフォンが同じ声を記録することを考えて。両方ピックアップスピーカー プラス相関バックグラウンド ノイズ。1つを減算その他からそして共有ノイズドロップアウト。音声は生き残る2つのシグナル相位またはそれによって十分異なるか振幅完全キャンセルを防ぐ。毎ヘッド`lambda`学ぶ正確なこのバランス。

### V1 vs V2:差

V1はベースラインのトランスフォーマーに等しいパラメータ数を保つ。ヘッドあたり2つのクエリを得るためにそれ頭部次元に半分にした。ヘッド表現性を費用、もっと痛い--頭部当たり値キャッシュを半分にした。デコード(毎ソフトマックス枝1回)値キャッシュ2倍ロード必要。結果:デコード遅いベースラインより遅いマッチ パラメータ数まで。

V2ヘッドクエリ数を2倍してKVヘッドを同じ(アップ投影からパラメータ借り)として保つ。ヘッド次元はベースラインのトランスフォーマーと同じ留まる。減算後、追加の次元は背O_W投影をマッチのベースラインのトランスフォーマー。3つのもの同時:

1. デコード スピード一致ベースライン(KVキャッシュは一度ロード)。
2. FlashAttention実行不変(カスタムカーネルなし)。
3. デコード時の演算強度(HBMからロード毎バイトより多くのコンピュート)。

V2も削除毎ヘッド RMSNorm V1その減算安定化使われたそれ。70B級事前訓練スケール、RMSNormの不安定化遅い訓練。V2はシンプル初期化スキームで置き換え、追加のモジュール訓練安定を保つ。

### いつそれに到達

値コンテキスト長とともに成長。4kトークンノイズ床は小さい十分標準注意は良い。128kそれは傷つけている。

## ビルドそれ

`code/main.py`実装 微分注意純 Python。既知シグナル プラス ノイズ構造付きおもちゃ クエリはノイズキャンセル 比率直接測定できます。

### ステップ1:標準ソフトマックス注意

標準ライブラリ行列オプ:リスト リスト、手動matmul、ソフトマックス数値安定性マイナス マックス。

```python
def softmax(row):
    m = max(row)
    exps = [math.exp(x - m) for x in row]
    s = sum(exps)
    return [e / s for e in exps]
```

### ステップ2:分割Q、K 2つ半

V1スタイル:ヘッド次元に半分。V2スタイル:ヘッド次元保つおよびヘッド倍数。おもちゃ実装は教育学的明確のV1をつかう--数学同一、簿記のみ異なり。

### ステップ3:2つソフトマックス枝+減算

```python
A1 = [softmax([dot(q1, k) / scale for k in K1]) for q1 in Q1]
A2 = [softmax([dot(q2, k) / scale for k in K2]) for q2 in Q2]
diff_weights = [[a1 - lam * a2 for a1, a2 in zip(r1, r2)] for r1, r2 in zip(A1, A2)]
out = [[sum(w * v[j] for w, v in zip(row, V)) for j in range(d_v)] for row in diff_weights]
```

注:出力の重みは負である可能。それが良い--値キャッシュはまだ符号貢献扱い。その後V投影は符号吸収。

### ステップ4:ノイズ キャンセル 測定

1024シーケンス 長さ構築合成。既知位置にシグナル トークンを置く、ノイズで残り埋める。計算(a)標準ソフトマックス注意重みシグナル位置および(b)微分注意重みで。各シグナルのシグナル ノイズ比率を測定 ノイズ比率。DIFF 注意確実に生成シグナル ノイズ 比率高い3x-10x係数依存する訓練済み2つ枝を異なるについて。

### ステップ5:V1 vs V2パラメータ会計

与えられたconfig(隠し=4096、頭=32、d_head=128)、印刷:

- ベースラインのトランスフォーマー:Q、K、V各サイズ`隠し * 隠し`、MLP 4 *隠し。
- DIFF V1:Q、K各サイズ`隠し * 隠し`、V サイズ`隠し * 隠し`(不変)、ヘッド 次元内に半分。毎ヘッド追加`lambda`パラメータ(O(頭* d_head))。
- DIFF V2:Q サイズ`2 * 隠し * 隠し`、K サイズ`隠し * 隠し`、V サイズ`隠し * 隠し`。追加次元プロジェクト背O_W前に。追加同じ`lambda`パラメータ。

おもちゃV2(大きさ約`隠し * 隠し`追加の注意ブロック当たり)の追加パラメータ費用を測定し、それを印刷。

## 演習

1. `code/main.py`を実行。微分注意のために報告されたシグナル ノイズ比率がより高いこと確認 標準ソフトマックス注意合成クエリで。ノイズ振幅を変動し、標準注意が使用できなくなる交差ポイントを表示。

2. ベースラインからDIFF V1およびベースラインからDIFF V2へのパラメータ カウント デルタを計算 7Bクラス モデルのため(隠し=4096、頭=32、d_head=128、32レイヤー)。どのコンポーネント獲得パラメータを表示、スカーレット同じ留まる。

3. DIFF V1紙のセクション3およびDIFF V2 Hugging Face ブログのセクション2を読む。2文で、説明なぜV1毎ヘッド RMSNorm必要および V2削除できる理由した訓練発散なし。

4. アブレーション実装:微分注意計算`lambda = 0`(純粋最初ソフトマックス)および`lambda = 1`(完全減算)。合成クエリ上、シグナル ノイズ変更方法はスイープ全体で変更を測定。特定`lambda`シグナル ノイズを最大化。

5. おもちゃをGQA + DIFF V2に拡張。8 KVヘッドおよび32 Qヘッドを選択。KVキャッシュサイズが一致ベースラインGQAモデルと同じ(8、32)構成を表示。

## キーターム

| 用語 | 人々が言う | 実際に意味するもの |
|------|----------------|------------------------|
| 微分注意 | "2つソフトマックスマイナス各その他" | Q、Kを半分に分割、2つのソフトマックス マップを計算、2番目を減算(lambda-スケールされた)最初から、Vで乗算 |
| ノイズ床 | "ソフトマックスの非ゼロ尾" | O(1/N)重みソフトマックスは一致していないすべてのトークンで置き、合計O(1)長いコンテキスト全体 |
| lambda | "減算スケール" | 毎ヘッド学習可能スカラー`exp(lq1.lk1) - exp(lq2.lk2) + lambda_init`としてパラメータ化;負である可能 |
| DIFF V1 | "ICLR 2025バージョン" | オリジナル微分トランスフォーマー;パラメータ数保つためにヘッド 次元に半分、カスタムカーネルが必要、遅いデコード |
| DIFF V2 | "2026年1月修正" | Qヘッド 2倍 KVヘッド保つ;ベースラインデコード速度と一致、FlashAttentionで機能 |
| 毎ヘッド RMSNorm | "V1安定化器" | 追加normV1が差の後に適用;V2削除遅い訓練不安定を防ぐ |
| シグナル ノイズ比率 | "どれほどアテンション無駄" | 真のシグナル位置の重みの比率平均 関係のない位置での重み |
| 中部での紛失 | "長いコンテキスト失敗モード" | 経験的現象ここに検索精度長いコンテキスト中ドキュメントのディップ--DIFF注意これを削減 |
| 演算強度 | "ロード 毎バイトフロップ" | 比率V2はデコード毎KVロード倍のクエリを2倍することによって増加;重要メモリ バウンド デコード |
