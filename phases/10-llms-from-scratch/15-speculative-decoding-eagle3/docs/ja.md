# 推測的デコーディングとEAGLE-3

> フェーズ7·レッスン16は数学を証明した:Leviathanの拒否ルールはベリファイアーの分布を正確に保持。このレッスンは2026年本番推測的デコーディングの訓練スタック視点。EAGLE-3はドラフトモデルをベリファイアーの独自の隠し状態で訓練された目的構築の小さいネットワークに変えて、訓練時テストループを追加して、訓練と推論の分布を合わせた。結果:3×から6.5×エンドツーエンドスピードアップ、チャット上0.9以上で受け入れられたトークンあたりレート、分布トレードオフなし。すべての本番推論スタック2026は既定でそれを出荷する。

**タイプ:** ビルド
**言語:** Python (標準ライブラリ)
**前提条件:** フェーズ7·16(推測的デコーディング数学)、フェーズ10·12(推論最適化)
**所要時間:** 約75分

## 学習目標

- Leviathanの定理を1文で述べ、推測ループはベリファイアーと同じように分布されたサンプルを生成することを証明
- 2年間のプログレッションを歩く(バニラ spec-decoding(Leviathan 2023)からEAGLE、EAGLE-2、EAGLE-3を通じて)および各ステップが削除した正確な制限に名前を付け
- 受け入れレート`α`とドラフトツーベリファイアーコスト比率`c`から予期されたスピードアップを計算し、各体制に対する最適なドラフト長`N`を選択
- スクラッチから完全な推測ループを実装:ドラフト、検証、残差からリサンプル拒否、拒否時KVキャッシュをロールバック、フル受け入れでボーナストークン放出。

## 問題

70Bモデル上のオートリグレッシブデコーディングはH100上で1秒あたり約35トークンで実行される。GPUはどこにも飽和している。メモリ帯域幅は天井:すべてのトークンはHBMから70Bの重みを読み込み、1ステップの算術を実行し、1フロートを生成。コンピュートユニットはほぼアイドルに座る。

推測的デコーディングはそれを実際に解くことができるスループット問題に変える。安い ドラフトは`N`トークンを`N`小さいフォワードパスで提案。ベリファイアーはプレフィックスプラスすべての`N`ドラフトで1回実行。ベリファイアーの分布が位置`i`で ドラフト(統計的意味で正確にする)と同意するなら、我々は受け入れる;そうでなければ、拒否し、残差分布からサンプル補正。単一ビッグモデル フォワード生成N+1受け入れられたトークンの代わりの1つ。

重要な定理はLeviathan、Kalman、Matias(ICML 2023):出力分布はベリファイアーから直接サンプリングするとき生成されたの同一。近似的に。同一に。これはその全理由推測的デコーディングが本番で受け入れられるのか--純粋なレイテンシ最適化品質トレードオフなし。

フェーズ7·レッスン16があなたに与えたのは数学だった。このレッスンが与えるのは訓練スタックだ。良いドラフトは安い ドラフトより2倍多いスピードアップの価値がある。EAGLE、EAGLE-2、EAGLE-3(Li et al.、2024-2025)は"ドラフト = 同じモデルのより小さいバージョン"を正確なエンジニアリング規律に変えた。2026年本番推論サーバーはEAGLE-3をデフォルトにする。

## コンセプト

### 不変:Leviathanの拒否サンプリング

`p(t)`を次のトークンのためのドラフトの分布をいくつかのプレフィックス、`q(t)`をベリファイアーの分布を持つとしよう。ドラフトトークン`d ~ p`をサンプル。確率`min(1、q(d) / p(d))`でアクセプト。拒否した、残差分布`(q - p)_+ / ||(q - p)_+||_1`からサンプル。結果のサンプルは`q`に従って分布される。これはどれほど悪い`p`だが真である--それより悪く、より多くをしばしば拒否するが、出力は正確に残る。

`prefix + d_1 + ... + d_N`上で1つのベリファイアー フォワード パスを使用して背の後ろにこれらの呼び出しの`N`を積み重ねて。ベリファイアーは`q_1、q_2、...、q_{N+1}`同時に戻る。左から右へ歩く。位置`j`の最初の拒否で、残差からサンプル`residual(q_j、p_j)`それを停止。完全受け入れで、ボーナストークンを`q_{N+1}`から一つサンプル。

### スピードアップを決定

`α`をドラフトされたトークンあたりの予期された受け入れレート。`c = cost(draft) / cost(verifier)`コスト比率。ベリファイアー フォワード毎受け入れられたトークンの予期される数:

```
E[accepted] = (1 - α^(N+1)) / (1 - α)
```

受け入れられたトークンあたりの予期される総ウォール時間は`(N * c + 1) / E[accepted]`。`N`のそれを最小化した甘い場所を得。`α = 0.8、c = 0.05`のための:最適`N`は約5-7、スピードアップ3.2×。`α = 0.95、c = 0.02`のための:最適`N`は約8-10、スピードアップ5×へ。

単一最大レバーは`α`。`α = 0.6`(バニラ ドラフト)から`α = 0.9`(EAGLE-3)行く固定`N = 5`で取る毎ベリファイアー コール受け入れられた2.2予期されたトークンから4.1。ほぼ2×スループット同じベリファイアーから。

### 2年間プログレッション

**バニラ推測(Leviathan、2023)。** ドラフトモデルは独立に訓練されたより小さいLLM同じファミリーから。ワイヤーアップ簡単、`α ≈ 0.6`、スピードアップ最高で約2×。

**EAGLE-1(Li et al.、2024)。** ドラフトは小さいトランスフォーマー--通常1-2層--ベリファイアーの最後層隠し状態を入力として次トークンを直接予測するため。ドラフトはベリファイアーの機能表現を見て、その分布ベリファイアーの多く近い。`α`0.7-0.8に登る。

**EAGLE-2(Li et al.、2024)。** 動的ドラフトツリーを追加:単一シーケンス`N`トークン提案の代わり、候補の小さいツリーを提案し、1つのベリファイアー フォワード パス(ツリー注意)でそれぞれをスコア、最高確率パスを歩く。ドラフト長はステップあたり適応になる。`α`受け入れられたパス トークンあたり0.85以上登る。

**EAGLE-3(Li et al.、2025、NeurIPS)。** 2つの詳細変更。最初、機能予測損失を完全にドロップ--EAGLE-1/2はベリファイアーの隠し状態に合致するためのドラフトを訓練し、どれほどのデータはキャップ助ける。EAGLE-3はトークン予測上直接訓練。2番目、訓練時テスト(TTT):ドラフト訓練中、複数のステップ上で自身前の予測フィードバック入力ドラフト、推論で運用する同じ方法。これは訓練とテスト分布を合わせ、エラー蓄積を停止。測定スピードアップ:チャット上6.5×まで、H100上のSGLangでバッチ64で38%スループット改善。

## ビルドそれ

`code/main.py`を参照。これは完全なLeviathanの推測ループ すべてのピースで:ドラフト N、ベリファイアー平行パス、毎位置拒否、残差サンプリング、ボーナストークン、KVロールバック、およびの出力分布が`q`から直接サンプリング一致することの経験的検証。

### ステップ1:拒否ルール

```python
def accept(q_prob, p_prob, u):
    if p_prob <= 0:
        return True
    return u < min(1.0, q_prob / p_prob)
```

### ステップ2:残差分布

```python
def residual(q, p):
    raw = [max(0.0, qi - pi) for qi, pi in zip(q, p)]
    s = sum(raw)
    if s == 0:
        return list(q)
    return [r / s for r in raw]
```

### ステップ3:完全な推測ステップ

`spec_step`機能ドラフト`N`トークン`p`から、その後検証すべてそれらを1つの平行`q`評価。それぞれの ドラフトされたトークンのための拒否ルールを適用し、最初の拒否で残差からサンプル。すべてが受け入れるなら、ボーナストークン`q_{N+1}`から放出。

### ステップ4:KVロールバック簿記

シミュレーターはワーカーあたり論理的`kv_length`を追跡。`k`ドラフトの受け入れで、`kv_length += k`。位置`j`での拒否で、キャッシュは既に`j`の過去に書かれているが、論理的長さはプレフィックス長+ j + 1に設定されている--1つの補正トークン過去。その後読取は論理的長さに切る。

### ステップ5:Leviathan チェック

50000推測ステップを実行。受け入れられたトークンの経験的分布をカウント。50000から`q`からの直接サンプルと比較。カイ二乗統計は臨界値の良く下にあるべき。定理はプラクティスで通される。

### ステップ6:スピードアップ対`α`

ドラフト品質`p`をさまざまな振幅で摂動することにより変動させる。`α`を測定し、そして予期されたトークンをベリファイアー呼び出しあたり関数`α`と`N`の関数としてプロット。コード印刷EAGLE-3クラスドラフト品質(`α ≈ 0.9`)でテーブルは4-5トークンベリファイアー呼び出しあたりをアンロック。

## 使う

本番レベル`vllm serve`とEAGLE-3:

```bash
vllm serve meta-llama/Llama-3.3-70B-Instruct \
  --speculative-config '{
    "model": "yuhuili/EAGLE3-LLaMA3.3-Instruct-70B",
    "num_speculative_tokens": 5,
    "method": "eagle3"
  }'
```

H100上でバッチ64でのSGLangとEAGLE-3:大体バッチ64バニラデコーディングより1.38×多いスループット、EAGLE-3ペーパーあたり。

推測的デコーディングに到達するとき:

- すべてのインタラクティブチャット負荷で p50レイテンシがピークスループットより重要。
- コード生成および構造化出力(JSON、SQL)。`α`は0.9以上、目標分布は予測可能。
- 長い形成成(数千トークン)。償却スピードアップは続く支払い。

しない:

- 非常に小さいモデル(< 3B)。ドラフトはベリファイアーより遥かに安くない。
- 小さいバッチ1 CPUの展開。ドラフト モデルのメモリオーバーヘッドは価値がないかもしれない。
- 非常に高い温度創造的サンプリング`α`の崩壊。

## 演習

1. `code/main.py`を実行。Leviathan分布チェック上のカイ二乗統計50000サンプルで95%臨界値の下に留まることを確認。

2. `α`を0.9に保ち`c`を0.04に保ちながら、`N`を1から10にスイープ。プロット予期されたトークンベリファイアー呼び出しあたり および毎トークン実際ウォール時間。ウォール時間を最小化する`N`を見つけ。カーブの形を説明。

3. コードを変更してEAGLE-2ツリー検索をシミュレート:各ステップで、ドラフトは[2、2、2](8候補パス)の形のツリーを提案。ベリファイアーは一度実行され、最高確率受け入れられたパス勝利。`α`葉あたり コンピュートし、総トークンベリファイアー呼び出しあたり。線形チェーン spec-デコーディング等しい計算と比較。

4. 2つの並行シーケンスのための一括KVロールバック シミュレーターを実装。シーケンスA すべてのドラフトを持つ受け入れられた;シーケンス B拒否位置2。正しい`kv_length`シーケンスごとに更新されることを示し、作業は無駄ではない。

5. EAGLE-3ペーパーのセクション4(訓練時テスト)を読む。素朴なドラフト訓練がTTTなしで被害暴露バイアスからなぜを2文で説明し、訓練中そのドラフト予測を解決なぜです。seq2seqリテラチャーに予定サンプリングをスケジュール。

## キーターム

| 用語 | 人々が言う | 実際に意味するもの |
|------|----------------|------------------------|
| Leviathan ルール | "min(1、qペア)" | ベルヌーイアクセプト/拒否確率`min(1、q(d)/p(d))`、残差からサンプリング上拒否時ベリファイアー分布を正確に保持 |
| 残差分布 | "(q マイナス p)プラス、正規化" | `(q - p)_+`ゼロでクランプされ、正規化--拒否上からサンプルへの正しい分布 |
| 受け入れレート α | "ドラフトがどれほど頻繁に正しい" | 拒否ルール下のトークンごとベルヌーイ成功確率を期待;すべてのスピードアップ数学を支配 |
| EAGLE-1 | "隠し状態ドラフト" | ベリファイアーの最後層隠し状態(Li et al.、2024)の小さいトランスフォーマー ドラフト |
| EAGLE-2 | "動的ドラフトツリー" | EAGLE-1プラス1つのベリファイアー パスのツリー注意でスコアされた候補継続のツリー |
| EAGLE-3 | "訓練時テスト" | 機能予測損失をドロップし、訓練中自身出力フィード ドラフトで直接トークン予測上訓練 |
| 訓練時テスト(TTT) | "暴露バイアス修正" | 訓練中オートリグレッシブ実行ドラフト、そのため訓練そしてテスト入力分布合致--seq2seqから予定サンプリングの直接類似 |
| KVロールバック | "拒否されたドラフトを元に戻す" | ベリファイアーのKVキャッシュを受け入れられたプレフィックス長に初期化後拒否の簿記 |
| ボーナストークン | "フリーの一つ" | すべての`N`ドラフトアクセプト時、追加のベリファイアー費用なしで`q_{N+1}`から1つサンプル |
| ツリー注意 | "1つのパスで多くの候補を検証" | ドラフトツリーのトポロジーを尊重する非因果的マスクの注意;ツリーのすべてのノードのための`q_i`を計算1フォワード パスで |
