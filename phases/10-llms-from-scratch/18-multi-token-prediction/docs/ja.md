# マルチトークン予測(MTP)

> すべてのオートリグレッシブLLM GPT-2からLlama 3毎位置1つの損失で訓練:次トークン予測。DeepSeek-V3は毎位置2番目の損失を追加:その後トークン予測。追加14B パラメータ(671Bモデル上)バックに蒸留メイン モデルに勾配フロー、訓練MTPヘッドが推論で推測的デコーディング ドラフター80%+受け入れにしたがって。1.8×生成スループット フリーで来た。このレッスンはDeepSeekテクニカル レポートから順序MTPモジュール構築、損失と共有ヘッド パラメータ レイアウトを計算、なぜMTP因果鎖を保つがGloeckle et al.の平行MTPを壊すのか。

**タイプ:** ビルド
**言語:** Python (標準ライブラリ)
**前提条件:** フェーズ10·04(ミニGPT事前訓練)、フェーズ10·15(推測的デコーディング)
**所要時間:** 約60分

## 学習目標

- MTR訓練目的を述べ、予測深全体で結合損失を導き。
- Gloeckle et al.の平行MTPヘッド(2024)とDeepSeek-V3の順序MTPモジュール間の違いを説明し、なぜ順序デザイン因果鎖を保持。
- MTP モジュールを事前訓練実行に追加するパラメータおよびメモリ オーバーヘッドを計算。
- スクラッチからMTPモジュール1つを実装:共有埋め込み、毎深トランスフォーマー ブロック、投影、共有出力ヘッド。

## 問題

次トークン予測はLLM訓練目的標準。すべての隠し状態は1つのものが正確に予測するために監督:直後トークン。これは驚いて弱いシグナル。シーケンス大部分の情報は1トークンを超える拡張--構造、コヒーレンス、ファクトアリティ、算術流。モデルは蓄積多くの1トークン シグナルを学ぶ必要がある何兆トークン。

MTP問えば:何も隠し状態が一度多くの将来トークン監督を予測されたなら? Gloeckle et al.(Meta、2024)これは助けることを示した。彼らの実装いくつかの独立出力ヘッド骨格上置き、各異なるオフセット予測。平行、シンプル、しかし頭見え同じ隠し状態いかなる階層的細緻化--予測なし因果チェーン、彼らは推測的デコーディング用いない。

DeepSeek-V3(2024年12月)MTP再設計順序モジュールとして因果鎖を各予測深で保つ。モデル予測`t+1`から`h_i^(0)`、その後予測`t+2`新しい隠し状態から`h_i^(1)`それ組み合わせ`h_i^(0)`で`E(t+1)`埋め込み、等々。各深度は独自小さいトランスフォーマー ブロック。共有埋め込みと共有出力ヘッド パラメータ オーバーヘッド保つ控えめ。DeepSeek-V3のスケール、14Bエキストラ パラメータ MTP モジュール全体671Bメイン モデル重み上。その2%オーバーヘッドが買った密な訓練シグナルAND準備できたドラフト推測的デコーディング推論。

このレッスンは単一MTPモジュール及びD深損失ビルド スクラッチから。数学が潮流。実装は150行。

## コンセプト

### 順序MTPレシピ

DeepSeek-V3追加`D`MTPモジュール メイン モデル上。各モジュール`k`(のため`k = 1..D`)トークン深k予測--すなわち、`t_{i+k}`プレフィックス位置iを通じ与えられ。

モジュール`k`構成する:

- トランスフォーマー ブロック`T_k`独自注意およびMLP。
- 投影行列`M_k`結合以前深隠し状態含む次深地トークン埋め込み。
- 共有埋め込み`E`(メイン モデルと同じ)。
- 共有出力ヘッド`Out`(メイン モデルと同じ)。

訓練で、プレフィックス位置iを通じ、毎深隠し状態:

```
h_i^(0) = メイン モデル骨格位置i
h_i^(k) = T_k( M_k * concat(RMSNorm(h_i^(k-1))、RMSNorm(E(t_{i+k}))) )   k >= 1
```

毎深予測:

```
logits_{i+k} = Out(h_i^(k-1))   k = 1..D
```

毎深損失は地トークン対クロスエントロピー`t_{i+k}`:

```
L_k = CE(logits_{i+k}、t_{i+k})
```

深全体 結合損失:

```
L_MTP = (lambda / D) * sum_{k=1..D} L_k
```

`lambda`は小さい重み係数 -- DeepSeek-V3はトレーニング最初10%に0.3および0.1その後 を使用。総訓練損失は`L_main + L_MTP`。

### なぜ順序、いいえ平行

Gloeckle元平行MTP D出力 頭、直接適用`h_i^(0)`に。各頭予測`t_{i+k}`同じ骨格隠し状態から。それトレーニング微調整、しかし予測ないコンディション互いに。ユースできない`head_1`出力助け`head_2`--頭火の平行。

DeepSeek-V3の順序デザイン ビルド`h_i^(k)`から`h_i^(k-1)`プラス実際次トークン埋め込み`E(t_{i+k})`。それは因果チェーンを保つ:予測`t_{i+k+1}`、深モジュール`k+1` (見れない何`t_{i+k}`)。これ構造的オートリグレッシブ デコーダが自身出力消費する方法と同一--MTPモジュール直接推測的デコーディング ドラフター 使用可能。

推論で:フィード`h_i^(k-1)`および ドラフト`t_{i+k}`モジュール`k+1`に、取得予測`t_{i+k+1}`。反復。それが正確EAGLE スタイル ドラフト、訓練MTPモジュール ドラフト ネットワークであろう。DeepSeek-V3報告80%+受け入れ最初MTPモジュールおよび1.8×スピードアップ。

### パラメータ会計

隠し`h`およびボキャブラリ`V`とのモデルのために:

- メイン モデル:数十億パラメータ、プラス1つ出力ヘッド`V * h`サイズ。
- 共有出力ヘッド:メイン モデルのヘッド 再利用。エキストラ パラ なし。
- 共有埋め込み:メイン モデルの埋め込み 再利用。エキストラ パラ なし。
- 毎MTPモジュール:
  - 投影`M_k`:`(2h) * h = 2h^2`。
  - トランスフォーマー ブロック`T_k`:注意(`4h^2` MHA)プラスMLP(通常`8h^2`SwiGLU 8/3比率)。約`12h^2`毎ブロック。

総エキストラ毎モジュール:~`14h^2`。DeepSeek-V3の`h = 7168`、D = 1モジュール:~`14 * 7168^2 = ~720M`ペーパーのパラメータ。DeepSeek-V3報告14B--差は大部分専門家レイヤー MoE MTPモジュール もいることによる。

### 推測的デコーディング ペイオフ

事前訓練中、MTPモジュール訓練は約10%遅くなる(より多くのフォワード計算、追加損失)。ペイオフが二重:

1. 密な訓練シグナル。各隠し状態見る D+1監督ターゲット。測定DeepSeek-V3アブレーション上の効果 MMLU、GSM8K、MATH、HumanEval:一貫数パーセント ポイント改良。

2. フリー推測的デコーディング ドラフト推論。MTPモジュール既に訓練次いくつ トークン予測。ドラフト ネットワークとして そこなわれ、80%+受け入れレート配信。その レベルで、N=3またはN=5推測 デコーディング給1.8×スループット。10%訓練時コスト給戻す最初を走る推論。

## ビルドそれ

`code/main.py`単一MTPモジュール エンドツーエンド構築:共有埋め込み、投影、トランスフォーマー ブロック、共有出力ヘッド。それから短い合成シーケンスで毎深クロスエントロピー損失を計算し、コンポーネント別パラメータ カウント印刷。32 トークン小さいボキャブラリは数字を保つ読める。

### ステップ1:共有埋め込みテーブル

単一`vocab_size x 隠し`テーブル メイン モデルおよびすべてのMTPモジュール各深で使用。2番目コピーなし--文字通り同じテンソル。

### ステップ2:毎深組み合わせ

```python
def combine(prev_hidden、next_token_embed、M_k):
    # 機能 dimに沿って連結、その後投影 隠しまで
    concat = rms_norm(prev_hidden) + rms_norm(next_token_embed)  # ベクトル追加スタンドイン
    projected = matvec(M_k、concat)
    return projected
```

リアルDeepSeek-V3を連結2つのRMSNormed ベクトル`[2h]`および投影h x 2h行列で。おもちゃは stdlib簡潔性のためのベクトル追加を使用。

### ステップ3:トランスフォーマー ブロック深k

自己注意プラスMLP。おもちゃで、1層線形注意ブロックおよびSwiGLU MLP構造 numpy なしで見える。

### ステップ4:共有出力ヘッド

メイン モデルの出力投影を再利用。ボキャブラリ上のロジット。

### ステップ5:毎深損失

クロスエントロピー softmax(ロジット)に対して地トークン オフセット`k`。深全体集計`lambda / D`スケーリング係数。

### ステップ6:パラメータ会計

総パラメータ カウント、共有(埋め込み、ヘッド)カウント、毎モジュール エキストラ カウントを印刷。MTPエキストラとメイン モデル サイズの比率を表示。

## 使う

MTPはDeepSeek-V3(2024年12月)および DeepSeek-R1系に統合。推論:

- DeepSeekが独自提供スタック消費MTPモジュール推測的デコーダ ボックス。
- vLLMおよびSGLang統合パスDeepSeek-V3 MTP 2026年4月スタンド。
- AMD ROCm SGLang チュートリアルは特定MTP推測的デコーディング config測定1.8×V3チェックポイント。

新しい事前訓練実行でMTP使用するとき:

- 完全事前訓練パイプラインを制御し、密な訓練シグナルをバンク したい。
- スケールでモデル提供知り、スペキュレーティブ デコーディング フリーしたい。
- ヘッド サイズは少なくとも4096。1Bスケール オーバーヘッド利益より傷つけ。

ない:

- 既存事前訓練密なモデルを微調整。MTPモジュール訓練いいえ。
- リサーチ モデルシンプル ベースライン を比較 したい。MTP アーキテクチャを変更。

## 演習

1. `code/main.py`を実行。毎深損失を表示合成シグナル強化モノトニックに減少。シンプル 修正パターンで合成を修正および確認両深1および深2損失収束。

2. 密な70Bモデル パラメータ オーバーヘッドを計算(隠し8192、80レイヤー)D=1 MTPモジュール。DeepSeek-V3報告14Bオーバーヘッドと比較。なぜDeepSeekの数が高い説明:MTPトランスフォーマー ブロック継承同じMoE構造、毎モジュール パラメータ カウント膨らます。

3. D=2をおもちゃで実装:`h^(1)`を取り予測`t_{i+2}`2番目MTPモジュール を追加。確認結合損失およびパラメータ会計マッチDeepSeekペーパーの方程式19-21。

4. おもちゃを平行MTP(Gloeckle スタイル)に切り替える:メイン隠し状態上D出力ヘッドを追加、各異なるオフセット予測。深毎損失が順序バージョン同じ合成シグナル上と比較する方法を測定。順序バージョン 深 k > 1低損失を生成する必要がある中間予測に条件。

5. ドラフトとしての訓練済みMTPモジュールを使用:呼び出しモジュール k推論で`t_{i+k}`提案。受け入れレート測定これら ドラフト トークン メイン モデルの予測ホールドアウト シーケンス。トークンの50%+ヒットなら、MTP ドラフト特性を複製した。
